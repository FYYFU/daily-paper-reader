## 摘要
我们提出了递归智能体优化（RAO），一种用于训练递归智能体的强化学习方法：这些智能体可以递归地生成新的自身实例并委派子任务。递归智能体实现了一种推理时缩放算法，该算法自然允许智能体扩展到更长的上下文，并通过分治法泛化到更困难的问题。RAO提供了一种训练模型以最好地利用这种递归推理的方法，教导智能体何时以及如何委派和通信。我们发现，以这种方式训练的递归智能体具有更好的训练效率，可以扩展到超出模型上下文窗口的任务，泛化到比训练任务困难得多的任务，并且与单智能体系统相比，可以减少挂钟时间。

## Abstract
We introduce Recursive Agent Optimization (RAO), a reinforcement learning approach for training recursive agents: agents that can spawn and delegate sub-tasks to new instantiations of themselves recursively. Recursive agents implement an inference-time scaling algorithm that naturally allows agents to scale to longer contexts and generalize to more difficult problems via divide-and-conquer. RAO provides a method to train models to best take advantage of such recursive inference, teaching agents when and how to delegate and communicate. We find that recursive agents trained in this way enjoy better training efficiency, can scale to tasks that go beyond the model's context window, generalize to tasks much harder than the ones the agent was trained on, and can enjoy reduced wall-clock time compared to single-agent systems.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有的大语言模型（LLM）智能体系统在处理长上下文、多步骤、复杂任务时，通常采用固定的单智能体或预定义的递归推理框架（如 Claude Code、ADaPT、Recursive Language Models）。但这些系统将递归仅视为推理时的脚手架（scaffold），而模型本身并未经过训练来**自主决定何时进行委派、如何构造有效子任务、如何跨层级通信以及如何整合子智能体输出**。这导致模型无法充分利用递归带来的分治优势。
- **整体含义**：本文提出**递归智能体优化（RAO）**，一种强化学习方法，通过端到端训练，使同一个策略模型（单一 LLM）在递归执行树的每个节点上都能学习解决任务、生成有用的子任务并协调递归副本。RAO 旨在让递归成为模型内化的推理时扩展原语，从而突破上下文窗口限制、提升训练效率和泛化能力。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 训练一个**共享策略 πθ**，它被实例化为递归执行树中的每个智能体节点。智能体在解决任务过程中，可以动态地调用 `launch_subagent` 函数创建子智能体实例，委派子任务。每个节点根据自身任务成功信号和子节点成功信号获得局部奖励，通过强化学习联合优化所有节点。

### 关键技术细节
1. **递归智能体推理**  
   - 智能体基于 Python REPL（类似 CodeAct）扩展，支持异步函数 `async launch_subagent(goal, ...)`，可顺序或并行（`asyncio.gather`）执行子任务。子智能体可以继续递归创建更深层子节点。
   - 执行树动态生成，分支模式由策略自主决定，深度受限制。

2. **RAO 奖励设计（局部节点奖励）**  
   - 每个节点的奖励由两部分组成：  
     \[
     R(X, \tau_X) = \tilde{s}(X, \tau_X) + \lambda \cdot \frac{1}{|C(X)|} \sum_{c \in C(X)} \tilde{s}(c, \tau_c)
     \]
     - 第一项：该节点自身的任务成功信号（可以是精确验证、LLM裁判或根任务成功代理）。
     - 第二项：**委派奖励**：子任务平均成功率，λ≥0 控制强度。避免单纯因数量而获得额外奖励，鼓励高质量委派。
   - 当子任务监督信号不可用（如无完美验证器）时，可使用代理或 LLM 裁判（如 GPT-5-mini）给出成功信号。

3. **策略优化目标**  
   - 递归执行产生跨深度的任务分布：根任务分布 D₀ 和策略生成的深度子任务分布 D_d(θ)。目标联合优化：
     \[
     J(\theta) = \sum_{d=0}^D \mathbb{E}_{X \sim D_d(\theta)} \left[ \mathbb{E}_{\tau_X \sim \pi_\theta(\cdot|X)} [R(X, \tau_X)] \right]
     \]
     这形成自诱导课程（self-induced curriculum）：子任务通常更简单，降低训练难度。

4. **优势估计与深度加权**  
   - 使用**留一法基线**：对同一根任务的 G 个独立递归展开，计算根奖励的留一均值作为基线，对所有节点（包括子节点）使用相同基线，保证无偏估计。
   - **深度级逆频率加权**：为缓解深层轨迹数量过多导致优化偏倚的问题，每个深度 d 的轨迹权重设为 \( w_d = \alpha \cdot 1/N_d \)，其中 α 为归一化常数，使总权重保持不变。对应梯度估计：
     \[
     \hat{\nabla} J(\theta) = \sum_{d=0}^D \sum_{\tau \in B_d} w_d A(\tau) \nabla_\theta \log \pi_\theta(\tau)
     \]

## 7. 优点

- **方法创新**：首次将递归执行树的**所有节点**通过 RL 联合训练，而不是仅训练根节点或固定层次结构。这使模型内化递归策略，而非仅依赖外部脚手架。
- **通用性**：适用于三种差异很大的场景（合成合成、长文档聚合、多跳搜索），且均表现优于单智能体基线。
- **实用优点**：训练可获得性高（可使用不完全的节点级监督，如 LLM 裁判），并带来推理时计算分配的自适应和并行优势。
- **公平基线**：单智能体基线在评估时也被给予足够长的上下文窗口（256K），确保对比侧重递归策略而非上下文限制本身。

## 8. 不足与局限

- **超参数敏感性**：奖励设计中的 λ 需要调整，虽实验中设定为 0 或 0.4，但未系统研究其影响。
- **训练成本与可扩展性**：递归展开的代价可能很高，尤其当树很深或分支很多时。论文仅在相对小模型（4B 和 30B）上实验，对更大模型（如 70B+）或更复杂环境的适用性未验证。
- **实验统计量不足**：未报告多次运行的方差或置信区间，仅报告单一曲线。消融实验仅运行一次，鲁棒性不明确。
- **实际瓶颈**：在 Oolong-Real 和 DeepDive 上，递归智能体速度更慢（挂钟时间 14-18 倍），因顺序依赖或需大量读上下文。加速仅出现在天然并行任务（TEXTCRAFT-SYNTH）。
- **任务类型局限**：实验中子任务大多为父任务的缩小版（同类递归），但真实世界常需异构子任务（检索、验证、实现等）。跨域迁移未实验。
- **缺乏异构递归研究**：所有节点使用同一模型参数，未探索更强模型监督更弱模型或专门化分工的潜力。
