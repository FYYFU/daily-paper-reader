## 摘要
GRPO风格的RLVR通过在每次提示时进行多次在线策略尝试来训练推理模型，但通常仅通过终端奖励利用这些尝试。我们发现，混合组包含更丰富的过程信号：正确的完成是当前策略如何解决问题的自生成证据，而错误的完成则提供了需要纠正的在线策略前缀。我们提出了自监督在线策略蒸馏（SSOPD），该方法将以最短正确完成为条件的教师分布蒸馏到最长错误完成的前缀中。这无需外部解决方案轨迹，即可将组内正确与错误的对比转化为密集的过程监督。停止时间视角解释了最短正确/最长错误规则作为编辑持续失败以趋向快速成功动作的有限组近似，而提示级前沿权重则将辅助损失集中在正确和错误分支共存的地方。在AIME 2024、AIME 2025和HMMT 2025上，SSOPD在所有九个模型-基准设置中均优于GRPO。在Qwen3-8B上，其宏平均Avg@12达到65.6，比GRPO高出1.6个点，比解决方案条件化的OPSD基线高出0.8个点。代码将发布在https://github.com/tzq1999/SSOPD。

## Abstract
GRPO-style RLVR trains reasoning models from multiple on-policy attempts per prompt, but typically uses these attempts only through terminal rewards. We show that a mixed group contains a richer process signal: a correct completion is a self-generated witness of how the current policy can solve the problem, while a wrong completion provides on-policy prefixes where the policy needs correction. We introduce \emph{Self-Supervised On-Policy Distillation} (SSOPD), which distills a teacher distribution conditioned on the shortest correct completion into prefixes of the longest wrong completion. This converts intra-group correct--wrong contrast into dense process supervision without external solution traces. A stopping-time view motivates the shortest-correct / longest-wrong rule as a finite-group approximation to editing persistent failures toward fast-success actions, and a prompt-level frontier weight concentrates the auxiliary loss where correct and wrong branches coexist. Across AIME 2024, AIME 2025, and HMMT 2025, SSOPD improves over GRPO in all nine model-benchmark settings. On Qwen3-8B, it reaches a macro Avg@12 of 65.6, outperforming GRPO by 1.6 points and the solution-conditioned OPSD baseline by 0.8 points. Code will be released at https://github.com/tzq1999/SSOPD.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：GRPO风格的RLVR通过每次提示采样多个完整输出（on-policy），但仅利用终端奖励（是否正确），这些尝试中包含了丰富的中间过程信号，GRPO却将其丢弃。
- **核心问题**：能否从在线策略采样的组内（同一提示的不同尝试）自动提取密集的过程监督，而不依赖外部解决方案轨迹？
- **整体含义**：混合组内存在正确的尝试（证明当前策略能解决该问题）和错误的尝试（暴露了策略需要修正的前缀）。SSOPD将这种组内正确-错误对比转化为前缀级别的蒸馏监督，从而提升训练效率和最终性能。

## 2. 论文提出的方法论

- **核心思想**：利用组内最短正确完成作为“自生成见证”来构建教师分布，再将该教师分布蒸馏到最长错误完成的前缀上，将末端奖励转化为密集的过程信号。
- **关键技术细节**：
  - **选择规则**：最短正确完成（y⁺）作为成功见证，最长错误完成（y⁻）作为需要修正的失败路径。该选择基于停止时间视角：较短的正确完成是“快速成功”的高权重观察，较长的错误完成更可能包含持久失败的前缀。
  - **教师-学生蒸馏**：在错误完成的前缀 s⁻_t 处，教师分布 q_T(·|s⁻_t, y⁺) 由停止梯度的策略 π̄_θ 计算（额外接入 y⁺作为特权信息），学生分布 p_θ(·|s⁻_t) 由当前学生策略 π_θ 计算，使用点级剪枝的前向KL散度 ℓ_OPSD 作为损失。
  - **前沿权重 λ_x**：λ_x = λ₀·4p̂ₓ(1-p̂ₓ)，其中 p̂ₓ 是组内正确率。该权重在组内同时有正确和错误时较大（前沿区域），在全部正确或全部错误时为零，从而集中监督于有效对比的提示。
  - **总损失**：L(x) = L_GRPO(x) + λ_x L_SSOPD(x)，L_SSOPD 对 y⁻ 的前 K 个 token 前缀计算蒸馏损失。
- **公式与算法**：
  - 蒸馏项：ℓ_OPSD(s,z) = Σ_a clip_{τ_clip}( q_T(a|s,z)[log q_T(a|s,z) - log p_θ(a|s)] )
  - 选择：y⁺ = argmin_{y∈C(x)} τ(y)（最短正确），y⁻ = argmax_{y∈W(x)} τ(y)（最长错误）
