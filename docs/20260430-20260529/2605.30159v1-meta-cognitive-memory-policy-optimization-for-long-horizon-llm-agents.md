## 摘要
增强记忆的大语言模型智能体通过将交互轨迹递归地总结为紧凑的记忆来处理复杂的长程任务。然而，现有方法通常使用基于结果的强化学习来训练这些记忆策略，未能定位中间记忆质量下降的位置。随着交互的展开，模糊的递归摘逐渐丢弃任务相关信息并引入语义噪声。这加剧了信念偏差，模糊了智能体对潜在任务状态的估计，最终使长程推理偏离轨道。因此，我们认为记忆优化不仅应关注轨迹层面的成功，还应关注中间摘引所诱导的信念清晰度。为此，我们引入了信念熵（Belief Entropy），这是一种自监督代理指标，用于探测量模型在给定当前记忆时对潜在任务状态的不确定性程度。基于此代理指标，我们提出了元认知记忆策略优化（MMPO）。MMPO不依赖稀疏的基于结果的信号，而是通过明确惩罚诱导高认知不确定性的摘要，提供细粒度的、针对记忆的监督。实验表明，MMPO在各种长程任务上持续优于现有方法，即使在扩展到175万tokens的上下文时仍能保持97.1%的性能。

## Abstract
Memory-augmented LLM agents tackle complex long-horizon tasks by recursively summarizing interaction trajectories into compact memory. However, existing approaches typically train these memory policies using outcome-based reinforcement learning, failing to localize where intermediate memory quality degrades. As interactions unfold, ambiguous recursive summaries progressively discard task-relevant information and introduce semantic noise. This exacerbates belief deviation, obscuring the agent's estimate of the latent task state and ultimately derailing long-horizon reasoning. We therefore argue that memory optimization should focus not merely on trajectory-level success, but on the clarity of the belief induced by intermediate summaries. To this end, we introduce Belief Entropy, a self-supervised proxy that probes how uncertain the model remains about the latent task state given its current memory. Based on this proxy, we propose Metacognitive Memory Policy Optimization (MMPO). Instead of relying only on sparse outcome-based signals, MMPO provides fine-grained, memory-specific supervision via explicitly penalizing summaries that induce high epistemic uncertainty. Experiments show that MMPO consistently outperforms existing methods on diverse long-horizon tasks, maintaining 97.1% performance even when scaled to 1.75M-token contexts.

## 1. 核心问题与研究动机

- **关键挑战**：长程任务（如多跳问答、工具交互）中，LLM智能体面临**有限上下文窗口**和“lost-in-the-middle”问题。现有方法采用**递归摘要**将交互轨迹压缩为紧凑记忆，以维持固定上下文预算。
- **现有缺陷**：当前记忆策略优化（如RL-MemAgent、MEM1）仅依赖**最终任务结果（稀疏奖励）**，无法定位**中间记忆质量退化**。递归摘要逐步累积语义噪声，导致**信念偏差（belief deviation）**——智能体对潜在任务状态（hidden task state）的估计逐渐偏离真实状态，最终引起长程推理崩溃。
- **核心论点**：记忆优化应从“轨迹成功”转向**中间摘要诱导的信念清晰度**。可靠的记忆应使潜在任务状态从当前摘要中保持可预测。

## 2. 方法论：核心思想与技术细节

### 2.1 信念熵（Belief Entropy, \( \mathcal{H}_{\text{BE}} \)）

- **理论基础**：将长程任务建模为**部分可观测马尔可夫决策过程（POMDP）**。由于真实状态不可观测，智能体根据**信念状态（belief）** \( b(s) = P(s \mid \text{history}) \) 行动。在摘要型智能体中，信念由压缩摘要 \( m_t \) 诱导：\( b^M_t(s) = P(s \mid m_t) \)。
- **优化目标**：最大化 \( \log P(s_t \mid m_t) \) 等价于最小化条件熵 \( H(s_t \mid m_t) \)，即保持摘要与潜在状态之间的**互信息**。
- **实际代理**：直接计算 \( H(s_t \mid m_t) \) 不可行（状态不可观测）。受认知科学启发，通过**锚问题 (anchor question)** 探测模型对任务状态的内部不确定性：
  - 锚问题格式：**“Based on current memory, what is current task progress and what information is still needed?”**
  - 信念熵定义：\( \mathcal{H}_{\text{BE}}(m_t) = H(y \mid m_t, q) \)，即模型对锚问题响应的**响应熵**。
  - 实际估计：对贪婪解码的每个token，计算其预测概率分布的熵，然后取均值（式5）。
- **验证实验**：
  - **轨迹动态**：成功轨迹的 \( \mathcal{H}_{\text{BE}} \) 随轮次降低，失败轨迹则升高。
  - **结果相关性**：总熵减少量 \( \Delta\mathcal{H}_{\text{BE}} \) 与任务F1分数的Pearson相关系数 \( r = -0.684 \)，显著相关。
  - **推理时筛选**：在5个候选轨迹中选择熵最低的路径，可提升准确率，证明信念熵是独立于训练目标的可用信号。

### 2.2 元认知记忆策略优化（MMPO）

- **核心思想**：为中间记忆状态提供**密集、记忆特异的奖励**，不仅依据最终结果，还依据信念熵。
- **训练流程**（Algorithm 1）：
  1. **轨迹采样**：对每个任务，当前策略采 \( G \) 条完整轨迹。
  2. **密集奖励计算**：对每条轨迹的每个轮次 \( k \)：
     - 计算信念熵 \( \mathcal{H}_{\text{BE}}(m_k) \)。
     - 子轨迹奖励：\( R_k = \alpha \cdot \sigma(-\mathcal{H}_{\text{BE}}(m_k)) + r_{\text{final}} \)，其中 \( \sigma \) 是sigmoid函数，将熵映射到(0,1)；\( r_{\text{final}} \) 是最终结果奖励（如F1分数）。
  3. **组相对优势估计**：在同一轮次 \( k \) 的 \( G \) 个奖励上标准化：\( \hat{A}_k = (R_k - \text{mean}(R_k)) / \text{std}(R_k) \)。
  4. **轮次层面优势聚合**：对于轮次 \( t \)，其优势 \( A_t \) 是所有包含该轮次的子轨迹优势的平均值（式8），确保对当前摘要的信用分配涵盖所有后续轮次。
  5. **策略优化**：使用截断PPO目标（式9），将 \( A_t \) 分配到摘要 \( m_t \) 的每个token上，并加KL散度惩罚。

- **关键创新**：将信念熵作为**内在奖励**与结果奖励结合，提供细粒度监督，避免稀疏信用分配问题。

## 5. 主要结论与发现

1. **信念熵是有效的中间记忆质量代理**：成功轨迹的信念熵持续下降，且总熵减少与F1分数强相关（r=-0.684）；无需训练即可通过筛选低熵轨迹提升性能。
2. **MMPO一致优于纯结果基线的记忆RL**：
   - 在RULER-HotpotQA上，MMPO-14B/7B在所有长度上均优于RL-MemAgent，尤其在超长上下文（224K~3.5M）平均提升约3.1%。
   - 在MEM1框架的多目标QA和WebShop上，MMPO进一步超越MEM1基线，且优势随目标数增加而扩大。
3. **锚问题设计至关重要**：同时询问“进展”和“信息缺口”的锚问题效果最佳，仅问“直接答案”会鼓励过早自信，效果反而差于纯结果奖励。
4. **MMPO增强了信念熵与任务准确率的相关性**（r从0.68提升至0.75），说明其训练使信念熵更准确反映中间记忆质量。
5. **可扩展性**：在1.75M token上下文中保持97.1%性能，在3.5M token下仍有76.47%准确率，远超无递归摘要的长上下文模型（后者在896K时已降至0%）。

## 7. 不足与局限

- **信念熵的有效性依赖于锚问题设计**：虽然实验表明“进度+缺口”锚有效，但最优锚问题可能因任务而异，未提供自适应锚问题生成方法。
- **信念熵是间接代理**：它测量的是模型对锚问题的响应不确定性，而非直接测量真实状态的条件熵 \( H(s_t \mid m_t) \)；当模型生成存在噪声或锚问题与状态弱相关时，信号可能失真。
- **计算资源信息缺失**：未报告GPU型号、数量、训练时间，增加了复现难度。
- **安全与风险考虑**：MMPO不保证事实正确性；高置信度的摘要仍可能包含错误信息，在长交互或高风险场景中需谨慎部署。
- **实验覆盖**：虽然任务类型多样，但未在纯生成式长程任务（如故事写作、代码项目）上验证；未与其他过程监督方法（如PRM）在记忆场景下直接对比。
- **训练复杂度**：每组采样G=16条轨迹，对于需要大量交互的真实环境（如机器人控制），采样成本可能较高。
