## 摘要
强化学习（RL）天然适用于智能体知识库问答（KBQA），其中模型必须发出可执行动作、观察知识库反馈并最终返回答案。然而，当前基于RL的KBQA系统主要优化来自最终答案的稀疏奖励，导致中间动作错误监督不足。这对于逻辑形式注释的KBQA基准尤其局限：金逻辑形式可以转换为可执行动作序列，但现有流水线主要将其用于热启动数据构建，而非用于在线策略RL更新。我们提出GAPD，一个训练时的金动作策略蒸馏框架，为基于结果的RL添加密集的词元级指导。为使金动作与在线策略学生轨迹对齐，GAPD使用MID-ANCHOR匹配：它将学生探索和金执行期间到达的中间实体视为状态锚点，并通过这些探索的实体集将学生状态与金状态匹配。当前策略基于对齐的金动作条件化，作为停止梯度的教师，其词元分布被蒸馏回普通学生策略的生成动作词元跨度上。GAPD在WebQSP、GrailQA和GraphQ上持续超越当前最先进水平。

## Abstract
Reinforcement learning (RL) is a natural fit for agentic knowledge base question answering (KBQA), where a model must issue executable actions, observe knowledge-base feedback, and eventually return an answer. However, current RL-based KBQA systems mainly optimize sparse rewards from the final answer, leaving intermediate action errors weakly supervised. This is especially limiting for logical-form annotated KBQA benchmarks: gold logical forms can be converted into executable action sequences, but existing pipelines use them mainly for warm-start data construction rather than for on-policy RL updates. We propose GAPD, a training-time Gold-Action Policy Distillation framework that adds dense token-level guidance to outcome-based RL. To align gold actions with on-policy student rollouts, GAPD uses MID-ANCHOR MATCHING: it treats the intermediate entities reached during student exploration and gold execution as state anchors, and matches student states to gold states through these explored entity sets. The current policy conditioned on this aligned gold action serves as a stop-gradient teacher, whose token distribution is distilled back to the ordinary student policy over generated action-token spans. GAPD consistently surpasses the current state of the art on WebQSP, GrailQA, and GraphQ.

- **研究背景**：知识库问答（KBQA）要求模型通过执行结构化操作（如实体、关系、约束的选择）在知识库中导航并返回答案。近年来，强化学习（RL）被用于智能体式KBQA，模型通过与环境交互获得最终答案的稀疏奖励。然而，现有RL方法（如基于GRPO）仅优化最终答案的奖励，中间动作的误差缺乏细粒度监督。
- **核心问题**：逻辑形式标注的KBQA基准（如WebQSP、GrailQA）中，黄金逻辑形式可转换为可执行的动作序列，但现有RL流水线仅在热启动数据构建（如SFT）中使用这些信息，而在在线策略RL更新中被丢弃，导致token级信用分配不足：当轨迹失败时，无法定位是哪个中间动作出错。
- **整体含义**：GAPD旨在利用黄金动作提供密集的token级指导，弥补结果奖励的稀疏性，提升RL训练效率和最终性能。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：在结果奖励RL（GRPO）基础上，增加基于黄金动作的密集token级蒸馏信号。具体而言，对于学生策略（当前策略）在探索中到达的中间状态，通过Entity-Anchor Matching与黄金执行状态对齐，将对齐后的黄金动作作为条件，构造“自我教师”（self-teacher）分布，然后将该分布蒸馏回学生策略的生成token上。
- **关键技术细节**：
  1. **黄金动作提取**：从黄金逻辑形式解析出动作序列，并执行得到中间状态实体集。
  2. **Entity-Anchor Matching**：计算学生状态与黄金状态中间实体集的Jaccard相似度，选择最匹配的黄金状态；仅当匹配分数≥0.95时接受。
  3. **金动作条件自我教师**：在学生上下文基础上插入对齐后的黄金动作，构造教师侧提示；用当前策略在该提示下的分布作为教师分布（stop-gradient）；学生分布为原上下文下的策略分布。
  4. **Token级蒸馏信号**：对于匹配的生成token，计算教师分布下log概率减去学生分布下log概率，作为token级优势，并裁剪到[-2,2]。
  5. **融合与更新**：将token级优势与GRPO优势相加，形成融合优势，用于PPO风格的策略更新。
- **公式与算法**（文字描述）：
  - 状态相似度：`sim = |E(student) ∩ E(gold)| / |E(student) ∪ E(gold)|`
  - 蒸馏优势：`Agapd = clip(log p_teacher(y) - log p_student(y), -c, c)`

## 8. 不足与局限
