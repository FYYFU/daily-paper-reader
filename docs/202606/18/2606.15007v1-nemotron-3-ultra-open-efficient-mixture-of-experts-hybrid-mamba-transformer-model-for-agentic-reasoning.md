## 摘要
我们推出了 Nemotron 3 Ultra，一个总参数5500亿、激活参数55亿的混合专家混合曼巴-注意力语言模型。我们在20万亿文本Token上预训练了Nemotron 3 Ultra，随后将上下文长度扩展至100万Token，并使用监督微调（SFT）、强化学习（RL）和多教师在线策略蒸馏（MOPD）进行后训练。Nemotron 3 Ultra是我们迄今为止能力最强的模型，采用了多项关键技术——LatentMoE、多Token预测（MTP）、NVFP4预训练、多环境RLVR、MOPD以及推理预算控制。与最先进的开源大语言模型相比，Nemotron 3 Ultra的推理吞吐量最高可提升约6倍，同时达到同等精度。最先进的精度、高推理吞吐量以及100万Token的上下文长度使Nemotron 3 Ultra非常适合长时间运行的自主智能代理任务。我们在HuggingFace上开源了基础版、后训练版和量化版检查点，以及训练数据和配方。

## Abstract
We introduce Nemotron 3 Ultra, a 550 billion total and 55 billion active parameter Mixture-of-Experts Hybrid Mamba-Attention language model. We pre-trained Nemotron 3 Ultra on 20 trillion text tokens, then extended the context length to 1M tokens, and post-trained using Supervised Fine Tuning (SFT), Reinforcement Learning (RL), and Multi-teacher On-Policy Distillation (MOPD). Nemotron 3 Ultra is our most capable model yet, employing multiple key technologies - LatentMoE, Multi Token Prediction (MTP), NVFP4 pre-training, multi-environment RLVR, MOPD, and reasoning budget control. Nemotron 3 Ultra achieves up to ~6x higher inference throughput as compared to state-of-the-art publicly available LLMs while attaining on-par accuracy. The state-of-the-art accuracy, high inference throughput, and 1M token context length make Nemotron 3 Ultra ideal for long-running autonomous agentic tasks. We open-source the base, post-trained, and quantized checkpoints, along with the training data and recipe on HuggingFace.

### 2.1 核心思想
- 采用 **Mixture-of-Experts (MoE)** 架构，激活参数仅占总量10%，提高参数效率。
- 使用**混合 Mamba-Attention** 架构：Mamba-2层提供线性复杂度的序列建模，减少KV缓存占用；注意力层提供全局信息交互。
- **LatentMoE**：在专家路由中引入潜在表示，提升每参数精度。
- **多Token预测（MTP）**：预训练时预测未来多个token，支持推理时投机解码，加速生成。
- **NVFP4预训练**：使用4位浮点（NVFP4）进行训练，显著降低内存和计算成本，且精度损失极小。
- **后训练**：结合监督微调（SFT）、强化学习（RLVR）、**多教师在线策略蒸馏（MOPD）**，融合多个领域专家的能力。
- **推理预算控制**：支持“medium-effort”模式，推理时可动态调整计算成本与精度。

- MOPD损失：最大化反向KL目标：E[∑(logπ_teacher - logπ_student)]，使用异步重要性采样比率和裁剪。
- MTP增强损失：温度缩放前向KL：T² * D_KL(σ(z_backbone/T) || σ(z_mtp/T))，固定主模型，只更新MTP头。

## 6. 论文的主要结论与发现

1. **效率与精度平衡**：Nemotron 3 Ultra在保持与GLM-5.1、Kimi-K2.6、Qwen-3.5等模型同等精度（多数基准差距在1-2点内）的同时，**推理吞吐量最高提升5.9倍**（8k输入/64k输出工作负载）。
2. **代理任务强大**：在Terminal Bench、GDPVal、SWE-Bench、BrowseComp等代理基准上表现突出，PinchBench和ProfBench（留出集）同样达到顶级水平，证明泛化能力。
3. **MOPD有效性**：多教师在线策略蒸馏在代理、指令跟随、安全性等任务上显著提升性能（恢复率最高达172%），但在纯推理任务（如HLE）上增益较小，归因于学生采样分布与教师支持域不匹配。
4. **NVFP4预训练**：在20万亿token上成功且稳定，与BF16训练损失差距低于0.4%，并发现两个训练发散事件及其解决方案（修改梯度累积精度、提前学习率衰减）。
5. **量化精度**：5.03 BPE的NVFP4量化版本在大部分基准上精度损失<1%，长上下文（AA-LCR）略有恢复，总体可部署。
6. **投机解码加速**：MTP增强后，平均接收长度从4.39提升至4.58（贪心解码），生成速度提升5-6%。
7. **测试时扩展**：结合搜索精炼，在IMO-ProofBench Advanced集上达到82.3%精度（5轮），接近基于Gemini的Aletheia智能体（91.9%）。

## 7. 优点

- **架构创新**：混合Mamba-Attention + LatentMoE + MTP的协同设计，在推理效率和参数效率上均取得收益。
- **训练革命**：NVFP4预训练在如此大规模上验证稳定，节省大量训练成本；MOPD算法有效融合多领域专家知识。
- **后训练全面**：覆盖从通用能力到高度专用（如CUDA、RTL、法律、竞编程）的多个领域，且使用多个环境框架避免过拟合单一工具。
- **开源彻底**：不仅开放模型检查点（BF16、NVFP4），还开源训练数据、配方、RL环境，极大促进社区复现和应用。
- **推理优化深入**：针对混合模型在Prefill-Decode解耦、MoE通信（FlashInfer all-to-all）、Mamba状态快照等方面的工程创新。
- **实验严谨**：大量消融实验、留出集验证、量化精度与吞吐量权衡的系统搜索。
- **文档详尽**：论文长达65页，详细记录架构、数据、训练细节、稳定性问题和修复、基础设施优化，可复现性强。

## 8. 不足与局限

- **MOPD在推理任务上增益有限**：如HLE（无工具）场景，MOPD2相对于RLVR仅提升1.1点（16.9%恢复率），说明多教师蒸馏更适合行为偏好（如工具选择、安全决策）而非新增知识。
- **训练稳定性**：预训练中遇到两次发散，虽然被定位和缓解（第一次归因于MTP梯度精度，第二次通过提前衰减学习率解决），但根本原因（第二个发散）未完全查明，仅通过实践手段规避。
- **量化长上下文性能**：NVFP4版本在AA-LCR上需恢复+2.4 BPE才能与BF16持平，表明4位量化对长距离检索任务仍有挑战。
- **低估的基准缺失**：部分重要基准（如IOI 2025）缺少对比模型数据；多语言覆盖主要限于MMLU-ProX和WMT，可能未充分评估低资源语言。
- **推理吞吐对比潜在偏差**：Nemotron使用TRT-LLM，对比模型使用vLLM，虽然都是各自最优，但不同框架的工程优化水平不等同，可能影响公平性。
- **后训练计算开销**：MOPD涉及十多教师模型并行训练，且需要多轮迭代，资源消耗巨大（尽管没给出具体数字），可能对中小团队不友好。
- **开源数据限制**：虽声称开源，但部分数据（如OpenResearcher过滤版）来源授权可能限制商用；部分训练数据（如内部专有数据集）仅部分公开。
