## 摘要
人形机器人在现实世界辅助中具有巨大潜力，但在非结构化环境中与物体的敏捷交互需要紧密耦合的全身协调。尽管最近取得了进展，但当前控制器面临关键的部署差距。它们严重依赖密集的参考轨迹和完美的状态可观测性，这本质上限制了物理泛化。我们提出了视觉引导的敏捷交互控制（VAIC），这是一个统一的框架，通过仅依靠机载深度、历史本体感受和解耦的用户命令接口来弥合这一差距。VAIC采用两阶段蒸馏范式。首先，特权教师策略利用精确的物体运动学和精确的环境状态掌握多样化的交互技能。其次，可部署的学生策略通过用跨多个轴的速度目标和每帧的交互指示器替换全身跟踪来蒸馏这些能力。学生利用循环物体适应模块从原始深度流和本体感受中隐式推断不可观测的物体动力学。在人形机器人上的评估和实际部署表明，单个VAIC策略成功执行了高度多样化的动态任务。这些任务包括搬运箱子、与手推车互动和玩滑板，始终优于基线，并推进了自主人形机器人的部署。

## Abstract
Humanoid robots hold immense potential for real-world assistance, yet agile interaction with objects in unstructured environments demands tightly coupled whole-body coordination. Despite recent advancements, current controllers face a critical deployment gap. They rely heavily on dense reference trajectories and perfect state observability, which inherently limits physical generalization. We present Vision Guided Agile Interaction Control (VAIC), a unified framework that bridges this gap by operating exclusively on onboard depth, historical proprioception, and a decoupled user command interface. VAIC employs a two-stage distillation paradigm. First, a privileged teacher policy masters diverse interaction skills using precise object kinematics and exact environmental states. Second, a deployable student policy distills these capabilities by replacing full body tracking with velocity targets across multiple axes and an interaction indicator for each frame. The student utilizes a recurrent object adaptation module to implicitly infer unobservable object dynamics from raw depth streams and proprioception. Evaluations and real-world deployments on the humanoid robot demonstrate that a single VAIC policy successfully executes highly diverse dynamic tasks. These tasks include box carrying, cart interaction, and skateboarding, consistently outperforming baselines and advancing autonomous humanoid deployment.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：采用 **两阶段蒸馏范式**，将高层次用户意图与低层交互执行解耦。
  - **第一阶段（教师训练与学生预热）**：在模拟器中训练一个特权教师策略，它可以使用精确的物体运动学、参考轨迹等完美信息，通过PPO掌握复杂接触动力学。同时，利用教师的rollout数据对学生模块进行行为克隆（BC）和监督学习，使学生初步学会从深度和本体感受中推断不可观测的状态。
  - **第二阶段（视觉引导的学生探索）**：移除一切特权信息和密集参考轨迹，学生策略仅依赖机载深度流、历史本体感受和用户解耦命令，通过PPO优化任务完成度和命令跟踪奖励。同时，物体适配器和特权适配器继续在线更新，防止表征漂移。

- **关键技术细节**：
  - **解耦命令接口**：用户命令 \(u_t = (v_t, c_t)\)，其中 \(v_t \in \mathbb{R}^3\) 是多轴速度目标（导航意图），\(c_t \in \{0,1\}\) 是每帧交互状态指示器（0表示接近/导航，1表示主动接触/交互）。此接口在训练时从参考轨迹中提取，部署时由用户提供。
  - **物体适配器（Object Adapter）**：使用CNN-GRU深度编码器提取时间深度特征，结合本体感受和命令，隐式预测物体状态 \(\hat{s}_{t}^{obj}\)。训练时监督真实状态。
  - **特权适配器（Privilege Adapter）**：利用预测的物体状态和几何模板 \(P\)，将空间占用先验投影到机器人坐标系，重建教师策略的隐层表征 \(\hat{z}_{t}^{priv}\)，用于驱动学生策略的动作生成。
  - **两阶段不对称输入设计**：教师跟踪所有参考状态；学生严格使用解耦命令和深度，完全脱离参考运动学。

- VAIC 通过对用户意图与交互执行进行解耦（使用速度命令 + 交互状态指示器），并采用两阶段蒸馏，成功在真实人形机器人上仅依靠机载深度和本体感受实现了高度多样化的动态交互任务（盒子搬运、推车、滑板）。

### 7. 优点（方法与实验设计亮点）

- **解耦命令接口**：将“去哪里”与“如何交互”分离，降低了部署难度，支持直观的遥操作和自主导航。
- **两阶段蒸馏范式**：让学生在不依赖密集轨迹和完美状态的前提下，同时保留教师策略的接触敏捷性。
- **递归物体适配模块**：通过CNN-GRU有效克服视觉遮挡和传感器噪声，从时序深度中重建物体动态。
- **统一策略**：单个策略即可跨多种动态任务泛化，无需任务特定重训练。
- **充分的消融与泛化实验**：验证了深度、物体适配器、GRU等组件的必要性，并在真实硬件上展示了对抗未建模变化的鲁棒性。

### 8. 不足与局限（包括实验覆盖、偏差风险、应用限制）

- **传感器限制**：深度适配器对透明或反光物体可能失效（传感器噪声大）。
- **极端分布外情况**：当物体质量发生剧烈且突然的变化时，循环网络可能来不及调整，导致表示崩溃。
- **缺乏精细操作能力**：解耦命令接口对粗粒度导航和大幅度操控高效，但不支持灵巧的手内精细操作（如旋转、抓取小物体）。
- **实验覆盖**：定量评估主要在模拟器中完成，真实部署虽展示但未给出与基线的直接对比数据。
- **可重复性风险**：需要大量域随机化（相机噪声、延迟、深度量化等）才能实现 sim-to-real，训练过程对随机种子和参数敏感。
