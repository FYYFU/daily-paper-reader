## 摘要
技能记忆可以通过将过去经验作为文本指导来改善智能体强化学习，但检索到的技能并非完美：它们可能在一种状态下有帮助，而在另一种状态下误导同一策略。这使得常见的特权教师假设变得脆弱，即技能条件提示可以被视为无技能提示的固定教师。我们引入了UCOB，一个通过信用感知的在策略双向自蒸馏来学习利用和演化智能体技能的框架。UCOB将技能条件提示和无技能提示视为同一模型的两个在策略上下文视图，比较它们在相同任务和锚定状态下的剩余回报，并将较高回报的视图作为局部教师。这种局部信用信号内化了有用的技能条件行为，纠正了误导性的技能使用，并指导任务/状态技能记忆更新、效用感知检索和反思自训练。在包括ALFWorld、WebShop和Search-QA的智能体任务上的实验表明，UCOB在不同模型规模上优于无技能强化学习、技能记忆基线和自蒸馏方法，在ALFWorld和WebShop上比最先进基线分别高出23.5和18.0个百分点。消融和分析进一步验证了其核心机制和效率。

## Abstract
Skill memories can improve agentic reinforcement learning by reusing past experience as textual guidance, but retrieved skills are not oracular: they may help in one state while misleading the same policy in another. This makes the common privileged-teacher assumption fragile, namely that a skill-conditioned prompt can be treated as a fixed teacher for the no-skill prompt. We introduce UCOB, a framework for learning to utilize and evolve agentic skills via credit-aware on-policy bidirectional self-distillation. UCOB treats skill-conditioned and no-skill prompts as two on-policy context views of the same model, compares their return-to-go within the same task and anchor state, and uses the higher-return view as the local teacher. This local credit signal internalizes useful skill-conditioned behavior, corrects misleading skill usage, and guides task/state skill memory updates, utility-aware retrieval, and reflection self-training. Experiments on agentic tasks, including ALFWorld, WebShop, and Search-QA, show that UCOB outperforms skill-free RL, skill-memory baselines, and self-distillation methods across model scales, with up to 23.5 and 18.0 point gains over SOTA baselines on ALFWorld and WebShop. Ablations and analyses further validate its core mechanisms and efficiency.

# UCOB: 学习利用和演化智能体技能

## 1. 核心问题与整体含义
**研究动机**：在长期交互任务中，智能体强化学习（RL）常利用过去经验形成可复用的技能记忆（如文本指导）。然而，检索到的技能并非总是有益的——同一技能在某个状态可能帮助策略，在另一状态可能误导。现有方法（如 Skill-SD、SDAR）通常将技能条件提示（skill-conditioned prompt）视为无技能提示（no-skill prompt）的固定特权教师，但这种假设脆弱，因为技能条件分支并非始终优于无技能分支。

**整体含义**：论文提出**UCOB**框架，将技能条件与无技能提示视为同一模型的两个在策略上下文视图，通过比较同一任务同一锚定状态（anchor state）下的剩余回报（return-to-go），选择较高回报的视图作为局部教师，从而实现有用技能行为的内化、误导技能的纠正，并闭环更新技能记忆与生成策略。这一视角将技能利用转化为信用分配问题，显著提升了智能体在复杂环境中的学习效率与最终表现。

## 2. 方法论
**核心思想**：UCOB 不预先固定教师方向，而是在每个锚定状态组内，根据两个视图（技能条件 vs. 无技能）的实际回报比较，动态决定谁教谁。高回报视图成为局部教师，通过自蒸馏将行为分布传递给另一视图；同时，利用该回报差异更新技能效用、指导检索，并训练反思式技能写入器。

**关键技术细节**：

1. **双层级技能记忆与UCB检索**：
   - 维护任务级和状态级技能池，技能包含文本原则、适用条件、检索键、效用分数和计数。
   - 对上界置信度（UCB）分数进行排序检索：`Rp(m; st) = α sim(qp(st), km) + (1-α)[U(m) + η sqrt(log(Np+1)/(nm+1))]`。

2. **混合技能/无技能轨迹采样**：
   - 每批次一半轨迹使用技能提示，另一半使用无技能提示，两者均通过在线RL优化。

3. **锚定状态分组**：
   - 对同一任务组内，将所有轨迹按规范状态分组，每个记录包含分支标识、回报等，形成比较集 `C(u, ˜s)`。

4. **信用感知双向自蒸馏（CBSD）**：
   - 在每个锚定状态组中，选择最高回报记录 `ρ*` 作为信用参考。
   - 对每个来自另一分支的记录，若回报差 `Δe > ε`，则形成教师-学生对。
   - 教师方向由 `c*` 决定（技能→无技能 或 无技能→技能），而非固定。
   - 在教师参考前缀上，计算 Top-K 令牌支持匹配的 KL 散度：`D_topK = KL(q_e,j || p_e,j)`。
   - 引入置信门 `g_e,j` 和回报差距权重 `w_e`，得到 CBSD 损失 `L_CBSD`。

5. **技能记忆演化与训练目标**：
   - 基于同一锚定状态组内高/低回报对比，写入状态级技能；任务级技能来自轨迹反思。
   - 技能效用通过指数移动平均更新：`U(m) ← (1-βU)U(m) + βU δm`。
   - 技能写入器通过含有效用归一化优势的反思自训练损失 `L_refl` 优化。
   - 联合损失：`L = L_RL + λ_CBSD L_CBSD + λ_refl L_refl + β_KL L_KL`。

**理论支撑**（Proposition 1）：当固定的低回报分支向高回报分支插值时，局部优势代理改善量为 `ηδ_x`，并提供一轮更新下的性能下界。
