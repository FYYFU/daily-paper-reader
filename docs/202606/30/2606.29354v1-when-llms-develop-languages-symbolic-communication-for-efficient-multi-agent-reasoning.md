## 摘要
思维链（CoT）提升了大型语言模型（LLMs）在困难推理任务上的表现，但其生成的冗长自然语言推理过程与高效的机器推理并不匹配。我们提出了通信语言符号路由（CLSR），这是一个测试时框架，其中多个LLM智能体自主发明、演化并共享紧凑的语言符号框架（LSFs），同时一个无潜在变量的路由器根据查询自适应地选择和组合这些语言，以优化准确性与token消耗之间的权衡。与优化表面指令的提示优化不同，CLSR将每个LSF视为可复用的符号协议，包含紧凑符号、使用规则和消息传递契约，并通过由正确性和token成本驱动的进化循环来改进它。在推理时，路由器可以调用单个低成本的LSF调用、集成多个LSF，或在更难的查询上执行多轮LSF组合协议。在具有挑战性的基准测试中，与标准CoT相比，CLSR将面向延迟的生成token消耗减少了3~6倍，同时保持了准确性。我们进一步推导了任意符号体系下token成本的信息论下界，并表明在解释器可实现的前提下，多轮LSF协议有条件地包含程序执行流水线。代码已公开（https://github.com/pzqpzq/LSF_MDia）。

## Abstract
Chain-of-Thought (CoT) improves large language models (LLMs) on difficult reasoning tasks, but it often incurs long natural-language rationales that are poorly aligned with efficient machine reasoning. We propose Communicative Language Symbolism Routing (CLSR), a test-time framework in which multiple LLM agents autonomously invent, evolve, and share compact Language Symbolism Frameworks (LSFs), while a latent-free router adaptively selects and composes these languages per query to optimize the accuracy-token trade-off. Unlike prompt optimization that refines surface instructions, CLSR treats each LSF as a reusable symbolic protocol with compact symbols, usage rules, and a message-passing contract, and improves it through an evolutionary loop driven by correctness and token cost. At inference time, the router may invoke a single low-cost LSF call, ensemble multiple LSFs, or execute a multi-round LSF composition protocol on harder queries. Across challenging benchmarks, CLSR reduces latency-oriented generated token completion by $3\sim 6\times$ compared to standard CoT while maintaining accuracy. We further derive an information-theoretic lower bound on token cost under arbitrary symbolism and show that, under an interpreter-realizability premise, multi-round LSF protocols conditionally subsume program-execution pipelines. Code is publicly available (https://github.com/pzqpzq/LSF_MDia).

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 2.1 核心思想
- **Language Symbolism Framework (LSF)**：一种由LLM自主生成的符号通信协议，包含紧凑的词汇表、语法规则、使用约束，以及成本-准确度档案。它被作为可复用的“专家”模块，而非一次性指令。
- **多智能体演化**：多个LLM实例作为“提案者”、“评论者”和“变异者”，并行生成、评价、筛选和重组LSF。每轮通过“正确性优先、token最少”的帕累托选择保留高杠杆推理轨迹，再让LLM变异产生下一代LSF，直至性能饱和。
- **潜变量自由路由器**：推理时，一个LLM-router（同样使用基础LLM）根据查询自适应选择三种模式：
    1. **单LSF直接回答**（低风险问题，1轮）
    2. **多LSF集成**（中等难度，多数投票）
    3. **隐式LSF组合**（困难问题，多轮、顺序调用不同LSF）
    路由器输出一个紧凑的协议计划（约几十token），指导后续推理过程，所有轮次和调用生成的token均计入总成本。

### 2.2 关键技术细节
- **LSF卡**：每个LSF以卡片形式存储：`[ID] TAG | perf:<acc>/<tok> | IO | STOP`，包含验证准确率、中位数生成token数、输出模式及停止条件。
- **进化循环**：训练集分为M组，依次用于生成、评估、选择/变异。每一代保留历史上最优的LSF（精英继承），其余由变异产生。默认使用第5代LSF池。
- **测试时流程**：
    1. 给定查询x，路由器先通过轻量级类别路由（约<10 token）缩小候选LSF范围。
    2. 路由器根据类别内LSF卡，输出完整计划（模式、LSF列表、轮数、采样数、聚合方式、停止条件）。
    3. 执行计划，所有生成token累积到成本C中。
- **理论分析**：
    - 将CLSR形式化为约束随机控制问题，证明存在最优策略（定理3.1）。
    - 推导信息论下界：预期token数 ≥ (H(Y|X) - h2(δ) - δlog(|Yx|-1)) / κθ(x)，其中κθ(x)是每个生成token携带的最大信息量（定理3.2）。解释：LSF提高每token信息密度，因此同等准确率下理论下界更低。
    - 在“解释器可实现性”假设下，证明多轮多LSF协议有条件地包含程序执行流水线（定理3.3），即CLSR可在不依赖外部解释器的情况下达到类似效果。

## 6. 论文的主要结论与发现

1. **符号语言可显著提升推理效率**：CLSR能自动生成适应LLM自身归纳偏好的紧凑符号协议，在多数基准上以相似准确率减少3-6倍生成token。
2. **自适应路由是关键**：简单问题单LSF快速完成；困难问题通过多LSF集成或多轮组合获得更高准确率，整体帕累托前沿优于固定长度基线。
3. **演化机制有效**：基于正确性和token成本的进化搜索能稳定收敛到高质量LSF，随智能体数量和进化深度提升而改善。
4. **信息论解释**：CLSR提高每生成token携带的任务相关信息量（κθ），因此相同信息需求下所需token更少，符合理论下界。
5. **条件通用性**：在解释器可实现前提下，多轮LSF协议可模拟程序执行管道，且不依赖外部运行时。

## 8. 不足与局限
