## 摘要
我们推出了 Nemotron 3 Ultra，一个总参数量为5500亿、激活参数为550亿的混合专家（Mixture-of-Experts）混合Mamba-Attention语言模型。我们在20万亿文本标记上预训练了Nemotron 3 Ultra，然后将上下文长度扩展到100万标记，并使用监督微调（SFT）、强化学习（RL）和多位教师同策略蒸馏（MOPD）进行后训练。Nemotron 3 Ultra是我们目前能力最强的模型，采用了多种关键技术——LatentMoE、多标记预测（MTP）、NVFP4预训练、多环境RLVR、MOPD以及推理预算控制。与公开可用的最先进大语言模型相比，Nemotron 3 Ultra的推理吞吐量最高提升约6倍，同时达到同等精度。最先进的精度、高推理吞吐量和100万标记上下文长度使Nemotron 3 Ultra非常适合长时间运行的自主智能体任务。我们在HuggingFace上开源了基座、后训练和量化检查点，以及训练数据和配方。

## Abstract
We introduce Nemotron 3 Ultra, a 550 billion total and 55 billion active parameter Mixture-of-Experts Hybrid Mamba-Attention language model. We pre-trained Nemotron 3 Ultra on 20 trillion text tokens, then extended the context length to 1M tokens, and post-trained using Supervised Fine Tuning (SFT), Reinforcement Learning (RL), and Multi-teacher On-Policy Distillation (MOPD). Nemotron 3 Ultra is our most capable model yet, employing multiple key technologies - LatentMoE, Multi Token Prediction (MTP), NVFP4 pre-training, multi-environment RLVR, MOPD, and reasoning budget control. Nemotron 3 Ultra achieves up to ~6x higher inference throughput as compared to state-of-the-art publicly available LLMs while attaining on-par accuracy. The state-of-the-art accuracy, high inference throughput, and 1M token context length make Nemotron 3 Ultra ideal for long-running autonomous agentic tasks. We open-source the base, post-trained, and quantized checkpoints, along with the training data and recipe on HuggingFace.

        *   分两阶段进行：第一阶段（15T tokens）侧重数据多样性；第二阶段（5T tokens）侧重数据质量。
    3.  **后训练**:
        *   **监督微调 (SFT)**：使用精心构建的多领域数据（代码、数学、科学、对话、搜索、终端使用等）进行初步能力构建。
        *   **统一可验证奖励强化学习 (RLVR)**：在多个环境（终端、软件工程、搜索、对话、安全等）上使用 GRPO 算法进行统一强化学习，提升综合能力。
        *   **多教师在线策略蒸馏 (MOPD)**：这是后训练的核心创新。
            *   **算法**：训练超过 10 个领域专家教师模型（如 SWE 教师、搜索教师、代码教师等）。然后，让学生模型（RLVR 后的模型）在自己生成的轨迹上（on-policy），以密集的 token 级 KL 散度作为奖励信号，向不同领域的教师模型学习。
            *   **核心优势**：与稀疏的 RLVR 奖励相比，MOPD 提供了密集、细粒度的学习信号。
            *   **流程**：学生模型 -> RLVR -> MOPD 预热 -> MOPD 迭代（多轮，教师和学生可以相互进化和提升）。
        *   **MTP 增强 (MTP Boosting)**：通过修改训练方式，让 MTP 头适应推理时的输入分布，提升投机解码接受长度。

### 3. 实验设计：使用了哪些数据集 / 场景，它的 benchmark 是什么，对比了哪些方法

*   **数据集/场景**:
    *   **预训练数据**：网络爬虫（FineWeb, Crawl++）、代码（GitHub）、数学、学术文本、法律文本、多语言数据等。后训练数据包括大量合成数据，覆盖代码、数学、科学、软件工程、对话、终端使用等。
*   **Benchmark（评估基准）**:
    *   **智能体能力**: Terminal Bench 2.1, SWE-Bench Verified, SWE-Bench Multilingual, GDPVal, TauBench V3, BrowseComp, ProfBench, PinchBench, Vals.ai Financial Agent。
    *   **推理与知识**: MMLU-Pro, GPQA, Humanity's Last Exam (HLE), LiveCodeBench v6, IMO-AnswerBench, Apex-Shortlist, IOI 2025, SciCode, CritPt, OmniScience。
    *   **对话与指令遵循**: IFBench, Multi-Challenge。
    *   **长上下文**: RULER (1M), AA-LCR, LongBench v2。
    *   **多语言能力**: MMLU-ProX, WMT24++。
*   **对比方法**:
    *   DeepSeek V3.2, Mistral Large 3, Kimi K2, GLM-4.5/5.1, Qwen-3.5, DeepSeek V4 Pro/Flash, MiniMax-2.7 等当时最先进的开源模型。
*   **消融实验**: 论文进行了多个消融实验以验证各组件有效性：
    *   **NVFP4 预训练稳定性**: 从不同检查点切回 BF16 训练，验证低精度损失的差距很小。
    *   **MOPD 预热**: 对比有无预热步骤对最终性能的影响。
    *   **MOPD vs. 仅 RLVR**: 展示 MOPD 在多个智能体基准上的显著提升。
    *   **量化位宽选择 (BPE Sweep)**：通过实验选择最优的 5.03 BPE 量化点，平衡精度与效率。
    *   **FP4 量化算法**：对比不同权重缩放算法（Max, MSE, Four-Over-Six）。
    *   **MTP 增强**：对比前后接受长度。
    *   **RLVR 训练数据混合**：高斯分布方法用于课程学习。

    *   **训练硬件**：NVIDIA GB200 NVL72 集群（每节点含 Grace CPU 和 Blackwell GPU）。

### 6. 论文的主要结论与发现

1.  **高效性**：Nemotron 3 Ultra 在 8K 输入 / 64K 输出 token 的推理场景下，吞吐量是 GLM-5.1 的 5.9 倍，Kimi-K2.6 的 4.8 倍，Qwen-3.5 的 1.6 倍。
2.  **高精度**：在精度上，Nemotron 3 Ultra 在所有基准上与同代顶级开源模型并驾齐驱，甚至在某些智能体任务上表现更优。
3.  **MOPD 有效性**：多教师在线策略蒸馏是提升模型普遍智能体能力的有效方法，特别是在工具使用、环境交互和任务执行方面。MOPD 能充分利用多个领域专家模型的优势。
4.  **稳定性**：成功演示了在 550B 模型上进行 NVFP4 预训练的稳定性和有效性，这是该技术迄今为止最大规模的展示。
5.  **长上下文**：通过长上下文扩展，模型成功支持了 1M token 的上下文长度，并在 RULER 等长期依赖任务上表现优异。
6.  **后训练管线的优越性**：通过 SFT -> 统一 RLVR -> MOPD 的管线，集成后的模型在整体能力上表现出色，特别是在未见过的智能体任务上（PinchBench, ProfBench）具有良好的泛化能力。

### 7. 优点：方法或实验设计上的亮点

*   **方法上的亮点**：
    *   **架构创新**：结合了 Mamba 的高效和 Attention 的强表达力，并配合 LatentMoE，为长序列推理提供了高效的解决方案。
    *   **MOPD 算法**：创新性地将多教师知识蒸馏与在线策略RL（on-policy RL）结合，提供密集、细粒度的信息，避免了统一 RLVR 在混合环境中的信号稀释问题。
    *   **全面的后训练管线**：SFT -> RLVR -> MOPD 的流水线设计清晰，各阶段目标明确，能够系统性地提升模型能力。
*   **实验设计上的亮点**：
    *   **全面性与深度**：从预训练、后训练、量化到推理的全方位评估，且每个环节都包含消融实验。
    *   **“保留验证”**：将 PinchBench 和 ProfBench 作为最终测试集，有效避免了过拟合和评估污染，可信度高。
    *   **开放性与可复现性**：开源了模型、训练数据、训练配方和评估工具，极大促进了社区研究和发展。

### 8. 不足与局限

*   **实验公平性与客观性受限**：
    *   正如前文所述，主要评估框架（NeMo Evaluator）和推理框架（TRT-LLM vs vLLM）的选择可能引入系统性偏差。
    *   对比模型可能已使用相似数据进行训练，存在数据泄露风险，论文虽尽力防范但无法完全避免。
*   **训练不稳定性**：作者承认在预训练过程中遇到了两次严重的训练发散问题（Loss divergence），虽然通过回滚和调整学习率等解决了，但这暴露了大规模 MoE 混合架构训练的固有困难和不稳定性。
*   **MOPD 的局限性**：
    *   **对纯推理任务提升有限**：MOPD 在HLE（人类终极考试）等纯推理任务上的恢复率较低，作者认为这是因为该优势来自模型未曾见过的离策略数据，学生生成的轨迹不在教师的支持范围内。
    *   **复杂性和资源成本**：训练10+个单独的教师模型，以及复杂的异步蒸馏流程，对计算资源和工程实现提出了极高要求。
*   **资源估算不精确**：虽然提到了使用 GB200 NVL72 集群，但并未给出总 GPU 小时数和训练时间，使得成本难以直接量化。
*   **评估的局限性**:
    *   **智能体任务评估**：大多数智能体基准测试仍依赖于预设的模拟环境，与真实的、动态变化的环境仍有差距。
    *   **缺乏不同模型族比较**：未与如 Mixtral（同为 MoE）等其他流派模型进行直接比较。
