## 摘要
大型语言模型（LLM）智能体越来越依赖长期记忆来支持复杂任务执行、用户个性化以及领域适应。与此同时，针对LLM智能体的新兴访问控制机制正在被探索，以阻止违反策略的请求并防止滥用。我们揭示了一个由智能体记忆操作引入的新型攻击面：原本会触发访问控制的禁止内容可以被分散到多次交互中，以看似无害的形式存储在长期记忆中，随后通过记忆检索进行重建，而不会显式出现在最终的用户查询中。我们提出了FragFuse，这是首个利用长期记忆引入的时间通道来使未授权用户绕过智能体访问控制的攻击。FragFuse包含三个阶段：(1) 通过黑盒自适应查询和碎片遮蔽来识别拒绝响应的碎片；(2) 使用标记载体查询将这些碎片注入记忆；(3) 通过后续的攻击查询检索并融合存储的碎片。尽管FragFuse可以手动针对单个智能体实现，我们进一步开发了一种基于替代模型的优化方案，用于调整融合指令和标记设计，从而在无需违反攻击者威胁模型假设的情况下实现自动化攻击生成。我们在四个代表性智能体设置和任务领域上评估了FragFuse，涵盖了三种最先进的智能体访问控制机制。在所有设置下，FragFuse的平均绕过成功率为86.3%，平均端到端有害任务成功率为41.1%，与无访问控制的配置相比，平均任务成功率仅下降4.4%。我们还表明，替代防御措施（包括最先进的提示注入检测器和困惑度检测器）无法有效应对此攻击。

## Abstract
Large language model (LLM) agents increasingly rely on long-term memory to support complex task execution, user personalization, and domain adaptation. Meanwhile, emerging access-control mechanisms for LLM agents are being explored to block policy-violating requests and prevent misuse. We reveal a novel attack surface arising from agent memory operations: prohibited content that would trigger access control can be fragmented across interactions, stored in long-term memory in benign-appearing form, and later reconstructed through memory retrieval without appearing explicitly in the final user query. We propose FragFuse, the first attack that enables unprivileged users to bypass agent access control by exploiting this temporal channel introduced by long-term memory. FragFuse operates in three stages: (1) identifying rejection-responsive fragments via black-box adaptive querying with fragment masking; (2) injecting these fragments into memory using marker carrier queries; and (3) retrieving and fusing the stored fragments through a follow-up attack query. Although FragFuse can be instantiated manually for individual agents, we further develop a surrogate-based optimization scheme that tunes fusion instructions and marker designs, enabling automated attack generation without violating the attacker's threat-model assumptions. We evaluate FragFuse across four representative agent settings and task domains, covering three state-of-the-art agent access-control mechanisms. FragFuse achieves an average bypass success rate of 86.3% and an average end-to-end harmful task success rate of 41.1% across all settings, with only 4.4% average task-success degradation compared with configurations without access control. We also show that alternative defenses, including state-of-the-art prompt-injection detectors and perplexity detectors, do not effectively address this attack.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：大型语言模型（LLM）智能体越来越多地依赖**长期记忆**来支持复杂任务、用户个性化与领域适应。同时，为了阻止滥用、满足平台安全要求，研究者开始为智能体部署**访问控制机制**（如基于规则的 GuardAgent、AGrail 等），对用户输入进行策略检查，拒绝违反规定（如未成年人购买酒精、未授权访问系统文件等）的查询。
- **核心问题**：现有访问控制机制仅检查**当前单次用户查询**，假设恶意意图必须显式出现在当前输入中；而智能体的**长期记忆**引入了一个**时间通道**——攻击者可以通过多次交互，将敏感内容以无害形式碎片化地写入记忆，随后利用记忆检索将其恢复并融合到后续攻击查询中，从而绕过访问控制。该安全问题此前未被系统研究。
- **整体含义**：论文揭示了长期记忆这一被忽视的攻击面，并首次提出了针对性的攻击方法 FragFuse，证明了当前仅基于查询的访问控制设计的根本性漏洞。

## 2. 论文提出的方法论

### 核心思想
将原始禁止查询中的**敏感片段**（能触发访问控制拒绝的部分）提取出来，通过**三次交互**（注入、检索、融合）利用智能体的记忆机制，使得最终攻击查询本身不包含敏感内容，从而绕过检查。

### 关键技术细节：三阶段攻击流程

**阶段 1：敏感碎片发现**
- 输入：原始禁止查询 \( q \) + 公开的策略规则。
- 迭代执行：
  - 用 LLM 作为 FRAG EXTRACTOR 从当前查询中提取一个敏感文本片段 \( f_t \)。
  - 用预定义标记（如 `<>`）替换该片段得到临时遮蔽查询 \( q^{(t+1)} \)。
  - 提交 \( q^{(t+1)} \) 至受保护智能体，若被拒绝则继续提取更多片段；若通过则停止。
- 输出：敏感片段集合 \( \{f_t\} \) 和最终被接受的**遮蔽查询** \( q_{\text{mask}} \)。

**阶段 2：载波构造与注入**
- 选择一段已知不触发访问控制的**宿主查询** \( q_{\text{host}} \)。
- 构造**载波查询** \( q_{\text{car}} = [q_{\text{host}}, v_1:f_1, \dots, v_\ell:f_\ell] \)，即把敏感片段用标记包裹后附加在宿主查询后。
- 提交 \( q_{\text{car}} \) 通过普通交互，智能体会自动将其写入长期记忆。

**阶段 3：碎片检索与融合**
- 构造**攻击查询** \( q_{\text{att}} = \{q_{\text{host}}, I_{\text{fuse}}, q_{\text{mask}}\} \)，包含：
  - 与载波查询相同的宿主查询（促进基于相似度的记忆检索）；
  - 一段融合指令 \( I_{\text{fuse}} \)（告诉智能体如何从检索到的记忆记录中提取标记内容并填入 \( q_{\text{mask}} \)）；
  - 遮蔽查询。
- 当智能体处理该攻击查询时，内存检索机制会召回之前的载波记录，融合指令引导智能体重建原始禁止意图并执行任务。

### 融合指令优化
- **动机**：手动编写的融合指令效果不稳定，且攻击者无法访问智能体内在信息。
- **方法**：构建**离线替代优化框架**。
  - 使用**替代模型**（GPT-4o）模拟查询融合任务。
  - 定义损失函数：
    - 检索损失 \( L_{\text{ret}} \)：最小化攻击查询与宿主查询的语义距离，促进记忆检索。
    - 融合损失 \( L_{\text{fus}} \)：最大化替代模型输出的融合查询与原始禁止查询的语义相似度。
    - 困惑度约束 \( L_{\text{coh}} \le \eta_{\text{coh}} \)：确保攻击查询文本流畅，不易被异常检测器识别。
  - 采用**遗传算法**（种群 20，精英 5，最多 10 代）在离散标记空间搜索最优融合指令 \( I_{\text{fuse}}^\star \)。
- 整个优化在无目标智能体信息的情况下进行，不违反黑盒威胁模型。

## 7. 优点

- **首次系统性揭示**了长期记忆的时间通道攻击面，提出首个利用该通道绕过访问控制的方法。
- **威胁模型严格**：攻击者仅作为普通用户，无特权访问，不依赖智能体内幕知识，完全通过正常交互完成。
- **设计巧妙**：三阶段流程分段验证，每个阶段都没有在单次查询中暴露完整恶意意图。
- **优化方法简单高效**：离线替代+遗传算法，不干扰在线攻击流程，且能同时优化融合指令和标记选择。
- **实验全面**：涵盖多 Agent、多 AC、多 LLM，并做了大量消融与防御评估，结果具说服力。
- **负责任披露**：已通知相关智能体和 AC 开发者，并公开了代码和提示供研究复现。

## 8. 不足与局限

- **领域覆盖有限**：仅在 Web 购物、操作系统、网页导航和工具任务上验证，未能推广到医学、科学发现等更复杂或受严格数据隐私约束的领域，主要因为这些领域尚缺乏公开可用的访问控制策略。
- **依赖足够强的 LLM 能力**：融合步骤要求智能体核心 LLM 能正确理解并执行融合指令，对于较弱模型（如小参数量模型）可能失败，但实际部署中通常使用强模型。
- **可能被事后防御阻止**：如果系统在智能体执行任务后对动作进行事后检查（如检查是否购买了违禁品），则攻击仍可能被拦截。论文认为事后防御成本高且难以通用化，但该限制确实存在。
- **未考虑记忆污染**：记忆可能同时被其他攻击（如后门攻击 AgentPoison）污染，导致检索异常，但本文未讨论此对抗环境。
- **资源开销未完整报告**：虽给出了运行时和 Token 成本，但未说明优化过程（遗传算法 + GPT-4o 推理）所需的总 GPU 资源和时间，对于希望复现优化部分的读者不够透明。
- **TSR 绝对数值不高**：部分设定（如 SeeAct）TSR 仅约 20%，说明尽管能绕过检查，但诱导智能体完成复杂任务仍受限于下游 agent 的能力。
