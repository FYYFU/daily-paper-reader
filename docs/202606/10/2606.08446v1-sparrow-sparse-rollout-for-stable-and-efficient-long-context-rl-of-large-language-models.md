## 摘要
尽管基于可验证奖励的强化学习（RLVR）功能强大，但会导致极长的思维链（COT），计算成本高昂。由于RLVR每步成本主要由长上下文展开生成主导，稀疏注意力提供了一种加速密集展开的有前景的方法。然而，稀疏展开需要在稳定性与效率之间进行精细权衡：过于激进的稀疏性会导致崩溃，而过于宽松的稀疏性则加速不足。在这项工作中，我们通过稀疏到密集的演员-策略不匹配来研究这种权衡。我们首先观察到稀疏展开崩溃并非由所有token的均匀退化驱动：即使在激进稀疏下，大多数稀疏token与密集token完全对齐。受此启发，我们假设如果每个token的演员-策略不匹配的下尾在轨迹中始终保持在某个临界阈值以上，则稀疏展开训练保持稳定。我们引入了一种动态稀疏调度，在生成过程中保持该尾部统计量恒定，并验证了我们的假设。在Qwen3思维家族模型中，将尾部不匹配统计量保持在一致阈值附近通常可以实现稳定训练。然后，我们使用一个成本模型找到在该不匹配阈值下实现最大加速的稀疏调度，在训练Qwen3-1.7B、Qwen3-4B和Qwen3-8B时分别实现了2.2倍、2.4倍和2.0倍的展开加速。实验表明，该阈值可泛化到更大模型（Qwen3-14B）和另一个RL领域（编码）。最后，我们的分析自然地催生了DistillSparse：基于轻量级LoRA的稀疏展开蒸馏，使得更激进的稀疏性能够达到相同的稀疏到密集不匹配阈值，从而实现更高的加速。

## Abstract
Despite being powerful, reinforcement learning with verifiable rewards (RLVR) induces extremely long COT, making it computationally expensive. Since RLVR per-step cost is dominated by long-context rollout generation, sparse attention offers a promising way to accelerate dense rollout. However, sparse rollouts require a delicate stability-efficiency tradeoff: overly aggressive sparsity causes collapse, while overly lenient sparsity gives insufficient speedup. In this work, we study this tradeoff through sparse-to-dense actor-policy mismatch. We first observe that sparse rollout collapse is not driven by uniform degradation across tokens: most sparse tokens align perfectly with dense even under aggressive sparsity. Motivated by this, we hypothesize that sparse rollout training remains stable if the lower tail of per-token actor-policy mismatch stays above a critical threshold throughout the trajectory. We introduce a dynamic sparsity schedule that keeps this tail statistic constant during generation and validate our hypothesis. Across Qwen3 thinking-family models, keeping the tail mismatch statistic near a consistent threshold generally enables stable training. We then use a cost model to find the sparsity schedule for maximum speedup under this mismatch threshold, achieving 2.2x, 2.4x, and 2.0x rollout speedups when training Qwen3-1.7B, Qwen3-4B, and Qwen3-8B. Empirically, we show the thresholds generalize to a larger model (Qwen3-14B) and another RL domain (coding). Finally, our analysis naturally motivates DistillSparse: lightweight LoRA-based distillation on sparse rollout lets more aggressive sparsity reach the same sparse-to-dense mismatch threshold, yielding higher speedup.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：强化学习可验证奖励（RLVR）在数学推理和代码生成中表现优异，但其训练成本极高，其中长上下文（Long CoT）的 rollout 生成占据每步成本的 70% 以上。稀疏注意力虽能有效加速长序列生成，但将其直接用于标准密集策略的 RL 训练时会出现稳定性与效率之间的权衡：过于激进的稀疏性会导致策略分布严重不匹配，引发训练崩溃；而过于宽松的稀疏性则加速效果有限。
- **整体含义**：本文旨在解决如何最大限度利用稀疏注意力加速 rollout，同时保证密集策略 RL 训练稳定性。核心洞察是稀疏 rollout 的崩溃并非源于所有 token 的均匀退化，而是由少数 token 的严重分布偏移所致。因此关注下尾统计量（如 5% 分位数的 per-token 接受率）比平均值更能准确反映训练稳定性。

## 2. 方法论：核心思想、关键技术细节和算法流程

### 核心思想
- 定义演员-策略分布不匹配的度量：使用 **5-percentile 的 per-token 接受率**（下尾统计量）来衡量稀疏与密集策略的分布差异。
- **假设**：如果在整个 rollout 轨迹中保持该尾部统计量高于某个临界阈值，则稀疏 rollout 训练的密集策略可以保持稳定。

### 关键技术细节
- **动态稀疏调度（Dynamic Sparsity Scheduling）**：
    - 固定稀疏预算下，随着生成序列长度增加，per-token 不匹配自然恶化。
    - 通过动态调整稀疏预算（KV budget），随序列长度增加使稀疏更宽松，从而将尾部不匹配统计量在整个生成过程中保持恒定。
- **成本模型**：
    - 分别对密集注意力和块稀疏注意力建模，公式化计算成本和内存成本。
    - 基于 eFLOPs（H200 环境）分析，寻找在给定不匹配阈值下使成本最小的稀疏配置（pagesize 和 KV budget 的选择）。
- **DistillSparse**：
    - 在 RL 训练的同时，利用稀疏 rollout 生成的轨迹和密集教师网络的 log-probs，对稀疏学生网络进行轻量级 LoRA 蒸馏。
    - 蒸馏损失结合 PPO 目标与 KL 散度，使稀疏 rollout 与密集策略更对齐，从而允许更激进的稀疏度达到同样的不匹配阈值，获得更高加速。
    - 算法流程（Algorithm 1）：
        1. 构建密集策略和带 LoRA 的稀疏策略。
        2. 用稀疏策略采样 rollout，计算奖励/优势。
        3. 用密集模型重算 log probs。
        4. 用稀疏模型计算 log probs。
        5. 更新基础模型（PPO）。
        6. 冻结基础模型，更新 LoRA 参数（PPO + KL 蒸馏损失）。
        7. 刷新推理引擎权重。

### 关键观察
- 稀疏 rollout 生成的大部分 token 与密集策略完美对齐（接受率 >0.999），只有小部分 token 出现较大偏差。平均接受率对序列长度和稀疏度变化不敏感，而下尾统计量（5%-percentile）与序列长度强相关（Pearson’s r ≈ -0.55），能有效区分稳定与不稳定场景。

## 6. 主要结论与发现

1. **尾部统计量是关键**：稀疏 rollout 的稳定性取决于下尾的 per-token 接受率（5%-percentile），而不是平均值。当该值低于约 0.86 时，训练易崩溃。
2. **最优阈值跨模型一致**：对于 1.7B、4B、8B 模型，使训练稳定的最优尾部不匹配阈值基本一致（≈0.86），且该阈值可泛化到 14B 模型和代码生成训练。
3. **动态稀疏调度有效**：通过动态调整 KV budget，可以在整个生成中维持恒定尾部不匹配，从而实现稳定训练。
4. **成本模型指导最优配置**：在给定不匹配阈值下，page size 16 在块稀疏注意力中占帕累托前沿，最优加速比随模型增大而减小（因为注意力的相对比重下降，且大模型需要更大 KV budget 维持同一不匹配水平）。
5. **DistillSparse 进一步突破**：通过轻量 LoRA 蒸馏，可显著提升稀疏-密集对齐，使得原本不稳定的激进稀疏达到相同阈值，获得更高加速（例如从 2.2x 提升到 2.5x）。
6. **实际加速**：在 Qwen3-1.7B、4B、8B 上分别实现 2.2x、2.4x、2.0x 的 rollout 加速（端到端速度提升约 1.8-2.1x）；14B 模型获得 1.48x rollout 加速。

## 7. 优点

- **创新洞察**：明确指出稀疏 rollout 崩溃由少数 token 的极端分布偏移导致，而非整体退化，并创新性采用下尾统计量（5%-percentile acceptance rate）作为稳定性指标，比平均 KL 散度更有效。
- **系统方法论**：通过动态稀疏调度隔离尾部不匹配的影响，结合成本模型进行系统搜索，找到稳定阈值和最优效率配置，方法论清晰且可迁移。
- **充分验证泛化性**：阈值不仅在多个模型大小间一致，还成功迁移到更大模型（14B）和另一个任务领域（代码生成），证明了方法的鲁棒性。
- **DistillSparse 低代价提升**：利用已有计算（密集 log-probs）进行轻量 LoRA 蒸馏，额外开销很小（仅增加约 55% 的策略更新时间，但总体端到端仍获 1.72x 加速），且可自然融入训练流程。
- **硬件感知的成本模型**：基于 eFLOPs 和内存带宽建模，便于迁移到不同 GPU 架构。

## 8. 不足与局限

- **模型和任务覆盖有限**：实验仅使用 Qwen3 思考系列模型，在 LLaMA、DeepSeek 等其他流行模型上未验证。任务仅包括数学推理和代码生成，未涉及多模态、Agent 交互等更复杂场景。
- **稀疏注意力类型限制**：仅考虑块稀疏注意力（page size ≥ 16），未探索 fine-grained 稀疏（如 token 级或 streaming attention）。结论可能不直接适用于其他稀疏模式。
- **阈值精确性依赖特定设置**：0.86 的阈值是在特定模型家族和训练数据（Polaris）下发现的，当模型结构、训练算法（如 PPO vs GRPO）或奖励函数改变时，该数字可能需要重新确定。
- **成本模型假设**：成本模型假设 Top-k 核函数开销极小，且页面大小 ≥16 时主导因素不变。实际中，对于更小的 page size 或更复杂的硬件，可能需要更精细建模。
- **训练时长和资源**：作者仅进行了 one epoch 训练（部分原因是现实工业中常用单 epoch），未探讨多 epoch 或更长时间训练的稳定性。对于 14B 模型，近 8 天的训练成本较高，小实验室难以复现。
- **DistillSparse 的稳健性**：该方法需要超参数 λ 调节蒸馏和 RL 目标的平衡，在更大规模或不同任务上的调优成本未充分讨论。
