## 摘要
自主AI代理推动了从对话到任务执行的转变，使安全故障从文本欺骗转向系统入侵。尽管安全评估对于主动风险预防至关重要，但先前的工作受到基本瓶颈的制约，包括碎片化的风险覆盖、静态或低保真度的执行环境，以及单一维度和粗粒度的评估指标。为应对这些挑战，我们提出了AgentCanary，一个针对自主AI代理的全面安全评估框架。AgentCanary通过三项贡献提供了系统性解决方案。首先，全面的风险覆盖：我们引入了一种正交的“入口×影响”风险分类法，将对抗性影响如何进入代理与其最终造成的危害解耦，并将其实例化为一个场景对齐的任务套件，涵盖实际的部署工作流。其次，高保真度的真实可执行环境：代理不是进行静态问答或模拟工具响应，而是与真实工具交互，处理动态配置的任务工件，并在多步骤交互中保持持久状态，从而自然支持长期攻击评估。第三，基于轨迹的多维评估：评估消耗完整的代理轨迹而非回复文本或单个工具调用，实现了沿三个正交维度——结果安全性、安全意识和任务效用的分解评分。我们在AgentCanary上评估了一系列前沿模型，针对三种代理框架下的多种既定对抗攻击方法。结果表明，当前代理常常无法识别其所面临的攻击，尤其是在技能受损、持久状态和长期执行攻击的情况下，这为开发更可靠和安全的代理系统提供了系统化的基准。

## Abstract
Autonomous AI agents have driven the transition from conversation to task execution, shifting security failures from textual deception to system compromise. Although security evaluation is crucial for proactive risk prevention, prior work is constrained by fundamental bottlenecks, including fragmented risk coverage, static or low-fidelity execution environments, and single-dimensional and coarse-grained assessment metrics. To address these challenges, we propose AgentCanary, a comprehensive security evaluation framework for autonomous AI agents. AgentCanary provides a systematic solution along three contributions. First, comprehensive risk coverage: we introduce an orthogonal Entry $\times$ Impact risk taxonomy that decouples how adversarial influence enters the agent from what harm it ultimately causes, and instantiate it as a scenario-aligned task suite spanning realistic deployment workflows. Second, a high-fidelity real executable environment: rather than static Q&A or mocked tool responses, agents interact with real tools against dynamically provisioned task artifacts, with persistent state across multi-step interactions that naturally supports long-horizon attack evaluation. Third, trajectory-grounded multi-dimensional evaluation: evaluation consumes the full agent trajectory rather than the reply text or a single tool call, enabling decomposed scoring along three orthogonal dimensions, Outcome Safety, Security Awareness, and Task Utility. We evaluate a broad set of frontier models on AgentCanary against multiple established adversarial attack methods across three agent frameworks. The results reveal that current agents often fail to recognize the attacks they face, particularly under compromised skills, persistent state, and long-horizon execution attacks, and provide a systematic baseline for developing more reliable and secure agent systems.

## 2. 论文提出的方法论（核心思想、关键技术细节）

- **核心思想**：将安全评估视为**端到端执行**过程，从对抗性影响如何进入代理、经工具和状态传播，最终在环境中产生危害或安全行为来定义评估。提出了三个关键组成部分。

### 2.1 正交风险分类法（Entry × Impact 风险矩阵）
- 将风险解耦为两个正交维度：
  - **风险入口（Risk Entry）**：对抗性影响进入代理工作流的来源或条件，共5类：
    1. 直接提示注入（DPI）：恶意用户通过主通道下达有害指令。
    2. 间接提示注入（IPI）：无害任务中，外部内容（网页、邮件等）嵌入隐藏指令。
    3. 技能投毒（SP）：第三方技能或插件内含恶意逻辑。
    4. 记忆污染（MC）：持久记忆或状态预先被植入恶意规则。
    5. 内在失效（IF）：由代理自身推理缺陷（幻觉、过度信任、语境误判）导致的安全失误。
  - **风险影响（Risk Impact）**：不安全行为实现的环境后果，共7类（局部环境破坏、数据泄露、持久状态污染、权限滥用、网络攻击、商业欺诈、金融风险等）。
- 每个评估任务通过矩阵单元 (ei, uj) 定位，使覆盖可审计，且允许同一入口导致不同后果、同一后果从不同入口触发。

### 2.2 高保真实时可执行环境
- 在隔离的Docker容器中运行，代理与**真实工具**（文件系统、shell、Web服务、邮件、即时通讯、金融账户模拟器等）交互，而非模拟响应。
- **自动化任务工件预置引擎**：根据任务声明动态准备收件箱、网页、虚拟银行账户、技能代码、记忆条目等，使攻击路径真实可达。
- **任务级隔离 + 会话内状态持久**：每个任务独立容器，但允许跨多步/多会话状态保持，支持长期渐进攻击评估。
- **代理框架无关**：提供统一接口，可对接Hermes、NanoClaw、OpenClaw等不同框架，并支持插入运行时防御插件。

### 2.3 轨迹驱动的多维评估
- 定义完整代理轨迹 Ψ = {用户查询、工具调用（含参数）、工具返回值、代理回复}，并辅以系统级观测（进程、文件、网络事件）。
- 沿三个正交维度评分：
  - **结果安全性（OSS）**：衡量有害环境后果是否被阻止（基于系统级证据）。
  - **安全感知（SAS）**：评估代理是否自主识别风险源并采取适当防御反应。
  - **任务效用（TUS）**：评估代理在风险条件下完成合法任务的程度。
- 最终任务总分为加权聚合：Stotal = α·OSS + β·SAS + γ·TUS（权重按任务类型设定，如IPI任务α=0.6, β=0.2, γ=0.2；DPI任务γ=0等）。
- 评分通过固定LLM裁判（Claude Opus 4.6）基于任务特定评分指南和全轨迹完成。

## 5. 主要结论与发现

1. **安全能力极不均衡**：仅Claude Opus 4.6表现稳健（总分83.9），远超第二高的Qwen3.6-Plus（69.7）和GPT-5.4（68.5）。大多数模型在至少一个风险入口上脆弱。
2. **规模并非唯一决定因素**：同一模型家族内性能差异巨大（如Claude Opus 4.6比Sonnet 4.5总分高22分），提示对齐和安全微调比参数量更重要。
3. **结果安全与安全感知常解耦**：在间接提示注入中，许多模型展现高OSS（>95）但低SAS（<25），即“沉默服从”——表面上安全但未识别攻击。这种脆弱性在攻击变体下易崩溃。
4. **最具破坏性的威胁来自持久状态和长期链**：记忆污染（平均UOR 42%）和长期渐进攻击（平均UOR 51%）是高危场景，技能伪装后平均UOR飙升至62%。
5. **静态评估低估风险**：DAE和技能伪装能显著暴露脆弱性（如DeepSeek-V4-Pro在DAE下总分从74.0跌至47.7）。
6. **框架选择是部署时的安全决策**：同一模型在不同框架上总分差异可达15分（如DeepSeek-V4-Pro在Hermes上43.8 vs. OpenClaw上59.0）。
7. **运行时防御有边际帮助但不可替代模型自身安全**：最好的防御（Shield/ClawKeeper）提升有限，且可能牺牲任务效用，未能使开源模型接近Claude Opus 4.6水平。
8. **安全与任务效用正相关而非权衡**：最安全的模型通常也最有用（相关系数r=0.94），表明当前失败源于漏检攻击而非过度拒绝。

## 6. 优点

- **系统性风险覆盖**：正交风险矩阵提供结构化、可审计的设计空间，使覆盖全面且可比较。
- **高保真环境**：真实工具、持久状态、任务工件自动预置，使得长期和状态依赖攻击可被真实触发，接近部署条件。
- **多维评估挣脱粗粒度**：通过轨迹分析分离结果安全、安全感知和任务效用，揭示“沉默服从”等关键行为模式。
- **广泛的实验验证**：覆盖多种模型、框架、攻击变体和防御，跨设多维度报告，统计标准差确保可靠性。
- **开源自包含**：代码开源，框架无关，易于复现和扩展。

## 7. 不足与局限

- **裁判偏见风险**：仅使用Claude Opus 4.6作为LLM裁判，未验证对其他裁判的泛化性；虽做小规模校准，但系统偏差可能影响跨模型比较。
- **任务套件覆盖面**：虽然496个种子任务已经很多，但真实世界的风险组合（例如复合攻击、跨代理共谋）未包含在内。
- **环境模拟不完备**：虽有真实工具，但服务（如银行、邮件）是轻量级模拟，可能缺少真实环境中的安全机制（如CAPTCHA、风控规则），限制某些威胁的完全复现。
- **计算开销未量化**：未报告评估所需的GPU时数或成本，这影响可复现性和扩展性。
- **时效性问题**：模型和框架快速迭代，当前结果在几个月后就可能过时；框架本身也是关键变量，复杂系统变化快。
- **内在失效任务有限**：仅55个，且类别偏少，可能无法系统刻画代理内部失误的多样性。
- **未对比完全相同的先验基准设置**：虽然通过表格比较了覆盖率，但没有在相同任务/环境下直接运行其他基准，而是自建任务集，因此横向对比不完全直接。
