## 摘要
我们提出了AgentJet，一个用于大型语言模型（LLM）智能体强化学习的分布式群体训练框架。与将智能体推出与模型优化紧密耦合的集中式框架不同，AgentJet采用解耦的多节点架构，其中群体服务器节点托管可训练模型并在GPU集群上运行优化，而群体客户端节点可在任意设备上执行任意智能体。这种设计提供了集中式框架难以支持的能力：（1）异构多模型强化学习，能够训练以多个LLM为大脑的异构多智能体团队；（2）具有隔离智能体运行时的多任务鸡尾酒训练；（3）容错执行，防止外部环境故障中断训练过程；（4）实时代码迭代，允许通过替换群体客户端节点在训练期间编辑智能体。为了支持多模型、多轮和多智能体设置中的高效RL，AgentJet引入了一个带有时间线合并的上下文跟踪模块，该模块整合冗余上下文并实现1.5-10倍的训练加速。最后，AgentJet引入了一个自动化研究系统，该系统以研究主题为输入，并在大规模集群上自主进行长时间、多天的RL研究。通过利用群体架构，该系统在没有人工干预的情况下再现了RL研究人员的关键探索工作流程。

## Abstract
We present AgentJet, a distributed swarm training framework for large language model (LLM) agent reinforcement learning. Unlike centralized frameworks that tightly couple agent rollouts with model optimization, AgentJet adopts a decoupled multi-node architecture in which swarm server nodes host trainable models and run optimization on GPU clusters, whereas swarm client nodes execute arbitrary agents on arbitrary devices. This design provides capabilities that are difficult to support in centralized frameworks: (1) heterogeneous multi-model reinforcement learning, enabling the training of heterogeneous multi-agent teams with multiple LLM as brains; (2) multi-task cocktail training with isolated agent runtimes; (3) fault-tolerant execution that prevents external environment failures from interrupting the training process; and (4) live code iteration, which allows agents to be edited during training by replacing swarm client nodes. To support efficient RL in multi-model, multi-turn, and multi-agent settings, AgentJet introduces a context tracking module with timeline merging, which consolidates redundant context and achieves a 1.5-10x training speedup. Finally, AgentJet introduces an automated research system that takes a research topic as input and autonomously conducts long-horizon, multi-day RL studies on large-scale clusters. By leveraging the swarm architecture, this system reproduces key exploratory workflows of RL researchers without human intervention during execution.

- **整体含义**：需要一种新的分布式架构，将模型优化与 agent 执行完全解耦，提供灵活、可扩展、容错、支持异构模型的训练框架。AgentJet 旨在解决这些挑战，并为 agentic RL 提供基础设施。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：采用群体（swarm）架构，包含两类节点：
  - **Swarm Server 节点**（优化节点）：运行在 GPU 集群上，负责模型权重更新、vLLM/SGLang 推理 API、上下文跟踪和样本池管理。每个 server 托管并训练一个模型。
  - **Swarm Client 节点**（采样节点）：轻量级 CPU 进程，可在任意设备上运行，执行任意 agent 工作流、读取数据、计算奖励。client 与 server 通过 OpenAI 兼容的 API 交互。
  - 这种解耦实现了：异构多模型 RL、混合任务鸡尾酒训练、容错执行、热替换调试、去中心化训练。
- **关键技术细节**：
  1. **上下文跟踪与时间线合并（Timeline Merging）**：
     - 每个 agent episode 中，server 透明地记录所有 LLM 调用的上下文，形成多个时间线（timelines）。
     - 时间线 Ti 由消息块组成，包含文本、token ID、log-prob 和 loss mask（仅 LLM 生成的 token 设为 1）。
     - 合并算法（Algorithm 1）从后向前遍历，若短时间线是长时间线的前缀（基于 token 或 text 级匹配），则将短时间线合并到长时间线中，保留 LLM 生成 token 的标签和 log-prob。这消除了冗余上下文，训练速度提升 1.5-10 倍。
     - 提供两种匹配策略：token 级（精确对齐）和 text 级（容忍 tokenizer 漂移，更激进）。
  2. **Episode 批处理策略（Section 3.3）**：支持五种触发策略（C1-C5）：
     - C1：基于累积 episode 数。
     - C2：基于完成的任务数（达到重复次数）。
     - C3：基于非“空任务”数（奖励方差非零）。
     - C4：所有活动 client 同意。
     - C5：任何 client 同意。
     - 这允许动态采样控制（如 DAPO 的零优势样本过滤）。
  3. **Swarm RL 工作流**：server 初始化 → client 连接并注入配置 → agent 执行 episode（通过 per-episode API key 跟踪上下文）→ 收集奖励 → 触发更新（从样本池提取，合并时间线，计算策略梯度）。
- **算法/流程**：Algorithm 2 描述了自动研究 A3R 流程，包括 leader agent 规划多阶段实验、生成实验蓝图、worker agent 并行执行、结果分析和自适应循环。

## 6. 论文的主要结论与发现

- **解耦架构有效**：群体架构解决了 agentic RL 的多个核心痛点：容错、异构多模型、混合任务、REPL 式调试、去中心化训练。
- **时间线合并显著加速**：在 AppWorld 上获得 6.25x 的 actor-update 加速，且不降低奖励曲线（图8）。
- **共享参数多 agent 训练成功**：狼人杀中训练狼人 faction 从 23% 提升至 64.7%（14B 模型vs 235B 对手）。
- **非共享参数带来多样性收益**：三个独立训练狼人模型获得更高胜率（66.5% vs 64.7%），因为行为多样性使对手更难识别。
- **多任务鸡尾酒训练可行**：Qwen3-8B 同时在 AppWorld 和 AIME 上训练，奖励曲线接近单任务专用模型，但混合训练产生通用模型且节省资源。不过 AppWorld 上存在一定性能损失（约 10 分）。
- **框架无关性验证**：四种 agent loop 框架在相同设置下训练曲线几乎一致（最终差异 <0.04）。
- **自动化研究（A3R）能力**：能够自主设计、执行和总结多天实验，发现最小稳定 batch size 和响应长度之间的交互（建议 bs=4 或 16，mr=12000 最佳）。

## 7. 优点
