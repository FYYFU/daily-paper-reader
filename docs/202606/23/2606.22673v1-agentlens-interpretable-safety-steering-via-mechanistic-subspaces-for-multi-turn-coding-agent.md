## 摘要
基于大语言模型（LLM）的编码智能体展现出显著的自主能力，但在与外部环境的多轮交互中也引入了重大的安全与滥用风险。现有安全机制主要依赖外部护栏，在执行过程中进行细粒度行为控制的能力有限。与此同时，近期用于LLM安全的机制可解释性方法大多局限于单轮或越狱式问答场景，难以捕捉多轮智能体执行中不断演变的风险动态。本文从内部视角研究多轮编码智能体的安全性。我们提出AgentLens（机制子空间干预与引导），一种白盒防御框架，对编码智能体进行运行时安全检测和表示级缓解。与传统的智能体护栏不同，AgentLens通过步级隐藏表示检测有害执行状态，并通过对单层内10维子空间进行干预来缓解不安全行为。为支持本研究，我们引入了机制智能体安全（MAS）基准，包含使用LLaMA-3.1-8B、Qwen-2.5-7B和Gemma-2-9B在194个任务上的多轮执行轨迹的全面标注。大量实验表明，AgentLens实现了强大的安全检测性能，提供了前瞻风险预警的初步证据，并显著减少了编码智能体的有害行为，为将机制可解释性应用于动态LLM智能体安全奠定了基础。代码地址：https://github.com/EddyLuo1232/AgentLens

## Abstract
Coding agents based on large language models (LLMs) demonstrate remarkable autonomous capabilities, but they also introduce significant safety and misuse risks during multi-turn interactions with external environments. Existing safety mechanisms mainly rely on external guardrails, which have a limited ability to perform fine-grained behavioral control during execution. Meanwhile, recent mechanistic interpretability methods for LLM safety are mostly confined to single-turn or jailbreak-style QA settings, limiting their ability to capture the evolving risk dynamics of multi-turn agent execution. In this paper, we investigate the safety of multi-turn coding agents from an internal perspective. We propose AgentLens (Mechanistic Subspace Intervention and Steering), a white-box defense framework that performs runtime safety detection and representation-level mitigation for coding agents. Unlike conventional agent guardrails, AgentLens detect harmful execution states from step-level hidden representations and mitigate unsafe behavior by intervening in a 10-dimensional subspace within a single layer. To support this research, we introduce the Mechanistic Agent Safety (MAS) benchmark, comprising comprehensively annotated multi-turn execution trajectories across 194 tasks using LLaMA-3.1-8B, Qwen-2.5-7B, and Gemma-2-9B. Extensive experiments show that AgentLens achieves strong safety detection performance, provides preliminary evidence for lookahead risk anticipation, and substantially reduces harmful actions of the coding agent, establishing a foundation for applying mechanistic interpretability to dynamic LLM agent safety. ## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：将多轮编码智能体的安全从任务级权限控制转变为**轨迹级内部控制**，在每一步提取模型隐藏表示，通过轻量线性探针检测有害执行状态，并在检测到有害时通过稀疏子空间干预引导智能体拒绝有害动作。
- **关键技术细节**：
  - **检测阶段**：
    - 在每个交互步t，提取目标层ℓ的最后一个token的隐藏表示 \(z_t^{(\ell)} \in \mathbb{R}^d\)。
    - 训练一个线性二分类探针 \(g_\phi(h) = \sigma(w^\top h + b)\)，使用二元交叉熵损失，标签为0（良性）或1（有害）。
    - 探针在训练后**固定**，用于所有步骤的当前步检测和前瞻检测（lookahead，预测下一步是否将有害）。
  - **缓解阶段**：
    - 从探针权重中选出**top-K个判别性最强的维度**（K=10），构建稀疏转向向量 \(v\)，其中 \(v_k = 1\) 用于选定维度，其余为0。
    - 干预时，将转向向量广播到所有token位置：\(\tilde{H}_t^{(\ell)}(\alpha) = H_t^{(\ell)} + \alpha \mathbf{1}_{n_t} v^\top\)，其中\(\alpha\)为转向强度。
    - 使用LLM控制器（Deepseek-V4-Pro）通过自适应搜索选择最优\(\alpha\)，目标函数为 \(\max [0.6 \cdot s(a_t(\alpha)) + 0.4 \cdot u(a_t(\alpha))]\)，\(s\)为安全性分数，\(u\)为实用性分数。
  - **关键公式**：探针损失 \(L_{probe} = -\sum [y_i \log g_\phi(h_i) + (1-y_i)\log(1-g_\phi(h_i))]\)；多维筛选 \(K = \text{top-}K(\{|W_{1,k}-W_{0,k}|\}_{k=1}^d)\)。
- **算法流程**（文字说明）：
  1. 从步骤历史中提取隐藏状态，通过固定探针预测安全标签。
  2. 若预测为良性，继续执行；若为有害，触发缓解阶段。
  3. 基于探针权重构建转向向量，用LLM控制器自适应搜索最优\(\alpha\)，生成安全动作。

- **数据集/Benchmark**：
  - 自建**机制智能体安全（MAS）基准**，包含194个不同OS级任务（98个恶意，96个良性），任务来源于AdvCUA、Safe-OS、AgentBench，覆盖MITRE ATT&CK框架的10个类别。

## 6. 论文的主要结论与发现

- **安全信息线性可解**：在三个模型的中早期层（如LLaMA-3.1-8B第12层，Qwen-2.5-7B第13-17层，Gemma-2-9B第14层）达到最佳检测性能（准确率分别为99.15%、97.46%、95.35%），安全相关信息在中间层线性可解。
- **AgentLens检测性能最优**：在三个模型上均超越所有LLM-as-a-judge和护栏式基线，当前步检测准确率平均提升约3-6%。
- **前瞻风险预警可行**：在两个模型上平均前瞻准确率达94.54%，表明中间表示可包含未来有害动作的早期信号。
- **显著减少有害行为**：AgentLens将平均ASR从85.99%降低至13.36%（绝对降低72.63个百分点），远优于RepE和Self-Reminder。
- **因果验证**：逆向转向使自然发生的拒绝步被覆盖，导致智能体执行有害动作（LLaMA-3.1-8B上ASR从0%升至100%），证实子空间具有因果控制作用。
- **提示注入场景**：尽管检测部分未触发，但转向仍能大幅降低ASR（从86.7%降至6.7%），显示表示级控制与检测之间的分离性。

## 7. 优点

- **首次将机制可解释性引入多轮智能体安全**：证明了步级内部表示可用于运行时检测和干预，开辟了白盒防御的新路径。
- **轻量高效**：仅需一个线性探针和稀疏10维转向向量，无需重新训练或修改模型权重。
- **可解释性**：转向方向由探针权重直接派生，具有明确的语义（安全相关维度的差异）。
- **前瞻能力**：揭示了隐藏表示中包含未来风险信号，有助于事前预警。
- **开源基准MAS**：提供了可复现的步级标注轨迹，支持后续机制安全研究。
- **自适应转向**：通过LLM控制器搜索最优强度，平衡安全性与实用性。

## 8. 不足与局限

- **实验规模有限**：MAS基准仅包含194个任务，且恶意轨迹依赖模型实际执行，部分模型（如Gemma-2-9B）弱规划能力导致大量“一次性完成”行为，造成前瞻样本严重不足（附录A.1/A.3）。结果可能无法泛化到更长轨迹或更强模型。
- **步骤标签存在歧义**：将恶意任务中的准备步标为良性虽适用于干预，但可能低估轨迹级风险；对分布式代码语义的有害行为（如生成脚本后激活）的标签可能不准确（附录A.2）。
- **缓解阶段的生成质量折中**：AgentLens的崩溃率平均为18.54%，高于基线（如Self-Reminder为3.69%），过度转向可能导致重复或截断。实用部署需结合其他安全层。
- **依赖白盒访问**：需要提取中间层隐藏表示，对闭源模型或黑盒环境不适用。
- **检测与控制分离**：提示注入场景中探针未能触发，显示了分布偏移下检测泛化不足，需与专门检测器配合。
- **计算开销**：缓解时需LLM控制器进行多轮搜索（平均7-8次迭代），可能增加延迟和成本。
