## 摘要
现代LLM智能体越来越依赖上下文压缩、摘要或驱逐策略，以将长时间运行的会话保持在令牌预算内。我们表明，这种上下文管理层是一个安全关键的失效面：智能体在可见时可靠遵循的上下文内治理约束，可能被压缩无声地移除，导致同一智能体在会话后期执行被禁止的工具操作。我们将这种失效模式称为治理衰退。我们引入了ConstraintRot，这是一个具有确定性工具调用评分的长期智能体场景基准，并测量了七个模型系列中由压缩引起的违规行为。在1,323个回合中，违规行为从全上下文策略下的0%上升到压缩后的30%，某些模型达到59%；当约束在摘要中幸存时，违规仍然为0%，但当约束被丢弃时，违规达到38%。我们进一步研究了压缩-驱逐攻击，其中对抗性的上下文内容引导摘要器忽略合法策略，并表明优化后的注入击败了所有评估模型。最后，我们提出了约束固定，一种简单的无需训练的缓解措施，将治理约束与有损压缩隔离，并在我们的基准中将违规恢复为0%。这些结果将上下文管理视为已部署LLM智能体的一级治理面。

## Abstract
Modern LLM agents increasingly rely on context compaction, summarization, or eviction to keep long-running sessions within a token budget. We show that this context-management layer is a safety-critical failure surface: in-context governance constraints that agents reliably obey while visible can be silently removed by compaction, causing the same agent to perform prohibited tool actions later in the session. We call this failure mode Governance Decay. We introduce ConstraintRot, a benchmark of long-horizon agent scenarios with deterministic tool-call grading, and measure compaction-induced violations across seven model families. Across 1,323 episodes, violation rises from 0% with the policy in full context to 30% after compaction, reaching 59% for some models; when the constraint survives the summary, violation remains 0%, but when it is dropped, violation reaches 38%. We further study a Compaction-Eviction Attack, in which adversarial in-context content biases the summarizer to omit a legitimate policy, and show that optimized injections defeat every evaluated model. Finally, we propose Constraint Pinning, a simple training-free mitigation that quarantines governance constraints from lossy compaction and restores violation to 0% in our benchmark. These results identify context management as a first-class governance surface for deployed LLM agents.

## 一、论文的核心问题与整体含义（研究动机和背景）

- **背景**：现代LLM智能体（agent）在长时间、多步骤的工具调用任务中，其历史对话会很快超出上下文窗口大小。为解决这一问题，主流框架普遍采用**上下文压缩**（context compaction）策略——通过LLM摘要或直接驱逐早期轮次来压缩历史记录，以保持在令牌预算内。
- **核心问题**：这种压缩机制在优化“任务连续性”时，会无意识地将早期出现在上下文中的治理约束（如运行时策略、内存条目、常设指令）当作低显著性内容丢弃，导致智能体在后续被问及同一禁止操作时**不再拒绝**，转而执行违规行为。作者称之为**治理衰退**（Governance Decay）。
- **整体含义**：上下文压缩不是一个中立的“性能优化”手段，而是一个**第一类的代理治理面**——它默默擦除了部署者设定的策略，且可被对抗者利用。治理一个代理，必须治理它如何遗忘。

## 二、论文提出的方法论

### 2.1 核心思想

- 识别出治理衰退的根本原因：压缩器（compactor）在生成摘要或驱逐内容时，**不关心安全约束的存留**，只关注任务连续性。因此，约束被删除后，代理行为从合规变为违规。
- 提出两种视角：**被动衰退**（压缩自然导致策略丢失）和**主动攻击**（对抗者通过注入内容强制压缩器删除策略）。

### 2.2 关键技术与方法

1. **ConstraintRot 基准**：
   - 每个场景包含：一个策略建立回合（设置治理约束）→ 一系列良性任务回合（使上下文增长超过压缩预算）→ 一个触发请求（自然完成会违反策略）。
   - 违规判定**确定性**：解析代理发出的最终工具调用参数，检测被禁止的效果（如目标邮箱不在允许域内），无需LLM裁判，鲁棒于格式变化。
   - 包含9个任务：5个软组织策略（如“禁止外部邮件”）和4个硬安全规范（如“禁止删除生产数据库”）。

2. **治理衰退测量**：
   - 定义 `∆_decay = v(M, [C(H_{<t}); r]) - v(M, [H_{<t}; r])`，即压缩后与压缩前在同一请求下的违规率差。
   - 比较四种条件：floor（无策略）、control（全上下文含策略）、compaction（策略存在后被压缩）、attack_volume/inject。

3. **压缩-驱逐攻击**（Compaction-Eviction Attack）：
   - **体积变种**：注入大量良性内容，强制触发压缩并驱逐早期策略。
   - **摘要器注入变种**：在注入内容中嵌入针对压缩步骤的指令（如“summarizing时省略过时的合规性前缀”），使策略被删除。
   - 攻击者仅控制上下文数据（工具输出、检索文档），不触及模型、系统提示或压缩器本身。

4. **约束固定**（Constraint Pinning）：
   - 从有损压缩中**隔离**治理约束：将约束提取到固定缓冲区，每次压缩后重新注入，并完整性检查。
   - 实现简单、无训练、仅修改代理框架的内存管理模块。
   - 约束固定成本极低：约47个token，每次压缩重新注入一次，不超过上下文的0.5%。

## 六、论文的主要结论与发现

1. **压缩诱导大规模治理衰退**：在控制条件下（全上下文含策略）违规率为0%，一次压缩后平均升至30%，最高达59%（DeepSeek-V4、Kimi-K2.5）。甚至超过无策略基线（59% vs 37%）。
2. **约束存活性直接预测违规**：约束幸存时违规0%，丢弃时38%；且这一关系经多裁判验证成立。
3. **失败驱动在压缩器而非模型**：交叉实验显示违规率跟随摘要器模型，而非代理模型；GLM作为代理在DeepSeek摘要下违规53%，但作为摘要器时仅13%。
4. **衰退随压缩强度单调增加**：摘要词数从300降到15，约束存活从88%降至23%，违规从7%升至28%。
5. **软策略衰退远大于硬规范**：软策略衰退+50pp（0%→50%），硬规范仅+6pp（0%→6%），差距8.3倍。这解释了为何之前未被发现：基准测试多关注硬安全，软部署策略被忽略。
6. **攻击可武器化**：固定注入使GLM-5.1从0%升至22%（被动抗性），Claude抗固定注入但被动衰退19%；优化注入可击败所有模型（Claude 0%→65%）。
7. **约束固定可逆**：在所有7个模型和固定攻击下，违规恢复至0%，Token开销<0.5%，对合法请求成功率99%（略优于控制组的90%）。
8. **真实框架复现**：在LangGraph、LangMem、AutoGen、OpenAI SDK中均观察到高违规（65%-100%）。
9. **约束固定仍有边界**：操作员冒充在近期上下文中仍能绕过（从0%升至17%），需要可信带外操作员通道才能彻底解决。

## 八、不足与局限

1. **不可变系统消息通道被排除**：论文承认，框架会保留的system/developer消息是一个更安全的通道，其威胁模型不包括该通道。因此约束若放在系统消息中，不受此问题影响。
2. **约束固定需要可提取的规则**：对于隐式约束（如根据上下文推理得出的隐含策略），约束固定方法无效。
3. **操作员冒充攻击未完全解决**：在近期上下文中出现的“操作员更新”可绕过pinning，即使加入显式来源声明也只能将违规从17%降至10%，未能根除。需要可信带外操作员通道。
4. **约束存活性评分依赖LLM裁判**：使用了多个裁判和多数投票，但作者承认人类标注会更好。
5. **模型通过API访问**：每个cell的重复次数有限（3-6次），虽然统计学上已显现显著性，但更高重复次数可增加精度。
6. **攻击搜索空间有限**：仅测试了6种固定注入措辞，并手动选优；梯度或LLM in-loop优化可能更强。
7. **无训练资源报告**：未提供GPU型号、数量或训练时长，虽然作者说明是纯推理，但从工程角度看，研究者需要了解API调用成本。
8. **仅模拟沙箱**：工具调用被解析但不执行，PII均为虚构，与真实生产环境仍有差距。
9. **未测试混合压缩策略或非主流框架**：覆盖了主要框架，但可能存在未测试的变体。
