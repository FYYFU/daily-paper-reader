## 摘要
在线策略自蒸馏（OPSD）在大语言模型（LLMs）的后训练中已被证明有效，但其在扩散LLMs（dLLMs）中的应用仍未探索。现有OPSD方法本质上是自回归中心化的，它们通过从左到右的前缀条件化结合词元级差异监督来注入特权信息，这种设计与dLLMs的任意顺序生成存在根本冲突。我们提出d-OPSD，首个针对dLLMs定制的OPSD框架。我们的方法包含两个核心贡献：首先，我们重新构建自教师模型，通过使用自生成答案作为后缀条件化，使学生模型能够从“自我未来经验”而非特权前缀中学习；其次，我们将监督从词元级转移到步骤级，使训练与dLLMs的迭代去噪过程对齐。在四个推理基准上的实验表明，d-OPSD在样本效率上持续优于RLVR和SFT基线，仅需RLVR约10%的优化步骤，为dLLM后训练开辟了一条有前景的路径。代码开源在https://github.com/xingzhejun/d-OPSD。

## Abstract
On-policy self-distillation (OPSD) has proven effective for post-training large language models (LLMs), yet its application to diffusion LLMs (dLLMs) remains unexplored. Existing OPSD methods are inherently autoregressive-centric. They inject privileged information via left-to-right prefix conditioning with token-level divergence supervision, a design that fundamentally conflicts with the arbitraryorder generation of dLLMs. We introduce d-OPSD, the first OPSD framework tailored for dLLMs. Our approach makes two core contributions. First, we reframe self-teacher construction by using self-generated answers as suffix conditioning, enabling the student model to learn from "self future-experience" rather than privileged prefixes. Second, we shift supervision from token-level to step-level, aligning training with the iterative denoising process of dLLMs. Experiments across four reasoning benchmarks show that d-OPSD consistently outperforms RLVR and SFT baselines with superior sample efficiency, requiring only around 10% of the optimization steps by RLVR and opening a promising pathway for dLLM posttraining. The code is available at https://github.com/xingzhejun/d-OPSD.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：在线策略自蒸馏（On-policy Self-distillation, OPSD）在自回归（AR）大语言模型后训练中展现出优异性能，能提供密集的 token 级监督，克服稀疏奖励瓶颈并避免暴露偏差（exposure bias）。然而，现有 OPSD 方法天然面向 AR 模型，依赖从左到右的前缀条件化（prefix conditioning）和 token 级 KL 散度监督。
- **核心问题**：扩散语言模型（dLLM）采用任意顺序的迭代去噪生成方式，与 AR 模型的 left-to-right 因式分解根本不同，因此现有 OPSD 设计不兼容 dLLM。
- **研究含义**：本文旨在为 dLLM 设计首个 OPSD 框架，使其能够利用自身生成的未来答案作为教师信号，实现自我改进，并与 RLVR（带可验证奖励的强化学习）和 SFT 进行对比，展现更好性能和样本效率。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想

- **从自我未来学习**：利用 dLLM 的双向能力，将学生自生成的完整答案（future）作为后缀条件（suffix conditioning）注入教师模型，使学生从“自我未来经验”中学习，而非 AR 式的前缀参考回答。
- **步级监督**：将监督粒度从 token 级转变为 step 级，与 dLLM 每步仅展示 top-k 置信 token 的迭代机制对齐。

### 2.2 关键技术细节

- **教师构造（Self-teacher Construction）**：
    1. 学生模型采样一条完整轨迹 Y = {y_T, ..., y_0} ~ p_θ(·|x)。
    2. 学生输入为当前含噪序列 y_{student,t} = y_t。
    3. 教师输入在保留比例 ρ_teacher 下，从 y_0 中随机揭示部分真实 token 到 y_t 的对应位置，形成 y_{teacher,t}。
    4. 教师和学生共享同一模型参数，但教师拥有更多关于未来答案的信息。
- **步级 KL 散度监督**：
    1. 对每个去噪步 t，只关注当前已掩码位置中 top-k 最置信 token（K_t 子集）。
    2. 计算这些位置上教师和学生分布之间的 KL 散度均值：L_t = (1/|K_t|) Σ_{i∈K_t} D_KL( P^i_{student,t} || P^i_{teacher,t} )。
    3. 总目标：L_OPSD(θ) = E_x [ (1/T) Σ_{t=1}^T L_t ]。
- **采样策略**：使用 pass@k（默认 k=8），持续采样直到出现正确答案或达到上限，确保教师构造基于相对正确的轨迹。这与 RLVR 的 group rollout 计算开销相同。
- **训练技巧**：固定教师策略到初始策略（stabilize training）、逐 token 点状剪裁（pointwise clipping，阈值 0.05）、输入拼接（将整条轨迹各步输入拼接为 batch 以节省显存）、仅对正确生成计算损失（correct-only generation）。

### 2.3 与 AR 式 OPSD 的差异

- AR 式 OPSD：将参考答案作为前缀拼接到 prompt，使用 token 级监督。
- d-OPSD：将自生成答案作为后缀条件，使用步级监督。
- 实验表明 AR 式 OPSD 的 Overlap Top-K 接近 1（教师知识冗余），而 d-OPSD 的 Overlap 适中，提供更多新知识。

- d-OPSD 在四个推理任务上一致优于或匹配 SFT 和 RLVR 基线，尤其在 Sudoku 等规划任务上提升显著（20.6 vs 12.9）。

- 步级监督、逆向 KL、适当保留比例、教师引导的 top-k 选择、点状剪裁等设计至关重要。

## 7. 优点

- **方法创新**：首次提出面向 dLLM 的 OPSD，突破 AR 封装的限制，利用 dLLM 的双向能力设计后缀条件的自教师。
- **监督对齐**：步级 KL 散度与 dLLM 的迭代去噪过程天然一致，避免 token 级不兼容。
- **样本效率极高**：相比 RLVR 节约大量梯度更新，后训练成本大幅降低。
- **实验全面**：在多个推理基准、多种消融设置下验证，代码开源。
- **强调公平性**：正确生成采样策略（pass@k）与 RLVR 计算开销持平，对比更公平。

## 8. 不足与局限

- **模型泛化性**：仅基于 LLaDA-8B-Instruct，未在更大规模 dLLM（如 Dream、LLaDA2.0）验证，结论可能受模型特性影响。
- **失败模式**：训练可能经历策略崩溃，类似 RLVR，缺乏有效缓解方案（论文仅提到未来工作）。
- **采样开销**：pass@k 虽然与 RLVR 计算量相同，但 k=8 仍需要多次模型推理，对长序列和大 batch 可能有显存压力。
- **超参数敏感性**：保留比例 ρ_teacher、点状剪裁阈值、是否固定教师等对性能有影响，未提供明确的自动选择策略。
- **仅限于推理任务**：未在通用语言任务（如文本生成、翻译）上实验，OPSD 对 dLLM 的其他能力提升未知。
- **评估限于准确率**：缺乏对生成多样性、自方差的报告，可能忽略教师过于确定带来的模式坍缩风险。
