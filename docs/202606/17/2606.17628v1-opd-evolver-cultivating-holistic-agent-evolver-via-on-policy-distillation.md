## 摘要
记忆已成为自我演化智能体的标准基础，但保留经验并不等同于学习如何通过它进行演化。现有的记忆智能体可以存储轨迹、检索反思或积累技能，但往往缺乏整体能力来选择有用经验、据此行动、编写可复用知识以及维护不断增长的存储库。我们提出OPD-Evolver，一个通过同策略自我蒸馏培养此类智能体演化器的慢-快协同演化框架。在快循环中，OPD-Evolver与四级记忆层次结构交互，以读取、使用、编写和维护经验，实现快速的测试时演化。在慢循环中，结果校准的记忆归因和特权事后分析将这些四种能力蒸馏到可部署的策略中。跨多领域基准测试，OPD-Evolver在性能上超越如ReasoningBank等记忆系统最多11.5%，以及如Skill0等基于训练的方法约5.8%。进一步分析表明，OPD-Evolver内化了高价值经验和记忆管理，使得OPD-Evolver-9B能够挑战如Qwen3.5-397B-A17B和Step-3.5-Flash等大型对手，指向超越记忆增强型智能体的真正合格智能体演化器。

## Abstract
Memory has become a standard substrate for self-evolving agents, yet retaining experience is not the same as learning how to evolve through it. Existing memory agents can store trajectories, retrieve reflections, or accumulate skills, but often lack the holistic competence to select useful experience, act on it, write reusable knowledge, and maintain a growing repository. We introduce OPD-Evolver, a slow-fast co-evolution framework that cultivates such an agent evolver through on-policy self-distillation. In the fast loop, OPD-Evolver interacts with a four-level memory hierarchy to read, use, write, and maintain experience for rapid test-time evolution. In the slow loop, outcome-calibrated memory attribution and privileged hindsight distill these four abilities into the deployable policy. Across multi-domain benchmarks, OPD-Evolver surpasses memory systems such as ReasoningBank by up to 11.5%, and training-based methods such as Skill0 by ~5.8%. Further analysis shows that OPD-Evolver internalizes high-value experience and memory management, enabling OPD-Evolver-9B to challenge giant counterparts such as Qwen3.5-397B-A17B and Step-3.5-Flash, pointing beyond memory-augmented agents toward genuinely qualified agent evolvers.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：当前自演化智能体的研究大多聚焦于让智能体“记住”经验（如存储轨迹、检索反思、积累技能），但“记住”不等于“学会如何利用经验进行进化”。现有代理缺乏**整体性演化能力**，即无法系统地将交互历史和反馈转化为持续的行为改进。
- **关键定义**：本文首次明确定义了一个**合格的智能体演化器**应具备四项耦合能力：
  1. **经验选择**（Experience Selection）：从不断增长的、含噪声的记忆库中识别有用记忆。
  2. **经验驱动执行**（Experience-grounded Execution）：将选中的经验转化为有效的多轮动作。
  3. **经验编写**（Experience Writing）：从新轨迹和反馈中提取可复用的知识并存入记忆。
  4. **经验管理**（Experience Management）：对记忆进行评分、整合、更新和淘汰。
- **研究动机**：这些能力相互耦合，弱化任何一项都会导致整体退化。现有方法多只优化生命周期中的某一环节，缺乏统一的训练框架。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
提出**OPD-Evolver**，一个**慢-快共演化框架**，通过**同策略自我蒸馏（On-Policy Self-Distillation）** 将特权信息转化为可部署策略的演化能力。

### 关键技术细节

#### 快循环（Fast Evolution Loop）——测试时在线演化
- **四级记忆层次**：
  - **轨迹（Trajectories）**：保留完整的回合证据。
  - **提示（Tips）**：存储局部的警告或启发式规则。
  - **技能（Skills）**：抽象出可复用的过程。
  - **工具（Tools）**：存储可执行的命令或代码模板。
- **检索与选择**：
  - 根据任务查询，从每层检索 top-K 候选记忆 → 产生候选集 C_t。
  - 选择器 Sel_θ 从 C_t 中精选出子集 S_t，并格式化为提示上下文 c_t。
- **经验驱动执行与编写**：
  - 智能体 π_θ 基于 c_t 与环境交互产生轨迹 τ_t 和回报 R_t。
  - 回合结束后，同一智能体决定向哪些层级写入新记忆 Δ_t（数量由智能体自主选择）。
  - 另外每 Q 个任务进行一次存储库维护（lookup/merge/delete）。

#### 慢循环（Slow Evolution Loop）——训练阶段特权蒸馏
- **结果校准记忆归因**：
  - 对每个记忆 m，基于其被选中/未被选中的任务回报差异，计算归因分数 Â(m)，再转换为记忆分数 V(m)。
  - 高 V(m) 的记忆表示本应被选择、使用和保留；低 V(m) 的记忆表示噪声，应被忽略。
- **统一事后自我蒸馏**：
  - 针对四个生命周期决策（选择、执行、编写、维护），教师观测到特权信息（如记忆真实价值、成功轨迹、未来效用等），学生仅有公共输入。
  - 学生采样 on-policy 输出，教师基于相同学生前缀提供分布。
  - 使用 token 级 KL 散度作为蒸馏损失 L_slow，联合优化四项能力。

### 算法流程（文字说明）
1. 对于每个任务，形成查询，检索并选择记忆，构建上下文。
2. 智能体执行任务，收集轨迹和回报，更新记忆（写入新条目、执行维护）。
3. 记录候选记忆、选择、回报、写入内容等日志。
4. 慢循环中，根据日志估计每个记忆的价值分数。
5. 为四项决策分别构建学生-教师对，计算蒸馏损失，更新策略参数。
6. 仅部署学生策略进行测试。
