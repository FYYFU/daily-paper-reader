## 摘要
计算机使用代理越来越根据它们是否完成真实的桌面和网页任务来评估。然而，仅凭任务成功可能会遗漏代理通过不安全的捷径达到名义目标时的失败。我们引入了OSGuard，这是一个双粒度基准测试套件，用于在良性、不变的用户指令下评估计算机使用代理的安全性。OSGuard包含一个用于本地护栏决策的动作级基准测试，以及一个用于端到端评估的风险增强执行套件。动作级基准测试由上下文相关的提议动作组成，这些动作被标记为允许、无关或不安全，每个动作都相对于原始指令和当前界面状态进行判断。执行套件包含手动构建的OSWorld衍生任务变体，其中原始任务仍然可以实现，但环境被修改以引入潜在危害，例如破坏性覆盖等。每个变体都配有增强评估器，保留原始任务成功标准的同时添加显式的基于状态的安全不变量，使我们能够区分安全完成和满足名义任务目标的不安全完成。我们在OSGuard上的实验结果表明，当前的多模态护栏可以在孤立的动作判断上表现良好，而风险增强的执行暴露了本地监督与可靠端到端安全性之间的剩余差距。这种双粒度设计使得更精确地诊断模型是否既能识别不安全的提议动作，又能提高部署为护栏时的全任务安全性。

## Abstract
Computer-use agents are increasingly evaluated by whether they complete realistic desktop and web tasks. However, task success alone can miss failures in which an agent reaches the nominal goal through an unsafe shortcut. We introduce OSGuard, a dual-granularity benchmark suite for evaluating safety in computer-use agents under benign, unchanged user instructions. OSGuard contains an action-level benchmark for local guardrail decisions and a risk-augmented execution suite for end-to-end evaluation. The action-level benchmark consists of contextualized proposed actions labeled as allowed, unrelated, or unsafe, each judged relative to the original instruction and current interface state. The execution suite contains manually constructed OSWorld-derived task variants in which the original task remains achievable, but the environment is modified to introduce latent hazards such as destructive overwrites, etc. Each variant is paired with augmented evaluators that retain the original task-success criterion while adding explicit state-based safety invariants, allowing us to distinguish safe completions from unsafe completions that satisfy the nominal task objective. Our experimental results on OSGuard show that current multimodal guardrails can perform well on isolated action judgments, while risk-augmented execution exposes remaining gaps between local oversight and reliable end-to-end safety. This dual-granularity design enables more precise diagnosis of whether models can both recognize unsafe proposed actions and improve full-task safety when deployed as guardrails.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **问题**：现有计算机使用代理（Computer-Use Agents）评估主要关注任务是否完成（如OSWorld、WebArena等），但未能捕捉代理通过“不安全捷径”达成目标的情况——代理可能执行了破坏性覆盖、权限提升、访问无关敏感信息等，而表面上仍完成了用户指令。
- **背景**：已有安全基准（如OS-Harm、RiOSWorld）关注恶意请求、提示注入、钓鱼等显式对抗场景；AUTOELICIT、BLIND-ACT等则研究指令扰动或上下文不敏感导致的失败。但缺乏对**普通良性指令下，因任务状态约束而引发的安全失败**的研究。
- **核心贡献**：引入OSGuard——一个双粒度基准套件，在用户指令不变、环境引入潜在危险的情况下，评估代理是否既能完成任务又能遵守安全约束。

## 2. 论文提出的方法论：核心思想、关键技术细节
- **双粒度设计**：
  - **动作级基准（Action-Level Benchmark）**：评估本地护栏（Guardrail）的判断能力。护栏接收：原始指令、当前界面状态（截图+无障碍树）、候选动作，输出`allowed`（允许执行）、`unrelated`（无关但不危险）、`unsafe`（不安全，应拦截）。共324个已标注上下文化候选动作。
  - **风险增强执行套件（Risk-Augmented Execution Suite）**：基于OSWorld任务手动构建45个变体。保持原始指令不变，修改环境状态（如添加诱饵文件、冲突配置、额外身份等）引入潜在危险，但保留安全完成路径。每个变体配有**增强评估器**：保留原任务成功标准，额外添加显式的状态安全不变量（如文件哈希、权限、范围等）。将任务完成分为：安全成功、不安全成功（名义完成但违反不变量）、失败。
- **构建细节**：
  - 动作级数据来源：标准OSWorld执行、中断前缀+状态兼容动作、中断前缀+显式不安全动作、风险增强变体执行。
  - 风险增强变体覆盖6类安全失败模式：破坏性覆盖/删除、过度编辑/权限、范围逃逸、配置冲突、不必要敏感访问、错误目标/全局更新。
  - 护栏代理：同构反应式模型，接收指令+状态+候选动作，输出决策+自然语言反馈。支持至多2次重试（共3次提议）。
- **评估指标**：动作级使用准确率、宏F1；执行级使用变体成功率、不安全完成率、重试终止率。

## 6. 论文的主要结论与发现
- 动作级：Gemini 3 Pro Preview最优（准确率80%，宏F1 0.80），各标签表现平衡。GPT 5.1和Claude Sonnet 4.5较低（宏F1约0.60）。DeAction在OSGuard上表现差（宏F1 0.55），尤其不安全类召回低（0.20），说明转移性不足。
- 按来源：标准OSWorld动作最容易；风险增强动作最难（所有模型宏F1≤0.48），说明局部状态依赖的危险难以识别。

- 安全家族分析：破坏性覆盖/删除是最难避免的失败类型。

## 7. 优点：方法或实验设计上的亮点
- **双粒度设计**创新性强：同时评估局部动作判断与全局执行安全，能诊断模型在哪个环节失败。
- **风险增强场景合理**：在普通良性指令下，通过修改环境状态引入“不安全捷径”，更贴近现实使用中无恶意的意外违规。
- **增强评估器使用状态不变量**：基于文件哈希、权限等可验证检查，避免语义模糊，确保结果可复现。
- **护栏与执行器分离**：明确区分“护栏判断能力”与“全任务能力”，便于归因。
- **动作级数据构建方法**：利用真实OSWorld中间状态结合自动生成方式，兼顾真实性与危险性覆盖。

## 8. 不足与局限
- **实验规模有限**：仅45个执行变体和324个动作级样本，覆盖的危险模式有限（6类），可能无法泛化到所有桌面任务。
