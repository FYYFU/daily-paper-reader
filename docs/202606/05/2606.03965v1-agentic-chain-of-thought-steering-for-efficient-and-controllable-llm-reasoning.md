## 摘要
大型语言模型通过扩展的链式思维推理提高了最终答案的准确性，但通常令牌使用效率低下，且推理时控制能力有限。现有的高效推理方法通过缩短、提前停止或压缩轨迹来控制思考长度，但模型如何思考仍是隐式的。在本文中，我们提出了智能链式思维引导（ACTS），将推理引导形式化为一个马尔可夫决策过程，其中控制器代理在推理过程中自适应地引导冻结的推理器。在每个步骤中，控制器观察推理轨迹和剩余思考预算，然后发出一个引导动作，该动作由推理策略和引导短语组成，用于启动下一步推理。这使得在保持推理器生成连续性的同时，能够进行预算感知的策略控制以实现高效推理。我们从构建的合成引导轨迹（具有多预算增强）初始化控制器代理，并通过带有预算条件奖励塑造的强化学习进一步优化。在多个基准上的实验表明，ACTS在节省大量令牌的同时匹配了完整思考的性能，并在不同推理器和任务上实现了可控的准确率-效率权衡。代码可在https://github.com/Andree-9/ACTS获取。

## Abstract
Large language models improve final-answer accuracy through extended chain-of-thought reasoning, but often spend tokens inefficiently and offer little inference-time control. Existing efficient reasoning methods control thinking length by shortening, early-stopping, or compressing traces, leaving how the model thinks implicit. In this paper, we propose Agentic Chain-of-Thought Steering (ACTS), which formulates reasoning steering as a Markov decision process where a controller agent adaptively steers a frozen reasoner during inference. At each step, the controller observes the reasoning trace and remaining thinking budget, then issues a steering action consisting of a reasoning strategy and a steering phrase that initiates the next reasoner step. This enables budget-aware strategy control for efficient reasoning while preserving the reasoner's generation continuity. We initialize the controller agent from our constructed synthetic steering trajectories with multi-budget augmentation, and further optimize it via reinforcement learning with budget-conditioned reward shaping. Experiments across multiple benchmarks show that ACTS matches full-thinking performance with substantial token savings, and enables controllable accuracy-efficiency trade-offs across different reasoners and tasks. The code is available at https://github.com/Andree-9/ACTS.

- **现有局限**：当前高效推理方法主要关注*控制思考长度*（如缩短、早停、压缩），但*未显式干预模型如何思考*。推理轨迹内部包含功能步骤（如理解、计划、执行、验证等），但既有方法未利用该结构性信息。
- **整体意义**：论文提出**ACTS**，将推理引导形式化为一个马尔可夫决策过程，由控制器代理在推理过程中自适应地引导冻结的推理器，实现对*推理策略层面*的显式控制，同时保持预算（token预算）感知，从而在保持准确率的同时大幅节省推理成本。

## 2. 方法论
### 核心思想
- **推理引导作为MDP**：给定问题 \(x\) 和思考令牌预算 \(B\)，控制器（controller）在每个步骤 \(t\) 观测到完整引导历史 \(H_{t-1}\)（包含之前动作、推理步骤、剩余预算），然后输出一个引导动作 \(a_t = (u_t, p_t)\)：
  - \(u_t \in U\)：高维推理策略，取自预定义集合（UNDERSTAND, PLAN, EXECUTE, EXPLORE, CHECK, SUMMARIZE, CONCLUDE）。
  - \(p_t\)：自由形式的自然语言引导短语，用于启动推理器的下一步。
- **状态转移**：推理器（冻结）根据当前轨迹和引导短语生成下一步推理内容 \(z_t\)；预算按实际消耗更新；当控制器选择 CONCLUDE 或推理器发出结束令牌时终止。
- **终端奖励**：基于答案正确性 \(c\) 和剩余预算分数 \(b_T\) 设计的预算条件奖励函数：
  \[
  R(\tau, \hat{y}) = \begin{cases}
  1 + \alpha \min(b_T, 0), & c = 1 \\
  -\alpha |b_T|, & c = 0
  \end{cases}
  \]
  该奖励同时惩罚*过度思考*（正确但浪费预算）和*过早终止*（错误但预算未用尽）。

### 关键技术细节
1. **行为初始化（SFT）**：
   - 从 DeepSeek-R1 生成的专家推理轨迹（OpenR1-Math数据集）中，用 LLM 注释器将每个推理步骤标注为策略 \(u_t\)，并提取开头短语作为引导短语 \(p_t\)，合成引导轨迹。
   - **多预算增强**：将同一轨迹的预算重新缩放至不同终止预算（如用完预算、提前终止），使控制器暴露于多种预算场景。
   - 用监督学习最小化控制器的动作对数似然损失。
2. **在线强化学习（GRPO）**：
   - 使用组相对策略优化（Group Relative Policy Optimization），每个问题采样 \(G=8\) 条轨迹，计算组内平均奖励作为基线，轨迹级优势广播至控制器动作令牌。
   - 奖励函数中加入 10% 的预算边界容差，避免微小超/欠预算触发惩罚。
3. **异步推理**：控制器和推理器作为两个独立的 SGLang 服务器，通过异步 HTTP 通信，使得多个样本并发推进，有效隐藏控制器调用开销。

## 6. 主要结论与发现
- **ACTS 匹配全思考性能并大幅节省 token**：在多数基准上准确率接近或超过 Vanilla，token 使用量减少 37%~64%（1.5B 模型）和 37%~57%（7B 模型）。
- **可控的准确率-效率权衡**：通过调整预算，ACTS 生成平滑的 Pareto 曲线，优于 Vanilla 和 NoThink 的简单插值。
- **控制器泛化性强**：训练于数学推理的控制器，可迁移至科学 QA（GPQA Diamond）和不同推理器（如 Qwen3-8B），且性能提升显著（尤其对弱推理器）。
- **异步架构低延迟**：吞吐量接近直接推理（±1% 以内），优于 DEER 的约 50% 吞吐量。
- **可解释的 token 节省**：节省主要来自“挽救”（将 Vanilla 错误的试次变正确）和“缩短”（将正确试次进一步精简），而“退化”（Vanilla 正确变错误）占比不足 5%。

## 7. 优点
- **创新性**：首次将推理引导建模为 MDP，实现推理策略层面的显式控制，而非仅长度控制。
- **训练流程巧妙**：通过合成轨迹 + 多预算增强实现行为初始化，再用预算条件奖励的在线 RL 优化，设计对称惩罚避免过度思考/过早终止。
- **异步推理设计**：双服务器架构有效隐藏控制器调用开销，实际部署中无明显延迟负担。
- **可解释性**：提供对 token 节省机制的细粒度分解，显示控制器学习的是结构化引导而非盲目截断。
- **泛化能力**：控制器能迁移至不同推理器和任务，表明其学习了与领域无关的推理结构。

## 8. 不足与局限
- **模型规模限制**：实验推理器最大为 8B 参数（Qwen3-8B），未验证在 70B 级或更大模型上的表现。控制器本身为 4B，是否可有效引导更大模型？
- **预算需外部提供**：用户或服务方需在推理时指定 token 预算，而非模型自适应推断问题难度后自动设定。
- **域外泛化仅测试了一个非数学基**：虽然 GPQA 显示了迁移能力，但仅涉及科学问答，未覆盖其他领域（如代码、常识推理等）。
- **视角偏差风险**：控制器训练数据来源于单一数据集（OpenR1-Math）和单一推理器（DeepSeek-R1），可能存在分布偏差。
- **计算开销**：尽管推理时延迟可接受，但控制器训练（SFT + RL）需要大量合成数据和强化学习迭代，训练成本较高。
