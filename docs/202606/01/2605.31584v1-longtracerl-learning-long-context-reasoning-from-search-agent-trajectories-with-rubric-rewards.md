## 摘要
长上下文推理仍然是大型语言模型的一个核心挑战，它们常常无法在大量干扰内容中定位并整合关键信息。基于可验证奖励的强化学习（RLVR）在此任务上展现出了潜力，但现有方法受限于低混淆度的干扰项以及稀疏的、仅关注结果的奖励信号，无法监督中间推理步骤。为解决这些问题，我们提出了\textsc{LongTraceRL}。在数据构建方面，我们通过知识图谱随机游走生成多跳问题，并利用搜索代理轨迹构建\emph{分层干扰项}：代理读取但未引用的文档（高混淆度）以及搜索结果中出现但从未打开的文档（低混淆度），从而生成比随机采样或单次搜索构建的训练上下文更具挑战性的内容。在奖励设计方面，我们提出了一种\emph{评分奖励}，利用每条推理链上的黄金实体作为细粒度的实体级过程监督。该评分奖励仅应用于最终答案正确的回复（正向策略），以区分正确回复之间的推理质量，并防止奖励作弊。在三个推理型大语言模型（4B-30B）上，基于五个长上下文基准的实验表明，\textsc{LongTraceRL} 持续优于强基线方法，并鼓励全面、基于证据的推理。代码、数据集和模型可在 \href{https://github.com/THU-KEG/LongTraceRL}{https://github.com/THU-KEG/LongTraceRL} 获取。

## Abstract
Long-context reasoning remains a central challenge for large language models, which often fail to locate and integrate key information in extensive distracting content. Reinforcement learning with verifiable rewards (RLVR) has shown promise for this task, yet existing methods are limited by low-confusability distractors and sparse, outcome-only reward signals that cannot supervise intermediate reasoning steps. To address these issues, we introduce \textsc{LongTraceRL}. For data construction, we generate multi-hop questions via knowledge graph random walks and leverage search agent trajectories to build \emph{tiered distractors}: documents the agent read but did not cite (high confusability) and documents that appeared in search results but were never opened (low confusability), producing training contexts that are far more challenging than those built by random sampling or one-shot search. For reward design, we propose a \emph{rubric reward} that uses the gold entities along each reasoning chain as fine-grained, entity-level process supervision. This rubric reward is applied only to responses with correct final answers (positive-only strategy), distinguishing the reasoning quality among correct responses and preventing reward hacking. Experiments on three reasoning LLMs (4B--30B) across five long-context benchmarks demonstrate that \textsc{LongTraceRL} consistently outperforms strong baselines and encourages comprehensive, evidence-grounded reasoning. Codes, datasets and models are available at \href{https://github.com/THU-KEG/LongTraceRL}{https://github.com/THU-KEG/LongTraceRL}.

# LongTraceRL 论文详细总结

## 1. 核心问题与整体含义

- **研究背景**：长上下文推理是大语言模型（LLM）的关键能力，但现有模型在包含大量干扰信息的长文本中往往无法准确定位和整合关键信息，出现幻觉、碎片化检索等问题。
- **现有不足**：强化学习方法（RLVR）在长上下文中存在两个主要局限：① 训练数据质量低——干扰项多为随机采样，与问题语义关联弱、混淆度低；② 奖励信号稀疏——仅基于最终答案正确性的结果奖励无法监督中间推理步骤，导致模型可能通过错误推理路径巧合得到正确答案。
- **论文目标**：通过构建更具挑战性的训练数据和更细粒度的过程奖励，提升LLM在长上下文中的推理能力。

## 2. 方法论：核心思想与关键技术

### 2.1 数据构建管道（四步流程）

1. **多跳问题生成**：基于KILT Wikipedia知识图，进行随机游走（8步）获得实体路径，再由LLM生成需逐步推理的多跳问题，并输出路径中的**黄金实体**。
2. **搜索智能体轨迹收集**：部署具有深度搜索能力的智能体，记录其尝试回答每个问题时的完整轨迹（搜索、打开、引用）。仅保留最终正确回答的轨迹。
3. **分层干扰项提取**：将智能体打开但未引用的文档作为高混淆度干扰项（Tier-1）；将搜索结果中出现但从未打开的文档作为低混淆度干扰项（Tier-2）。
4. **长上下文组装**：优先填充Tier-1干扰项，再补Tier-2，直至目标长度128K token，然后打乱文档顺序。

### 2.2 强化学习框架（GRPO + 复合奖励）

- **结果奖励**：r_oc ∈ {0,1}，由LLM评判最终答案是否正确。
- **评分奖励**：
  - 原始评分：r̂_rb = （响应中出现的黄金实体数）/（总黄金实体数）
  - 组级归一化：将每个组内的r̂_rb除以该组最大值（若为正），使其落在[0,1]。
- **正仅策略**：仅对答案正确的响应应用评分奖励，错误响应奖励为0：
  ```
  r = (1-α)·r_oc + α·r_rb  （若r_oc>0）
  r = 0                      （若r_oc=0）
  ```
  默认α=0.3。

## 6. 主要结论与发现

- **性能提升**：LongTraceRL在所有模型上均实现一致改进。以Qwen3-4B为例，平均分从53.3提升至59.0（+5.7），超越最强基线LongRLVR 2.5分。
- **评分奖励的关键作用**：移除评分奖励（降为纯GRPO）后，平均分从59.0降至53.7，说明过程监督是主要驱动力。
- **干扰项质量影响**：基于智能体轨迹的分层干扰项（traj-tiered）显著优于随机、单次搜索或混合采样，更多干扰项包含黄金实体（50.03% vs 随机1.35%）。
- **正仅策略防止奖励作弊**：将评分奖励仅用于正确响应，避免模型通过列举实体而不真正推理来获得高奖励。
- **模型行为的改善**：训练后模型生成长度更长、更谨慎的推理路径，能识别问题中的矛盾、正确解析代词指代、理解细微限定条件。

## 7. 优点

1. **创新的数据构建方法**：利用搜索智能体的真实行为轨迹生成高混淆度干扰项，更贴近实际应用场景，显著提升训练数据质量。
2. **细粒度的过程奖励**：在实体级别提供监督，比文档/块级别奖励更精细，且无需额外评分模型。
3. **正仅策略设计**：有效防止奖励作弊，保证模型真正学习推理而非投机。
4. **跨模型/跨基准的通用性**：在三个不同架构和规模的模型上均有效，在五个差异化的长上下文基准上表现稳定。
5. **开源贡献**：提供代码、数据集和模型，促进复现和研究。

## 8. 不足与局限

1. **知识源单一**：所有训练数据基于KILT Wikipedia，问答模式限于百科知识，可能缺乏金融、法律、代码等领域的多样性。
2. **依赖智能体能力**：干扰项质量受限于所用搜索智能体的性能，不同智能体产生的轨迹分布不同，可能影响数据泛化性。
3. **计算资源要求高**：32×H800 GPU的训练配置对一般研究者门槛较高。
4. **未覆盖更多基线**：未与最新长上下文RL方法（如LongR、Evidence-augmented Policy Optimization）直接比较。
5. **超参数搜索有限**：仅对α做了三个值的选择，未系统探索其他超参数（如组大小、训练迭代数）的影响。
6. **基准局限性**：LongReason为合成基准，AA-LCR规模较小（100题），实际场景的泛化能力仍需进一步验证。
