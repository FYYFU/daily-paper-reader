## 摘要
LLM 智能体越来越依赖社区贡献的技能，这些技能扩展了智能体的操作能力集。我们研究了智能体 AI 系统中的一个核心安全问题：个体安全的技能是否可能组合成不安全的已安装技能集。我们提出了 SkillReact，一个组合安全测量框架，包含三个组件：一个确定性静态组合基准、一个双评分者 LLM 辅助的人类裁决流程，以及一个基于动作的可利用性测试工具。在 1,520 个 ClawHub 技能中，651 个通过了个体检查，并形成了 211,575 个技能对；基准将这些对中的 22.25% 标记为结构候选。我们将这个原始比率视为一个面向召回率的扫描上限，并根据人类判断进行校准：在一个按模式分层的审计中，大约每五个被标记的对模式命中中有一个被确认为真实的组合风险（人口加权有效性 18.2%，这是我们的主要结果），这意味着在一个单一注册表中，由于每对技能都是个体安全的，因此按构建方式会遗漏约 14,000 个真实风险成员（这是每技能扫描所固有的遗漏）。然后，基于动作的测试工具探测这些候选何时成为模型发布的工具调用，并发现实现受主机模型倾向的制约：在一个锚定条件的投放子集上，Haiku-4-5 在所有 39 次直接提示试验中发布了投放阶段的工具调用（其中 36 次为完整的下载后执行链，3 次仅为下载），Opus-4-7 在下载阶段停止，而 Sonnet-4-6 直接拒绝。一个保持请求固定、仅改变已安装技能的对照实验发现，未安装任何技能时合规性最高：组合决定了哪些能力可访问，而主机模型决定是否使用它们。这些共同表明，安装时的组合检查和能力隔离是每技能扫描的补充。

## Abstract
LLM agents increasingly rely on community-contributed skills that expand an agent's operational capability set. We study a core safety problem in agentic AI systems: whether individually safe skills can compose into unsafe installed skill sets. We present SkillReact, a compositional security measurement framework with three components: a deterministic static-composition benchmark, a two-rater LLM-assisted human-adjudication pipeline, and an action-based exploitability harness. On 1,520 ClawHub skills, 651 pass individual inspection and form 211,575 pairs; the benchmark flags 22.25% of these as structural candidates. We treat this raw rate as a recall-oriented scanner ceiling and calibrate it against human judgment: in a pattern-stratified audit, roughly one in five flagged pair-pattern hits survives as a real compositional risk (population-weighted validity 18.2%, our headline result), implying about 14K genuine risk memberships in a single registry that per-skill scanning misses by construction, since every pair is individually safe. An action-based harness then probes when these candidates become model-issued tool calls, and finds realization gated by host-model disposition: on an anchor-conditioned dropper subset, Haiku-4-5 issues the dropper-stage tool call on all 39 direct-prompt trials (36 of them the full download-then-execute chain, 3 download-only), Opus-4-7 stops at the download, and Sonnet-4-6 refuses outright. A control that holds the request fixed and varies only the installed skills finds compliance highest with no skills installed: a composition fixes which capabilities are reachable, while the host model decides whether to use them. Together these motivate install-time compositional checks and capability isolation as complements to per-skill scanning.

### 论文核心问题与整体含义

**核心问题**: 大语言模型（LLM）智能体依赖社区贡献的“技能包”来扩展其能力。虽然每个独立的技能包经过安全检查可能被认为是安全的，但如果用户同时安装多个安全技能，这些技能的**能力集合**（并集）在运行时可能会组合成不安全的配置，从而触发本不应存在的风险。

**研究动机与背景**:
1.  **固有的安全漏洞**：现有的安全机制（如恶意技能检测、漏洞扫描器）主要针对单个技能进行检查。然而，智能体在一个会话中会同时加载所有已安装的技能，因此，安全的单位应该是“已安装的技能集”，而非单个技能。
2.  **组合性盲区**：一个提供文件读取能力的“科研助手”技能和一个提供网络外部访问能力的“API助手”技能，各自都符合“禁止数据泄露”的安全策略。但当它们被同时安装使用时，其能力并集（文件读取 + 网络外联）就构成了一个违反数据泄露策略的风险组合。这种由个体安全元素组合成不安全整体的问题，是现有逐技能扫描方法**结构性遗漏**的。
3.  **研究目标**：量化在真实的技能生态系统中，这种结构性组合风险候选有多普遍；以及哪些候选最终会转化为模型实际发出的、旨在实现被禁止目标的工具调用。

### 论文提出的方法论：SkillReact 框架

**核心思想**: SkillReact 是一个组合安全测量框架，它区分了三个不同层次的证据：
1.  **个体安全技能**：单个技能的自身能力不违反任何策略。
2.  **结构组合风险候选**：每个都安全的两个技能，它们的能力集并集违反了一个或多个禁止模式。
3.  **模型工具调用尝试**：主机模型在实际运行时，是否真的调用了实现被禁止目标的工具。

**关键技术细节**:
*   **静态组合基准**:
    *   **能力提取**: 对每个技能，使用确定性的、面向召回率的正则表达式（regex）从代码和文档中提取其能力（如文件读/写、网络出站/入站、凭据访问等）。
    *   **禁止模式**: 定义了 10 个禁止组合模式（如数据泄露: `文件读取 ∧ 网络出站`；投毒者：`网络出站 ∧ 文件写入 ∧ 进程生成`）。
    *   **算法**: 对于所有“个体安全”的技能对，计算其能力并集（`Cap(SkillA) ∪ Cap(SkillB)`），检查该并集是否满足任何一个禁止模式。如果满足，则该对是一个“结构组合风险候选”。
    *   **核心指标**: 组合候选率（CCR），即结构候选对占所有安全技能对的比例。

*   **LLM辅助的人类裁决流水线**:
    *   **目的**: 校准静态基准的原始 CCR（一个高召回率上限），得到一个更准确的有效性比率。
    *   **流程**:
        1.  **分层抽样**: 从被标记的候选对中，按 10 个禁止模式分层抽取 200 个“对-模式”单元（每个模式 20 个）。
        2.  **双 LLM 评分**: 每个单元被两个来自不同供应商的 LLM（Claude 和 Codex）独立标记，输出结构化的判断（有效、误报、上下文无害等）。
        3.  **人类裁决**: 一个标准库用户界面显示两个 LLM 的标记和原始证据，由一名人类审计员对每个单元进行最终仲裁。
        4.  **校准**: 基于人类审计结果，计算人口加权有效比率，这是论文的核心结论。

*   **基于动作的可利用性测试工具**:
    *   **目的**: 测试结构候选在模型“眼里”是否真的会转化为工具调用尝试。
    *   **设置**: 在权限尽可能宽松的运行时环境中，向模型展示特定技能对的文本（`SKILL.md`），记录模型是否发出如 `Bash`、`Read` 等旨在实现被禁止目标的工具调用。
    *   **关键发现**: 工具调用的现实化程度取决于**主机模型**的倾向，而非具体的风险组合。

### 论文主要结论与发现

1.  **静态风险普遍性**: 在所有个体安全的技能对中，有 22.25%（95% 置信区间 [19.98%, 24.61%]）成为结构组合风险候选。
2.  **校准后的真实风险**: 经过人类仲裁，这些候选对的“对-模式”级别的加权有效比率为 **18.2%**（95% 置信区间 [11.3%, 27.4%]），换算后约 **14,000 个**真实风险成员存在于一个注册表中，而这些风险是逐技能扫描无法发现的。
3.  **主导风险模式**: 数据泄露（文件读取 + 网络出站）是压倒性的主导风险模式，占所有结构候选的 71.6%。
4.  **模型行为的关键决定因素**: 模型是否将静态风险转化为实际工具调用，主要取决于**主机模型的倾向**，而非具体的技能组合。在相同的配置和请求下，Haiku-4-5 执行了完整的恶意链，Opus-4-7 仅执行下载部分，而 Sonnet-4-6 则完全拒绝。
5.  **组合控制重要性**: 组合确定能力的**可达性**，而模型则决定是否**使用**这些能力。
6.  **安装时检查的必要性**: 基于结果，论文提出了一个可在安装时运行的、亚毫秒级响应的组合检查器，以快速识别和警告由新技能引入的组合风险。

### 优点：方法或实验设计上的亮点

1.  **问题定义清晰准确**: 准确定位了当前安全范式的结构性盲区，并提出了“已安装技能集”作为新的安全分析单元。
2.  **多层次证据体系**: 分离了“扫描结果”、“人类确认的风险”和“模型行为”三个层次，使得结论更加严谨、不夸大。
3.  **校准而非单纯报告原始数据**: 明确将 22.25% 的原始比率定位为“召回上限”，并通过人类审计报告 18.2% 的有效比率，避免了对结果过度解读。
4.  **创新的实验设计**: 特别是“组合控制实验”（固定请求，仅改变技能），巧妙地将“组合能力”与“模型对请求的响应”的影响解耦，结论极具说服力。
5.  **可操作的产出**: 最终提出了一个算法原型（安装时组合检查器），为实际工程防护提供了直接指导。

### 不足与局限

1.  **静态过度近似**: 能力并集不代表一定会被执行，CCR 是一个上限估计。即使被标记为风险组合，实际是否被利用还取决于复杂的提示和运行时环境。
2.  **能力提取噪音**: 基于正则表达式的能力提取虽然是确定性的，但存在大量的假阳性（如文档描述了能力但代码并未真正实现），这是论文承认的主要噪音来源。人类校准可以缓解但无法完全消除。
3.  **人类裁决偏差**: 单一人审计员、在 LLM 评分后做裁决，引入了潜在的偏见。
4.  **成对分析的局限**: 论文仅分析了成对组合。三个或更多技能的复杂组合可能带来新的风险（或冗余），这超出了当前模型的分析范围。
5.  **样本偏差**: 动作测试仅针对一个模式（投毒者）和一个锚定技能，其普遍性有待验证。
6.  **语料库范围有限**: 仅分析了 ClawHub 这一个技能生态系统，结论可能不适用于其他注册表（如 GPT Actions, MCP 服务器）。
