## 摘要
随着语言模型承担起代理角色，涉及调用外部API、读取工具输出以及根据第三方内容中的嵌入指令采取行动，其攻击面远超用户输入。模型是否以相同方式处理恶意指令，无论其来自何处，尚未得到系统研究。我们引入了安全不对称性得分（SAS），通过匹配的载荷对（恶意文本相同，仅传递上下文不同）衡量模型对对抗性内容的敏感性随其出现在用户消息、工具元数据或工具输出中的变化。在6个生产级LLM和三种攻击族上的评估显示，存在一致且信息量的不对称性：当对抗性内容通过工具描述而非用户消息传递时，代理原生模型显著更易受影响，而通用模型则相反。当相同内容通过工具输出而非描述传递时，这种不对称性进一步反转，表明模型隐含地将工具元数据视为可信指令，将工具结果视为普通数据。对Llama 3.3 70B的机制研究表明，安全相关表示因果性地存在于网络的中后层，但以非线性方式编码，解释了线性探针为何无法检测到它。这些发现揭示了当前使用工具的语言模型在应对对抗性内容时存在系统性、渠道依赖的盲点。

## Abstract
As language models take on agentic roles that span calling external APIs, reading tool outputs, and acting on instructions embedded in third-party content, their attack surface expands well beyond what users type. Whether a model treats a malicious instruction the same way regardless of where it arrives has not been systematically studied. We introduce the Safety Asymmetry Score (SAS), which measures how much a model's susceptibility to adversarial content shifts depending on whether that content arrives in the user message, tool metadata, or tool output, using matched payload pairs that keep the malicious text identical and vary only the context of delivery. Evaluated across 6 production LLMs and three attack families, we find a consistent and informative asymmetry: agent-native models are substantially more vulnerable when adversarial content arrives via tool descriptions than via user messages, while general-purpose models show the reverse. This asymmetry further inverts when the same content is delivered through tool outputs rather than descriptions, suggesting models implicitly treat tool metadata as trusted instructions and tool results as ordinary data. A mechanistic study on Llama 3.3 70B reveals that the safety-relevant representation is causally present at mid-to-late network depths but non-linearly encoded, explaining why linear probes fail to detect it. These findings expose a systematic, channel-dependent blind spot in how current tool-using models handle adversarial content.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）在作为自主代理使用时，面临来自多个输入通道（用户消息、工具元数据、工具输出）的攻击。当前研究尚未系统回答：**同一恶意指令通过不同通道送达时，模型的安全性表现是否不同？**
- **研究动机**：随着MCP（Model Context Protocol）等标准的推广，工具使用LLM的攻击面急剧扩大。攻击者可以通过控制工具描述、工具返回值或跨工具指令来实施攻击，但现有基准仅测量单个通道的绝对漏洞，忽略了通道间的相对脆弱性差异。
- **整体含义**：该研究揭示了模型对对抗性内容的敏感性存在**系统性的通道依赖**，并提出了量化指标（SAS），为设计通道感知的安全防御提供了方法论基础。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **安全不对称性得分（Safety Asymmetry Score, SAS）**：定义 SAS(\(M\)) = ASR_tool(\(M\)) − ASR_chat(\(M\))，即同一模型在工具通道与聊天通道上的攻击成功率之差。正分表示工具通道更脆弱，负分表示聊天通道更脆弱。
- **匹配载荷对（matched payload pairs）**：针对每个攻击案例，构建两组提示（prompt）：一组将恶意指令嵌入工具描述/工具输出，另一组将其嵌入用户消息。保证恶意文本字节相同，仅传递通道不同，从而隔离通道作为唯一实验变量。

### 关键技术细节
- **攻击家族**：构造了98个案例，覆盖三种攻击类型：
  - **工具投毒（Tool Poisoning, TP）**：50例，将恶意指令嵌入工具描述元数据（分三个子类型P1/P2/P3）。
  - **间接提示注入通过工具输出（IPI via tool output）**：24例，将恶意指令嵌入工具返回值。
  - **跨工具影子攻击（Cross-Tool Shadowing, CTS）**：24例，在一个工具描述中嵌入指向另一工具的指令。
- **攻击成功率（ASR）计算**：使用六类评分体系（SUCCESS, FAILURE-IGNORED, FAILURE-REFUSED, FAILURE-DIRECT-EXECUTION, AMBIGUOUS, ERRORED），排除AMBIGUOUS和ERRORED后计算ASR。
- **评分流程**：工具通道通过确定性工具调用检查；聊天通道通过拒绝正则表达式+非名册LLM评判（Cohen's κ=0.722），确保SUCCESS类不受影响。
- **外部验证**：在MCPTox基准的379例提取案例上复现模型排名，Spearman ρ=0.54。
- **机制分析**：对Llama 3.3 70B进行线性探针（ℓ2正则逻辑回归，5折交叉验证）和因果激活修补（前向/反向修补，余弦相似度得分），定位安全表示所在的网络深度。

## 6. 主要结论与发现

- **核心行为结果**：agent-native模型（Nemotron、GPT-OSS、Kimi）的SAS显著为正（平均+24.8 pp），通用模型（Llama、GLM）的SAS为负（平均−5.6 pp），组间ΔSAS=+30.4 pp。该差距主要由工具投毒驱动。
- **通道信任不对称性**：模型信任策略不同——agent-native模型更信任工具描述（视作指令），通用模型更信任用户消息；而当恶意内容通过工具输出传递时，所有模型都表现得更不信任（IPI下SAS均为负），表明模型区分“工具描述=指令” vs “工具输出=数据”。
- **机制发现**：Llama 3.3 70B的线性探针在所有4个深度上聊天模式分离度高于工具模式（与SAS矛盾），而因果激活修补显示在L48和L64双向修补产生显著正向位移（必要且充分），说明安全表示因果存在但非线性编码，线性探针无法捕获。
- **跨通道探针迁移性差**：聊天模式训练的探针迁移到工具模式时准确率接近随机（L64仅0.50），说明分离方向是通道特定的。

## 7. 优点

- **创新度量**：提出SAS作为量化通道间安全不对称性的标准化指标，概念清晰，计算简单。
- **匹配载荷设计**：严格控制恶意文本相同，仅改变通道，这是隔离变量的关键创新，优于先前绝对漏洞测量。
- **因果机制研究**：结合线性探针和因果激活修补，揭示了表示的存在性和非线性编码方式，解释了行为不对称机制，超越了纯行为观察。
- **多攻击族覆盖**：同时研究工具投毒、间接提示注入和跨工具影子攻击，发现了不同家族可能反转不对称方向，加深了对通道信任的理解。
- **公开承诺**：将发布所有代码、痕迹和机制制品，便于复现和后续研究。

## 8. 不足与局限

- **样本量有限**：仅6个模型、98个案例，统计检验力不足；模型规模与类别存在混淆（agent-native普遍更大）。
- **机制范围狭窄**：仅对单个模型（Llama 3.3 70B）和单一攻击家族（TP）进行机制研究，未验证IPI和CTS的通向性。
- **评分不对称性**：工具通道ASR基于确定性工具调用检查，聊天通道ASR依赖LLM评判，存在潜在偏差（尽管已通过评判替换和协议检验缓解）。
- **技术限制**：仅使用贪婪解码（temperature=0），无系统提示；单轮静态攻击，未考虑多轮自适应攻击。
- **通道与能力混杂**：聊天模式无工具注册，因此被攻击能力受限；虽然IPI反转表明存在非纯能力效应，但工具通道和聊天模式的动作可能性本质上不同。
