## 摘要
为了提升大型语言模型的推理能力，通常在最终答案之前生成中间标记来扩展测试时的计算量。然而，这将推理与自回归生成耦合，从而将内部计算与外部通信混为一谈。相比之下，人类认知能够利用工作记忆在内部持有和操作信息，而无需将中间思维外部化。基于这一原理，我们引入了记忆推理（RiM），一种潜在推理方法，用记忆块取代推理步骤的自回归生成。这些记忆块是固定序列的特殊标记，能够解锁大型语言模型的工作记忆容量。由于它们是固定的而非生成的，可以在单次前向传播中处理，从而实现计算高效的潜在推理。为了操作这些记忆块，我们采用了两阶段课程。首先，在每个记忆块之后预测明确的推理步骤来对其进行接地。其次，我们丢弃这种步骤级别的监督，并在每个记忆块之后迭代地细化最终答案。我们在推理基准上的实验表明，跨不同家族和大小的语言模型，RiM 匹配或超过了现有的潜在推理方法，同时避免了思维的自回归生成。这些结果证明，大型语言模型可以被训练成使用工作记忆作为潜在推理的有效机制。

## Abstract
To improve the reasoning capabilities of large language models, test-time compute is typically scaled by generating intermediate tokens before the final answer. However, this couples reasoning to autoregressive generation and thereby conflates internal computation with external communication. In contrast, human cognition can use working memory to hold and manipulate information internally without the need to externalize intermediate thoughts. Drawing on this principle, we introduce Reasoning in Memory (RiM), a latent reasoning method that replaces the autoregressive generation of reasoning steps with memory blocks. These memory blocks are fixed sequences of special tokens that unlock the working-memory capacity of large language models. Since they are fixed rather than generated, they can be processed in a single forward pass, enabling compute-efficient latent reasoning. To operationalize these memory blocks, we employ a two-stage curriculum. First, we ground them by predicting explicit reasoning steps after each memory block. Second, we discard this step-level supervision and iteratively refine the final answer after each memory block. Our experiments on reasoning benchmarks show that, across language models of different families and sizes, RiM matches or exceeds existing latent reasoning methods while avoiding the autoregressive generation of thoughts. These results demonstrate that large language models can be trained to use working memory as an effective mechanism for latent reasoning.

- **人类认知的启发**：人类在解决复杂问题时不需要将每个中间步骤都表达出来，而是利用工作记忆（working memory）在内部持有和操作任务相关表征。发展心理学也观察到，儿童起初依赖“出声思考”，但随后内化为内部处理。这启发作者将 LLM 的推理从自回归生成中间步骤转向在专用的内部工作空间中进行潜在推理。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：用固定序列的特殊 token（记忆块）替代自回归生成的推理步骤。记忆块是固定的输入 token，不参与自回归生成，因此可以在单次前向传播中处理整个序列，从而解耦内部推理与自回归生成。
- **记忆块定义**：每个记忆块 `mk` 由 `<b>`、若干 `<m>` 标记（默认 2 个）、`</b>` 组成。其中 `<m>` token 形成潜在工作空间，`<b>` 和 `</b>` 界定块边界。新增特殊 token，并冻结预训练 token 的嵌入，仅更新特殊 token 的嵌入，以避免干扰预训练语义。
- **两阶段课程（Two-Stage Curriculum）**：
    - **Stage 1（推理步骤监督）**：将推理踪迹分割为 T 个步骤，每个步骤后放置一个记忆块，并训练模型在每个记忆块后预测下一个推理步骤。自定义注意力掩码（图2）使得每个读出头只能关注问题及之前的记忆块，而不能关注其他推理步骤，从而迫使中间计算必须编码在记忆块中。损失函数为加权推理步骤负对数似然 `LS1 = -∑ λt log p(rt+1 | x, m≤t)`，其中 λt 按步骤索引线性退火，实现软多阶段课程。
    - **Stage 2（最终答案精炼）**：丢弃步骤级监督，改用固定数量 K 个记忆块（默认 8 个），每个记忆块后训练模型预测最终答案。损失函数 `LS2 = -∑ αk log p(y | x, m≤k)`，对后几个记忆块赋予更大权重。此阶段将接地后的记忆块转化为固定序列的潜在计算，用于精炼答案。
- **优化细节**：使用标准下一 token 预测目标，无需多个通路或辅助损失。阶段切换时重置优化器和学习率调度器，并使用较低学习率和更高 Dropout 防止过拟合。

## 7. 优点

- **创新性**：首次提出将固定记忆块作为 LLM 的工作记忆，完全解耦内部推理与自回归生成，突破了现有“思考外化”范式。
- **计算效率**：记忆块在单次前向传播中处理，推理延迟与直接答案生成相同，相比 Coconut 和 CoT 大幅提升。

- **实验严谨**：引入交叉验证和独立验证集选择，避免选择过拟合，并提供表示分析、消融实验等多角度证据。
- **可扩展性**：推理时记忆块数量和大小可灵活调整，且性能稳定。

## 8. 不足与局限

- **领域局限性**：仅在数学推理（GSM8K 系列）上验证，未涉及常识推理、符号推理、编程等更广泛任务。工作记忆机制是否适用于其他领域尚不明确。
- **可解释性降低**：潜在空间的内部计算难以直接解读，相比显式 CoT 可追溯性更差，可能阻碍在需要可解释性的场景中的部署。
- **两阶段训练依赖显式推理步骤**：Stage 1 需要每个样本提供完整的推理步骤序列，这在缺乏步骤级标注的任务中难以获得，限制了适用范围。
- **对比基线的不完全公平性**：DART 的对比使用官方报告数字而非可复现的重新实现，且训练配置（更多 epochs、全参数微调）不同。尽管 RiM 在较低训练成本下表现更好，但非完全控制比较。
- **参数高效微调可能不足**：仅使用 LoRA 适配器，未探索全参数微调是否带来进一步提升；且仅微调新增特殊 token 的嵌入，可能未完全释放模型潜力。
- **长链推理验证不足**：GSM8K-Aug 最多 13 个推理步骤，未测试更长的推理链（如数学竞赛题），内存块数量固定为 8，是否适用于更长推理有待验证。
