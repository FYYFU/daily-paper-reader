## 摘要
大型推理模型（LRM）通常通过最终答案准确率或标记数量等指标进行评估。然而，在这些指标上得分相同可能会掩盖根本不同的推理结构。为解决这一局限性，我们引入了一个可扩展的逻辑谜题LRM基准测试，以及一个将非结构化轨迹转换为可验证的声明与依赖关系推理图的流水线。这使推理成为结构化的、可测量的对象，其拓扑能够被定量分析。在此基础上，我们定义了一个推理效率指标，用于量化模型逻辑流的集中程度。我们对开源推理模型的分析表明，结构度量能够区分标记数量和准确率混淆的行为，为诊断故障模式和比较推理如何随谜题难度扩展提供了实用工具。

## Abstract
Large reasoning models (LRMs) are often evaluated using metrics such as final-answer accuracy or token count. However, identical scores on these metrics can hide fundamentally different reasoning structures. To address this limitation, we introduce a scalable LRM benchmark of logic puzzles and a pipeline that converts unstructured traces into verifiable reasoning graphs of claims and dependencies. This turns reasoning into a structured, measurable object whose topology can be quantitatively analyzed. Building on this, we define a reasoning efficiency metric that quantifies how concentrated the model's logical flow is. Our analysis on open-source reasoning models shows that structural measurements separate behaviors that token count and accuracy conflate, providing a practical tool for diagnosing failure modes and comparing how reasoning scales with puzzle difficulty.

## 2. 方法论
### 核心思想
- 将LRM生成的非结构化文本推理轨迹转换为**有向无环图（DAG）**，其中节点表示原子声明（Claims），边表示演绎依赖关系（Deductive Dependencies）。
- 图结构通过可执行谜题环境进行验证，从而将推理过程转化为可测量、可分析的对象。

### 关键技术细节
1. **推理图构建**：
   - **声明提取**：采用两阶段混合方法：结合确定性模式匹配和LLM提取，平衡精确率和召回率。提取后经LLM筛选、去重、规范化。
   - **规则提取**：对每个非临时声明，通过LLM从推理轨迹中识别一条显式规则应用，连接前提与结论。如果没有规则，则视为直接陈述。
   - **声明验证**：每个可验证声明通过执行谜题环境确定性检查其正确性（正确/错误/不可验证）。
2. **推理效率指标 η**：
   - 基于吸收马尔可夫链，将逻辑流视为从源节点出发、沿边传播的概率分布。
   - 计算结构熵 H_str(G)，量化逻辑流在图中分散或集中的程度。
   - 效率 η = (log|V| - H_str(G)) / (log|V| - log|C*|)，其中|V|是节点数，|C*|是最小声明集大小。η∈[0,1]，越大表示逻辑流越集中于核心解。
3. **子图定义**：
   - 最小解子图 G_sol：包含直接或间接支持最终解的节点和边。
   - 验证子图 G_ver：包含从解节点出发的所有后代及其祖先，用于度量验证开销。

### 公式或算法流程（文字说明）
- 推理图提取流程：轨迹分块 → 确定性提取 + LLM补充 → LLM从零提取 → 归并筛选 → 声明集 → 规则提取（边缘） → 图构建。

## 6. 主要结论与发现
- **难度扩展导致准确率急剧下降，尽管令牌数增加**：所有模型在最高难度下几乎完全失败（GPT‑5从83.8%降至5.7%，其他开源模型降至0%左右），同时令牌数大幅增加（达到20k-61k）。更多标记并不带来更好性能。
- **令牌数不是推理质量的代理**：效率η与令牌数几乎不相关（r=-0.05, p=0.64）。
- **η优于简单图统计**：η是唯一与准确率正相关且与令牌数无关的指标，而图宽度、|V|、令牌数都与难度相关且互相关。
- **额外令牌主要用于验证开销**：令牌数与|V_ver|/|V_sol|强正相关（r=0.53, p<0.001）。
- **效率反映对解的专注并惩罚图膨胀**：η与解支持部分占比正相关（r=0.55），与图大小负相关（r=-0.33）。
- **正确性有帮助但结构更重要**：声明准确率仅与η中等相关（r=0.33）。
- **早期错误与低效轨迹相关**：首次错误深度与η正相关（r=0.28）。
- **某些冗余可能有益**：每个唯一声明的平均重述次数与η正相关（r=0.27），可能是“锚定”行为。
- **η信号延伸到错误轨迹**：失败轨迹的η远低于成功轨迹（57%更低），且图更大、更发散。

## 7. 优点
- **创新性方法论**：将自由文本推理轨迹转化为结构化推理图，使推理拓扑可测量，超越了仅靠准确率和令牌数的评估范式。
- **可扩展基准**：21种逻辑谜题，支持难度可控扩展，避免数据污染和基准饱和。
- **新颖效率指标η**：基于马尔可夫链的信息论方法，能够区分专注推理与发散探索，且与令牌数无关，与准确率和结构质量相关。
- **自动化流程**：图提取完全自动，谜题无关的部分（图构建、效率计算）可迁移至其他领域（数学推理、代码生成、智能体工具使用）。
- **验证机制**：声明通过可执行谜题环境确定性验证，保证了结构分析的可靠性；LLM提取的稳定性经过多次消融检验。
- **全面消融分析**：包括提取器鲁棒性、参数灵敏度、图扰动稳定性等，增强了结果可信度。

## 8. 不足与局限
- **LLM组件的偏差**：声明和规则提取依赖LLM（GPT‑5.2和GPT‑5‑mini），不是完全静态；尽管做了消融，但仍可能引入微小偏差。
- **声明类型和规则是谜题特定的**：每个谜题族需要领域特定的声明和规则类型，框架的可迁移性仅限于“图构建和η计算”这一层，验证必须重新实现。
- **图提取的脆弱性**：对于几乎没有显式推理的轨迹，无法提取有意义的图；此时η失去意义。
- **实验覆盖有限**：结构化分析仅在Tents谜题上完成，未扩展到全部21种谜题；其他谜题可能存在不同行为。
- **模型覆盖有限**：仅分析了三个开源模型；闭源模型（如GPT‑5）无法提取轨迹，无法全面对比。
- **边缘验证的局限性**：规则应用仅通过人工抽样检查200条（75.5%正确），并非完全自动化验证；部分错误源于缺失前提而非虚假边。
- **无法验证的状态相关声明**：依赖于部分棋盘状态的声明被排除验证（约15%），可能轻微延迟首次错误检测并削弱η与准确率的相关性。
- **解码策略固定**：仅使用T=1；作者承认低温可能导致重复循环，且策略可能改变模型排名，但未探索替代策略。
- **无法区分“有用冗余”与“无用冗余”**：η对重构和验证部分都记作分散，可能对合理的验证步骤也给予较低效率分数。
