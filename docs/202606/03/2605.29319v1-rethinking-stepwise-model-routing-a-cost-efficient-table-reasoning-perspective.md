## 摘要
大型推理模型（LRMs）在表格推理任务上表现出色，但由于其较长的推理轨迹，推理成本高昂。逐步模型路由通过动态地将推理步骤分配给较小或较大的模型来缓解这一问题。然而，针对表格推理的逐步模型路由仍未得到充分探索。通过实证分析，我们发现涉及表格的推理步骤包含两种具有不同不确定性分布的标记：基于表格结构的标记（如单元格值和标题）和表示周围自然语言推理的文本标记。两种标记类型的不确定性与模型在下一步推理中出错的风险相关。然而，现有方法未能分别对它们进行建模，导致了次优的路由决策。为解决此问题，我们提出EcoTab，一种针对高效表格推理的表格感知逐步路由框架。在每个推理步骤中，EcoTab分别估计表格标记和文本标记的不确定性，将它们映射到小模型下一步的失败风险，并将两种风险结合起来进行路由决策。在多个表格推理基准上的实验表明，EcoTab始终优于强基线，并在准确性和效率之间实现了更好的平衡。

## Abstract
Large Reasoning Models (LRMs) achieve strong performance on table reasoning tasks but incur substantial inference cost due to long reasoning traces. Stepwise model routing mitigates this issue by dynamically assigning reasoning steps to smaller or larger models. However, stepwise model routing for table reasoning remains underexplored. Through empirical analysis, we find that reasoning steps involving tables contain two types of tokens with distinct uncertainty distributions: table tokens grounded in table structure, such as cell values and headers, and text tokens representing surrounding natural-language reasoning. The uncertainty of both token types is correlated with the risk that the model makes an error in the next reasoning step. However, existing methods fail to model them separately, leading to suboptimal routing decisions. To address this, we propose EcoTab, a table-aware stepwise routing framework for efficient table reasoning. At each reasoning step, EcoTab separately estimates the uncertainties of table tokens and text tokens, maps them to next-step failure risks for the small model, and combines the two risks for routing. Experiments on multiple table reasoning benchmarks show that EcoTab consistently outperforms strong baselines and achieves a better balance between accuracy and efficiency.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
EcoTab在每个推理步骤中，将令牌分为表格令牌和文本令牌，分别计算其平均熵（Shannon entropy）作为不确定性度量，然后通过离线拟合的sigmoid映射转换为下一步失败风险，最后使用Noisy-OR融合两个风险得到路由分数，与阈值比较决定下一步由SRM还是LRM生成。

### 关键技术细节
1. **表格Trie构建**：对输入表格内容（列标题、单元格值）进行归一化（小写化、去空格、标准化数字和标点）后构建词级Trie。对每个推理步骤，使用最长前缀匹配识别表格令牌，得到布尔掩码，从而划分V_tab和V_text。
2. **步骤级不确定性估计**：
   - 令牌级不确定性：\(c_{i,j} = -\sum_{v \in V} p_{i,j}(v) \log p_{i,j}(v)\)
   - 步骤级不确定性：分别对表格令牌集和文本令牌集的令牌级熵求平均，得到 \(\Phi^{(i)}_{\text{tab}}\) 和 \(\Phi^{(i)}_{\text{text}}\)。
3. **路由分数（Noisy-OR）**：
   - 离线构建两个sigmoid风险映射：\(d^{(i)}_{*} = \sigma(a_* \Phi^{(i)}_{*} + b_*)\)，其中 \(* \in \{\text{tab}, \text{text}\}\)，参数通过验证集上的交叉熵学习，监督标签通过后缀替换法（suffix replacement）构造（确定触发LRM切换的临界步骤）。
   - 融合风险：\(d^{(i)}_{\text{final}} = 1 - (1 - d^{(i)}_{\text{tab}})(1 - d^{(i)}_{\text{text}})\)。
   - 比较 \(d^{(i)}_{\text{final}}\) 与阈值 \(\tau\) 决定下一步模型。
4. **整体流程**：
   - 初始步骤由SRM生成，计算第一步骤路由分数，若超过阈值则重生成（由LRM做第一步精炼）。
   - 后续迭代：根据当前步骤路由分数选择模型生成下一个步骤，直至产生最终答案或达到迭代限制。

## 6. 主要结论与发现

1. 现有自由文本路由方法在表格推理上效率低下，因为无法正确路由表格特定步骤（表检索、表操作），导致接近80%的LRM FLOPs才能达到LRM-Only准确率。

3. EcoTab通过分别建模两类令牌的不确定性、映射为失败风险并Noisy-OR融合，在多个表格推理基准上显著优于所有基线，实现了更好的准确率-效率权衡。
4. EcoTab具有轻量级路由开销（与STEER相当），并能自动根据样本难度分配计算资源。

## 7. 优点

- **新颖的视角**：首次系统研究表格推理的逐步模型路由，发现并利用表格令牌与文本令牌的差异性。

## 8. 不足与局限

- **无法控制输出长度**：路由决策仅选择下一步模型，无法强制推理过早停止，仍可能产生不必要的长推理轨迹。作者建议未来引入表格感知的早停机制。
