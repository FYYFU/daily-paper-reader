## 摘要
混合智能体（Mixture-of-Agents, MoA）架构通过将多个LLM智能体组织成层级推理流水线来改进推理时缩放。然而，现有的MoA变体随着深度增加无法持续保持增益，表现出退化、早期平台期或饱和。我们提出ReM-MoA，一种记忆增强的MoA框架，通过两种机制维持缩放：（1）一种分级推理记忆，使用比较型评审智能体持久存储并对所有层的推理轨迹进行排序；（2）一种精选多样化记忆路由方案，将不同的智能体暴露于成功和失败轨迹的不同组合中，在传播高质量推理的同时保持探索多样性。我们进一步引入可选的多领域评审蒸馏流水线，通过前沿模型监督提高排序质量。在涵盖数学、形式逻辑、代码、知识和常识的五个推理基准上，ReM-MoA在深度和宽度缩放上均持续优于先前的MoA变体，且其优势随深度增加而扩大，建立了结构化的跨层推理记忆作为可扩展多智能体推理的关键缺失机制。

## Abstract
Mixture-of-Agents (MoA) architectures improve inference-time scaling by organizing multiple LLM agents into layered reasoning pipelines. However, existing MoA variants fail to sustain gains as depth increases, exhibiting degradation, early plateauing, or saturation. We propose ReM-MoA, a memory-augmented MoA framework that sustains scaling through two mechanisms: (1) a Ranked Reasoning Memory that persistently stores and ranks reasoning traces from all layers using a comparative Reviewer Agent, and (2) a Curated Diversified Memory Routing scheme that exposes different agents to distinct combinations of successful and failed traces, preserving exploration diversity while propagating high-quality reasoning. We further introduce an optional multi-domain Reviewer distillation pipeline that improves ranking quality through frontier-model supervision. Across five reasoning benchmarks spanning math, formal logic, code, knowledge, and commonsense, ReM-MoA consistently outperforms prior MoA variants across both depth and width scaling, and its advantage widens with depth, establishing structured cross-layer reasoning memory as a key missing mechanism for scalable multi-agent inference.

## 1. 核心问题与整体含义（研究动机和背景）
- **背景**：Mixture-of-Agents (MoA) 架构通过将多个LLM智能体组织成层级推理流水线来提升推理时的缩放效果。理论上增加深度和宽度可带来可扩展的集体推理能力，但实践中现有MoA变体在深度增加时无法持续增益，表现出三种失败模式：
  - **退化**：标准MoA在浅层深度达到峰值后性能下降（如MMLU-redux上损失近10个百分点）。
  - **早期平台期**：RMoA通过相邻层残差避免退化，但无法检索非相邻早期层的高质量推理，较早饱和。
  - **饱和**：AttentionMoA通过层内注意力改进，但智能体间收敛导致探索多样性崩溃，进一步加深时增益接近于零。
- **根本原因**：原始层间拼接传播错误；仅局部通信无法保留早期强推理轨迹；所有智能体消费相似累积上下文导致多样性崩塌。
- **核心动机**：需要一个既能跨层保留高质量推理、又能维持层内智能体间探索多样性的框架，联合解决质量与多样性问题，以实现可持续的MoA缩放。

## 2. 方法论
### 2.1 核心思想
将MoA缩放视为**质量-多样性联合保持问题**，而非单纯的聚合或交互问题。通过结构化跨层推理记忆和差异化路由同时满足两类要求：
- **质量保持 (Q)**：以评分排序形式保留每层推理轨迹，并可跨任意层距离访问，使深层构建于高质量推理之上。
- **多样性保持 (D)**：同一层内不同智能体收到记忆中的不同参考集，避免所有智能体共享相同输入而收敛到冗余路径。

### 2.2 关键组件
- **Ranked Reasoning Memory (分级推理记忆)**：一个只读追加的共享子系统，记录每层产生的N个轨迹及其由Reviewer Agent分配的整数分数（1-10）和针对性理由。记忆可被所有后续层查询，不仅限于相邻层。
- **Reviewer Agent**：基于LLM的比较型评审器，每一层对层内N个轨迹进行相互比较评分（同时给出理由）。采用比较评分而非独立评分以获得更可靠结果。
- **Curated Diversified Memory Routing (精选多样化记忆路由)**：从第2层开始，每个proposer agent收到一个精心策划的参考集（从记忆中选择），同一层内不同agent获得不同的子集。路由函数来自三个家族：
  - **对比型**：Top a ∪ Bot b（a+b=n）
  - **正强化型**：Top a（只选高分）
  - **失败模式聚焦型**：Bot a（只选低分）
  例如N=3时：一个agent获得Top1∪Bot1，一个获得Top2，一个获得Bot2。更大N时有系统扩展。
- **可选的多领域Reviewer蒸馏**：用GPT-5.5作为教师，对ReM-MoA自身运行中产生的（问题、轨迹对）进行高质量评分-理由标注；然后用LoRA微调Qwen2.5-7B-Instruct（学生）来模仿教师。训练数据覆盖多个推理领域，使学生学习通用的判断能力。

### 2.3 流程说明
1. 第一层：N个proposer直接根据问题生成轨迹。
2. 每层生成后，Reviewer Agent对所有N个轨迹进行评分和理由生成，存入记忆。
3. 第l层（l≥2）：每个proposer根据差异化路由获得参考集（包含轨迹+分数+理由），结合问题生成新轨迹。
4. 最终层后，聚合器同样从记忆中获得参考集，合成最终答案。

## 6. 主要结论与发现
- **深度缩放**：ReM-MoA在所有层数上均优于所有基线，且优势随深度增加而扩大。例如在L=9时，ReM-MoA在Formal Logic上比AttentionMoA高7.3个百分点，在MATH上高4.1个百分点。基线高峰后退化或饱和，ReM-MoA持续提升。
- **宽度缩放**：ReM-MoA同样在宽度上领先，但差距增长较深度更平缓。
- **蒸馏增强**：ReM-MoA*进一步提升了性能，在三个基线饱和的基准（MMLU-redux、HellaSwag）上将趋势从↑→转为↑↑，实现全面可持续缩放。
- **消融分析**：跨层访问是最大贡献因素（L=9时平均损失-4.28），路由多样化其次（-2.50），理由提供微小但持续的改进（-1.52）。证明记忆+差异化路由是可持续缩放的核心。
- **蒸馏泛化**：LOO（未见领域）恢复Multi-FT平均增益的44%，表明蒸馏学习了可转移的评价技能。

## 7. 优点
- **问题诊断清晰**：通过实验揭示了三种具体失败模式（退化、平台期、饱和），为设计提供了明确方向。
- **设计简洁有效**：仅增加一个比较评审器和差异化路由，即实现了持续的缩放增益，改动小且机制可解释。
- **多样性保持机制新颖**：不是让所有智能体共享相同记忆，而是通过差异化路由维持层内探索多样性，解决了先前方法忽略的关键点。
- **蒸馏实用性**：利用自身运行轨迹进行蒸馏，使得评审器学会判断自身分布中的轨迹质量，且多领域训练带来跨域泛化，开销可控（仅一次性训练成本）。
- **实验全面**：同时评估深度和宽度两种缩放轴，并在多个推理领域验证，结论稳健。

## 8. 不足与局限
- **宽度探索有限**：宽度仅到N=8，且深度固定为L=3，未探索更大宽度或更高深度下的行为，可能掩盖新的瓶颈。
- **单尺度Proposer池**：仅使用7-8B模型，未验证在更大（如30-70B）或更小（1-3B）模型上的可迁移性。质量-多样性权衡可能随模型尺度变化。
- **Reviewer额外开销**：每层额外一次Reviewer调用（L+1次），虽然低于AttentionMoA的2L·N，但仍带来计算成本。蒸馏虽是一次性成本，但需访问GPT-5.5等前沿模型，并非所有场景可用。
- **继承LLM偏见**：Reviewer Agent作为LLM可能存在位置偏见、自偏好等已知问题，对比评分和对比路由虽能缓解但未完全消除。在重要场景需额外安全措施。
- **成本限制**：框架每次查询需L·N+L+1次LLM调用，能效高于单次生成，但在不需要深度缩放的任务中可能得不偿失。
