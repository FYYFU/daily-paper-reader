## 摘要
混合智能体（MoA）架构通过将多个大语言模型智能体组织成分层推理流程来改进推理时扩展。然而，现有MoA变体随着深度增加无法持续保持收益，表现出退化、早期平台期或饱和。我们提出ReM-MoA，一种记忆增强的MoA框架，通过两种机制维持扩展：（1）排序推理记忆，使用比较评审智能体持续存储并对来自所有层的推理轨迹进行排序；（2）策划多样化记忆路由方案，使不同智能体接触到成功和失败轨迹的不同组合，在传播高质量推理的同时保持探索多样性。我们进一步引入可选的跨领域评审蒸馏流程，通过前沿模型监督提高排序质量。在涵盖数学、形式逻辑、代码、知识及常识的五个推理基准上，ReM-MoA在深度和宽度扩展上均一致优于先前的MoA变体，且其优势随深度增加而扩大，确立了结构化跨层推理记忆作为可扩展多智能体推理的关键缺失机制。

## Abstract
Mixture-of-Agents (MoA) architectures improve inference-time scaling by organizing multiple LLM agents into layered reasoning pipelines. However, existing MoA variants fail to sustain gains as depth increases, exhibiting degradation, early plateauing, or saturation. We propose ReM-MoA, a memory-augmented MoA framework that sustains scaling through two mechanisms: (1) a Ranked Reasoning Memory that persistently stores and ranks reasoning traces from all layers using a comparative Reviewer Agent, and (2) a Curated Diversified Memory Routing scheme that exposes different agents to distinct combinations of successful and failed traces, preserving exploration diversity while propagating high-quality reasoning. We further introduce an optional multi-domain Reviewer distillation pipeline that improves ranking quality through frontier-model supervision. Across five reasoning benchmarks spanning math, formal logic, code, knowledge, and commonsense, ReM-MoA consistently outperforms prior MoA variants across both depth and width scaling, and its advantage widens with depth, establishing structured cross-layer reasoning memory as a key missing mechanism for scalable multi-agent inference.

# ReM-MoA: 论文详细总结

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：Mixture-of-Agents (MoA) 架构通过将多个大语言模型（LLM）智能体组织成分层推理管线，利用推理时计算扩展来提升性能。理论上增加层数（深度）和每层智能体数量（宽度）应能持续提升集体推理能力，但实际中现有MoA变体在深度增加时性能退化、过早进入平台期或饱和，无法持续扩展。
- **核心动机**：识别出三种导致扩展失败的关键机制：
  - 直接层间拼接会同时传播有用推理和累积错误；
  - 仅依赖局部层间通信的架构无法保留早期层产生的高质量推理轨迹；
  - 智能体消费越来越相似的累积上下文，探索多样性崩塌，导致冗余解路径。
- **研究目标**：设计一种能同时保持跨层推理质量与智能体多样性的MoA框架，实现深度和宽度上的持续扩展，并将此作为多智能体推理扩展的关键缺失机制。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 将MoA扩展视为**质量与多样性联合保持问题**，而非仅聚合或交互问题。通过引入结构化**跨层推理记忆**，让后续层能够访问经过排序和筛选的先前轨迹，同时通过多样化路由保持同层智能体间的探索多样性。

### 2.2 关键组件

**（1）Ranked Reasoning Memory（排序推理记忆）**
- 每个层：`N`个提议智能体生成推理轨迹`r_l`，然后一个**Reviewer Agent**（基于LLM）对所有轨迹进行**比较式评分**（1-10分），并附带每个轨迹的针对性理由（rationale）。
- 每个层提交一个记忆条目`M_l = {(r_l,j, s_l,j, ρ_l,j)}`，累积记忆`M_≤l`包含所有先前层的条目，支持后续层跨层读取。
- 记忆是追加只读的，实现了**质量保持**：轨迹以排序形式存储而非原始拼接，且能跨任意层距离访问。

**（2）Curated Diversified Memory Routing（策划多样化记忆路由）**
- 从第2层起，每个提议智能体除了问题`x`外，还会收到一个从记忆`M_≤l-1`中提取的**策划参考集**`S_l,j`。
- 设计原则：
  - **质量感知**：包含高得分轨迹（正例）、低得分轨迹（负例）或两者皆有（对比式）。
  - **多样化**：同一层内不同智能体接收不同的参考集，以避免探索同质化。
- 路由函数分为三类：
  - **对比式**（Con）：Top a ∪ Bot b，同时给出高、低得分示例。
  - **正强化**（Top）：仅高得分。
  - **失败模式聚焦**（Bot）：仅低得分。
- 例如N=3时：一个智能体用Con 2 (Top1∪Bot1)，一个用Top 2，一个用Bot 2。更大N时在附录B中给出了分配表。

**（3）Reviewer Distillation（可选增强：评审蒸馏）**
- 默认使用现成LLM作为Reviewer，可选通过**前沿模型蒸馏**增强。
- 用ReM-MoA自身在训练问题上运行收集轨迹，然后让强教师模型（GPT-5.5）对每个（问题，轨迹）对生成评分和理由作为监督标签。
- 用**LoRA**（参数量仅0.18%）微调较小的基础Reviewer（Qwen2.5-7B-Instruct），在多个推理领域上联合训练，使其能泛化到未见过领域。

**（4）聚合器**：最后一层后，聚合器从记忆中通过三类路由获取参考集，生成最终答案。

### 2.3 流程（文字说明）
1. 第一层：每个提议智能体只接收问题，生成推理轨迹。
2. Reviewer对第一层的N个轨迹进行比较评分并生成理由，存入记忆。
3. 后续每层：根据路由方案，每个提议智能体收到问题 + 各自的策划参考集（包含轨迹、分数、理由），生成新轨迹。
4. Reviewer再次评分并入记忆。
5. L层后，聚合器从记忆中选择参考集，综合生成最终答案。

## 3. 实验设计

## 6. 主要结论与发现

1. **ReM-MoA在所有配置下均优于所有基线**，且优势随深度增加而扩大（gap从L=1时的0.84%增长到L=9时的4.92%，相对最强基线）。
2. **标准MoA在深度增加时退化**（如MMLU-redux下降9.1个百分点）；RMoA早停；AttentionMoA饱和。ReM-MoA则持续提升（↑↑趋势）。
3. **消融证实关键组件**：跨层访问丢失导致最大退化（−4.28平均，峰值−6.2）；路由多样化次之（−2.50）；理由移除影响较小（−1.52），但对趋势影响不大。
4. **蒸馏带来额外提升**：Multi-FT相比默认Reviewer平均提升+1.92（L=9）；LOO（未见域）恢复44%的提升，证明蒸馏学到了可迁移的评估技能。
5. **在知识密集型/常识类基准中（MMLU-redux、HellaSwag），增益虽然较小但仍为正，且蒸馏后能在HellaSwag上转为持续扩展（从↑→变为↑↑）。**

## 8. 不足与局限

- **宽度探索有限**：最大宽度N=8，虽然匹配前人工作，但未展示更大N下的行为（是否存在饱和或退化？）。
- **提议模型规模单一**：仅使用7-8B模型，未验证在更大（30B-70B）或更小（1-3B）规模上的迁移性，质量和多样性权衡可能变化。

- **应用限制**：当前主要针对单个问题实例内的跨层记忆，而非跨任务累积学习；任务间知识迁移未探索。
- **统计不确定性未报告**：实验虽多次采样（温度>0），但未给出置信区间或方差，结果可靠性可能受随机性影响。
