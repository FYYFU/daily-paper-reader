## 摘要
大型语言模型（LLM）智能体通过将语言模型与外部工具和环境集成，日益自动化地执行复杂任务。然而，它们的自主性带来了显著的安全风险：智能体可能执行破坏性指令、泄露敏感数据或违反领域约束。现有的安全方法面临一个基本的权衡：手工制定的规则具有可解释性但脆弱，过于保守的规则会阻止安全操作（高误报率），而宽松的规则则会遗漏不安全行为（高漏报率）。神经分类器缺乏安全关键部署所需的可解释性。我们提出了AutoSpec框架，该框架通过反例引导的归纳综合（CEGIS），结合归纳逻辑编程（ILP），从用户安全/不安全标注中自动演化出已部署的专家设计安全规则。从专家规则和带标注的轨迹流开始，AutoSpec迭代评估规则，挖掘误报和漏报的反例，利用ILP学习哪些谓词能够区分它们，生成候选规则编辑，并验证候选规则以选择最佳修订版本。关键洞察在于，ILP能够高效识别那些在漏报中频繁出现但在误报中很少出现（或反之）的谓词，从而大幅剪枝规则编辑的指数级搜索空间。此过程持续直至收敛，生成平衡精度与召回率的可解释规则。我们在涵盖代码执行和具身智能体两个领域的291条执行轨迹上评估了AutoSpec。AutoSpec将两个领域的规则F1分数分别提升至0.98和0.93，在保持高召回率的同时实现了高达94%的误报率降低，并在4-5次迭代内收敛。与启发式CEGIS相比，ILP引导的方法实现了高达4.8倍的F1提升。学习到的规则具有可读性、可审计性，并能泛化到未见场景。

## Abstract
Large language model (LLM) agents increasingly automate complex tasks by integrating language models with external tools and environments. However, their autonomy poses significant safety risks: agents may execute destructive commands, leak sensitive data, or violate domain constraints. Existing safety approaches face a fundamental tradeoff: hand-crafted rules are interpretable but brittle, with overly conservative rules blocking safe operations (high false positives) while permissive rules miss unsafe behaviors (high false negatives). Neural classifiers lack the interpretability required for safety-critical deployments.   We present AutoSpec, a framework that automatically evolves deployed expert-designed safety rules from user safe/unsafe annotations through counterexample-guided inductive synthesis (CEGIS) guided by inductive logic programming (ILP). Starting from the expert rules and a stream of annotated traces, AutoSpec iteratively evaluates rules, mines false-positive and false-negative counterexamples, uses ILP to learn which predicates discriminate them, generates candidate rule edits, and verifies candidates to select the best revision. The key insight is that ILP efficiently identifies predicates that appear frequently in false negatives but rarely in false positives (or vice versa), dramatically pruning the exponential search space of rule edits. This continues until convergence, producing interpretable rules that balance precision and recall.   We evaluate AutoSpec on 291 execution traces spanning code execution and embodied agent domains. AutoSpec raises rule F1 to 0.98 and 0.93 across the two domains, achieving up to 94% false positive reduction while maintaining high recall, and converges within 4-5 iterations. The ILP-guided approach achieves up to 4.8x higher F1 than heuristic CEGIS. The learned rules are human-readable, auditable, and generalize to unseen scenarios.

- **现有方法的局限**：
  - **静态规则**：可解释但脆弱，保守规则导致高误报（FP），宽松规则导致高漏报（FN），且需专家持续维护。
  - **神经分类器**：适应性强但黑箱，缺乏可解释性，不适合安全关键部署。
- **关键观察**：部署的智能体会产生大量工具调用轨迹和用户标注（安全/不安全），这些反馈可以用来自动演化专家初始设计的规则，而非从头编写或丢弃。
- **解决的问题**：如何从持续积累的标注轨迹中自动、可解释地更新安全规则，实现规则的自适应演进，同时保持可审计性。

## 2. 方法论：核心思想、关键技术细节

- **总体框架**：将规则演化建模为**反例引导的归纳综合（CEGIS）** 问题，利用**归纳逻辑编程（ILP）** 高效引导搜索空间。
- **输入**：初始规则集 \(R_0\)、带标签的轨迹数据集 \(D_L\)（每个轨迹标记为安全/不安全）、谓词库 \(P\)（如 `destructive_os_inst`，`near_danger_zone`）。
- **输出**：精炼的规则集 \(R^*\)，以 F1 为主要优化目标。

### 四个阶段（每轮迭代）：

1. **阶段 1：评估与反例挖掘**
   - 用当前规则集 \(R_i\) 评估 \(D_L\)，得到预测标签。
   - 与真实标签比较，提取**漏报（FN）** 和**误报（FP）** 作为反例。
   - FN 指示规则过窄（需要放宽），FP 指示规则过宽（需要收紧）。

2. **阶段 2：ILP 引导的谓词学习**
   - 将 FN 作为正例，FP 作为负例，利用 ILP 求解器（ILASP）学习最简逻辑假设 \(H\)，形式为：\((p_{11} \land p_{12} \land ...) \lor ...\)。
   - ILP 找出**最能区分 FN 和 FP 的谓词**，并给出排名及建议编辑方向（tighten / broaden）。
   - 通过贪婪集合覆盖选择少数高质量建议，避免冗余。

3. **阶段 3：候选生成与选择**
   - 将 ILP 建议转化为具体编辑操作（表 2）：
     - `AddConjunct`（添加合取） / `AddException`（添加例外） → 减少 FP（收紧）
     - `Relax`（删除合取） / `AddDisjunct`（添加析取分支） → 减少 FN（放宽）
   - 三种生成策略：
     - **单步编辑**：应用单个建议。
     - **混合修复链**：先收紧后放宽，处理 FN 修复引入新 FP 的情况。
     - **失败驱动分支**：针对残留 FN 添加高精度析取分支。
   - 对所有候选在 \(D_L\) 上评估，按字典序选择最优：**F1 → 精确率 → -未解决反例数 → -规则数量**。

4. **阶段 4：验证与收敛**
   - 如果候选得分优于当前规则，则更新 `R ← best`，否则停止。
   - 终止条件：达到目标阈值 / 无改进 / 达到最大迭代次数 \(K\)。
   - 加入**软新颖性压力**：对已包含在规则中的谓词降权，避免重复建议。

### 关键算法（Algorithm 1）：
- 输入：初始规则、标签轨迹、谓词库、最大迭代 \(K\)、目标得分 \(\theta\)。
- 循环直至收敛或达到上限：
  1. 挖矿反例。
  2. ILP 学习建议。
  3. 生成候选并选最优。
  4. 若更优则更新，否则返回。

## 7. 优点

- **方法创新**：首次将 ILP 与 CEGIS 结合应用于安全规则演化，巧妙利用反例信号自动修剪指数搜索空间。
- **保持可解释性**：最终规则为逻辑谓词组合，可直接部署、审计和手动微调。
- **极低的资源需求**：单 CPU 机器运行，无需 GPU，适合离线异步更新。
- **实用设计**：直接对接现有 AgentSpec 框架，编辑操作简单且语义清晰；软新颖性压力避免循环。
- **用户中心验证**：不仅报告数值指标，还评测从业者的主观感受，强化实际可用性论证。

## 8. 不足与局限

1. **领域覆盖有限**：仅测试代码执行和具身智能体，其他常见领域（如 Web 代理、数据库代理）未涉及。
2. **数据规模较小**：总计 291 条轨迹，每个域约 100-191 条；更大规模、更多样化的轨迹集合可能暴露更多失败模式。
3. **谓词库依赖性**：规则的质量严重依赖谓词库的完备性；若关键谓词缺失，AutoSpec 无法学习。
4. **无全局最优保证**：算法采用贪心上升保证局部最优，不保证全局最优；理论收敛性质为 NP-hard。
5. **时序与状态依赖**：当前谓词为单事件属性；更复杂的时序模式（如 LTL）需在谓词内编码，论文未原生支持。
6. **标注偏差风险**：轨迹标签依赖人工判断，可能存在不一致或错误，影响学习效果。
7. **对比基线局限**：未包含基于微调的神经分类器（如 Llama Guard 微调版本），以及更先进的符号规则学习系统。
8. **缺少对成本（标注量）的分析**：未讨论达到目标性能所需最小标注量。
