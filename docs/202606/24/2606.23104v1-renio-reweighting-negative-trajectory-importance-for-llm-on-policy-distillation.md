## 摘要
在线策略蒸馏（OPD）通过在模型自身生成的输出上训练学生模型来提升大语言模型的推理能力，但标准 OPD 对所有学生生成输出（SGOs）一视同仁，而不考虑其信息量。我们在受控筛选实验中发现了一致的不对称性：在 OPD 和在线策略自蒸馏（OPSD）中，仅使用错误 SGOs 进行训练的效果优于仅使用正确 SGOs。进一步分析表明，仅用正确 SGOs 训练的模型倾向于生成更短的推理链，并表现出较弱的反思行为，而错误 SGOs 则更好地保留了接近模型能力边界的探索性推理。为了利用这一信号而无需完整包含答案的轨迹，我们引入了 ReNIO，它通过重加权负面轨迹重要性来改进大语言模型在线策略蒸馏。通过使用学生与教师的概率比，ReNIO 识别出导致错误推理链的关键 token，并将其信息聚合为归一化的样本权重，从而在无需观察最终答案正确性的情况下，自然地赋予可能为负面轨迹更大的权重。由于 ReNIO 仅使用前缀条件化的 token 概率，它保留了 OPD 相对于完整轨迹强化学习的前缀训练优势。在数学推理和代码生成任务中，ReNIO 同时提升了 OPD 和 OPSD，在数学推理基准上，Qwen3-1.7B 的相对增益达到 8.90%，R1-Distill-Qwen-7B 达到 10.00%。代码仓库：https://github.com/BDML-lab/ReNIO。

## Abstract
On-policy distillation (OPD) improves LLM reasoning by training a student model on its own generated outputs, but standard OPD treats all student-generated outputs (SGOs) equally regardless of their informativeness. We observe a consistent asymmetry in controlled filtering experiments: in both OPD and on-policy self distillation (OPSD), training only on incorrect SGOs outperforms training only on correct ones. Our further analysis suggests that models trained on correct-only SGOs tend to generate shorter reasoning traces and show weaker reflection behavior, while incorrect SGOs better preserve exploratory reasoning near the model's capability boundary. To exploit this signal without requiring full answer-containing rollouts, we introduce ReNIO, which Reweights Negative trajectory Importance for LLM On-policy distillation. By using the student-to-teacher probability ratio, ReNIO identifies pivotal tokens leading to wrong reasoning traces and aggregates their information into a normalized sample weight, inherently assigning larger weights to likely negative trajectories without observing the correctness of final-answer. Since Re-NIO only uses prefix-conditioned token probabilities, it preserves OPD's prefix training advantage over full-rollout reinforcement learning. Across both mathematical reasoning and code generation tasks, ReNIO improves both OPD and OPSD, with representative relative gains of up to 8.90% for Qwen3-1.7B and 10.00% for R1-Distill-Qwen-7B on mathematical reasoning benchmarks. Code repo: https://github.com/BDML-lab/ReNIO.

## 1. 核心问题与整体含义

- **研究背景**：在线策略蒸馏（OPD）是提升大语言模型推理能力的主流范式之一，它与强化学习（RL）相比具有密集监督信号和训练成本优势（可在短前缀上训练，无需完整答案轨迹）。然而，标准 OPD 和在线策略自蒸馏（OPSD）对所有学生生成输出（SGOs）赋予相同权重，忽略了不同 SGO 所含学习信号量的巨大差异。
- **核心问题**：如何在不依赖最终答案正确性的前提下，有效识别并强调那些对提升推理能力更有价值的（尤其是错误的）轨迹，从而更高效地利用 OPD 的短前缀训练优势。
- **整体含义**：论文发现并验证了一个反直觉的不对称性——仅用错误 SGOs 训练优于仅用正确 SGOs，并据此提出一种基于前缀可计算信号（学生-教师概率比）的样本级重加权方法 ReNIO，在保持计算效率的同时显著提升推理性能。

## 2. 方法论

- **核心思想**：利用学生-教师概率比（student-to-teacher log ratio）定位导致错误推理的关键 token（学生高概率、教师低概率的 token），将这些 token 的信息聚合为样本权重，从而在无需观察最终答案的情况下，自动赋予可能为负面（错误）的轨迹更大的训练权重。
- **关键技术细节**：
  - **Token 级重要性信号**：每个 token 的 `ℓ_t = log( π_S(y_t|x,y_<t) / π_T(y_t|x,y_<t) )`。
  - **关键 Token 选择**：设定固定阈值 τ，保留 ℓ_t > τ 的 token 构成集合 `K(x,y)`，过滤掉多数对齐良好的 token。
  - **聚合权重**：对集合内 token 的 log 比率取几何平均（即算术平均后指数化）：`w(x,y) = exp( (1/|K|) Σ_{t∈K} ℓ_t )`。若集合为空则 w=1。
  - **批归一化**：每批内归一化，使平均权重为 1，保持梯度尺度不变：`ŵ = w / (1/B Σ w_i)`。
- **算法流程**（文字说明）：
  1. 对每个 SGO 计算所有 token 的 ℓ_t；
  2. 选出 ℓ_t > τ 的 token 作为关键 token；
  3. 对关键 token 的 ℓ_t 进行截断（clip），然后取平均并指数化得到初始权重 w；
  4. 若无关键 token，则 w=1；
  5. 每批内对 w 进行归一化得到最终权重 ŵ；
  6. 将 ŵ 作为样本权重应用到蒸馏损失函数中：`L_ReNIO = E[ ŵ·D(π_S||π_T) ]`。

## 6. 主要结论与发现

- 在 OPD 和 OPSD 中，**仅用错误 SGOs 训练始终优于仅用正确 SGOs**，因为错误轨迹有助于维持模型探索性推理和反思行为，而正确轨迹可能导致模型过于自信、推理链缩短。
- ReNIO 通过学生-教师概率比识别关键 token 并重加权，在数学推理和代码生成任务上一致提升了 OPD 和 OPSD 性能，最高相对增益达 10.00%（R1-Distill-Qwen-7B 在 AIME25），且能稳定在自蒸馏信号噪声较大时的训练。
- ReNIO 保留了 OPD 的短前缀训练优势（无需完整答案），在 1024 token 前缀下效果甚至优于部分 4096 token 设置，且计算成本显著低于 GRPO。
