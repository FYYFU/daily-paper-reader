## 摘要
语言模型推理可以通过在测试时利用脚手架来显著提升，该脚手架将推理计算扩展到不同的原语上——包括单个追踪链内的顺序推理、独立采样的并行追踪链、以及将多个推理追踪链聚合为最终响应。然而，在后训练阶段，语言模型仅针对单个追踪链内的顺序推理进行优化。我们提出了序列-并行-聚合强化学习（SPIRAL），这是一个框架，其中语言模型被训练以使用所有三种原语，作为统一推理计算流水线的一部分。具体来说，语言模型首先并行采样一组独立的追踪链，每个追踪链通过顺序的思维链推理生成，然后基于这些追踪链生成最终的聚合追踪链；所有组件都针对最终聚合响应的奖励进行端到端优化。为了训练该系统，SPIRAL使用集合强化学习来教导模型生成一组对聚合器整体有用的追踪链，并使用标准强化学习来教导模型将集合聚合为改进的最终响应。我们在推理任务上的实验表明，当所有三种计算原语都进行扩展时，SPIRAL能够有效地随推理计算扩展，在扩展效率上比GRPO高出高达11倍，在性能上高出15%。

## Abstract
Language model reasoning can be substantially improved at test time via scaffolds that scale inference compute across different primitives -- sequential reasoning within a trace, independently sampled parallel traces, and aggregation of multiple reasoning traces into a final response. During post-training, however, language models are optimized only for sequential reasoning within a single trace. We introduce Sequential-Parallel-Aggregative Reinforcement Learning (SPIRAL), a framework in which a language model is trained to use all three primitives, as part of a unified inference compute pipeline. Concretely, the language model first samples a set of independent traces in parallel, each produced through sequential chain-of-thought reasoning, and then generates a final aggregation trace conditioned on those traces; all components are optimized end-to-end against the reward of the final aggregated response. To train this system, SPIRAL uses set reinforcement learning to teach models to produce a set of traces that are collectively useful for an aggregator and standard reinforcement learning to teach models to aggregate the set into improved final responses. Our experiments on reasoning tasks show that SPIRAL effectively scales with inference compute, outperforming GRPO by up to 11$\times$ scaling efficiency and 15% higher performance when all three compute primitives are scaled.

### 1. 核心问题与整体含义（研究动机和背景）

- **瓶颈**：当前语言模型的后训练（post-training）仅针对单一轨迹内的**顺序推理**（sequential compute）进行优化（如思维链），而测试时实际常使用**并行采样**（parallel compute）和**轨迹聚合**（aggregative compute）来扩展计算。这种训练与部署之间的不匹配导致模型无法有效利用额外的推理计算资源。
- **表现**：当只扩展顺序计算时，模型常将计算浪费在琐碎细节上；并行采样时陷入冗余重复；聚合时则难以筛选、验证和组合不同思路。
- **目标**：提出一个统一的训练框架，使模型学会协同使用三种推理原语（顺序、并行、聚合），自动优化整个推理流水线，从而更有效地扩展测试时计算。

### 2. 论文提出的方法论

- **核心思想**：将推理过程建模为一个两阶段的强化学习问题。
  - 第一阶段：给定问题 \(x\)，模型 \(\pi_\theta\) 并行采样 \(n\) 个搜索轨迹 \(y_1,\dots,y_n\)（顺序推理）。
  - 第二阶段：模型以问题 \(x\) 和这些搜索轨迹为条件，生成一个聚合轨迹 \(y^*\)（聚合推理），最终奖励 \(r(x, y^*)\)。
  - 整体目标：最大化 \(\mathbb{E}_{y_{1:n}\sim\pi_\theta}[\mathbb{E}_{y^*\sim\pi_\theta(\cdot|x,y_{1:n})}[r(x,y^*)]]\)。
- **关键技术细节**：
  - **搜索轨迹优化**：使用**集合强化学习（Set RL）**。每一组搜索轨迹得到一个集合奖励（基于该组聚合轨迹的平均奖励），所有轨迹共享同一学习信号，鼓励集体有用性和多样性。
    - 实际实现中，采样 \(N_1\) 个搜索轨迹，从中无放回抽取 \(K\) 个大小为 \(n\) 的子集，每个子集采样 \(N_2\) 个聚合轨迹，计算集合优势 \(A^\sharp(x, G_i) = \hat{f}(x, G_i) - \text{baseline}\)，再为每个搜索轨迹计算边际优势 \(A^\sharp_{\text{marg}}(x, y_j)\)，用于梯度更新。
  - **聚合轨迹优化**：使用**标准强化学习（Standard RL）**。每个聚合轨迹的优势在子集内部计算（减去子集内平均奖励），以降低方差。
  - **梯度计算**：总梯度 = 集合RL梯度（搜索轨迹项） + 标准RL梯度（聚合轨迹项），两者联合优化。
  - **理论保证**：证明了当从有序元组中采样时，该梯度估计是无偏的（命题3.1）。
- **训练流程**（Algorithm 1）：
  1. 对每个问题，采样 \(N_1\) 个搜索轨迹。
  2. 随机采样 \(K\) 个不重复的大小为 \(n\) 的子集。
  3. 对每个子集，采样 \(N_2\) 个聚合轨迹，并计算奖励。
  4. 计算集合优势、边际优势，分别得到搜索和聚合轨迹的梯度。
  5. 合并梯度并更新模型参数。
