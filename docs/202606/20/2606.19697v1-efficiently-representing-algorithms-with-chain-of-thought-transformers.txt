## Abstract 

The increasing popularity of reasoning models—language models that output a series of reasoning or thought tokens before producing an answer—is justified, in part, by theoretical results showing that chain-of-thought (CoT) transformers can simulate Turing machines, and thus perform arbitrary computation. However, the Turing machine, while suitable for complexity-theoretic analysis, is not convenient, intuitive, or efficient for discussing algorithms. Algorithms are typically designed and analyzed at a higher level of abstraction, captured by the Word RAM model with random-access memory and unit-cost operations on O(log n)-bit words. As a result, Word RAM algorithms can be substantially more efficient than their Turing machine counterparts, raising the question: Can CoT transformers efficiently simulate Word RAM algorithms? For instance, can they sort n items in O(n log n) steps or run Dijkstra’s algorithm in O(E + V log V ) steps? We answer affirmatively, up to poly-logarithmic overhead. We first establish this for finite-precision transformers with poly-logarithmic width and rightmost unique hard attention, then strengthen the result to two more practical settings with finite width and log-precision: continuous 

CoT, where reasoning takes the form of vectors rather than tokens, and a hybrid 

architecture in which transformer layers sit atop a recurrent (linear RNN) layer. In all three cases, we find that CoT can efficiently simulate any Word RAM algorithm with only a poly-logarithmic overhead in n. This overhead reduces to log-square when the Word RAM has a “flat” instruction set, and only logarithmic for multiplication-free flat instructions—in stark contrast to known CoT simulations of Turing machines, which require quadratic overhead over Word RAM. 

## 1 Introduction 

It has been known for a while that chain of thought (CoT) makes transformers Turing complete —able to simulate any algorithm in principle (Pérez et al., 2021). This justifies the empirical practice of training reasoning models to solve complex tasks (e.g., Snell et al., 2024; DeepSeek-AI, 2025; OpenAI, 2026). However, the ability to simulate a Turing machine (TM) does not immediately translate to being able to implement interpretable algorithms efficiently: while TMs provide an established framework for analyzing computational complexity, they are not practical for designing real-world algorithms. For instance, sorting n items takes only O(n log n) steps on a random-access machine (RAM)—an idealized computer whose memory cells can be read and written in unit time at arbitrary addresses, formalized in §2.1—because each comparison and swap touches an arbitrary array entry in a single step. A TM has no such random access: to reach the addressed cell it must move its head there, scanning a tape region of length up to O(n + t) per access. Simulating these

This overhead is not a quirk of the analysis—it reflects a fundamental mismatch between TMs and algorithms that rely on random memory access or word-level arithmetic. This paper asks: can CoT transformers efficiently simulate algorithms as written in textbooks? We answer affirmatively. The natural model for describing textbook algorithms efficiently and concisely is the Word RAM .A Word RAM program stores data in a random-access memory (RAM) partitioned into words of 

w = Θ(log( n + t)) bits—wide enough to address any of the O(n + t) cells the program may touch during a t-step execution—and it executes a program one instruction at a time, where each instruction runs in O(1) steps. Its primitive operations—memory reads and writes, integer arithmetic, bitwise operations, and comparisons—correspond directly to the operations of modern programming languages and the algorithms textbooks describe. Thus, the standard complexity bounds practitioners know about correspond naturally to the algorithms expressed in the Word RAM model. For example, sorting takes O(n log n) Word RAM steps and Dijkstra’s algorithm takes O(E + V log V ).Existing CoT expressivity results establish computational universality by simulating Turing machines (Pérez et al., 2021; Merrill and Sabharwal, 2024). These constructions, however, operate entirely at the TM-instruction level: each CoT token encodes one tape symbol, and each decoding step simulates one TM transition. Routing a Word RAM program of t steps through this construction therefore costs 

O(t2) CoT tokens: a Word RAM on a TM already incurs an O(t2) blowup, 1 and the CoT spends one token per TM step. What is missing is a direct simulation at the Word RAM-instruction level—one that would let CoT transformers execute textbook algorithms at close to their textbook complexity. We close this gap with direct Word RAM simulations for decoder-only CoT transformers in three settings. (i) First, finite-precision transformers of polylogarithmic width using rightmost-unique hard attention—an interpretable and pedagogical construction emitting O(w2) tokens per Word RAM instruction, hence O(t w 2) = O(t log 2(n + t)) tokens overall, with the per-instruction overhead drop-ping to O(w) for multiplication-free instruction sets. This is essentially tight for discrete CoT: just as a Python program hard-coding a p-bit integer must write Ω( p) characters, any CoT transcript represent-ing a w-bit word must emit Ω( w) tokens from a fixed alphabet, giving an Ω( t w ) = Ω( t log( n + t)) 

lower bound. Our Word RAM simulation matches this bound up to one logarithmic factor in general and exactly when multiplication is excluded—already far below the Ω( t2) of TM-based CoT simula-tions. (ii) Second, fixed-width transformers with continuous CoT —which reasons with hidden vectors in place of discrete tokens. These models can invoke a TM oracle for each Word RAM instruction, generalizing the finite-precision setting. (iii) Third, hybrid models—transformers augmented with a linear RNN layer—which can implement the same simulation with explicit CoT. For an instruc-tion set whose individual operations run in time κ(w) on a TM, the latter two constructions take 

O(t κ (w)) CoT steps, generalizing the results to arbitrary instruction sets. This recovers the same poly-logarithmic overhead as the first setting: for the flat instruction sets we consider, κ(w) = O(w)

when multiplication, division, and modulo are excluded and O(w2) otherwise, so O(t κ (w)) is 

O(t log( n + t)) or O(t log 2(n + t)) accordingly.

## 2 Preliminaries 

2.1 The Word RAM Computational Model 

There are several different variants of Word RAMs in the literature with different instruction set details, ranging from C-like Word RAMs (Hagerup, 1998; Thorup, 1996; Morin, 2013) to the weaker multiplication-free basic instruction set of Brodnik et al. (1997). While classical literature has often explored various minimal , assembly-like instruction sets under which the model is Turing-complete, our goal here is different: we would like to argue that transformers and hybrid models, when equipped with CoT, can directly simulate programs even when they are written in a much higher-level language, such as “textbook algorithms” or simple Python programs, with only a log or log-square overhead. Note that this is a stronger claim than simulating a minimal instruction set with instructions like 

load , store , increment , etc., with a log or log-square overhead. Keeping this overhead small is important; after all, every Word RAM program can be converted to a Turing machine if one disregards the overhead (which, in general, would be quadratic), and then simulated using standard transformers with CoT via known constructions (Pérez et al., 2021; Merrill and Sabharwal, 2024). 

Syntax. To this end, we define the following grammar for specifying Word RAM programs. Let n be the input length, k as the maximum number of variables (realized as registers) allowed in the Word RAM program, and c1, . . . , cm as the fixed constants (independent of n) allowed in the program.

The above grammar allows nesting of expressions (e.g., mem [mem [mem [7]]]) as well as composition of logical operators over conditionals via the Cond rule. We require that the nesting depth of every expression and condition be bounded by a constant independent of n. This is what makes each line 3executable in O(1) Word RAM steps. We also consider a simplified version of this grammar that doesn’t allow nesting. This is obtained by changing the grammar expansion rules for Expr and Cond to not be recursive, as follows:           

> FlatExpr →Input | Const | Var Expr →FlatExpr | FlatExpr Op FlatExpr | UnOp FlatExpr | FlatExpr if Cond else FlatExpr Cond →Bool | FlatExpr Comp FlatExpr

We call a Word RAM program flat if it uses only the flat grammar above. We further distinguish two flat instruction classes by which operators from Op appear: full-arithmetic flat programs may use any operator from Op, while simple-arithmetic flat programs restrict Op to {+, −, ≪, ≫, &, |, ⊕} ,i.e., it excludes ×, ÷, mod .

Semantics. A specific Word RAM machine is specified by a finite-length program of the form above, with the standard semantics when executed on a string x ∈ Σ∗. In more detail, the machine has three memory arrays: a read-only one inp that contains the input x, mem is a standard read-write tape initialized to all zeros, and out is an output tape that can only be written to, where written values are stored mod |Σ|. The machine state also specifies the value of the program counter pc and finite registers r1 , . . . , rk , which all start as 0. When an instruction is run, pc is incremented, unless it is otherwise assigned by the instruction (e.g., pc = r2 + 1 ). The program runs until halt is executed, at which point the contents of out is deemed the output. When considering language recognition, we say that a Word RAM accepts a string iff out [0] = 1 .Without loss of generality, we can view the program counter pc and each register ri as stored in memory via pc ≜ mem [0] and ri ≜ mem [−i], with mem being 1-indexed for standard memory.

## 3 Simulating Word RAMs with Polylog-Width Transformers 

We begin by showing how transformers with polylogarithmic width and fixed precision can simulate Word RAMs—a stepping stone towards more uniform transformer models, which do not require new parameters as the input length grows. Thus, we use the construction in this section to cleanly outline the simulation; the main ideas will re-appear in the subsequent sections. 

The transformer model. We use a log 2-width finite-precision transformer model; see Remark 2 for a discussion of why log 2 width is needed. The constructions build on the log-width machinery formalized by existing work (Li et al., 2024; Svete et al., 2025; Svete and Sabharwal, 2026). 2 This allows us to reuse existing gadgets in our constructions. 

Representation of values. In the polylogarithmic-width setting, word-sized integers are stored in the residual stream as binary vectors denoting their binary expansions: for a value v ∈ { 0, 1}w, the

These two primitives are the key gadgets exploited throughout this section; the subsequent sections will define different representations to maintain fixed width. ## 4 Towards a Uniform Fixed-Width Construction: Bottlenecks and Plan 

In §3, storing bits( v) in the residual stream gives three things for free: (i) equality checking via inner products, (ii) bit extraction via bits( v)⊤et = bt, and (iii) a clean serialization/deserialization mechanism. Moving to fixed width forces us to abandon the bits representation, and each of these primitives must be rebuilt from scratch. We describe the resulting challenges below. 

Bottleneck: Reading and writing values. The deepest challenge in moving to fixed width is that we must adopt a different representation of numbers in the residual stream. For fixed-width constructions, any such representation repr of values must satisfy two properties simultaneously: (a) Equality gap: repr( i)⊤repr( j) = 1 if i = j, and repr( i)⊤repr( j) < 1 − ϵ otherwise. (b) Read/write: We can serialize repr( v) to a binary CoT encoding and recover repr( v) from it. The first property is needed for the indexed-read attention mechanism; the second is needed both 

(a) to pass pointers computed at a higher layer back to lower layers in future steps, and (b) to hand binary-encoded operands to the TM oracle for complex arithmetic. Past work has established the 

layer-norm hash ϕ(i) as a fixed-width representation satisfying the equality-gap property (Merrill and Sabharwal, 2024), but reading and writing via ϕ presents a challenge. 4.1 Writing Solution: Equivalent RAMs with Bounded Word Values 

We define a c-dimensional Word RAM as a Word RAM where the tape is replaced by a c-dimensional array of c-dimensional tuples. The following lemma shows that such Word RAMs are as powerful as standard Word RAMs, up to a constant factor in runtime (see §B for a proof): 

Lemma 1. Let M be a 1-dimensional Word RAM with word size w = c log( n + t) for some c ∈ N.Then there exists a c-dimensional restricted Word RAM M ′ with word size w′ = log( n + t) and the same instruction set as M that simulates t steps of M using O(t) steps. 

This shows that we can reduce any M using word size w = c log( n + t) to M ′ using word size 

w = log( n + t). Now, define a Word RAM as restricted if, for any step i, any value v represented in any register satisfies v ≤ i. Any Word RAM M can be converted to be restricted by making it do nothing for the first v steps, and then simulating the i-th step of M at step v + i:

Lemma 2. Let M be any c-dimensional Word RAM where all values are at most v(n) on inputs of size n. Then there exists M ′ that simulates t steps of M using v(n) + t steps such that, at each step i,all values in M ′ are at most i.

We can first invoke Lem. 1 and then apply Lem. 2 with v(n) = n + t to obtain the following: 

Theorem 2 (Restricted Word RAM Equivalence) . A standard Word RAM running in time t(n) on inputs of size n can be simulated by a restricted c-dimensional Word RAM in time O(n + t(n)) .

This will be useful because simulating restricted Word RAMs with transformer-like models will be easier than simulating unrestricted ones. Specifically, the property v ≤ i will be useful for writing numbers in binary as it allows us to divide numbers using attention via the following special case of the “division lemma” of Merrill and Sabharwal (2024) where the divisor is 2 (see §B for a proof): 

Lemma 3 (Simplified Division Lemma) . There exists a transformer block that, for any ϕ(v) present in the residual stream at position t for v ≤ t, adds ϕ(⌊v/ 2⌋) and ϕ(v mod 2) to the residual stream. 

This will be useful for writing numbers ϕ(v) onto the CoT in binary because it allows computing the least significant bit via ϕ(v mod 2) , or ϕ(v mod 2 i), which can be useful for recovering other bits. 

## 5 Simulating Word RAMs with Continuous CoT 

The bounded-value normal form from §4.1 ensures a crucial bit-extraction precondition: during execution, every word v at RAM step s satisfies v ≤ s. Since each simulated step produces at least one transcript symbol, this implies v ≤ τ whenever the simulator begins emitting a bit-string segment at transcript position τ . Thus, whenever we need to stream a word as bits we may apply the simplified division primitive (Lem. 3) to compute ϕ(⌊v/ 2⌋) and ϕ(v mod 2) (the first bit) from ϕ(v), and then repeat the process for the next bit by conditioning the computation on ϕ(⌊v/ 2⌋), and so on. The remaining obstacle is persistence : after streaming w discrete bits, later computation must still have access to a constant-dimensional word code that supports exact-match attention (for RAM addressing) and can be carried forward across time. We show that continuous CoT can be used to achieve both conditioning and persistence. 

Continuous CoT and soft tokens. In the continuous-CoT model, each decoding step emits both (i) a discrete token from a fixed alphabet and (ii) a constant-dimensional soft token pi ∈ Rds . The next decoding step receives the entire prefix, including the past soft tokens, so pi can be attended to by future layers. In particular, a constant-depth transformer block can copy designated coordinates of

## 7 Conclusion 

We show that poly-logarithmic-width transformers can simulate a Word RAM; while it is unclear whether fixed-width transformers can also do so, we show that two minimal extensions to fixed-width transformers (continuous CoT transformers and hybrid models) both can. Our results show that CoT—at least for certain architectures—allows simulating algorithms with their general textbook runtimes up to a polylogarithmic factor, removing the quadratic overhead that would be incurred by going through previously established Turing completeness results (Pérez et al., 2021; Merrill and Sabharwal, 2024). Since certain realistic extensions (continuous CoT or RNN layers) to transformers are crucial for the fixed-width constructions, it is an interesting open question to what degree these extensions are necessary for efficient Word RAM simulation. It would also be interesting to study to what degree these extensions benefit algorithmic reasoning in transformer models in practice.

Lemma 3 (Simplified Division Lemma) . There exists a transformer block that, for any ϕ(v) present in the residual stream at position t for v ≤ t, adds ϕ(⌊v/ 2⌋) and ϕ(v mod 2) to the residual stream. Remark 2 (Why log 2 width is necessary) . The Read step above uses w = Θ(log n) heads, each of width Θ(log n), for a total model dimension of Θ(log 2 n). Both factors are forced. 

Logarithmically many heads. To reconstruct bits( v) at a single position, one must gather w bits from w distinct earlier positions. Compressing information from a log-growing number of positions into fewer constantly-many heads is infeasible at constant precision: the attention weight at each position is a scalar, so with o(w) heads the aggregated output is a fixed-dimensional summary of o(w)

positions, which cannot faithfully recover all w bits. One could in principle design highly bespoke positional encodings (with L-uniform PEs; cf. London and Kanade (2025) and Svete and Sabharwal (2026)) that pack multiple bits into a single head’s key-query score, but this would substantially complicate the construction and break the clean one-bit-per-head structure; we therefore use Θ( w)

heads. 

Logarithmic width per head. Each head Ht must attend to a specific position i + t, which it identifies by forming the query PE( i + w) − bits( w − t) = PE( i + t) and matching it against position keys. Carrying the positional offset bits( w − t) in the query already requires θ(1) bits of information per head; a constant-width head cannot encode this offset. Hence each head must itself be of growing width. Together, Θ( w) heads of width Θ( w) give total dimension Θ( w2) = Θ(log 2 n). Note that the structure is highly uniform: all w heads are simple affine transformations of the same log n-wide positional encoding PE , differing only in a constant additive offset bits( w − t).
