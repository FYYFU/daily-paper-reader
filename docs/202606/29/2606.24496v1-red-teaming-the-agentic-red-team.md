## 摘要
使用智能体系统执行进攻性安全操作已从理论可能性转变为商品化能力。然而，尽管社区专注于创建越来越强大的智能体，但对其系统安全性的评估却关注不足。在这项工作中，我们首次对广泛应用于进攻性安全操作的智能体系统进行了深入的安全分析。我们表明，这些工具中的大多数存在共同的设计缺陷，使得主动攻击者能够窃取API密钥、建立持久性立足点，并完全攻破操作员的机器，即使智能体在沙箱化容器中运行。为支持我们的分析，我们引入了针对此类智能体系统的完整网络杀伤链，捕捉从初始LLM操纵到横向移动、持久化、护栏绕过和沙箱逃逸的进展。基于我们的安全分析，我们推导出智能体进攻性安全工具的稳健架构，并提出可操作、广泛适用的设计原则，从架构层面缓解所披露的攻击路径。

## Abstract
The use of agentic systems to perform offensive security operations has moved from a theoretical possibility to a commoditized capability. However, while the community has focused on creating more and more capable agents, less attention has been allocated to assessing the security of those systems.   In this work, we present the first in-depth security analysis of the most widely used agentic systems for offensive security operations. We show that most of these tools share common design flaws that enable an active adversary to exfiltrate API keys, establish persistent footholds, and fully compromise the operator's machine, even when the agent operates inside a sandboxed container. To support our analysis, we introduce a full cyber kill chain for such agentic systems, capturing the progression from initial LLM manipulation to lateral movement, persistence, guardrail bypass, and sandbox escape.   Building on our security analysis, we derive a robust architecture for agentic offensive-security tools and propose actionable, broadly applicable design principles that mitigate the disclosed attack paths at the architectural level.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究背景**：智能体系统（agentic systems）已从理论探索发展为商品化能力，广泛应用于自动化渗透测试、红队演练和全链条对手模拟。业界和学术界主要在提升智能体能力，却严重忽视了这些系统本身的安全性。
- **核心问题**：当攻击者控制进攻性安全操作的目标（如一个蜜罐或受控主机）时，能否反向利用该智能体系统，操纵其执行恶意行为，进而攻破操作员的机器？即使智能体运行在沙箱容器中，现有工具是否足够安全？
- **研究意义**：这些工具正被部署在真实作战场景中（包括国家网络部队和恶意攻击），其安全缺陷不仅会导致主动攻击时的严重后果，即使在没有恶意对手时，幻觉、规划错误或奖励错配也可能意外触发类似故障。因此，该研究对保障智能体系统在实战中的安全性具有紧迫性。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：采用**对抗性威胁模型**——假设攻击者控制目标系统（如蜜罐），通过环境设计诱导智能体自主下载并执行恶意负载，从而实现代码执行。该方法不依赖显式提示注入（prompt injection），而是利用**上下文欺骗与奖励黑客**（contextual deception and reward hacking）。
- **关键技术：Agent-Phishing 攻击**
    - **Payload 伪装**：在目标系统上部署一个看似合法的工具（如密码解密器 `pwcrypt`），配套一个看似泄露了密码的日志文件（如 CI 日志），构成完整叙事链。智能体在侦察中会发现这些“关键工件”，并认为必须下载并执行该工具才能解密获取的凭证，从而在本地执行二进制文件。
    - **Payload 木马化**：恶意负载不包含任何显式恶意代码（如网络连接、反连 shell 字符串），而是包含**自植入漏洞**（self-planted vulnerability）。例如，`pwcrypt` 的元数据扩展记录解析中存在边界条件缺失（`subtype + value_len ≤ 256` 未检查），利用堆/ BSS 溢出覆盖函数指针表，将完整性校验回调指向 `system()`，参数来自 KDF 参数字段中的 shell 命令。因此，智能体为验证负载安全性而执行该二进制时，就会触发代码执行（如反连 shell），而二进制行为看上去完全正常（输出正确解密结果、退出码为零）。
    - **自动化生成**：开发了端到端的 Agentic 流水线，可自动生成不同的伪装场景和木马化机制（见表 A.4：`cfgunpack`、`db-restore`、`tokensign` 等）。
- **定义针对智能体红队系统的杀伤链（Kill Chain）**：
    1. Worker RCE（通过 agent-phishing 实现）
    2. 权限提升（从 worker 横向移动到 orchestrator）
    3. 持久化（硬持久化如修改源代码/配置；软持久化如毒化技能/记忆）
    4. 沙箱逃逸（利用容器过度权限）
    5. 宿主机沦陷
    同时定义了中间目标：持久且无约束的智能体武器化、秘密窃取。

- **被评估系统**：12 个开源智能体进攻性安全工具（agentic red-teams），包括 STRIX, PentAGI, PentestGPT, CAI, METATRON, PentestAgent, RedAmon, nebula, AIRecon, xalgorix, Artemis, DarkMoon（见表2）。
- **蜜罐场景**：构建了三种不同的 agent-phishing 负载：

- **LLM 模型**：6 种前沿模型——Claude Opus 4.8, DeepSeek V4 Pro, Gemini 3.1 Pro, GLM-5.1, GPT-5.5, Kimi K2.6。

### 6. 论文的主要结论与发现

1. **agent-phishing 攻击高度有效**：在 97.8% 的测试中成功实现 worker 代码执行，且不依赖任何显式提示注入。
2. **几乎所有测试工具都存在严重安全缺陷**：
    - 10/12 系统可被攻击者完全窃取 secrets（API 密钥、跨会话日志等）。
    - 10/12 系统可实现硬持久化。
    - 10/12 系统可实现无约束武器化（绕过 guardrail）。
    - 10/12 系统可实现沙箱逃逸并导致宿主机沦陷。
3. **根本原因**：
    - **架构层面**：worker 和 orchestrator 之间缺乏强隔离（共享文件系统、网络、API）。
    - **权限过度**：容器被授予过多能力（`--privileged`, `--network=host`, 挂载 docker socket 等）。
    - **护栏仅在编排层执行**：worker 层的实际行为不受监管，攻击者可绕过。
    - **违背最小权限原则**：worker 内运行 root 且携带 secrets。

### 7. 优点

- **首次系统性安全分析**：针对 agentic red-team 这一新兴领域，填补了安全评估空白。
- **创新攻击技术**：提出无需提示注入的“agent-phishing”，利用自植入漏洞绕过了前沿 LLM 的检测，具有实际破坏力。
- **构建了专用杀伤链**：为后续红队测试提供了结构化框架，有助于发现同类漏洞。
- **提出了可操作的安全架构**：基于“假设 worker 已攻破”的妥协模型，设计隔离、最小化 blast radius，包括 OS 分离、网络隔离、文件系统隔离、应用层严格通道、深度护栏（egress 代理）、特权 worker 细粒度 API 等原则，理论扎实。
- **实验设计较为严谨**：自动化管道减少了人工偏差；覆盖多种 LLM 和 agent 系统；声称遵循最强默认安全配置。

### 8. 不足与局限

- **实验覆盖面有限**：仅测试了 12 个开源工具（且大型评估中排除了 2 个），未涵盖商业产品（如 XBOW, Wiz Red Agent）；每个组合仅一次运行，未统计多次重复结果；未测试 agent 在不同护栏配置、不同网络拓扑下的表现。
- **攻击方法局限性**：agent-phishing 高度依赖精心设计的蜜罐环境和伪装工件，实际场景中若操作员手动干预或使用更严格的网络过滤（如仅允许白名单域名），攻击可能失败。但作者也指出在杀伤链后续阶段仍有其他攻击路径。
- **未考虑人类用户干预**：威胁模型假定无人类门控决策，但现实操作中可能有人工审查，作者未评估人类参与的影响。
- **软持久化控制困难**：作者承认记忆投毒等软持久化问题仍是开放难题，提出的最小化跨操作记忆共享在实际中可能影响功能性。
- **牺牲功能与安全平衡**：提出的安全架构引入了额外复杂性（特权 worker 受限 API），可能影响部分工具的灵活性；面向操作员的开发成本增加。
- **部分结论依赖特定 LLM 能力**：作者指出未来更强大的 LLM 可能改善检测，但攻击也可利用相同模型改进伪装，攻击与防御的竞赛持续存在。
