## 摘要
大型语言模型（LLM）推理中的长思维链（CoT）轨迹因键值（KV）缓存快速增长而导致严重的推理瓶颈。当前的解码时压缩方法通过令牌驱逐缓解此问题，但通常假设所有层和头的预算均匀分配。相比之下，现有的非均匀预算分配方法主要设计用于静态提示预填充阶段，无法捕捉自回归推理的逐步上下文需求。为弥合这一差距，我们提出ReasonAlloc，一种无需训练的框架，将解码时KV压缩重新定义为层级式预算分配问题。ReasonAlloc在两个互补层面运作：离线层间预分配策略捕捉一种架构驱动的需求模式，我们称之为“推理波”；而在线头间策略则根据实时效用将资源重新分配给解码过程中信息丰富的头。在数学推理基准（MATH-500、AIME 2024）上使用DeepSeek-R1-Distill-Llama-8B、DeepSeek-R1-Distill-Qwen-14B和AceReason-14B的评估表明，ReasonAlloc优于均匀预算的R-KV、SnapKV和Pyramid-RKV（一种强制静态单调递减层预算的基线），在较小预算（128-512个令牌）下增益最大。ReasonAlloc可即插即用，与现有令牌驱逐策略兼容，并引入可忽略的推理时开销。

## Abstract
Long chain-of-thought (CoT) trajectories in large language model (LLM) reasoning cause severe inference bottlenecks due to rapid key-value (KV) cache growth. Current decoding-time compression methods mitigate this issue via token eviction, but typically assume a uniform budget distribution across all layers and heads. In contrast, existing non-uniform budget allocation methods are predominantly designed for the static prompt prefill phase, and they do not capture the stepwise context demands of autoregressive reasoning. To bridge this gap, we propose ReasonAlloc, a training-free framework that recasts decoding-time KV compression as a hierarchical budget allocation problem. ReasonAlloc operates at two complementary levels: an offline layer-wise preallocation strategy captures an architecture-driven demand pattern which we call ``\textit{Reasoning Wave}'', while an online head-wise strategy reallocates resources during decoding to information-rich heads based on real-time utility. Evaluations on mathematical reasoning benchmarks (MATH-500, AIME~2024) using DeepSeek-R1-Distill-Llama-8B, DeepSeek-R1-Distill-Qwen-14B, and AceReason-14B show that ReasonAlloc outperforms uniform-budget R-KV, SnapKV, and Pyramid-RKV (a baseline enforcing a static, monotonically decreasing layer budget), with the largest gains at small budgets (128-512 tokens). ReasonAlloc is plug-and-play with existing token-eviction policies and introduces negligible inference-time overhead.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：大型推理模型（如 DeepSeek-R1）生成的长链式思维（CoT）轨迹导致 KV 缓存快速膨胀，成为推理时内存与吞吐量的主要瓶颈。
- **现有局限**：当前解码时 KV 压缩方法（如 SnapKV、R-KV）多采用**均匀预算分配**（每层/每头分配相同容量），但实证表明不同层与注意力头对缓存的真实需求高度异构。另一类非均匀方法（如 PyramidKV）主要面向**提示预填充阶段**，依赖静态单调递减假设，无法适应解码时逐步变化的上下文需求，导致推理关键路径被“饿死”。
- **核心贡献**：提出**ReasonAlloc**，一种无需训练的层级式预算分配框架，将解码时压缩重新定义为“离线层间预分配 + 在线头间动态路由”的两级资源分配问题。

## 2. 方法论：核心思想与关键技术

### 2.1 总体框架
- **分层架构**：
  - **层级别（Layer-wise）**：离线校准，捕捉架构固有的“推理波”需求模式。
  - **头级别（Head-wise）**：在线动态调整，根据实时效用在每层内重新分配预算。
- **底层令牌评分**：可兼容任意令牌驱逐策略（如 SnapKV 的注意力分数、R-KV 的联合重要-冗余分数）。论文实例化采用 R-KV 的 `S = αI + (1-α)R` 公式。

### 2.2 离线层间预分配
- **“推理波”需求模式**：通过轻量校准（在少量随机提示上前向一次），计算每层达到目标注意力质量 `ρ=0.93` 所需的最小令牌数，得到原始需求向量 `B_raw`。
  - 浅层高需求（全局语义锚定）、中层低振荡（局部逻辑演绎）、深层尖峰（生成前全局验证）。
- **稳健化算子 `R`**：包含三步：
  1. **幂平滑**：`˜x = x^ν`，ν=0.5（平方根收缩），压缩极端峰值。
  2. **裁剪**：限制在 `[0.25·B̄, 2·B̄]`，防止单层垄断或饿死，其中 `B̄ = B/L`。
  3. **归一化**：按比例重缩放并取整，使总和等于总预算 `B`。
- 输出每层预算 `{B^(ℓ)}`，离线一次计算，后续推理复用。

### 2.3 在线头间动态路由
- 每 `Δ=128` 步刷新一次，捕捉解码时头的效用变化。
- **三步流程**：
  1. **全局阈值**：合并当前层所有头所有候选令牌的分数，取第 `B^(ℓ)` 大分数作为阈值 `τ`。
  2. **原始需求**：统计每头内分数 ≥ τ 的令牌数 `ˆr_i`。
  3. **稳健化**：对 `ˆr_i` 向量再次应用算子 `R`（头级指数 β=0.5，安全底限 `B_min = μ(B^(ℓ)/H)`，μ=0.25），得到最终每头预算 `{B^(ℓ,i)}`。
- **防止饥饿循环**：头级稳健化确保每头即使暂时效用低也保留最低预算，避免陷入“分配少→保留少→上下文缺失→评分低→分配更少”的恶性循环。

3. **组件贡献**：头间动态路由在小预算起主导，层间预分配在中高预算扩大优势，两者结合最稳定。
4. **效率**：层次化分配增加可忽略开销，吞吐量匹配均匀基线；相比 FullKV，在 16K/1024 预算下实现 5.52× 加速。
5. **静态启发式局限**：Pyramid-RKV 在中等预算尚可，但整体低于 ReasonAlloc，证实单调递减假设不适合 CoT 解码。

## 7. 优点

- **训练无关，即插即用**：不修改模型权重，兼容现有令牌驱逐算法（如 R-KV、SnapKV）。
- **分层设计贴合实际**：基于实证的“推理波”和头部异质性，理论动机清晰。
- **稳健化机制**：幂平滑 + 裁剪 + 最小值保护，有效防止极端分配和饥饿崩溃。
- **动态性**：头分配在线更新（每 128 步），适应逐步骤效变化；层分配虽离线但跨任务稳定，可一次校准复用。
- **推理开销极小**：路由仅涉及排序和简单统计，在整个解码中占比极低。
- **全面实验**：三个模型、两个基准、多预算点、消融、效率、案例，证明框架有效。

## 8. 不足与局限

- **领域覆盖有限**：主实验仅在数学推理（MATH/AIME），未评估常识推理、长文档QA 等场景，通用性需扩展验证。
