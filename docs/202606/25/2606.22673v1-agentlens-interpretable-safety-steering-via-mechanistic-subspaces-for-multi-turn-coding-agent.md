## 摘要
基于大语言模型（LLM）的编程代理展现出卓越的自主能力，但也带来了与外部环境多轮交互过程中的重大安全与滥用风险。现有的安全机制主要依赖外部护栏，在执行过程中进行细粒度行为控制的能力有限。同时，近期针对LLM安全的机制可解释性方法大多局限于单轮或越狱式问答场景，难以捕捉多轮代理执行中不断变化的风险动态。在本文中，我们从内部视角研究多轮编程代理的安全性。我们提出AgentLens（机制子空间干预与引导），一种白盒防御框架，可为编程代理提供运行时安全检测和表示级缓解。与传统代理护栏不同，AgentLens从步骤级隐藏表示检测有害执行状态，并通过在单层内的10维子空间中干预来缓解不安全行为。为支持本研究，我们引入了机制代理安全（MAS）基准，包含使用LLaMA-3.1-8B、Qwen-2.5-7B和Gemma-2-9B在194个任务上全面标注的多轮执行轨迹。大量实验表明，AgentLens实现了强大的安全检测性能，为前瞻性风险预测提供了初步证据，并显著减少了编程代理的有害行为，为将机制可解释性应用于动态LLM代理安全奠定了基础。代码可在以下网址获取：https://github.com/EddyLuo1232/AgentLens

## Abstract
Coding agents based on large language models (LLMs) demonstrate remarkable autonomous capabilities, but they also introduce significant safety and misuse risks during multi-turn interactions with external environments. Existing safety mechanisms mainly rely on external guardrails, which have a limited ability to perform fine-grained behavioral control during execution. Meanwhile, recent mechanistic interpretability methods for LLM safety are mostly confined to single-turn or jailbreak-style QA settings, limiting their ability to capture the evolving risk dynamics of multi-turn agent execution. In this paper, we investigate the safety of multi-turn coding agents from an internal perspective. We propose AgentLens (Mechanistic Subspace Intervention and Steering), a white-box defense framework that performs runtime safety detection and representation-level mitigation for coding agents. Unlike conventional agent guardrails, AgentLens detect harmful execution states from step-level hidden representations and mitigate unsafe behavior by intervening in a 10-dimensional subspace within a single layer. To support this research, we introduce the Mechanistic Agent Safety (MAS) benchmark, comprising comprehensively annotated multi-turn execution trajectories across 194 tasks using LLaMA-3.1-8B, Qwen-2.5-7B, and Gemma-2-9B. Extensive experiments show that AgentLens achieves strong safety detection performance, provides preliminary evidence for lookahead risk anticipation, and substantially reduces harmful actions of the coding agent, establishing a foundation for applying mechanistic interpretability to dynamic LLM agent safety. # 方法论：核心思想、关键技术细节

## 核心思想
利用多轮编码代理每一步的隐藏状态（last-token 表示）作为安全信号的载体，训练轻量线性探针进行实时检测，并从探针权重中提取判别性最强的维度构成稀疏引导向量，对同一层的隐藏表示进行干预，从而在推理时无权重修改地抑制有害行为。

## 关键技术细节

1. **安全检测（Stage 1）**
   - 在固定层 ℓ 提取当前步骤的隐藏表示 \( z_t^{(\ell)} = F_\theta^{(\ell)}(c_{t-1})[-1] \in \mathbb{R}^d \)。
   - 训练逻辑回归探针 \( g_\phi(h) = \sigma(w^\top h + b) \)，使用二元交叉熵损失。
   - 训练数据由当前步骤标签 $y_{i,t}$（有害=1，良性=0）构成，包含恶意任务中的关键执行步骤（标记为1）和安全感知/拒绝步骤（标记为0），以及所有良性任务步骤（0）。
   - 前瞻检测：用同一探针预测下一步是否有害（标签 $y_{i,t+1}$）。

2. **风险缓解（Stage 2）**
   - 当探针输出$ \hat{y}_t = 1$时触发。
   - 构建引导向量 $v \in \mathbb{R}^d$：选择探针权重差绝对值最大的 Top-K 维度（$K=10$），并令这些维度上的分量为1，其余为0。
   - 将引导向量广播到所有 token 位置的隐藏表示：$ \tilde{H}_t^{(\ell)}(\alpha) = H_t^{(\ell)} + \alpha \mathbf{1}_{n_t} v^\top $。
   - 自适应搜索引导强度 $\alpha$：使用 LLM 控制器（Deepseek-V4-Pro）对每个候选 $\alpha$ 生成的行动评估安全得分 $s(a_t(\alpha))$ 和效用得分 $u(a_t(\alpha))$，优化目标 $J(\alpha) = 0.6 s + 0.4 u$，选最优 $\alpha^*$。
   - 引导方向符号由训练集标定（正方向促使安全拒绝）。

# 实验设计

## 数据集 / 场景

- **MAS（Mechanistic Agent Safety）基准**：自主构建，包含 194 个 OS 级任务（98 个恶意，96 个良性），来源于 AdvCUA、Safe-OS、AgentBench，覆盖 MITRE ATT&CK 框架 10 类攻击技术。
- **执行环境**：Docker 容器 + ReAct 代理，使用三个白盒 LLM（LLaMA-3.1-8B、Qwen-2.5-7B、Gemma-2-9B），贪心解码，生成完整多轮轨迹。
- **标注**：严格步骤级标签：关键有害执行步骤 = 1，安全感知/拒绝 = 0。采用关键词匹配 + GPT/Claude 标注 + 人工审核 + 反事实增强。

## 对比方法

- **检测基线**：LLM-as-Judge（GPT-4o, GPT-5.2, Claude-4.6-Sonnet, Claude-4.6-Opus, Gemini-3-Pro, DeepSeek-V4 系列）、外部护栏（AGrail with GPT-4o, LLaMA-Guard-4-12B）。
- **缓解基线**：RepE（表示工程）、Self Reminder（自我提醒）。
- **检测指标**：Accuracy、F1-score（当前步骤和前瞻）。
- **缓解指标**：Attack Success Rate (ASR)、Collapse Rate（重复循环或突然截断）。

# 资源与算力

- **硬件**：8 块 NVIDIA A100 GPU（每块 40GB VRAM）。
- **训练超参数**：AdamW 优化器，100 epochs，batch size 16，学习率 LLaMA-3.1-8B 为 0.01，Qwen-2.5-7B / Gemma-2-9B 为 0.001。
- **推理开销**：探针检测 <1s；缓解阶段平均搜索迭代次数约 8 次（LLaMA 8.26, Qwen 8.0, Gemma 7.67），依赖于 LLM 控制器（Deepseek-V4-Pro）。

# 主要结论与发现

1. **安全信息线性可解码**：在多轮代理的中间层隐藏状态中，安全相关信号可通过线性探针高精度提取（LLaMA-3.1-8B 达 99.15% 准确率）。
2. **运行时检测有效**：AgentLens 平均当前步骤检测准确率 97.32%，F1 较高，显著优于所有 LLM-Judge 和护栏基线。
3. **前瞻风险预测可行性**：在可用数据上，前瞻检测准确率最高达 96.77%（Qwen-2.5-7B），表明中间表示包含未来有害行为的早期信号。
4. **表示级缓解显著降低 ASR**：平均 ASR 从 85.99% 降至 13.36%，降低 72.63 个百分点，优于 RepE 和 Self Reminder。
5. **引导方向具有双向因果性**：负方向引导可将原本拒绝的有害任务转化为执行有害命令（ASR 达 100%），证明该子空间具有因果控制作用。
6. **检测-控制差距**：在提示注入分布偏移下，探针检测失效但相同引导方向仍能大幅降低 ASR（86.7% → 6.7%），说明表示控制能力可能超越检测泛化性。

# 优点

- **创新视角**：首次将机械可解释性方法应用于多轮代理安全，突破单轮问答局限。
- **轻量级**：探针仅需一步线性分类，引导仅修改单层中极少数维度（10维），无需修改模型权重，推理开销低。
- **自适应引导**：引入 LLM 控制器在线搜索最优强度，平衡安全性与效用。
- **前瞻能力**：提供初步证据表明内部表示可预测未来有害行为，为主动防御奠基。
- **完整基准**：构建 MAS 基准，包含多模型、多任务、步骤级标注及反事实增强，支持可复现研究。

# 不足与局限

1. **白盒依赖**：需要访问模型内部隐藏状态，不适用于闭源或仅提供 API 的商业模型。
2. **生成质量折衷**：缓解阶段平均 collapse 率升至 18.54%，高于基线（2.24%–7.25%），可能因过度引导导致输出退化。
3. **实验规模有限**：MAS 仅含 194 个任务，每个模型轨迹数有限（如 Gemma-2-9B 仅 146 个训练步骤），有害步骤更少，统计显著性受限。
4. **前瞻数据稀疏**：自然产生的“准备→攻击”过渡实例稀少，前瞻结论仅为初步可行性证据，缺乏大样本鲁棒验证。
5. **标签歧义**：当前步骤标签难以区分任务级恶意与步骤级直接有害，且预备步骤虽标为良性但实际构成攻击链部分。
6. **模型能力限制**：所用开源模型（尤其是 7B/8B/9B 量级）在长程规划和复杂攻击任务中表现弱，导致大量轨迹中途拒绝或失败，限制了有害轨迹的多样性。
7. **提示注入未见分布**：探针在提示注入场景检测失效，表明跨分布泛化不足；缓解虽有效但只能在外触发下使用，无法形成闭环防御。
8. **防护范围局限**：仅聚焦 CLI 攻击，未覆盖 Web 操作、文件编辑、API 调用等其他代理能力，也未考虑多代理协作场景。
