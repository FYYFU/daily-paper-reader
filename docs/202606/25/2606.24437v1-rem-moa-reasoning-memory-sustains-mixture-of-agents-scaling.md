## 摘要
混合智能体（MoA）架构通过将多个LLM智能体组织成分层推理流程，提升了推理时的扩展性。然而，现有的MoA变体在深度增加时无法维持增益，表现出性能退化、早期平台期或饱和。我们提出ReM-MoA，一种记忆增强的MoA框架，通过两种机制维持扩展性：（1）排名推理记忆，持续存储并使用比较型评审智能体对所有层的推理轨迹进行排序；（2）策展多样化记忆路由方案，将不同的智能体暴露于成功与失败轨迹的不同组合中，在传播高质量推理的同时保持探索多样性。我们进一步引入可选的多领域评审精炼流程，通过前沿模型监督提升排序质量。在涵盖数学、形式逻辑、代码、知识和常识的五个推理基准测试中，ReM-MoA在深度和宽度扩展上均持续优于先前的MoA变体，且其优势随深度增加而扩大，将结构化的跨层推理记忆确立为可扩展多智能体推理的关键缺失机制。

## Abstract
Mixture-of-Agents (MoA) architectures improve inference-time scaling by organizing multiple LLM agents into layered reasoning pipelines. However, existing MoA variants fail to sustain gains as depth increases, exhibiting degradation, early plateauing, or saturation. We propose ReM-MoA, a memory-augmented MoA framework that sustains scaling through two mechanisms: (1) a Ranked Reasoning Memory that persistently stores and ranks reasoning traces from all layers using a comparative Reviewer Agent, and (2) a Curated Diversified Memory Routing scheme that exposes different agents to distinct combinations of successful and failed traces, preserving exploration diversity while propagating high-quality reasoning. We further introduce an optional multi-domain Reviewer distillation pipeline that improves ranking quality through frontier-model supervision. Across five reasoning benchmarks spanning math, formal logic, code, knowledge, and commonsense, ReM-MoA consistently outperforms prior MoA variants across both depth and width scaling, and its advantage widens with depth, establishing structured cross-layer reasoning memory as a key missing mechanism for scalable multi-agent inference.

## 1. 核心问题与研究动机

- **问题背景**：大规模语言模型（LLM）的推理时计算已成为除模型参数扩展外另一重要扩展轴。混合智能体（Mixture-of-Agents, MoA）架构通过将多个提案智能体（proposer agents）组织成分层流水线，实现逐步精化，理论上可通过增加层数（深度）和每层智能体数（宽度）持续提升集体推理能力。
- **研究动机**：现有MoA变体（如标准MoA、RMoA、AttentionMoA）在深度增加时无法维持性能增益，具体表现为三种失效模式：
  - **退化**（标准MoA）：原始跨层直接拼接导致错误累积，性能随深度先升后降；
  - **早期平台期**（RMoA）：仅保留相邻层残差，无法回溯远端高质量推理轨迹；
  - **饱和**（AttentionMoA）：层内智能体间交互导致多样性丧失，探索收敛至冗余解路径。
- **核心问题**：如何设计一种MoA框架，使其在深度和宽度扩展时能同时保持**推理质量**（高品质轨迹持续传递）和**探索多样性**（层内智能体保持差异化），从而维持甚至扩大性能增益。

## 2. 方法论：ReM-MoA

### 核心思想
将MoA扩展视为**质量-多样性联合保持问题**，通过引入结构化的跨层推理记忆和差异化路由机制，替代原始拼接、局部残差或层内注意力交互。

### 关键技术细节

#### (1) 排序推理记忆（Ranked Reasoning Memory, M）
- 每个层（layer l）产生N条推理轨迹r_{l,j}后，调用一个**评审智能体（Reviewer Agent）** 对所有N条轨迹进行**比较式评分**（1–10分），并输出每条的针对性理由（rationale）。
- 存储格式：M_l = {(r_{l,j}, s_{l,j}, ρ_{l,j})}_{j=1}^N，其中s为分数，ρ为理由。
- 跨层可访问：后续层可从累积记忆M_{≤l}中任意读取（不仅仅是相邻层），满足**质量保持（Q）** 需求。

#### (2) 策展多样化记忆路由（Curated Diversified Memory Routing）
- 从第2层起，每个提案智能体接收一个**策展参考集**S_{l,j}，该集合从记忆中提取，且**不同智能体获得不同子集**，以保持多样性（D）。
- 路由函数分为三类家族：
  - **对比式（Con）**：高分与低分轨迹配对（Top_a ∪ Bot_b）；
  - **正向强化（Top）**：仅选高分；
  - **故障模式聚焦（Bot）**：仅选低分。
- 例如N=3时：智能体1获Con(1+1)，智能体2获Top2，智能体3获Bot2。更大N的分配见附录B。
- 该机制同时满足**Q**（质量信号被筛选传递）和**D**（层内智能体输入互异）。

#### (3) 评审蒸馏（可选增强）
- 使用前沿模型（如GPT-5.5）作为教师，对ReM-MoA自身运行产生的（问题，轨迹）对进行高质量评分/理由标注，构成训练数据D_train。
- 通过LoRA（低秩适配）微调一个小型评审模型（以Qwen2.5-7B为基座），使其模仿教师，得到更优的评审智能体R_θ*。
- 多领域训练（MATH、ARC-Challenge、MedQA、MBPP、HellaSwag各100题）使蒸馏后的评审通用性强，可迁移至未见领域。

### 算法流程说明（文字描述）
1. 输入问题x，设定层数L、每层智能体数N。
2. 对于层1：N个提案智能体分别独立生成轨迹r_1；评审智能体评分并存储至M；输出r_1。
3. 对于层l≥2：从M_{≤l-1}中按路由方案Π为每个智能体选参考集；智能体基于x+参考集生成新轨迹；再次评审并存储。
4. 最后一层后，聚合智能体从记忆中选取参考（类似路由），合成最终答案。
