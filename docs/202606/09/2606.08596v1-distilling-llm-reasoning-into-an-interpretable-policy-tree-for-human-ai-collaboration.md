## 摘要
构建高效可靠的政策以协助人类，对于人机协作至关重要。现有方法主要沿两条研究路线。大多数先前的工作依赖多智能体强化学习（MARL）学习黑盒策略，这限制了可解释性并引发安全问题。近期方法在每一步决策都查询大型语言模型（LLM），导致响应缓慢和推理成本高昂。我们提出协作策略树（Co-pi-tree），这是一种闭环方法，学习一个可执行的策略树，由伙伴行为预测树和智能体动作选择树组成。Co-pi-tree通过将LLM推理提炼为策略树代码来构建策略。然后通过与伙伴互动评估策略，获取反馈，并使用自然语言总结互动反馈以改进有问题的分支。在Overcooked-AI中的实验表明，Co-pi-tree相对于基线平均值将平均奖励提高了35.4%，同时将LLM查询次数减少了77.7%，测试时间延迟减少了97.1%。项目页面：https://beiwenzhang.github.io/Co-pi-tree/

## Abstract
Constructing efficient and reliable policies to assist humans is indispensable for human-AI collaboration. Existing methods mainly follow two lines of work. Most prior work relies on multi-agent reinforcement learning (MARL) to learn black-box policies, which limits interpretability and raises safety concerns. Recent methods query large language models (LLMs) at each decision step, causing slow responses and high inference costs. We propose Collaboration Policy Tree (Co-pi-tree), a closed-loop method that learns an executable policy tree consisting of a partner-behavior prediction tree and an agent-action selection tree. Co-pi-tree constructs a policy by distilling LLM reasoning into policy tree code. It then evaluates the policy through partner interaction, obtains feedback, and uses natural language to summarize the interaction feedback to improve problematic branches. Experiments in Overcooked-AI show that Co-pi-tree improves average reward by 35.4% over the baseline average, while reducing the number of LLM queries by 77.7% and test-time latency by 97.1%. - **现有矛盾**：人机协作中，多智能体强化学习（MARL）方法能提供低延迟执行，但生成的策略是黑盒，缺乏可解释性，难以审计且存在安全风险；而直接查询大语言模型（LLM）的方法（如 ProAgent、CausalPlan）能在决策中融入语言推理，但每一步都要调用 LLM，导致响应延迟高、推理成本大，不适合实时交互。
- **核心目标**：提出一种既能保留 LLM 基于语言的协调推理优势，又能大幅减少测试时 LLM 查询次数和延迟，同时保持策略可解释性的方法。

### 2. 论文提出的方法论

- **核心思想**：将 LLM 的推理能力蒸馏为一个结构化的、可执行的策略树（Policy Tree），该树由两个子树组成：
  1. **伙伴行为预测树**（Partner-Behavior Prediction Tree）：预测合作者下一步可能采取的行为。
  2. **智能体动作选择树**（Agent-Action Selection Tree）：根据预测的伙伴行为选择自身的协调动作。
- **关键技术细节**：
  - 策略树以显式的 if/elif 条件分支表示决策逻辑，便于人类理解和局部修改。
  - 训练过程是一个闭环，包含三个循环阶段：
    1. **策略构建**：使用 Planner 模块（基于 LLM）根据任务知识、历史经验和反思生成两个文本策略树，再用 Coder 模块（基于 LLM）将其转换为可执行的 Python 代码。
    2. **环境接地与评估**：Executor 在模拟环境中运行代码，将原始张量状态转化为文本状态，将树输出动作转化为可执行的动作，并记录交互轨迹和团队奖励。如果当前策略奖励不低于历史最优，则接受；否则回退。
    3. **策略细化**：Summarizer 模块（基于 LLM）分析执行轨迹，定位有问题的分支，生成自然语言反思（reflection）用于下一轮构建，同时将高分策略摘要存入记忆池。
  - 整个流程只需要在训练阶段调用 LLM，测试时直接执行策略树，无需再查询 LLM。

### 7. 优点

- **高效性**：测试时无需 LLM 调用，延迟极低（仅执行 if/elif 条件代码），适合实时协作。
- **可解释性**：策略为显式条件树，每个分支解释“为什么预测该伙伴行为”和“为什么选择该动作”，便于人类审计和局部修复。
- **闭环改进**：通过自然语言反馈自动定位和修改问题分支，无需人工标注。
- **泛化能力**：在未见 AI 伙伴和真实人类伙伴上均表现优异；跨布局迁移能力突出。
- **资源友好**：不需要大量 GPU 训练，主要开销在训练阶段的 LLM API 调用，且可通过蒸馏大幅减少。

### 8. 不足与局限

- **实验场景局限**：仅验证了 Overcooked-AI 烹饪游戏环境，在物理具身协作（如机器人）等更复杂场景中的有效性尚未确认。
- **动作空间依赖**：当前需要手动定义动作空间和执行器，将树输出动作映射为环境可执行动作；未来需探索更自动化的动作接地方法以减少人工设计。
- **伙伴协作敏感**：在 Forced Coordination 布局上，策略性能对合作伙伴的习惯匹配较为敏感，可能导致波动。
- **LLM 依赖**：训练阶段仍需要 API 调用（GPT-4o），存在成本和对第三方模型的依赖；若 LLM 能力不足，可能影响策略质量。
- **可扩展性**：策略树的复杂度随任务状态空间增长可能变大，但论文未探讨极端大规模场景的表现。
