## 摘要
长上下文任务要求大语言模型从大量上下文中识别并保留与答案相关的信息。分块记忆智能体通过顺序读取文档块、更新紧凑记忆，并从累积记忆中生成最终答案来解决这一问题。然而，现有的基于强化学习的分块智能体要么依赖稀疏的最终答案奖励，要么使用词汇级中间奖励来指导记忆和检索动作。这些信号监督任务成功或局部重叠，但并未直接评估最终记忆是否支持真实答案。我们提出InfoMem，一种用于训练分块记忆智能体的奖励机制，通过答案条件信息来评估最终记忆的效用。InfoMem衡量最终记忆增加模型对真实答案的每标记对数似然的程度。为了稳定强化学习优化，InfoMem仅将这一信号应用于成功的轨迹，并在奖励组合前对其进行归一化。在相同的GRPO框架和训练预算下，InfoMem在长上下文记忆智能体性能上优于可比的记忆智能体强化学习基线。分析表明，有效的最终记忆奖励应作用于成功轨迹，在奖励组合前归一化，并以答案而非查询为条件。我们的代码可在https://github.com/GenSouKa1/InfoMem获取。

## Abstract
Long-context tasks require LLMs to identify and preserve answer-relevant information from large contexts. Chunk-wise memory agents address this issue by sequentially reading document chunks, updating a compact memory, and generating the final answer from the accumulated memory. However, existing RL-based chunk-wise agents either rely on sparse final-answer rewards or use lexical intermediate rewards for memory and retrieval actions. These signals supervise task success or local overlap, but do not directly evaluate whether the final memory supports the ground-truth answer. We propose InfoMem, a reward mechanism for training chunk-wise memory agents that evaluates final-memory utility using answer-conditioned information. InfoMem measures how much the final memory increases the model's per-token log-likelihood of the ground-truth answer. To stabilize RL optimization, InfoMem applies this signal only to successful trajectories and normalizes it before reward composition. Under the same GRPO framework and training budget, InfoMem improves long-context memory-agent performance over comparable memory-agent RL baselines. Analyses show that effective final-memory rewards should operate on successful trajectories, be normalized before reward composition, and be conditioned on the answer rather than the query. Our code is available at https://github.com/GenSouKa1/InfoMem.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题背景**：大语言模型在处理长上下文时，需要从大量文本中识别并保留与答案相关的证据。分块记忆智能体（chunk-wise memory agents）通过顺序读取文档块、更新紧凑记忆，最终从累积记忆生成答案，是一种有效范式。
- **现有不足**：训练这类智能体的强化学习方法（RL）通常使用两种奖励信号：①稀疏的最终答案正确性奖励（outcome reward）；②基于词汇重叠的中级记忆/检索奖励。但这些信号均不直接评估最终记忆是否在语义上支持真实答案（ground-truth answer）。即使成功轨迹，不同记忆的质量差异无法被区分；成功记忆可能包含冗余甚至误导信息，而稀疏奖励无法对此建模。
- **研究动机**：需要一种直接衡量最终记忆对答案支持程度的奖励信号，从而更高效地指导分块记忆智能体的记忆塑造。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将最终记忆的效用定义为 **答案条件信息增益**（answer-conditioned information gain）。即衡量最终记忆在多大程度上增加了模型对真实答案的每标记对数似然（per-token log-likelihood）。
- **关键公式**：  
  \[
  r_{\text{gain}}(x, M, y^*) = \frac{1}{|y^*|}\log P_\theta(y^* \mid x, M) - \frac{1}{|y^*|}\log P_\theta(y^* \mid x, \emptyset)
  \]  
  其中 \(\emptyset\) 表示空记忆，\(P_\theta\) 在教师强制（teacher-forcing）下计算。该值越高，说明记忆越能提升正确答案的似然。
- **技术细节**：
  - **仅限于成功轨迹**：信息增益奖励只施加于已获得正确答案的轨迹（\(R_{\text{outcome}}=1\)），失败轨迹只使用原始结果奖励。理由：失败轨迹中的记忆可能与错误答案纠缠，信号不可靠。
  - **预组合归一化**：对同一prompt分组中成功轨迹的原始 \(r_{\text{gain}}\) 进行标准化（减去均值、除以标准差），得到 \(\tilde{r}_i\)，再与结果奖励组合：\(R_i = R_{\text{outcome}, i} + \beta \tilde{r}_i\)（仅对成功轨迹加 \(\beta \tilde{r}_i\)）。这防止不同prompt间增益尺度差异导致的训练不稳定。
  - **算法框架**：基于 GRPO（Group Relative Policy Optimization），每组采样 \(n=8\) 个轨迹，计算奖励后，用GRPO的标准优势函数更新策略。
  - **与QueryPMI对比**：将真实答案替换为查询本身进行条件化，发现会导致模型记忆重复查询而非保留答案支持证据。

### 3. 实验设计：数据集、Benchmark、对比方法

- **训练数据**：RULER-HotpotQA（基于RULER范式和HotpotQA构造），每个样本包含200个文档，训练子集采样512个样本（原训练集32768个），验证集128个样本。
- **评估Benchmark（4个）**：
  - **MRCR-8needle**：多针检索任务，序列匹配准确率。
  - **RULER合成QA**：稀疏检索式问答，F1分数。
  - **CorpusQA**：语料级证据聚合（128K token子集），LLM-as-judge（Kimi-K2.6）。
  - **LongMemEval**：长对话记忆（115K token LongMemEval-S子集），同样LLM-as-judge。
- **对比方法**：
  - **Outcome-only GRPO**：去掉信息增益奖励项，其余相同。
  - **ReMemR1**：带有回调检索机制的分块记忆智能体，使用其原始设置（词汇级中间奖励）。
  - 初始模型（Qwen2.5-1.5B-Instruct）作为参考。
- **额外诊断实验**：
  - 合成幻觉上下文判别：基于SQuAD构造真正支持上下文 vs 两个幻觉上下文，比较信息增益与嵌入相似度（BGE-M3, E5, Qwen3-0.6B）、注意力评分（ATTN-MASS, ATTN-TOP1）。
  - 消融实验：监督侧面（成功/错误/两者）、归一化去除、查询条件化（QueryPMI）。

### 6. 论文的主要结论与发现

- InfoMem在所有四个长上下文基准上一致优于Outcome-only GRPO和ReMemR1，表明答案条件信息增益是有效的最终记忆监督信号。
- **三个关键性质**：
  1. **监督应只用于成功轨迹**（正面记忆塑造），失败或双侧混合会导致训练不稳定。
  2. **信息增益奖励必须在组合前归一化**，否则奖励尺度不稳定，损害性能。
  3. **奖励必须以真实答案为条件**，而非查询（QueryPMI导致记忆重复查询，降低性能）。
- 合成诊断实验显示，信息增益在区分真实上下文与幻觉上下文方面比嵌入相似性和注意力得分具有更高的MRR和信噪比。

### 7. 优点：方法或实验设计上的亮点

- **理论动机清晰**：从互信息到点态代理的自然递进，直接关联记忆效用与答案支持。
- **实验控制严谨**：所有比较方法共享相同训练框架、数据、模型、rollout数，消融实验独立验证各组件。
- **诊断实验创新**：构造包含幻觉上下文的合成任务，客观比较不同评分指标的判别力。
- **实用性强**：仅需在现有GRPO框架中修改奖励计算，无需改变架构或增加训练预算。
- **识别关键约束**：通过消融揭示成功侧、归一化、答案条件三个必要条件，为未来设计提供指导。

### 8. 不足与局限

- **规模限制**：训练数据仅512样本，模型仅1.5B参数，未验证向更大模型和大规模数据的可扩展性。
- **任务泛化性**：方法仅适用于分块记忆智能体，不适用于检索-only系统或全上下文单遍模型。
- **奖励仅在最终步骤**：虽然GRPO将轨迹级优势传播到所有token，但奖励本身只评估最终记忆和最终答案，未扩展到中间记忆步骤（过程监督）。
- **潜在风险**：若真实答案有误或来源有偏，可能鼓励记忆放大误导证据；可能过度优化支持答案而省略关键限定条件，不适合高风险领域（法律、医疗、金融）直接使用。
- **对比方法公平性**：ReMemR1使用不同推理框架（回调检索）和温度采样，虽保持其一致，但并非完全同设置对比；未与更多近期长上下文RL方法（如LongRLVR）比较。
- **评估指标**：MRCR-8needle得分普遍较低，可能因模型小、任务难；LLM-as-judge虽经人工校准，但仍存在一定自动评判偏差。
