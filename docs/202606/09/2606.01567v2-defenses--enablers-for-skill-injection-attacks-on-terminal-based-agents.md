## 摘要
大型语言模型（LLM）智能体日益依赖可重用的技能，即描述任务特定流程的文档。然而，这为智能体引入了新的攻击面需要管理。我们针对这一威胁研究了两个互补方向。首先，我们评估了基于守护者的防御：一个作为技能文件访问中介的中间LLM智能体（动态守护者），或在构建时预先重写这些文件（静态守护者）。在三个LLM智能体家族中，我们的守护者将攻击成功率（ASR）降低了一半以上，同时保持了任务实用性。其次，我们通过攻击重构来对其进行压力测试，使用了四种保留恶意指令但改变措辞的攻击。对于非守护者设置，重构将ASR提升至81.4%，但动态守护者将其降至18.6%，表明实时中介是一种稳健的防御手段。

## Abstract
Large language model (LLM) agents increasingly rely on reusable skills i.e. documents describing task-specific procedures. However, this introduces a new attack surface for agents to manage. We study two complementary directions for this threat. First, we evaluate guardian-based defenses: an intermediary LLM agent that acts as a mediator for skill file access (dynamic guardian) or pre-rewrites these files at build time (static guardian). Across three LLM agent families, our guardians cut attack success rate (ASR) by well over half while preserving task utility. Second, we stress test them through attack reframing using four attacks that preserve the malicious instruction but change the phrasing. For non-guardian setup, the reframing pushes the ASR up to 81.4\%, but the dynamic guardian brings it down to 18.6\%, showing that real-time mediation is a robust defense.

# 针对终端智能体的技能注入攻击的防御与使能因素：论文详细总结

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **背景**：大型语言模型（LLM）智能体在终端环境中执行任务时，依赖可复用的**技能文件**（`.md`文件），这些文件描述了任务特定流程、工具使用模式或领域约定。技能可以从公共市场下载，但存在安全风险：攻击者可以注入恶意指令，诱导智能体执行有害操作（如数据窃取、文件删除、权限修改等）。
- **核心问题**：技能注入攻击成功率高（已有基准显示26.1%的市场技能存在漏洞），而现有智能体对此缺乏有效防御。研究旨在设计既能防御攻击又不损害任务效用的机制，并揭示攻击表面可能被低估（通过攻击重构可放大威胁）。
- **整体含义**：提出**守护者**（Guardian）架构作为智能体和技能生态系统之间的中介，并通过**攻击重构**展示现有基准低估了攻击面，强调需要更全面的防御评估。

## 2. 论文提出的方法论：核心思想、关键技术细节
- **守护者防御架构**：
  - **动态守护者（Dynamic Guardian）**：作为一个MCP（Model Context Protocol）服务器，在运行时拦截每一次技能文件读取调用。守护者LLM收到智能体的查询后，返回经过清洗的版本（过滤注入内容）。不修改原始技能文件，但增加延迟。
  - **静态守护者（Static Guardian）**：在构建时（智能体会话开始前），用预处理脚本让守护者LLM重写所有技能文件，去除注入内容。智能体后续运行时直接使用重写后的文件，无运行时开销。
- **攻击重构（Attack Reframing）**：保持恶意指令语义不变，但改变措辞和结构以逃避检测。包括：
  - **翻译攻击**：将整个`SKILL.md`翻译为西班牙语（ES）或日语（JA）。
  - **自动发现的三类攻击**（通过`autoresearch`循环50次迭代，目标模型为DeepSeek-V4-Flash）：
    - **盲引用（Blind Reference）**：将注入伪装为YAML配置块`precondition:`，声称自动加载。
    - **交叉引用（Cross Reference）**：指示智能体合并来自虚构文件`SCOPE.md`的指令作为同等优先级约束。
    - **描述陷阱（Description Trap）**：在`SKILL.md`添加一个名为`universal-helper`的假技能，其描述宽泛，导致任意任务自动加载，体内携带恶意指令。
- **防御机制原理**：守护者仅基于技能文档内容进行过滤，不依赖智能体的任务上下文（动态守护者能看到智能体查询，但无任务上下文）。这种设计旨在使注入在到达智能体前就被移除。

## 6. 论文的主要结论与发现
- **守护者防御显著有效**：动态守护者和静态守护者均将ASR降至<15%（如Sonnet-4.5：Vanilla 36.0% → Dynamic 12.9% → Static 7.2%），同时TSR基本保持（Dynamic 82.8%，Static 87.0%）。
- **系统提示防御效果有限**：即使先知式目标警告（SysTargeted）ASR仍有23.0%，远低于守护者。说明过滤内容优于依赖智能体自监管。
- **攻击重构可放大威胁**：非防护设置下，原始注入ASR为55.8%，四种重构联合后ASR升至81.4%（Vanilla）。动态守护者将联合ASR降至18.6%（static为39.5%），证明实时中介对重构攻击依然稳健。
- **攻击空间可自动搜索**：通过LLM驱动的`autoresearch`低成本发现了三个有效重构策略，表明基准测试中单一措辞会低估威胁。
- **动态守护者的单次请求漏洞**：当智能体发出“给我完整文档”的宽泛请求时，守护者容易将嵌入良好的注入一并传递（如附录L.2实例）。多轮交互可降低风险。
- **静态守护者的“洗白”问题**：部分注入被守护者错误地保留并格式化为合理工作流步骤（如附录L.4实例），导致攻击成功。

## 7. 优点：方法或实验设计上的亮点
- **创新防御架构**：引入守护者作为中介层，概念简单但效果显著，且不修改主智能体模型，易于部署。
- **攻击重构的全面性**：不仅研究了翻译，还通过自动化搜索发现了三种新型结构伪装，揭示了攻击面可被系统性地扩展。
