## 摘要
大型语言模型（LLMs）通过遵循固定的深度和顺序、非循环地执行所有层来进行推理。我们发现了一种广泛存在的、无需训练的、灵活的动态程序化层执行（PoLar），其中预训练的层可以作为模块打包，然后跳过或循环，为每个输入形成定制的程序。对于大多数输入，显著更短的程序执行可以达到相同或更好的准确率，而原始LLM的错误预测可以通过使用更少层的替代程序来纠正。这些观察表明，推理中存在多种有效的潜在计算，超出了标准前向传播。为了在实践中高效实现PoLar，我们提出了一个轻量级的PoLar预测网络，它学习生成执行程序，动态地为每个输入跳过或重复预训练层。在数学推理基准上的实验表明，PoLar持续提高了标准推理和先前动态深度方法的准确率，通常同时执行更少的层，并且这些增益在分布外评估下仍然保持。我们的结果表明，固定深度执行只捕获了LLM潜在推理能力的一个狭窄子集。

## Abstract
Large language models (LLMs) perform inference by following a fixed depth and order, non-recurrent execution of all layers. We reveal the wide existence of training-free, flexible, dynamic program-of-layers (PoLar), where pretrained layers can be packed as modules and then skipped or looped to form a customized program for each input. For most inputs, substantially shorter program executions can achieve the same or better accuracy, while incorrect predictions of the original LLM can be corrected by alternative programs with fewer layers. These observations indicate that inference admits multiple valid latent computations beyond the standard forward pass. To efficiently achieve PoLar in practice, we propose a lightweight PoLar prediction network, which learns to generate execution programs that dynamically skip or repeat pretrained layers for each input. Experiments on mathematical reasoning benchmarks demonstrate that PoLar consistently improves accuracy over standard inference and prior dynamic-depth methods, often while executing fewer layers, and that these gains persist under out-of-distribution evaluation. Our results suggest that fixed-depth execution captures only a narrow subset of an LLM's latent reasoning capacity.

*   **整体含义**：论文首次系统性地从“程序”视角审视LLM推理，认为预训练模型的每一层都可视为一个原子函数，而推理则是执行一个由这些函数组成的“程序”。作者通过实验证明，对于同一个输入，存在多种有效的“层程序”（Program-of-Layers, **PoLar**），这些程序可以有选择地**跳过（Skip）** 或**循环（Loop）** 某些层，并且往往比标准的前向传播更短或更准确。这表明固定深度推理只利用了LLM潜在推理能力的一个狭窄子集，为更高效、更强大的推理提供了新的可能性。

### 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想**：将LLM推理看作是根据输入动态执行的“程序”。这个程序由对预训练层模块的**跳过**（避免计算）和**循环**（重复计算）操作构成，旨在为每个输入定制计算路径。
*   **关键技术细节**：
    1.  **程序表示**：每个执行程序由两部分组成：
        *   **分段（Segmentation）**：将连续的层分组为模块（Segment）。用一个二进制掩码 `z_seg(x)` 标记层的边界。
        *   **操作（Operation）**：对每个模块指定一个操作（`skip`, `keep`, 或 `repeat`），用一个标签向量 `z_op(x)` 表示。
    2.  **PoLar 预测网络**：这是一个轻量级的神经网络，用于在推理时直接“预测”出每个输入对应的程序，从而避免昂贵的搜索。
        *   **输入编码**：使用一个冻结的嵌入模型（如 Qwen3-Embedding）处理输入，得到token表示。
        *   **层查询与交叉注意力**：为每个层索引学习一个嵌入向量作为查询，与token表示进行交叉注意力，得到输入条件化的层表示。
        *   **跨层编码器**：使用一个轻量级Transformer编码器处理层维度的表示，捕捉层间的依赖关系。
        *   **预测头**：两个线性层，分别预测分段边界（`ℓ_seg`）和操作（`ℓ_op`）。
    3.  **离线监督与训练**：用**蒙特卡洛树搜索（MCTS）** 作为诊断工具，在小型数据集上离线搜索出有效的执行程序，并从中提取出分段和操作的真值标签来训练PoLar预测网络。

### 6. 论文的主要结论与发现

1.  **有效程序普遍存在**：通过MCTS搜索发现，对绝大多数输入，都存在比标准前向传播更好（更短或更准确）的执行程序。
2.  **跳过+循环组合最优**：单独使用“循环”优于“跳过”，但两者组合能取得最好的效果。
3.  **奥卡姆剃刀原则**：大部分有效的执行程序比标准前向传播更短，即存在冗余计算。
4.  **测试时计算缩放**：增加“循环”等潜在执行步骤，能系统性提高发现有效程序的可能性，并能显著提升模型在难题上的表现。
5.  **程序的结构偏好**：有效的程序倾向于使用**短的、连续的层段**（Segment），并且每个段通常**只需循环一次**。这揭示了预训练LLM在作为程序执行器时的固有限制。
6.  **PoLar的实用性**：提出的轻量级预测网络能有效学习这些程序，在数学推理任务上，**PoLar 持续超越所有基线**，且在分布外数据上也展现出强泛化能力，甚至能在减少总执行层数的情况下提升准确率。

### 7. 优点

1.  **新颖的视角**：将LLM推理抽象为“程序执行”，提供了理解模型内部计算机制的新颖且富有洞察力的框架。
2.  **方法优雅实用**：使用轻量级预测网络替代昂贵的MCTS搜索，在保持灵活性的同时显著降低了计算开销，使其具有实际部署价值。
3.  **完全的“冻结模型”**：PoLar无需修改或微调基础LLM的任何参数，这保留了模型的通用性，并避免了灾难性遗忘等问题。
4.  **统一了“跳过”和“循环”**：不同于先前的单操作策略，PoLar在一个统一框架内同时支持跳过和循环，并通过实验证明其远优于独立使用任一操作。
5.  **实验证据充分**：通过大量MCTS搜索实验，不仅验证了假设，还揭示了有效程序的结构特性，为学习预测网络的设计提供了有力的理论指导。

### 8. 不足与局限

1.  **依赖离线搜索数据**：PoLar预测网络的训练依赖于MCTS搜索出的程序。虽然搜索是在较小的数据集上离线进行，但MCTS过程本身仍有计算成本，且搜索空间受到约束（如最长段长度、最大循环次数）。
2.  **程序空间受限**：论文将程序限制为“连续段 + 最多一次循环”，虽然基于实验结果，但这仍然是一个强假设，可能忽略了更复杂、非局部或高次循环的程序结构。
3.  **解释性不足**：虽然方法名为“程序”，但预测网络如何学习决策以及“为什么”要跳过/循环特定层，其内在机制仍不透明，缺乏可解释性。
4.  **理论分析缺失**：尽管实验现象丰富，但论文缺少严格的数学或理论分析来解释为什么层跳过和循环能带来性能提升，或者固定深度执行为何是次优的。
5.  **实验数据与领域偏倚**：所有实验主要基于数学推理任务（DART-Math及其分布外数据集）。该方法的有效性在更通用、更复杂的自然语言理解和生成任务上尚未得到验证，可能存在领域偏倚。
6.  **安全性隐患被提及但未深入研究**：论文在“Impact Statement”中提到了动态执行路径可能被错误利用或难以审计的风险，但并未对此进行深入探讨或实验验证。
