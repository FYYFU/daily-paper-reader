## 摘要
在高风险领域，智能体安全性需要正式的策略执行，但现有大多数方法要么依赖无法提供形式化保证的概率性护栏（微调分类器、基于提示的引导），要么依赖手工编码的符号执行，该方法无法扩展到真实策略规范的广度。我们提出了一种自动形式化流水线，该流水线利用基于LLM的生成器-评判器循环，将智能体提示、MCP工具描述和自然语言策略文档转换为经过形式化验证的策略。生成的策略采用Cedar策略语言编写。在MedAgentBench基准测试上，我们的自动形式化策略覆盖了比先前工作中手工编码的符号执行多得多的源自然语言规范内容。

## Abstract
Agent safety in high-stakes domains requires formal policy enforcement, but most existing approaches either rely on probabilistic guardrails (fine-tuned classifiers, prompt-based steering) that offer no formal guarantees, or on hand-coded symbolic enforcement that does not scale to the breadth of real policy specifications. We present an autoformalization pipeline that translates agent prompts, MCP tool descriptions, and natural language policy documents into formally verified policies using an LLM-based generator-critic loop. The resulting policies are written in the Cedar Policy Language. On the MedAgentBench benchmark, our autoformalized policies cover substantially more of the source natural-language specification than the hand-coded symbolic enforcement in prior work.

## 2. 论文提出的方法论

- **核心思想**：利用LLM的生成能力与硬/软评判器循环，将自然语言意图自动转化为形式化策略代码（Policy-as-Code）。整个流程称为“Verification Sandwich”（验证三明治），包含三层：
  - **Grounding Layer（基础层）**：从智能体卡片（系统提示、工具JSON模式）中提取实体、工具模式，定义主体-资源-动作本体，并程序化生成Cedar模式（Schema），确保所有动作映射到真实实体。
  - **Model Layer（模型层）**：使用LLM（Gemini 3 Pro）作为策略生成器，基于系统提示、MCP工具定义和自然语言策略文档，输出候选Cedar策略集。
  - **Safety Layer（安全层）**：包含两个评判器：
    - **Hard Critic（硬评判器）**：确定性检查，利用Cedar CLI进行语法验证、模式合规检查、静态分析（检测空策略、逻辑矛盾），可重试最多3次。
    - **Soft Critic（软评判器）**：LLM-as-a-Judge（Gemini 2.5 Flash），评估策略与自然语言语义的对齐程度，并按照评分准则进行质性评价。
- **关键技术细节**：生成器-评判器循环迭代优化，直到策略同时通过硬、软评判。最终策略由外部确定性策略引擎在运行时强制实施，实现“fail-closed”机制（出错则默认阻止）。Cedar语言被选为目标语言，因为它支持形式化验证、人类可读且具备静态分析能力。

## 7. 优点

- **形式化保证**：硬评判器确保语法、模式、逻辑正确性，软评判器确保语义对齐，结合后提供了比纯概率方法更强的安全保证。
- **可扩展性**：自动生成策略，无需人工编写大量规则，大幅降低手工工作。
- **可审计性**：Cedar策略人类可读，安全工程师可以审核生成的规则。
- **安全架构**：外部确定性策略引擎与LLM分离，支持fail-closed机制，鲁棒性高。
- **实用创新**：引入软评判器解决语义模糊性问题，避免策略过于严格或宽松，达到“Goldilocks”状态。
