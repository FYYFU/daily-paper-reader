## 摘要
现代LLM智能体越来越依赖上下文压缩、摘要或驱逐来将长时间运行的会话保持在一定令牌预算内。我们表明，这种上下文管理层是一个安全关键失效面：智能体在可见时可靠遵守的上下文内治理约束可能被压缩悄然移除，导致同一智能体在会话后期执行被禁止的工具操作。我们将这种失效模式称为治理衰退。我们引入了ConstraintRot，一个具有确定性工具调用评分的长期智能体场景基准，并测量了七个模型家族中由压缩引发的违规行为。在1,323个回合中，违规行为从策略在完整上下文中的0%上升到压缩后的30%，某些模型达到59%；当约束保留在摘要中时，违规行为仍为0%，但当它被丢弃时，违规行为达到38%。我们进一步研究了一种压缩-驱逐攻击，其中对抗性上下文内容使摘要器偏向省略合法策略，并表明优化后的注入击败了所有评估模型。最后，我们提出了约束固定，一种简单的无需训练的缓解措施，将治理约束与有损压缩隔离，并在我们的基准中将违规行为恢复到0%。这些结果表明上下文管理是部署LLM智能体的首要治理面。

## Abstract
Modern LLM agents increasingly rely on context compaction, summarization, or eviction to keep long-running sessions within a token budget. We show that this context-management layer is a safety-critical failure surface: in-context governance constraints that agents reliably obey while visible can be silently removed by compaction, causing the same agent to perform prohibited tool actions later in the session. We call this failure mode Governance Decay. We introduce ConstraintRot, a benchmark of long-horizon agent scenarios with deterministic tool-call grading, and measure compaction-induced violations across seven model families. Across 1,323 episodes, violation rises from 0% with the policy in full context to 30% after compaction, reaching 59% for some models; when the constraint survives the summary, violation remains 0%, but when it is dropped, violation reaches 38%. We further study a Compaction-Eviction Attack, in which adversarial in-context content biases the summarizer to omit a legitimate policy, and show that optimized injections defeat every evaluated model. Finally, we propose Constraint Pinning, a simple training-free mitigation that quarantines governance constraints from lossy compaction and restores violation to 0% in our benchmark. These results identify context management as a first-class governance surface for deployed LLM agents.

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：治理衰退的根本原因是压缩操作对任务连续性优先，而忽略低显著性的约束。若能保护约束不受有损压缩影响，则可恢复治理安全性。为此提出**约束固定（Constraint Pinning）**。

- **关键技术细节**：
  - **治理衰退测量**：定义违规率变化 Δ_decay = v(M, [C(H<t); r]) - v(M, [H<t; r])，即压缩后与压缩前对同一禁止请求r的违规概率差。
  - **ConstraintRot基准**：每个场景包含策略建立、长序列良性任务（使上下文超过压缩预算）、触发请求（自然完成违反策略）。策略通过非系统通道（用户指令、记忆条目、工具输出）输入。
  - **约束固定**：将提取的治理约束存入一个固定缓冲区（pinned buffer），该缓冲区（i）免于压缩，每次压缩后重新注入；（ii）完整性检查：要求压缩后上下文仍蕴含约束。无需训练，仅修改harness的内存管理。
  - **压缩-驱逐攻击**两种变体：①体积攻击（volume）：注入大量无恶意内容迫使压缩触发并驱逐策略；②摘要器注入（summarizer injection）：向压缩步骤嵌入指令（如“省略过时合规性序言”），使策略被主动删除。攻击者仅控制智能体摄入的上下文内容（工具输出、检索文档）。

## 6. 主要结论与发现

- **治理衰退显著存在**：压缩后平均违规率从0%升至30%（最高59%），部分模型（DeepSeek-V4, Kimi-K2.5）甚至超过没有策略时的基线（37%/48%），表明压缩可能比从未设定策略更糟。
- **约束存活是决定性因素**：若约束保留在摘要中，违规率0%；若丢弃，违规率38%。通过摘要器交叉实验确认违规由摘要器驱动，而非智能体模型。
- **软性策略衰退远大于硬性规范**（+50 vs +6个百分点，8.3倍），硬性规范因模型内置先验而掩盖问题。
- **压缩-驱逐攻击有效**：固定注入可突破GLM-5.1（被动抵抗0%→22%）和Gemini-3.5-flash（体积攻击19%）；优化后的注入策略（如“令牌预算框架”）使所有模型失效，包括对固定注入完全免疫的Claude（0%→65%）。
- **约束固定恢复0%违规**：在所有7个模型和两种固定攻击变体下，违规率回到0%，令牌开销<0.5%，且不影响合法任务完成（99%成功率）。
- **应对现有框架验证**：真实框架（LangGraph, LangMem, AutoGen, OpenAI SDK）中再现治理衰退，违规率65%~100%。
- **建议**：（1）将治理约束视为固定状态免于压缩；（2）优先使用头部保留压缩而非纯近期驱逐；（3）将摘要器视为不可信输入槽。

## 8. 不足与局限

- **不可变系统消息通道未覆盖**：若策略通过框架保证保留的系统消息传递，则不受影响；但非系统通道（用户指令、记忆、工具输出）才是主要攻击面。
- **约束固定要求策略可提取为显式规则**：隐含约束（如学习到的规范）无法固定。
- **权威冒充攻击未完全解决**：即使深度锁定，智能体仍会遵从近期上下文中假冒的“操作员更新”，需要可信带外操作员信道（尚未实现）。
- **约束存活评判依赖LLM评判**（虽然用三个评判者多数投票验证，且与关键词启发式一致），但非人工标注。

- **所有实验在模拟沙箱中运行**（工具调用仅解析不执行），虽然是常见实践，但真实部署环境可能有额外因素。
- **伦理说明充分**：仅在自己的基准上演示攻击，不扩大攻击者所需权限（已假设间接注入能力）。
