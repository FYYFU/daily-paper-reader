## 摘要
虽然基于大语言模型的多智能体系统（LLM-MAS）通过编排专门的智能体和外部工具在解决复杂任务方面展现了卓越的能力，但对工具输出的隐式信任造成了关键的攻击面。现有工具攻击受限于领域特异性或固定的静态模板。为应对这些挑战，我们提出了Evo-Attacker，它将工具攻击建模为一个自进化的、记忆增强的强化学习过程。Evo-Attacker构建动态攻击记忆，并采用深思熟虑的推理来检索对抗模式，并在关键时刻策略性地修改干预。此外，我们引入了Attack-Flow GRPO以通过终端结果优化中间推理步骤，解决了长时域信用分配挑战。全面的实验表明，Evo-Attacker始终优于基线模型，凸显了其泛化与进化能力，以及防御性工具防护的迫切需求。

## Abstract
While Large Language Model-based Multi-Agent Systems (LLM-MAS) demonstrate remarkable capabilities in solving complex tasks by orchestrating specialized agents and external tools, the implicit trust in tool outputs creates a critical attack surface. Existing tool attacks are limited by domain specificity or fixed and static templates. To address these challenges, we propose Evo-Attacker, which formulates the tool attack as a self-evolving, memory-augmented reinforcement learning process. Evo-Attacker constructs a dynamic attack memory and employs deliberative reasoning to retrieve adversarial patterns and strategize modifying interventions at critical moments. Furthermore, we introduce Attack-Flow GRPO to optimize intermediate reasoning steps via terminal outcomes, addressing the long-horizon credit assignment challenge. Comprehensive experiments demonstrate that Evo-Attacker consistently outperforms baselines, highlighting its generalization and evolutionary capabilities and the urgent need for defensive tool safeguards.

- **现有方法的局限**：
  - 单智能体工具攻击（如InjecAgent、Forced Output）未考虑多智能体间的复杂交互，简单注入易被下游智能体识别为上下文不一致而失效。

- **核心挑战**：

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将工具攻击建模为**自进化的记忆增强强化学习过程**。通过构建动态攻击记忆（Attack Memory）和审慎推理（Deliberative Reasoning）管道，并利用专用的强化学习算法（Attack-Flow GRPO）优化整个推理管道，实现策略的持续演化。
- **关键技术细节**：
  - **攻击记忆生成**：初始为空，通过探索阶段收集成功导致系统失败的完整交互轨迹，并将每个成功攻击封装为记忆条目 `m(e) = <X_ctx, T_trace>`，其中包含任务上下文和对抗交互序列。攻击记忆 `M_A` 不断积累经验。
  - **记忆增强攻击（三阶段）**：
    - **检索（Retrieve）**：根据当前状态 `x_t`（包含当前工具调用和交互历史）和任务上下文，生成查询 `q_t`，从 `M_A` 中检索top-k最相似记忆。
    - **反思（Reflect）**：评估检索到的记忆是否可迁移、当前状态是否易受攻击，生成推理摘要 `c_t` 和决策 `d_t`（ATTACK / CONTINUE / NOOP）。若为CONTINUE，则更新上下文并重新检索；若为ATTACK，则进入修改阶段。
    - **修改（Modify）**：定位目标工具调用索引，生成修改指令 `ϕ_{t, i_t}`，并将原始返回值修改为 `r’_{t, i_t}`。
  - **优化 via Attack-Flow GRPO**：
    - 将整个攻击轨迹 `ζ` 视为优化单元，定义终端奖励 `R(ζ) = I(系统失败) + λ·R_struct`（结构惩罚）。
    - 使用**组相对优势**：对同一任务上下文采样G个并行轨迹，计算每个轨迹的优势 `A_i`，然后将该终端优势广播到攻击者生成的所有token。
    - 优化目标：最大化期望收益，同时通过KL散度约束策略更新，且仅计算攻击者生成token的损失，屏蔽工具返回和受害者智能体消息的token。
- **算法流程**（文字说明）：算法1描述审慎推理循环：初始化决策为CONTINUE后循环检索-反思，达到最大循环次数或决策为ATTACK/NOOP时退出；若决策为ATTACK且预算充足，则修改指定工具返回。算法2描述优化循环：每步采样任务，执行G个并行攻击（通过算法1），计算终端奖励，计算组相对优势，更新策略参数。
