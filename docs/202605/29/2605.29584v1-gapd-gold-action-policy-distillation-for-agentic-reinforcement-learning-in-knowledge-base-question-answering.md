## 摘要
强化学习（RL）天然适用于智能体知识库问答（KBQA），其中模型必须执行可执行动作、观察知识库反馈并最终返回答案。然而，当前基于RL的KBQA系统主要优化来自最终答案的稀疏奖励，导致中间动作错误弱监督。这对于具有逻辑形式标注的KBQA基准尤其受限：金牌逻辑形式可以转换为可执行的动作序列，但现有流程主要将其用于热启动数据构建，而非用于在策略RL更新。我们提出GAPD，一种训练时的金牌动作策略蒸馏框架，为基于结果的RL添加密集的令牌级指导。为了将金牌动作与在策略学生展开对齐，GAPD使用中间锚点匹配：它将学生探索和金牌执行期间达到的中间实体视为状态锚点，并通过这些探索到的实体集将学生状态与金牌状态匹配。基于该对齐后的金牌动作的条件化当前策略作为停止梯度教师，其令牌分布被蒸馏回生成动作令牌跨度上的普通学生策略。GAPD在WebQSP、GrailQA和GraphQ上持续超越当前最先进方法。

## Abstract
Reinforcement learning (RL) is a natural fit for agentic knowledge base question answering (KBQA), where a model must issue executable actions, observe knowledge-base feedback, and eventually return an answer. However, current RL-based KBQA systems mainly optimize sparse rewards from the final answer, leaving intermediate action errors weakly supervised. This is especially limiting for logical-form annotated KBQA benchmarks: gold logical forms can be converted into executable action sequences, but existing pipelines use them mainly for warm-start data construction rather than for on-policy RL updates. We propose GAPD, a training-time Gold-Action Policy Distillation framework that adds dense token-level guidance to outcome-based RL. To align gold actions with on-policy student rollouts, GAPD uses MID-ANCHOR MATCHING: it treats the intermediate entities reached during student exploration and gold execution as state anchors, and matches student states to gold states through these explored entity sets. The current policy conditioned on this aligned gold action serves as a stop-gradient teacher, whose token distribution is distilled back to the ordinary student policy over generated action-token spans. GAPD consistently surpasses the current state of the art on WebQSP, GrailQA, and GraphQ.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：基于强化学习（RL）的知识库问答（KBQA）系统通常只优化最终答案的稀疏奖励（如准确率），导致中间动作步骤缺乏有效监督。即使数据集提供了标注好的逻辑形式（gold logical forms），现有方法也仅将其用于监督微调（SFT）或数据构建，在策略优化（on-policy RL）阶段被丢弃。
- **背景**：KBQA 可建模为智能体与知识库的交互过程：模型依次发出结构化动作（如 Find_relation、Compare 等），知识库返回实体/状态，最终给出答案。RL 天然适合这一流程，但稀疏奖励使得模型难以识别具体哪个动作导致错误。
- **目标**：利用已有的逻辑形式标注，在在线 RL 训练中为中间动作生成令牌级（token-level）稠密监督信号，弥补稀疏奖励的不足，提升 KBQA 智能体的决策准确性。

## 2. 方法论：核心思想、关键技术细节与算法流程

### 核心思想
- 将 gold 逻辑形式解析为动作序列，在训练时通过**实体锚点匹配（MID-ANCHOR MATCHING）** 将学生（student）的当前执行状态与 gold 状态对齐，然后用对齐后的 gold 动作条件化当前策略作为教师（teacher），计算教师分布与学生分布在对齐令牌上的差异，并将该差异作为令牌级优势（advantage）融入 GRPO 优化。

### 关键技术细节

1. **金牌动作序列生成**：每个训练样本的 gold 逻辑形式被解析为动作序列 g = (g₁, …, gM)，并执行一次得到 gold 中间状态 uₘ 及其达到的实体集 E(uₘ)。

2. **实体锚点匹配（MID-ANCHOR MATCHING）**：
   - 对于学生当前状态 sₜ（基于学生动作执行后得到的实体集 E(sₜ)），与所有 gold 状态计算 Jaccard 相似度：sim(sₜ, uₘ) = |E(sₜ) ∩ E(uₘ)| / |E(sₜ) ∪ E(uₘ)|。
   - 选择最大相似度的 gold 状态 m*(t) = argmaxₘ sim(sₜ, uₘ)，若超过阈值（τₛₜₐₜₑ=0.95）则接受匹配。
   - 匹配后，下一个 gold 动作 g_{m*(t)+1} 用于构造教师侧提示。

3. **自教师蒸馏（Self-Teacher Distillation）**：
   - 教师侧提示：在原始学生上下文 xₜ 中加入对齐后的 gold 动作，形成教师上下文 x̃ₜ。
   - 教师分布：pᵗᵉᵃᶜʰ(·|x̃ₜ) = stop-gradient(πθ(·|x̃ₜ))（采用当前策略，停止梯度，避免教师随学生更新而漂移）。
   - 学生分布：pˢᵗᵘᵈ(·|xₜ) = πθ(·|xₜ)。
   - 令牌级分数：dᵢₖ = log pᵗᵉᵃᶜʰ(yᵢₖ) - log pˢᵗᵘᵈ(yᵢₖ)，经 clip 得到 Aᵍᵃᵖᵈᵢₖ。

4. **融合与策略更新**：
   - 最终令牌级优势：Âᵢₖ = Aᵍʳᵖᵒᵢ + λᴳᴬᴾᴰ · Aᵍᵃᵖᵈᵢₖ，其中 Aᵍʳᵖᵒᵢ 来自 GRPO 的轨迹级优势。
   - 采用 PPO 风格裁剪损失优化策略 πθ。

### 算法流程（文字描述）
1. 对每个提示 q，学生策略 πθ 进行多轮交互，得到轨迹 τ（包含动作和状态）。
2. 执行学生动作序列和 gold 动作序列，收集各自的中间实体集。
3. 对每个学生 turn t，通过实体锚点匹配找到对齐的 gold 状态，若匹配则构建教师侧提示，记录该 turn 的学生令牌位置加入蒸馏掩码 I。
4. 根据最终答案奖励计算 GRPO 优势。
5. 对 I 中的令牌，用 stop-gradient 教师计算 log 概率差异，得到令牌级 GAPD 优势。
6. 融合优势，用 GRPO + GAPD 损失更新策略。

## 6. 主要结论与发现

- GAPD 在**所有三个数据集**上均超越当时 SOTA（KBQA-R1）：
  - WebQSP F1：83.4 → 86.1（+3.2% 相对提升）
  - GrailQA 总体 F1：86.1 → 89.5（+3.9% 相对提升），在 compositional 和 zero-shot 子集上提升更大（分别 +5.5% 和 +4.1%）
  - GraphQ F1：53.8 → 55.1（+2.4% 相对提升）
- 令牌级蒸馏显著优于仅用结果奖励，且**不依赖外部大模型**（自教师复用当前策略），训练后推理无额外开销。
- 实体锚点匹配是对齐的关键；仅按 turn 索引对齐会导致性能暴跌。
- GAPD 在实现更高精度的同时，交互次数更少，说明学到更优的动作序列。

## 7. 优点

- **创新性**：首次将 gold 动作序列作为稠密监督融入在策略 RL 更新，通过实体锚点匹配解决状态对齐问题，设计自教师避免引入外部模型。
- **有效性**：在多个基准上取得显著一致的提升，消融实验验证各组件贡献。
- **实用性**：推理时无额外 oracle，训练时自教师不增加模型参数；交互效率更高。
- **实验完整性**：覆盖主流数据集、多种基线、消融、超参数分析、效率对比和案例剖析，结论可靠。

## 8. 不足与局限

- **训练成本**：比 outcome-only RL 慢约 1.5 倍（12.4h vs 8.2h 在 WebQSP 上），虽然可接受但仍有提升空间。
- **数据依赖**：需要数据集提供可执行的金牌逻辑形式（动作序列），适用于有标注的 KBQA 基准，对无标注场景不直接适用。
- **匹配假设**：实体锚点匹配依赖执行后可达的实体集，需要知识库可精确返回实体集合；若实体不完整或稀疏（如长尾问题），匹配可能失效。
- **参数规模**：仅使用 8B 模型（Llama-3.1-8B），在更大模型上未验证，但文中对比了商业大模型，GAPD 仍然更优。
- **任务泛化**：方法针对 KBQA 设计，动作空间和状态表示依赖 KB 结构，迁移到其他智能体任务（如网页导航、工具使用）需要适配。
- **负信号处理**：仅对匹配的 turn 施加 GAPD 优势，未探索对未匹配 turn 的额外处理（如直接丢弃或降权）。
