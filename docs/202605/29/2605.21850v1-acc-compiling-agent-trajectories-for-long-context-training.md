## 摘要
近期智能体的发展重新激发了对大语言模型长上下文推理能力的需求。然而，训练LLM的这种能力需要昂贵的长文档整理或启发式上下文合成。我们观察到，智能体在解决问题时会生成大量轨迹，跨多轮调用工具并接收环境观察结果。因此，回答原始问题所需的证据分散在这些轮次中，需要整合远距离的上下文片段。然而，标准的智能体SFT会掩码工具响应，仅训练轮次级别的工具选择，造成监督盲点，使得这些分散的信号未被利用。我们提出智能体上下文编译（ACC），它将来自搜索、软件工程和数据库查询智能体的轨迹转换为长上下文问答对，这些问答对将原始问题与跨多轮收集的工具响应和环境观察结果相结合，训练模型无需使用工具即可直接回答。这使得问题与证据之间的依赖关系明确，能够直接监督对远距离片段的长上下文推理，无需额外标注。ACC是一种简单但有效的方法，可与任何现有的长上下文扩展或训练方法结合，提供可扩展的监督微调数据。我们通过MRCR和GraphWalks验证了ACC在长距离依赖建模任务上的效果，这些是具有挑战性的基准测试，需要跨轮指代消解和在扩展上下文中的图遍历。使用ACC训练Qwen3-30B-A3B在MRCR上达到68.3（+18.1），在GraphWalks上达到77.5（+7.6），结果与Qwen3-235B-A22B相当，同时在GPQA、MMLU-Pro、AIME和IFEval上保持通用能力。进一步的机制分析表明，ACC训练的模型展现出任务自适应的注意力重构和专家专业化。

## Abstract
Recent development of agents has renewed demand for long-context reasoning capacity of LLMs. However, training LLMs for this capacity requires costly long-document curation or heuristic context synthesis. We observe that agents produce massive trajectories when solving problems, invoking tools and receiving environment observations across many turns. The evidence needed to answer the original question is thus scattered throughout these turns, requiring integration of distant context segments. Nevertheless, standard agent SFT masks tool responses and only trains turn-level tool selection, creating a supervision blind spot where these scattered signals go unused. We propose Agent Context Compilation (ACC), which converts trajectories from search, software engineering, and database querying agents into long-context QA pairs that combine the original question with tool responses and environment observations gathered across multiple turns, training the model to answer directly without tool use. This makes the dependencies between the question and the evidence explicit, enabling direct supervision of long-context reasoning over distant segments without additional annotation. ACC is a simple but effective approach that can be combined with any existing long-context extension or training method, providing scalable supervised fine-tuning data. We validate ACC on long-range dependency modeling tasks through MRCR and GraphWalks, challenging benchmarks requiring cross-turn coreference resolution and graph traversal over extended contexts. Training Qwen3-30B-A3B with ACC achieves 68.3 on MRCR (+18.1) and 77.5 on GraphWalks (+7.6), results comparable to Qwen3-235B-A22B, while preserving general capabilities on GPQA, MMLU-Pro, AIME, and IFEval. Further mechanism analysis reveals that the ACC-trained model exhibits task-adaptive attention restructuring and expert specialization.

- **背景**：大语言模型（LLM）的智能体（agent）应用日益广泛，智能体通过多轮工具调用（如搜索、代码执行、数据库查询）解决问题，但这也对 LLM 的长上下文推理能力提出了更高要求。
- **痛点**：现有训练长上下文能力的方法成本高昂，要么依赖昂贵的长文档人工标注，要么使用启发式上下文合成（如随机拼接短文本），但这些合成样本缺乏真实问题求解所产生的复杂跨段依赖关系。
- **核心问题**：智能体产生的多轮轨迹中，回答原始问题需要的证据分散在不同轮次的工具响应中，但**标准智能体监督微调（SFT）掩码掉了工具响应，只监督轮次级别的工具选择**，导致这些分散证据的监督信号被浪费——即“监督盲区”（supervision blind spot）。模型无法学会整合远距离片段进行推理。
- **研究含义**：提出一种低成本、可扩展的方法，利用现有的智能体轨迹自动生成高质量长上下文训练数据，直接训练模型不依赖工具进行跨段推理。

## 2. 方法论

### 2.1 核心思想：Agent Context Compilation（ACC）

将智能体的多轮轨迹**编译**成单一的长上下文问答对：把原始问题与所有相关工具响应和环境观察拼接成一个上下文 \( C \)，训练模型直接基于 \( C \) 推理出最终答案，而无需中间的工具调用步骤。这样，最终答案的监督信号可以直接反向传播到每个证据令牌，不受中间动作滤波。

### 2.2 关键技术细节

1. **监督盲区形式化**：
   - 标准智能体 SFT 损失：\( \mathcal{L}_{\text{agent}} = -\sum_{t=1}^{k} \sum_{j \in I_t} \log P(\text{token}_j \mid H_{<t}, \text{token}_{<j}) \)，其中 \( I_t \) 仅包含推理和动作令牌，工具响应被排除。这使得工具响应的梯度只能通过后续短链传导，难以影响最终答案。
   - ACC损失：\( \mathcal{L}_{\text{ACC}} = -\sum_{j \in r \cup y} \log P(\text{token}_j \mid q, C, \text{token}_{<j}) \)，无中间动作项，监督直接覆盖所有证据。

2. **上下文构建**：
   - 对三类智能体轨迹提取证据片段：
     - **搜索（Search）**：访问页面的全文，加上未访问但返回的候选结果作为干扰项。
     - **软件工程（SWE）**：修复过程中涉及的文件内容，加上调试期间查看的其他文件作为干扰项。
     - **数据库查询（SQL）**：所有查询过的表的完整内容。
   - 证据片段被随机排列后拼接成上下文 \( C_i \)，长度不超过预算 \( B \)（论文中为 128K tokens）。随机排列迫使模型靠语义关联定位信息，而非位置顺序。
   - 推理轨迹：使用 DeepSeek-V3.2-Thinking 为每个问答对生成推理链，仅保留能导出正确答案的轨迹（搜索通过率接近100%，SQL约50%，SWE约10%）。

3. **整体流程**：
   - 输入：多轮轨迹 \( \tau = (q, (r_1,a_1,o_1), \dots, (r_k, y)) \)
   - 输出：长上下文训练样本 \( (x_i = (q_i, C_i), y_i, r_i) \)
   - 训练：直接监督 \( r_i \) 和 \( y_i \) 的生成。

## 6. 主要结论与发现

1. **ACC显著提升长距离依赖建模**：
   - MRCR 总体得分 68.28（+18.09），超过 235B 模型（67.51）。
   - GraphWalks 总体得分 77.51（+7.59），与 DeepSeek-V3.2（85.39）和 Kimi-K2（80.04）还有差距，但超过许多大模型。
2. **通用能力未见下降甚至小幅提升**：GPQA (+2.49)、MMLU-Pro (+1.50)、AIME'25 (+3.33)，不可认为是负迁移。
3. **监督盲区存在且被ACC解决**：原始搜索轨迹用标准 SFT 训练后 MRCR 反而下降（-8.03），验证了盲区假设。
4. **多类型智能体数据互补**：仅 SQL 数据对 GraphWalks 提升显著（+5.58），而 Search 和 SWE 不提升甚至下降；混合数据取得最好结果。
5. **机制分析发现模型学到任务自适应模式**：
   - GraphWalks：注意分布在近距和极远距都有增强；多个专家分担远程跳跃计算。
   - MRCR：注意主要增强近距（验证局部指代）；少数专家高度激活，其他抑制。
   - 关键层在两个任务中完全不同，说明不是统一调整。

## 7. 优点

- **简单高效**：仅需收集已有智能体日志，无需额外人工标注，编译过程直接可得监督信号。
- **可扩展性**：可与任何现有长上下文扩展方法（如位置插值、稀疏注意）或训练方法（包括RL）结合。
- **数据多样性**：覆盖搜索（文本检索）、SWE（代码）、SQL（结构化表）三类完全不同的证据结构，提升泛化性。
- **严谨性**：明确指出监督盲区的理论形式化，并通过实验验证了原始 SFT 确实损伤能力，逻辑闭环。
- **机制分析丰富**：从注意力和专家路由两个角度解释提升来源，增加了方法可信度。
- **完整性**：公开了数据集和检查点，可复现。

## 8. 不足与局限

- **模型泛化性不足**：仅基于 Qwen3-30B-A3B 这一个基座模型验证，未测试其他架构或规模的模型（如 LLaMA、Mixtral），结论可能受模型特性影响。
- **上下文长度限制**：训练上下文上限 128K，未探索更长（如 1M tokens）场景下的效果。
- **推理合成偏差**：依赖 DeepSeek-V3.2 生成推理轨迹，可能引入教师模型的偏见或错误，且 SWE 通过率仅 10%，导致大量轨迹被丢弃，数据效率低。
- **隐私与版权风险**：直接使用智能体日志可能包含私人信息（如代码仓库中的密钥）或受版权保护的内容，文中仅建议“仔细数据过滤”，未提供具体方案。

- **评估维度有限**：长距离依赖建模仅用两个任务（MRCR、GraphWalks），虽然难度高，但还不足以覆盖所有长上下文应用场景（如长文档摘要、多文档推理）。
