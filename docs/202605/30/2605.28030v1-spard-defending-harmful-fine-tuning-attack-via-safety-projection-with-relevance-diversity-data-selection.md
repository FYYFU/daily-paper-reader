## 摘要
微调大型语言模型往往会削弱其安全对齐，而有害微调攻击进一步放大了这一问题，其中对抗性数据会移除安全防护并诱发不安全行为。我们提出SPARD，一种防御框架，它将安全投影交替优化与基于相关性和多样性的数据选择相结合。SPARD采用SPAG，它在效用更新和利用一组安全数据进行显式安全投影之间交替优化以强制执行安全约束。为了策划安全数据，我们引入了一种相关性与多样性的行列式点过程来选择紧凑的安全数据，平衡任务相关性与安全覆盖范围。在四种有害微调攻击下对GSM8K和OpenBookQA进行的实验表明，SPARD始终实现最低的平均攻击成功率，显著优于最先进的防御方法，同时保持高任务准确率。代码可在https://github.com/shuhao02/SPARD获取。

## Abstract
Fine-tuning large language models often undermines their safety alignment, a problem further amplified by harmful fine-tuning attacks in which adversarial data removes safeguards and induces unsafe behaviors. We propose SPARD, a defense framework that integrates Safety-Projected Alternating optimization with Relevance-Diversity aware data selection. SPARD employs SPAG, which optimizes alternatively between utility updates and explicit safety projections with a set of safe data to enforce safety constraints. To curate safe data, we introduce a Relevance-Diversity Determinantal Point Process to select compact safe data, balancing task relevance and safety coverage. Experiments on GSM8K and OpenBookQA under four harmful fine-tuning attacks demonstrate that SPARD consistently achieves the lowest average attack success rates, substantially outperforming state-of-the-art defense methods, while maintaining high task accuracy. Code is available at https://github.com/shuhao02/SPARD.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：大语言模型（LLM）的微调过程容易破坏其安全对齐，特别是当微调数据包含恶意或对抗性样本时（有害微调攻击），安全机制会被快速移除，导致模型输出不安全内容。
- **背景**：现有防御方法（如SafeInstr、Lisa、SafeLoRA、PTST等）要么依赖软惩罚（soft penalty）导致安全约束间接且弱，要么随机选取安全数据而忽略其与下游任务的相关性和多样性，难以平衡安全性与任务效用。
- **目标**：提出一种既能显式约束安全（而非仅靠软惩罚），又能智能选择安全数据（兼顾任务相关性和行为多样性）的防御框架，以在保持下游性能的同时大幅降低攻击成功率。

## 2. 提出的方法论

### 核心思想
将微调建模为带有安全约束的优化问题，通过交替进行效用更新和显式安全投影来强制执行约束；同时使用基于行列式点过程（DPP）的优化目标选择安全数据子集，平衡相关性与多样性。

### 关键技术细节
- **安全投影交替优化（SPAG）**：
  - 定义安全约束：`min_θ L(D_ft, θ)  s.t. L(D_safe, θ) ≤ τ`，其中`τ`为预设阈值（如初始模型在安全集上的平均损失）。
  - 每次迭代：先执行效用更新 `θ⁺ = θ - η_ft ∇L(D_ft, θ)`。
  - 检查`θ⁺`是否违反安全约束：若`L(D_safe, θ⁺) ≤ τ`则保留；否则，对安全损失做一阶泰勒展开，得到线性化半空间，并投影到该约束边界上：
    - 投影公式：`θ_new = θ⁺ - α * g_safe`，其中`α = min( (L(D_safe,θ⁺)-τ) / ‖g_safe‖², η_safe )`，`g_safe = ∇L(D_safe, θ⁺)`，`η_safe`为信任域半径。
  - 算法1给出了完整流程：交替采样minibatch，若安全条件满足则直接采用`θ⁺`，否则进行投影校正。

- **相关-多样性DPP数据选择**：
  - 首先计算候选安全样本`x_i`与微调数据`D_ft`的最大余弦相似度作为相关性得分`q_i`。
  - 构造核矩阵：`\tilde{L}_{ij} = (q_i·q_j)^β * K(x_i, x_j)`，其中`K`为样本间相似度，`β`控制相关性权重。
  - DPP的subset概率正比于`det(\tilde{L}_C)`，分解后为`∏_{i∈C} q_i^{2β} · det(L_C)`，第一项鼓励高相关性，第二项（行列式）鼓励多样性。
  - 采用贪心近似算法（每次选择增益最大的样本）来高效求解，维持Cholesky分解，每步复杂度O(m)。

### 算法流程（文字说明）
1. 准备：从通用安全池中通过相关性-多样性DPP选择大小为`p·|D_ft|`的安全子集作为`D_safe`。
2. 对每个训练步骤：
   - 从`D_ft`采样minibatch，计算梯度并更新参数至`θ⁺`。
   - 从`D_safe`采样minibatch，计算安全损失`ℓ_safe`。
   - 若`ℓ_safe ≤ τ`，则接受`θ⁺`；否则计算安全梯度，按投影公式（含信任域限制）修正参数。
3. 重复直至收敛。

## 8. 不足与局限
