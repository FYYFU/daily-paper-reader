## 摘要
语言智能体的强化学习越来越依赖于定制化框架，这些框架需管理长上下文运行、多轮工具使用以及多智能体编排。然而，将这些框架移植到强化学习环境接口中仍然困难，且常丢失重要的训练信号。我们通过Polar弥合了这一差距，它是一个面向任意智能体框架的可扩展异步强化学习部署框架。Polar将智能体框架视为黑盒：它代理LLM API调用，记录token级模型交互，并重建用于训练的忠实于token的轨迹。每个部署节点并行高效管理运行时预热、智能体执行、轨迹重建和评估，暴露异步服务端点，可被独立的大规模训练器消费。这种解耦设计使Polar与智能体框架、训练基础设施和强化学习算法无关，同时提高了长时间运行智能体工作负载的计算利用率。我们通过使用流行的编码框架在软件工程任务上训练智能体来验证Polar。使用简单的GRPO，Polar在SWE-Bench Verified上分别使用Codex、Claude Code、Qwen Code和Pi框架，将Qwen3.5-4B提升了22.6、4.8、0.6和6.2个百分点。我们还展示了Polar在自定义框架上的离线数据生成，并消融了轨迹重建策略。Polar重写了其前身Prorl Agent，并已注册为NeMo Gym环境之一。

## Abstract
Reinforcement learning for language agents increasingly depends on custom harnesses that manage long-running context, multi-turn tool use and multi-agent orchestration. However, porting these harnesses into RL environment interfaces remains difficult and often loses important training signals. We bridge this gap with polar, a rollout framework for scalable asynchronous RL over arbitrary agent harnesses. Polar treats the agent harness as a black box: it proxies LLM API calls, records token-level model interactions, and reconstructs token-faithful trajectories for training. Each rollout node efficiently manages runtime prewarming, agent execution, trajectory reconstruction, and evaluation in parallel, exposing asynchronous service endpoints that can be consumed by independent trainers at scale. This decoupled design makes Polar agnostic to agent harnesses, training infrastructure, and RL algorithms while improving compute utilization for long-running agent workloads. We validate polar by training agents on software-engineering tasks with popular coding harnesses. Using simple GRPO, polar improves Qwen3.5-4B by 22.6, 4.8, 0.6 and 6.2 points on SWE-Bench Verified with the Codex, Claude Code, Qwen Code and Pi harnesses, respectively. We further demonstrate Polar for offline data generation over custom harnesses and ablate trajectory reconstruction strategies. Polar rewrites its preceding work, Prorl Agent, and has been registered as one of NeMo Gym environments.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：语言智能体的强化学习正从短时单步任务转向长程上下文、多轮工具使用和多智能体编排的“智能体”场景。这些场景的轨迹长、步数多、token 量巨大，导致训练目标本身成为一个系统挑战。
- **核心问题**：传统的 RL 训练框架要求将 agent 或 harness（自定义框架）重写为 RL 环境的标准化接口，但现有 agent harness（如 SWE 工具、浏览器、操作系统等）往往实现复杂、语言多样甚至闭源，移植难度极大且容易丢失原生执行路径的训练信号。现有方案（如 SkyRL-Agent、Agent Lightning、rLLM）虽降低了侵入性，但仍要求 agent 符合特定接口，未能完全消除集成负担。
- **整体含义**：Polar 提出“不打开黑盒也能训练”的范式——利用 agent 和 LLM 模型之间普遍的 API 边界作为统一接口，通过代理监听模型调用来捕获 token 级交互，从而在不修改 harness 的前提下实现可扩展的异步 RL 训练。该框架将 harness 视为黑盒，与训练基础设施和 RL 算法解耦，显著降低了智能体 RL 落地的工程成本。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将集成边界从 harness 内部迁移到 LLM 模型 API 端点。所有 LLM 基 agent 都会调用模型，该 API 边界提供了一个公共接口，Polar 通过一个网关代理来监听、记录并重构成 RL 轨迹。Agent harness 无需改动，只需将其模型 base URL 指向代理。
- **架构组成**：
  - **Rollout Server**：接受 TaskRequest，扩展为多个独立 session，负责调度、持久化、回调。
  - **Gateway Node**：管理 session 生命周期，包括运行时准备、harness 执行、代理捕获、轨迹重建、评估和清理。代理与推理服务器同驻，避免额外采集服务。
- **关键技术细节**：
  - **模型 API 代理**：支持 Anthropic、OpenAI Chat/Responses、Google 等协议；自动检测请求类型，归一化为 OpenAI Chat 格式转发给本地推理服务；注入 `logprobs=true`；记录请求、响应、token ID、logprobs、完成原因；对 streaming 请求通过非 streaming 上游响应合成流。
  - **异步阶段隔离（Rollout Staging）**：每个网关内划分 INIT（运行时准备）、READY（缓冲）、RUNNING（执行 harness）、POSTRUN（轨迹重建/评估/回调/清理）四个独立 worker 池。运行时预热和后处理不阻塞 GPU 绑定 agent 执行。评估器可在 agent 运行期间预清洁运行时。
  - **轨迹重建**：
    - **Per-request**：每个完成独立成一条 trace（保守基线，无丢失但 fragment 严重）。
    - **Prefix Merging**：基于严格的 token 前缀匹配自动合并相邻完成，恢复 append-only 对话链。对于每个链，只保留原始采样 token 为可训练（loss mask=1），插页非生成 token 被 mask（loss mask=0）。支持子 agent、并行分支、上下文压缩等自然形成独立链。
  - **评估与奖励传播**：注册式评估器策略，支持 session 奖励、测试输出评估、SWE-Bench 专用评估器；可跨 trace 广播或按 trace 分配过程奖励。
- **算法流程（文字描述）**：
  1. 用户提交 TaskRequest，包含任务指令、agent 配置、评估器、回调查址。
  2. Rollout Server 拆分为多个 session，调度到空闲 Gateway。
  3. Gateway INIT worker 启动运行环境（Docker/Apptainer），准备 harness。
  4. READY 缓冲后，RUNNING worker 执行 harness：harness 通过代理调用 LLM，代理记录完整 token 级数据。
  5. 执行完成后，POSTRUN worker 依据构建策略（如 prefix_merging）重建轨迹，运行评估器，发送回调并清理资源。
  6. 训练器（如 Slime）异步接收结果，进行策略更新。

## 6. 主要结论与发现

- **Polar 能在不修改 harness 的前提下进行有效的 RL 训练**，且在不同 harness 上带来一致提升。
- **最大收益出现在与模型原生对齐较弱的 harness**：Codex 上提升 22.6 个百分点（3.8% → 26.4%），表明 RL 可将模型适应陌生执行协议。
- **Prefix merging 大幅减少训练样本数**（从 1,185 降至 218），训练时间缩短 5.39 倍，GPU 利用率从 20.4% 提升至 87.7%。
- **Polar 可复用于离线 SFT 数据生成**：单次运行产出 504 条通过验证的轨迹（接受率 30.8%），且可扩展至其他 harness 和教师模型。
- **Polar 的异步服务架构支持现代 RL 训练框架（如 Slime）无缝集成**。

## 7. 优点

- **创新性**：以模型 API 边界作为唯一集成点，将 agent harness 视为完全黑盒，彻底消除了对 harness 内部实现的依赖。这一思路简洁且鲁棒。
- **工程解耦**：异步阶段分离、rollout-as-a-service 设计，使运行时准备、执行、后处理和训练解耦，提升整体计算效率。
- **Token fidelity**：通过代理精确记录原始 token ID 和 logprobs，避免了反分词化漂移（retokenization drift），保证训练信号正确。
- **通用性**：与 harness（CLI/SDK/二进制）、训练框架（Slime/SkyRL/PRIME-RL）和 RL 算法（GRPO 等）无关，易于集成到现有生态。
- **可扩展性**：支持多 gateway 横向扩展，适合大规模集群分布式作业。
- **离线数据生成能力**：同一基础架构可用于在线 RL 和离线数据生产，验证了框架的实用价值。

## 8. 不足与局限

- **实验覆盖有限**：仅使用 GRPO 一种 RL 算法和 SWE-Bench 领域（软件工程）。未在网页浏览、OS 操作等长程任务上验证。
- **轨迹重建策略消融不够全面**：只对比了 per-request 和 prefix_merging，未考虑更复杂的 reward 分配方式；per-request 导致的 reward hacking 仅被提及而未系统处理。
- **资源与算力细节缺失**：GRPO 训练的具体 GPU 型号、数量和耗时未报告，影响复现。
- **离线数据实验缺乏对比基线**：未与其他离线生成方法（如拒绝采样基线、其他 harness）比较质量和效率。
- **假设限制**：Prefix merging 依赖于严格的 token 前缀匹配，当 harness 修改 prompt 前缀（如加系统消息注入）时可能无法合并；模型 API 代理需要支持 harness 所使用的确切协议（虽已支持常见几种，但可能不覆盖所有闭源或自定义 API）。
- **未讨论大规模部署瓶颈**：比如代理/网关在高并发下的延迟、内存、网络开销；分布式运行时管理（如容器启动时间）的优化。
- **公平性风险**：不同 harness 的基线分数差异大，且训练使用相同迭代数而 harness 任务复杂度不同，可能导致部分 harness 欠拟合；未报告多次运行的标准差。
