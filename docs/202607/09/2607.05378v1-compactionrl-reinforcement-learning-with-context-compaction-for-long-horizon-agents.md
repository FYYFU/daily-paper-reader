## 摘要
长程智能体LLM日益受到有限上下文窗口的限制，因为扩展的交互轨迹可能在任务完成前就超过最大上下文长度。上下文压缩通过总结之前的交互状态并在压缩上下文下继续执行，提供了一种自然的解决方案，但将压缩融入强化学习仍未被充分探索。我们提出CompactionRL，一种训练具有上下文压缩的长程智能体LLM的强化学习策略。我们的方法通过token级损失归一化和跨轨迹广义优势估计，联合优化任务执行和摘要生成。该设计使LLM智能体能够从压缩的长程轨迹中学习。我们在开源模型上训练CompactionRL，并在智能体编码任务上观察到一致的性能提升。CompactionRL使开源GLM-4.5-Air模型（106B-A30B）在SWE-bench Verified上达到66.8%的Pass@1得分，在Terminal-Bench 2.0上达到24.5%，分别获得7.0和3.1个百分点的绝对提升。基于GLM-4.7-Flash（30B-A3B）构建的CompactionRL将Pass@1分别提升了5.5和6.8个百分点，在SWE-bench Verified上达到56.0%，在Terminal-Bench 2.0上达到20.2%。因此，CompactionRL被部署在训练开源GLM-5.2模型（750B-A40B）的RL流程中。

## Abstract
Long-horizon agentic LLMs are increasingly limited by finite context windows, as extended interaction trajectories can exceed the maximum context length before a task is completed. Context compaction offers a natural solution by summarizing previous interaction states and continuing the rollout under a compressed context, but incorporating compaction into reinforcement learning remains underexplored. We propose CompactionRL, a reinforcement learning strategy to train long-horizon agentic LLMs with context compaction. Our approach jointly optimizes task execution and summary generation with token-level loss normalization and cross-trajectory generalized advantage estimation. This design enables the LLM agents to learn from compacted long-horizon trajectories. We train CompactionRL on top of open models and observe consistent performance gains on agentic coding tasks. CompactionRL enables the open GLM-4.5-Air model (106B-A30B) to achieve Pass@1 scores of 66.8% on SWE-bench Verified and 24.5% on Terminal-Bench 2.0, with absolute gains of 7.0 and 3.1 points, respectively. Built upon GLM-4.7-Flash (30B-A3B), CompactionRL improves Pass@1 by 5.5 and 6.8 points, reaching 56.0% on SWE-bench Verified and 20.2% on Terminal-Bench 2.0, respectively. CompactionRL is thus deployed in the RL pipeline for training the open GLM-5.2 model (750B-A40B).

# CompactionRL 论文详细中文总结

## 1. 核心问题与整体含义（研究动机与背景）

- **核心问题**：长程智能体（long-horizon agentic LLMs）在交互过程中，累计的历史上下文（如工具输出、中间推理、错误信息等）可能超过模型有限上下文窗口，导致任务中断。简单的扩展上下文长度成本高且无法完全解决长序列利用率下降的问题。
- **背景意义**：上下文压缩（context compaction）通过将早期历史总结为更短的状态，使智能体在固定上下文预算下继续执行，是一种自然的解决方案。然而，先前压缩多被用作推断时的启发式或外部记忆操作，**将压缩融入强化学习（RL）训练仍未被探索**。CompactionRL 旨在让 RL 训练也能在压缩环境下进行，从而突破峰值上下文长度的限制，同时训练智能体生成高质量摘要以支持后续决策。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
将上下文压缩作为轨迹收集（rollout collection）的内置环节：当历史接近上下文预算阈值时，策略模型生成一个压缩摘要，之后从“摘要+最近少量交互历史”重建上下文并继续执行。摘要生成和任务执行共享最终任务奖励，从而**联合优化执行策略与摘要策略**。

### 关键技术细节
- **可训练上下文压缩**：
  - 表示历史：`h_t = (s, u, z_1, …, z_t)`，其中 `z_i = (a_i, o_i)`（助手响应 + 环境反馈）。
  - 压缩触发条件：`C - |h_t| < T_comp`（剩余上下文低于阈值）。
  - 摘要生成：附加固定总结指令 `q_sum`，从策略 `π_θ` 采样摘要 `S_t`。
  - 重建上下文：`\bar{h}_t = (s) ⊕ u_resume(S_t) ⊕ (z_{t-k+1}, …, z_t)`，保留最近 k 步（默认 k=2）。
- **训练段（segments）**：完整轨迹被划分为多个段（执行段和摘要段），所有段共享最终任务奖励 `R(τ)`。摘要由同一可训练策略生成，列入 RL 目标。
- **优化策略**：
  - **PPO 框架**而非组归一化方法（如 GRPO）：因为压缩导致可变段数，组归一化不再适用。CompactionRL 使用 PPO 及其值函数进行优势估计。
  - **Token 级损失归一化（Token-Level Loss Normalization）**：对所有优化 token 计算 PPO 损失，而非按样本或段平均，消除因段数和段长不均带来的偏差。

### 算法流程（文字描述）
1. 初始化策略 `π_θ` 和值函数 `V_φ`，对值函数进行预训练（50步）。
2. 轨迹收集：智能体与环境交互，当上下文剩余低于阈值时触发压缩，策略生成摘要并重建上下文继续执行。记录所有执行段和摘要段。

## 6. 主要结论与发现

1. **CompactionRL 一致提升压缩评估下性能**：在固定峰值上下文下，CompactionRL 在 SWE-bench Verified 和 Terminal-Bench 2.0 上取得最优压缩评估结果，超过基础模型和无压缩 RL。
   - GLM-4.7-Flash：SWE-bench 从 50.5% → 56.0%（+5.5）；Terminal-Bench 从 13.4% → 20.2%（+6.8）。
   - GLM-4.5-Air：SWE-bench 从 59.8% → 66.8%（+7.0）；Terminal-Bench 从 21.4% → 24.5%（+3.1）。
2. **联合训练执行与摘要关键**：包含摘要 token 在 RL 损失中比掩码摘要训练（w/o sum.）有稳定提升，且摘要变得更长更详细，说明直接优化摘要质量有助于后续执行。
3. **Token 级损失归一化和跨轨迹 GAE 均必要**：去掉任一组件都会导致性能下降，其中 token 级损失影响更大，表明纠正段数和段长偏差非常重要。
4. **压缩行为改进**：CompactionRL 相比 w/o sum. 变体所需压缩次数更少，工具调用更少，说明摘要更有效，减少了重复探索。
5. **训练动态**：摘要长度和推理 token 数随训练增加，策略熵上升更慢，显示优化更稳定、更可控。

## 7. 优点（亮点）

- **方法创新性**：首次将上下文压缩作为 RL 可训练部分，而不是推断时启发式或外部模块，使摘要生成与任务执行协同优化。
- **技术贡献**：提出 token 级损失归一化和跨轨迹 GAE，有效解决了压缩轨迹中段长度不均和时间信用分配问题。
- **实验充分性**：在多个模型规模、两个基准、多种消融下验证，结果一致且清晰。
- **实际部署**：CompactionRL 已被用于训练更大模型 GLM-5.2 (750B-A40B) 的 RL 流程，表明方法具有可扩展性和实用价值。
- **分析深入**：提供了行为分析和训练动态，直观展示压缩优化如何影响摘要质量、工具调用和推理行为。

## 8. 不足与局限

1. **单窗口评估性能不匹配**：CompactionRL 在单窗口（无压缩）评估下并不稳定且有时会下降，说明训练与测试之间存在 mismatch，不适用于禁止压缩的场景。
2. **Cross-trajectory GAE 仍是近似**：虽然纠正了部分时间错位，但可能无法完全捕捉早期摘要经过多次压缩后的长期影响。
3. **领域覆盖局限**：实验主要集中于代码导向的长程任务（SWE-bench、Terminal-Bench），其他智能体领域（如网络交互、机器人控制）未验证。
4. **奖励设计依赖最终任务正确性**：没有引入独立的摘要质量奖励，完全依赖任务奖励，可能无法充分引导摘要的细节保留。
5. **计算成本**：采用 PPO 需要维护值函数和额外的 GAE 计算，相比无 critic 的方法（如 GRPO）可能增加训练开销。论文未报告具体 GPU 时间。
6. **评估统计**：仅报告 2 次运行均值，未提供方差或置信区间，统计显著性不明确。
