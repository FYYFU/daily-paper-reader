## 摘要
大型语言模型（LLM）可能通过多轮策略被诱导生成有害内容，其中单条用户消息看似安全。现有的运行时防护措施通常将提示或响应作为孤立消息进行评估，这限制了它们恢复累积意图、验证声称的权限或检测跨对话分解的有害目标的能力。本文提出了认知防火墙，这是一种主动的运行时监督框架，在用户与受保护的目标模型之间插入一个独立的监督模型。该框架将安全评估分解为四个分类门控：意图门控（识别请求的操作目标）、零信任上下文门控（将声称的角色和权限视为未验证的证据）、一致性门控（检测跨轮次的升级和分解）以及输出风险门控（在发布前检查候选响应）。门控决策通过升级而非分数平均进行组合，使得任何可信的危险信号都能阻止交互，同时保留可审计的理由。在四个越狱基准测试和一个良性安全测试集上的实验表明，认知防火墙显著降低了单轮、多轮、基于权威和人工制作的攻击的成功率。它在三个攻击集上将攻击成功率降低到2%或以下，在最困难的人工制作攻击集上降低到14%，同时保持了8%的过度拒绝率。这些结果表明，分解的、对话级别的监督可以改进LLM安全的主动遏制和可审计性。

## Abstract
Large language models (LLMs) can be induced to produce harmful content through multi turn strategies in which no single user message appears clearly unsafe. Existing runtime safeguards commonly evaluate prompts or responses as isolated messages, which limits their ability to recover ac-cumulated intent, verify asserted authority, or detect harmful objectives decomposed across a dialogue. This paper presents the Cognitive Firewall, a proactive runtime oversight framework that interposes an independent oversight model between a user and a protected target mod l. The framework decomposes safety assessment into four categorical gates: an intent gate that identi-fies the operational objective of a request, a zero trust context gate that treats claimed roles and permissions as unverified evidence, a consistency gate that detects escalation and decomposition across turns, and an output risk gate that inspects candidate responses before release. Gate decisions are combined through escalation rather than score averaging, allowing any confident danger signal to block an interaction while preserving an auditable rationale. Experiments on four jailbreak benchmarks and a benign safety test set show that the Cognitive Firewall substantially reduces attack success across single turn, multi turn, authority based, and human crafted attacks. It lowers attack success to 2 percent or below on three attack sets and to 14 percent on the most difficult human crafted set, while maintaining an 8 percent over refusal rate. These results indicate that decomposed, conversation level oversight can improve proactive containment and auditability for LLM safety.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）在运行时面临的安全威胁不仅来自单条恶意消息，更常见于多轮对话中**累积的恶意意图**、**用户声称的虚假权威/角色**、以及**分解到多个无害子问题中的有害目标**。现有的运行时防护（如 Llama Guard、ShieldGemma 等）仅对每条消息进行独立分类，无法恢复跨轮的意图、验证声称的身份，也无法检测渐进式升级攻击。
- **整体含义**：该论文认为 LLM 安全防护本质上是一个**理解问题**而非分类问题，需要对话级别的监督。作者提出**认知防火墙（Cognitive Firewall）**，通过在用户和目标模型之间部署独立的监督模型，进行分解式、主动的安全评估，以实现对多轮攻击的主动遏制和可审计性。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将安全判断分解为四个独立的“门控”（Gate），每个门控输出一个分类性的安全/危险标签，采用**升级决策规则**（任何门控一旦判定危险即阻断交互），避免分数平均导致的信号稀释。
- **关键技术细节**：
  - **威胁模型**：假设攻击者可以控制每一轮用户输入，使用任意措辞、断言权威、多轮策略。系统提示和部署栈可信，目标模型视为黑盒。
  - **四门控设计**：
    - **意图门 G1（预生成）**：识别请求的实际操作目标是否涉及可转移的有害能力（如武器合成、恶意代码），而非基于表面措辞判断。采用两步推理：先找到字面指代物（如“结束进程”是安全的），再判断该指代物是否本身有害。
    - **零信任上下文门 G2（预生成）**：专门检测用户是否声称角色、身份、权限、覆盖指令等以获取信任。不作为一般内容安全检测，而是聚焦“声称的可信度”。返回“操纵性”或“合理”。
    - **一致性门 G3（预生成）**：读取整段对话历史，检测逐渐升级、目标漂移、任务分解等时间性操作。首轮用户输入不触发（安全）。
    - **输出风险门 G4（后生成）**：检查候选响应是否实际交付了有害能力或有害制品（如仇恨言论、钓鱼消息）。返回“有害”或“无害”。
  - **升级决策规则**：预生成门控先评估，任何门控返回危险则**在调用目标模型前阻断**，无须等待其他门控。只有所有门控均为安全时才调用目标模型，再经过输出门控决定是否发布响应。每个门控输出一个标签和一句理由，无分数阈值。

## 6. 主要结论与发现

- 认知防火墙在所有攻击集上大幅降低 ASR：`jbb` 从 45% → 1%，`crescendo` 从 10% → 0%，`safemt` 从 30% → 2%，`mhj` 从 74% → 14%。
- **大部分阻断发生在生成前**（预生成门控），优于任何消息级防护或轨迹感知防火墙。
- **过度拒绝率仅 8%**，显著低于多数基线（如 ShieldGemma 18%、Granite Guardian 21%）。
- **每个门控覆盖不同攻击模式**：意图门控负责单轮直接请求；一致性门控负责多轮升级；输出门控作为后防线捕获分解攻击的载荷；上下文门控在权威探测中独占作用（阻断率 75% 且零误报）。
- **预生成门控和后生成门控互补**：仅用预生成门控或仅用输出门控都不如完整防火墙。
- 在多个种子、不同目标/监督模型、自适应攻击下性能稳定，表明框架模块有效。

## 7. 优点

- **分解式评估**：避免将复杂安全判断压缩为单一分数，每个门控做出窄化、可辩驳的决策，增强可审计性（每步都有标签和理由）。
- **主动阻断**：在目标模型生成前即可安全地阻断，降低计算浪费和有害输出风险。
- **零信任上下文件门控创新**：明确隔离权威冒充这一威胁通道，而现有系统多将其混入一般内容检测。
- **升级决策规则**：避免分数平均导致敏感信号被稀释，特别适合检测升级式或分解式攻击。
- **低过度拒绝**：通过意图门控的字面指代物解析，避免因表面用语危险而误拒良性提示。
- **广泛评估**：覆盖多种攻击类型，并进行消融、鲁棒性、泛化测试，方法稳健。

## 8. 不足与局限

- **实验规模有限**：攻击集样本量小（50-100），权威探测仅 16 个，自适应攻击仅 20 个，统计信度有限。
- **基线比较不完整**：未对所有基线进行多种子重复，部分基线（如 THRD、TCA）为自行复现且略去组件，比较结果可能低估这些基线的性能。
- **同家族偏差**：评判模型（`qwen3.5:122b`）与监督模型（`qwen3:8b`）同属 Qwen 系列，可能存在同家族偏好，需不同家族评判和人工审核验证。
- **门控提示在样本内开发**：数据集的提示是在部分实验样本上开发的（in-sample），可能高估性能；未来需进行彻底留出评估。
- **未测试对监督模型的提示注入**：一个模型执行所有四个门控，攻击者可能通过注入操控监督模型，论文未对此进行压力测试。
- **过度拒绝仅测量单轮良性集**：对多轮良性对话的误报成本未测量。
- **延迟问题**：每轮需要 3-4 次串行监督调用，可能引入可感知延迟，论文未进行延迟剖析。
- **主观边界**：有害性判定（如虚构框架、双用途答案）依赖门控提示和评判模型的策略参数，不同操作者可能有不同标准。
- **残留失败**：`mhj` 集上仍有 14% 的 ASR，特别是有 12 个绕过轨迹门和输出门的人工载荷，需要更强或任务蒸馏的监督模型。
