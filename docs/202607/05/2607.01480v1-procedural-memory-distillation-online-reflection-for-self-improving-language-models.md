## 摘要
带有可验证奖励的强化学习（RLVR），以及最近的自蒸馏变体如SDPO，基于验证器评估每次推出，并根据该回合级信号更新策略。然而，推出中更丰富的程序性信息很少被保留或重用。在跨回合和跨历元中，模型在变化的策略下反复遇到相关问题，产生跨回合信号，而回合局部更新无法捕获这些信号：哪些策略持续通过验证，哪些失败模式持续存在，哪些模式重复出现。我们提出程序记忆蒸馏（PMD），它将跨回合信号转换为可重用的程序记忆，并在训练期间将其蒸馏到策略权重中。该记忆作为训练支架，被吸收到策略本身中，从而在推理时产生无记忆模型。PMD在三个抽象级别组织记忆：原始轨迹、自我反思的策略和经验教训，以及跨问题重复出现的更高级行为模式，所有这些都从模型自身的轨迹中在线提取。一个记忆条件的自我教师利用累积的经验来监督学生自己的推出，使学生能够在其参数内逐步内化程序性知识。核心设计原则是共同进化：策略生成更新记忆的推出，而记忆塑造更新策略的监督。实验上，在Qwen3-8B和OLMo3-Instruct-7B上，PMD在SCIKNOWEVAL上比SDPO提高了3.8-5.5%，在LIVECODEBENCH上提高了7.9-13.6%。共同进化驱动了这些增益：冻结记忆或策略在SCIKNOWEVAL各领域上比PMD落后超过10%。

## Abstract
Reinforcement learning with verifiable rewards (RLVR), along with recent selfdistillation variants such as SDPO, evaluates each rollout against a verifier and updates the policy from that episode-level signal. However, the richer procedural information in the rollout is rarely retained or reused. Across episodes and epochs, the model repeatedly encounters related problems under a changing policy, producing cross-episode signals that episode-local updates cannot capture: which strategies consistently pass verification, which failure modes persist, which patterns recur. We propose Procedural Memory Distillation (PMD), which converts these crossepisode signals into reusable procedural memory and distills it into the policy's weights during training. This memory functions as a training scaffold, absorbed into the policy itself, yielding a memory-free model at inference. PMD organizes the memory at three levels of abstraction: raw trajectories, self-reflected strategies and lessons, and higher-level behavioral patterns that recur across problems, all extracted online from the model's own trajectories. A memory-conditioned self-teacher draws on the accumulated experience to supervise the student on its own rollouts, enabling student to progressively internalize procedural knowledge within its parameters. The central design principle is co-evolution: the policy generates rollouts that update the memory, and memory shapes the supervision that updates the policy. Empirically, across Qwen3-8B and OLMo3-Instruct-7B, PMD improves over SDPO by 3.8-5.5% on SCIKNOWEVAL and 7.9-13.6% on LIVECODEBENCH. Co-evolution powers these gains: freezing either the memory or the policy trails PMD by more than 10% across SCIKNOWEVAL domains.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：现有的强化学习与可验证奖励方法（如RLVR、PPO、DPO、GRPO）以及自蒸馏变体（如SDPO）都基于**单次推出（episode）**的验证信号来更新策略。然而，在训练过程中，模型会反复遇到相同或相关的问题，但跨推出、跨周期的信息（如哪些策略持续成功、哪些失败模式重复出现、哪些模式在多个问题中反复出现）被完全丢弃，未被系统性地利用。
- **核心问题**：如何将模型在重复尝试中积累的**跨推出程序性知识**（procedural knowledge）转化为可重用的学习信号，并蒸馏到策略参数中，使模型在推理时无需外部记忆即可内化这些知识。
- **整体意义**：提出了一种新的训练范式——程序记忆蒸馏（PMD），通过在线构建三级记忆层次（经验、洞察、行为）来引导自我教师（self-teacher），实现策略与记忆的**共同进化**（co-evolution），显著提升语言模型在科学推理和代码生成等可验证任务上的自我改进能力。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **程序记忆蒸馏（PMD）**：在SDPO的在线蒸馏框架基础上，加入一个在线构建的程序记忆库，该记忆库由模型自身在训练过程中的推出（rollouts）生成，并组织为三个抽象层次。记忆用于条件化自我教师（teacher），教师生成更丰富的监督信号来训练学生（student）策略。推理时学生无需任何记忆。
- **共同进化设计原则**：策略生成推出→推出更新记忆→记忆影响教师→教师训练新策略→新策略再次生成推出。两者紧密耦合，保持教师信号与政策当前状态对齐。

### 关键技术细节
- **三级记忆层次**：
  - **经验记忆（Level-0）**：按问题存储原始轨迹、奖励和反馈。每个问题最多保留5个成功和3个失败轨迹，通过嵌入余弦相似度进行多样性去重。
  - **洞察记忆（Level-1）**：对每个问题的累积经验进行反思，提取**策略**（成功原因）和**教训**（失败原因）。同时利用成功/失败对比进行提取，并引入反捷径过滤规则，防止输出元语言、选项字母引用等。
  - **行为记忆（Level-2）**：基于问题嵌入（Qwen3-Embedding-0.6B）进行语义聚类，将同一簇内的问题的洞察和经验进一步抽象为可重用的行为模式（技能、常见错误、准则）。行为库通过检索-决定机制进行演化（新增、更新、删除）。
- **记忆条件化教师**：教师接收当前问题的问题特定记忆（经验+洞察）以及从全局行为库中检索的top-K相关行为（K=3），结合当前批次的上下文（如验证器反馈、成功兄弟姐妹响应），形成增强的提示，重新生成教师分布。
- **训练目标**：采用SDPO的逆向KL散度，让学生匹配记忆条件化教师在同一前缀状态上的分布。教师通过stopgrad固定。
- **更新流程**：
  1. 学生从当前策略采样推出，获得奖励和反馈。
  2. 更新经验记忆和洞察记忆（异步）。
  3. 定期（每个epoch内固定，每K步）更新行为记忆。
  4. 构建教师提示，计算损失，更新策略参数。

## 3. 实验设计

### 数据集与Benchmark
- **SciKnowEval**：科学多选题（生物、化学、物理、材料科学），每个问题有4个选项，使用规则奖励（提取字母匹配）。训练/测试划分：生物450/50，化学1890/210，材料841/94，物理720/80。
- **LiveCodeBench v6**：代码生成基准，使用执行单元测试作为稀疏奖励（全部通过得1，否则0）。训练集924题，测试集131题。

### 对比方法
- **Base Policy**：初始策略（Qwen3-8B、OLMo3-Instruct-7B，thinking模式关闭）。
- **GRPO**：仅使用稀疏奖励的强化学习方法。
- **SDPO**：使用反馈条件化自蒸馏，无显式程序记忆。
- **PMD**：本文方法，在SDPO基础上加入在线程序记忆。
- **消融变体**：
  - PMD-Transient：每步后丢弃记忆，仅保留单步反思。
  - Evolving Memory + Frozen Policy：记忆更新但策略固定。
  - Frozen Memory + Evolving Policy：记忆固定一次构建，策略更新。
  - 不同记忆层次（仅经验、经验+洞察、完整三级）。

### 评估指标
- SciKnowEval：avg@16（16次推出中正确比例的平均值）。
- LiveCodeBench：score@4（4次推出中至少有一次全部通过的比例）。

## 6. 论文的主要结论与发现

1. **PMD一致优于SDPO和GRPO**：在SciKnowEval上相对SDPO提升3.8-5.5%，在LiveCodeBench上提升7.9%-13.6%。
2. **共同进化是核心驱动因素**：冻结记忆或策略导致性能下降超过10%，证明策略与记忆的在线耦合至关重要。
3. **持久性（跨步记忆累积）贡献最大**：尤其是在代码生成中，成功模式稀疏，需要跨多个尝试积累。
4. **反射（单步结构化提取）也有帮助**：即使没有持久性（PMD-Transient），也比SDPO略好（+1.3pp在SciKnowEval）。
5. **记忆层次影响**：完整三级记忆（经验+洞察+行为）在PMD下达到最佳平均性能；行为记忆提供了跨问题可转移知识，但需要蒸馏到策略才有效（冻结策略时行为记忆帮助不显著）。
6. **推理扩展性更好**：PMD保持更宽的验证器headroom（maj@k到best@k的差距），覆盖更多问题，且继续从更多推出中受益，而SDPO很快饱和。
7. **记忆可跨规模迁移**：从8B训练的记忆可以提升更小或更大模型的表现，且共同进化的记忆优于冻结策略产生的记忆。
8. **内部化证据**：训练后学生解码时越来越多地使用“strategy”“lesson”“behavior”等术语，且生成了更长的、包含教师侧推理结构的响应，说明程序知识被吸收到了参数中。

## 7. 优点

- **设计优雅**：记忆仅在训练时作为支架使用，推理时无额外依赖，保持推理效率。

## 8. 不足与局限

- **任务局限性**：仅评估了固定任务分布下的反复训练（科学MCQ和代码生成），未在长程agent环境（如工具使用、多任务、开放世界）中验证。
