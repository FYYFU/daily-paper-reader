## 摘要
多模态大语言模型通常遵循固定的“先思考后回答”范式，这在异构多任务设置中效率低下，因为简单输入可能不需要显式推理，而困难输入则可以从中大幅受益。在后期训练中，学习何时思考也不稳定，不平衡的 rollout 可能会导致模型倾向于始终思考或始终直接回答。我们提出了 Switch-Reasoner，一种基于 GRPO 的框架，能够自适应地选择 MLLMs 的推理模式。它将思考视为虚拟工具调用，并允许模型直接回答或在回答前调用显式推理。为了稳定这一决策，我们引入了一种双层调节机制，在平衡思考模式和直接模式整体使用的同时，基于两种选择的相对收益提供样本级监督。在 11 个多模态任务上的实验表明，Switch-Reasoner 在保持强劲性能的同时减少了不必要的推理，实现了更好的准确性与效率之间的权衡。

## Abstract
Multimodal Large Language Models (MLLMs) often follow a fixed Think-then-Answer paradigm, which is inefficient in heterogeneous multitask settings because simple inputs may not require explicit reasoning while difficult ones can benefit substantially from it. Learning when to think is also unstable during post-training, where imbalanced rollouts can drive the model toward always-thinking or always-direct behavior. We propose Switch-Reasoner, a GRPO-based framework that learns to adaptively select reasoning modes for MLLMs. It treats thinking as a virtual tool invocation and allows the model to either answer directly or invoke explicit reasoning before answering. To stabilize this decision, we introduce a dual-level regulation mechanism that balances the overall use of Thinking Mode and Direct Mode while providing sample-level supervision based on the relative benefit of the two choices. Experiments on 11 multimodal tasks show that Switch-Reasoner reduces unnecessary reasoning while maintaining strong performance, achieving a better accuracy-efficiency trade-off.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：多模态大语言模型（MLLMs）在异构多任务场景中通常采用固定的“先思考后回答”（Think-then-Answer）范式，但不同任务对显式推理的需求差异巨大：简单输入可能不需要推理，而困难输入则能从推理中获益。这种固定模式导致效率低下，且在后训练阶段（尤其是使用GRPO时），由于不均衡的采样（rollout）容易引发模式坍塌（始终思考或始终直接回答），无法稳定学习“何时思考”。
- **动机**：理想的多模态模型应能自适应地在“思考模式”（Thinking Mode）和“直接模式”（Direct Mode）之间切换，仅在必要时调用推理，从而在保持性能的同时降低计算成本。现有工作（如early-exit、reasoning-pruning、自适应推理）要么只缩短轨迹而非决定是否推理，要么依赖复杂的多阶段训练或精心设计的调度，缺乏在GRPO下稳定学习实例自适应策略的方法。
- **整体含义**：本文旨在解决GRPO训练下异构多模态混合任务中实例自适应模式选择的稳定性问题，防止过度推理或推理不足，实现更好的精度-效率权衡。

## 2. 方法论：核心思想、关键技术细节、算法流程
- **核心思想**：将显式推理视为一种**虚拟工具调用**（Thinking-as-Tool），通过可切换提示（switchable prompt）使模型能在直接回答和调用推理工具之间进行显式选择；并引入**双层调节机制**（dual-level regulation）来稳定该决策，避免模式坍塌。
- **关键技术细节**：
  - **Thinking-as-Tool 可切换提示**：模型先输出格式化的响应：要么直接输出 `<answer>...</answer>`（Direct Mode），要么输出工具调用 `<tool_call> {"name":"require_think", ...} </tool_call>` 触发思考。后者会触发后续思考工具响应，引导模型生成 `<thinking>...</thinking>`（Thinking Mode）。从而将模式选择转化为离散的、可优化的动作。
  - **全局模式平衡控制（Global Mode Balance Control）**：基于上一批次rollout的统计量（两种模式的使用比例、准确率差异），生成一个全局控制信号B，对当前批次中暂时不受欢迎的模式施加惩罚（乘以因子η），以防止两种模式的使用比例失衡。
  - **样本级细粒度优化（Sample-level Fine-grained Optimization）**：对每个输入样本，分别强制在Direct Mode和Thinking Mode下采样K次（反事实rollout），评估两种路线的经验成功率，判断样本属于“must-think”、“safe-direct”或“uncertain”三类。然后根据选择是否匹配偏好，施加正负路由奖励（route reward）。
  - **奖励计算**：最终奖励由三部分组成：基础奖励（正确性+格式合规）、全局调整后的奖励（乘以惩罚因子）、样本级路由奖励。该奖励用于GRPO的组相对优势估计。
- **算法流程（文字描述）**：
  1. 采样一批多模态输入。
  2. 对每个输入，从当前策略生成G个可切换rollout（允许选择模式）。
  3. 对每个输入，分别强制生成K个Direct Mode rollout和K个Thinking Mode rollout（反事实采样），计算经验成功率，划分样本类别（must-think/safe-direct/uncertain）。
  4. 计算每个rollout的基础奖励（正确性+格式）。
  5. 应用上一批次的全局控制信号B，得到全局调整后的奖励。
  6. 根据样本类别和实际选择模式，计算路由奖励并加到最终奖励上。
  7. 更新全局控制器（使用当前批次统计计算新B）。
  8. 使用最终奖励计算组相对优势，通过GRPO目标优化策略。
