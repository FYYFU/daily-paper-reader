## 摘要
LLM智能体通过外部工具自主执行任务的能力日益增强，带来了复杂且不断演变的安全风险。然而，现有的安全测试主要针对专家设计的安全违规行为，且相应的结果通过硬编码规则评估，这使得随着智能体的发展，测试扩展的成本高昂。为此，我们提出了Vera，一个端到端自动化安全测试框架，它通过三阶段自我强化的流水线，将软件工程测试原则应用于非确定性智能体。首先，文献驱动的探索持续发现并结构化新兴风险，形成安全风险、攻击方法和工具执行环境的分类体系。其次，跨分类维度的组合组合生成可执行的安全用例，每个用例指定具体的安全目标、通过编程方式构建的初始状态，以及基于可观察伪影的确定性验证谓词。第三，自适应执行在隔离沙箱中运行异构智能体，其中控制智能体基于运行时观察引导多轮交互，而基于证据的验证器根据环境状态和工具调用证据（而非模型自我报告）判断结果。我们在四个生产级智能体框架（OpenClaw、Hermes、Codex、Claude Code）上评估了Vera，揭示了显著的安全弱点，在多通道攻击下平均攻击成功率达到93.9\%；我们还发布了Vera-Bench，包含跨越三种执行环境的124个风险类别的1600个可执行安全用例。这些结果表明，模块化、可执行的测试基础设施对于快速发展的智能体系统的大规模、严格且可维护的安全评估至关重要。代码已在https://github.com/Yunhao-Feng/Vera公开。

## Abstract
LLM agents increasingly perform autonomous actions through external tools, leading to complex and evolving safety risks. However, existing safety testing targets expert-designed safety violations, and the corresponding outcomes are evaluated by hard-coded rules, making them costly to extend as agents evolve. To this end, we present Vera, an end-to-end automated safety testing framework that instantiates software engineering testing principles for non-deterministic agents through a three-stage, self-reinforcing pipeline. First, a literature-driven exploration continuously discovers and structures emerging risks into taxonomies of safety risks, attack methods, and tool execution environments. Second, combinatorial composition across taxonomy dimensions produces executable safety cases, each specifying a concrete safety goal, a programmatically constructed initial state, and a deterministic verification predicate grounded in observable artifacts. Third, adaptive execution runs heterogeneous agents in isolated sandboxes where a control agent steers multi-turn interaction based on runtime observations, while evidence-grounded verifiers judge outcomes from environment state and tool-call evidence rather than model self-report. We evaluate Vera on four production agent frameworks (OpenClaw, Hermes, Codex, Claude Code), revealing substantial safety weaknesses, with average attack success rates reaching 93.9\% under multi-channel attacks; we also release Vera-Bench, comprising 1600 executable safety cases spanning 124 risk categories across three execution settings. These results indicate that modular, executable testing infrastructure is essential for rigorous and maintainable safety evaluation of rapidly evolving agentic systems at scale. The code is publicly available at https://github.com/Yunhao-Feng/Vera.

## 6. 论文的主要结论与发现

1. **Vera 能够有效发现安全弱点**：在四个生产级智能体框架上，多通道攻击平均执行成功率（ESR，即攻击成功实现）达到 93.9%，单通道为 90.6%，说明大多数安全违规场景可以被实现。
2. **覆盖广泛的风险空间**：Vera-Bench 包含 1600 个可执行用例，覆盖 124 个风险类别、77 种攻击方法、30 种环境，且各维度的组合 ESR 均高于 40%，证明组合生成产生了有意义的测试用例而非退化组合。
3. **自适应测试驱动明显增益**：从良性（ESR 70.5%）到单通道（90.6%）的提升（+20.1 个百分点）量化了自适应交互的价值——多轮调整可有效绕过单轮静态提示难以突破的防御。
4. **多通道测试揭示渠道级鲁棒性差异**：总体多通道比单通道提高 3.3 个百分点，但不同智能体差异显著（OpenClaw +6.3，Claude Code -2.1），说明仅测试单通道无法反映完整的攻击面。
5. **“能力-脆弱性对齐”现象**：更强的指令遵循和工具编排能力（如 Claude Code、Hermes）反而使智能体更容易被多轮自适应攻击操纵，表明有用性与安全性之间存在结构性张力。
6. **利用Vera数据微调的 guard 模型显著提升**：在 Vera-Bench 上准确率从 67.0%（Qwen3Guard）提升至 93.0%，并在 R-Judge 上达到最高准确率（61.7%），证明安全信号具有泛化能力。

## 7. 优点：方法或实验设计上的亮点

- **将软件工程测试原则系统应用于非确定性Agent**：提出了“可执行安全用例”概念，包含确定的初始状态、验证谓词，使测试可重复、可验证。
- **文献驱动、持续演化的风险分类**：通过自动摘要智能体从文献中构建三维分类树，并支持更新操作，使得测试覆盖随风险演化而扩展，降低人工维护成本。
- **组合式测试用例生成**：利用三个正交分类树的笛卡尔积生成大规模候选目标，并通过兼容性过滤和去重保留有意义组合，保证覆盖面同时避免退化。
- **自适应多轮交互机制**：控制智能体根据实时观察调整攻击策略，克服了静态提示的局限性，更真实地模拟了对抗性使用场景。
- **基于环境状态证据的验证**：优先使用不可篡改的环境状态（如文件、数据库记录）来判断安全违规是否实现，而非依赖模型自我报告，降低了误报率。
- **统一执行契约支持异构Agent**：通过适配器将不同框架统一接入，实现了公平比较，且代码开源，有利于复现和扩展。
- **大规模实证评估**：在四个生产框架、多种模型、多威胁模型下进行了系统评估，并提供了丰富的覆盖分析、成本分布和跨测试泛化分析。
- **开放基准和代码**：发布了 Vera-Bench（1600个用例）和代码，有助于社区后续研究和安全性改进。

- **未与现有端到端测试框架直接比较**：论文没有提出一个基线测试框架作为对照，而是通过详细分析Vera的覆盖和效能来论证其优势，缺乏同类系统的量化比较。
- **ESR定义本身的局限**：执行成功率衡量的是测试用例在框架上是否可执行并满足谓词，但实际上攻击成功率还可能受控制Agent策略影响，可能与真实攻击者策略存在差异。
