## 摘要
尽管推理时扩展提高了大语言模型（LLMs）的推理能力，但生成长思维链（CoTs）的需求却成为了计算瓶颈。因此，与CoT等顺序扩展方法不同，最近的并行扩展技术转而使用分叉与合并（FJ）原语将工作分配到多个LLM线程中。然而，在分叉-合并范式中，线程通常是临时的，且彼此之间不进行点对点通信，这限制了可扩展性。为了解决这个问题，我们引入了消息传递语言模型（MPLMs），这是一个用于LLM推理的框架，其中线程通过轻量级的发送和接收原语直接通信。MPLMs通过两种关键机制实现高效扩展：（1）减少通信成本，通过避免冗余的上下文共享实现；（2）抢占，允许线程基于来自同伴的部分信息提前终止。我们在三类任务上展示了MPLMs的前景。首先，在数独谜题上，我们证明MPLMs所需的上下文比串行CoT和并行FJ渐近地更小。然后，我们微调一个单一模型来解决25x25的数独谜题，这些谜题对于标准的CoT和FJ方法以及没有工具的前沿推理模型仍然具有挑战性。其次，在3-SAT问题上，抢占能力允许终止没有希望的分支，从而提高了效率。最后，我们表明，适当提示的预训练大模型遵循MPLM协议，在长上下文问答任务上与流行的分叉-合并方法相比取得了有竞争力的结果。

## Abstract
While inference-time scaling has improved the reasoning abilities of large language models (LLMs), the need to generate long chains-of-thought (CoTs) is a computational bottleneck. Thus, in contrast to sequential scaling methods like CoT, recent parallel scaling techniques instead use fork and join (FJ) primitives to divide work across multiple LLM threads. However, in the fork-join paradigm, threads are typically transient and do not communicate pointwise with one another which limits scalability. To tackle this, we introduce Message Passing Language Models (MPLMs), a framework for LLM reasoning in which threads communicate directly via lightweight send and receive primitives. MPLMs enable efficient scaling through two key mechanisms: (1) reduced communication costs, achieved by avoiding redundant context sharing, and (2) preemption, which allows threads to terminate early based on partial information from their peers.   We demonstrate the promise of MPLMs on 3 classes of tasks. First, on Sudoku puzzles, we show that MPLMs require an asymptotically smaller context than both serial CoT and parallel FJ. We then fine-tune a single model to solve 25 x 25 puzzles that remain challenging for standard CoT and FJ approaches, as well as frontier reasoning models without tools. Second, on 3-SAT puzzles, the capability of preemption allows termination of unpromising branches, which results in improved efficiency. Finally, we show that appropriately prompted large pre-trained models follow the MPLM protocol, achieving competitive results on long-context question answering relative to popular fork-join approaches.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：大型语言模型（LLMs）的推理能力提升依赖于测试时计算扩展，但现有的串行思维链（CoT）方法需要生成长序列，成为计算瓶颈；并行方法如分叉-合并（Fork-Join, FJ）虽然可并行解码多个线程，但线程之间缺乏点对点通信，所有信息必须通过父线程集中传递，导致通信瓶颈和隐式串行化，限制了可扩展性。
- **核心挑战**：如何设计一种并行推理机制，使得线程可以低开销地直接通信，从而避免冗余上下文共享、降低计算复杂度，并支持基于部分信息的提前终止（抢占）。
- **研究意义**：提出消息传递语言模型（MPLM），将分布式计算中的消息传递接口（MPI）思想引入LLM推理，为大规模结构化推理任务提供更高效、可扩展的并行范式的理论依据和实证基础。

## 2. 方法论：核心思想、关键技术细节、算法流程
### 2.1 核心思想
- MPLM 将推理分解为多个**持久、半独立**的 LLM 线程，这些线程通过显式的点对点消息传递（send/recv）进行协调，而非依赖父线程集中聚合。
- 每个线程拥有独立上下文，可通过批处理推理引擎并发解码，并通过轻量控制器管理线程状态和消息路由。

### 2.2 关键技术细节
- **四个执行控制原语**：
  - `<spawn[ids]> prompt </spawn>`：创建新线程并赋予指定ID和初始提示。
  - `<send[ids]> message </send>`：向指定线程发送消息。
  - `<recv[ids]>`：阻塞等待来自指定线程的消息（支持 wait-for-all 或 wait-for-any 两种实现）。
  - `<stop>`：终止当前线程。
- **推理引擎**：基于现有的批处理推理引擎（如 vLLM），在其上叠加轻量控制器。控制器循环执行：解码所有可运行线程直到遇到指令边界、EOS 或最大步数；解析并处理指令（spawn/send/recv/stop）。
- **关键机制**：
  - **减少通信成本**：每个线程只处理局部子任务，只与邻居（有信息依赖的线程）通信，避免全局上下文复制。
  - **抢占（preemption）**：异步消息允许父线程在收到一个子线程的答案后立即终止其他仍在执行的子线程，从而减少无效计算。
- **可选的扩展功能**：线程可以“重生”（respawning）：自己创建一个包含压缩摘要的新替代线程，重置上下文长度以避免无限增长；支持不同的 recv 实现（待全部或待任意）。

### 2.3 算法流程示意图（文字说明）
- 初始：一个根线程接收待解问题。
- 根线程执行 `<spawn>` 创建多个工作线程（如每个单元格一个线程），传递初始状态。
- 每个工作线程处理局部子问题，根据情况选择：如果信息不足则执行 `<recv>` 等待邻居消息；若可确定局部结果，则通过 `<send>` 向相关线程（邻居和父线程）发送消息，然后 `<stop>`。
- 父线程接收全部消息后，合成最终答案并输出。

## 6. 主要结论与发现
1. **MPLM 在结构化推理中的渐近优势**：在数独上，MPLM 的顺序令牌数和最大上下文长度的缩放指数明显低于 Serial CoT 和 FJ（α≈1.1 vs α≈1.8），实现了更优的扩展性。
2. **超越前沿推理模型**：MPLM 能解决 25×25 数独（准确率 72%），而 GPT-5 Pro 仅解决 20%（4/20），DeepSeek-R1 在 9×9 上即失败。这表明结构化的并行推理能突破串行 CoT 的能力上限。
3. **抢占带来效率提升**：在 3-SAT 上，MPLM 通过异步 recv 允许在发现满足解后立即终止其他分支，比 FJ 延迟降低 1.6–3.4 倍，尤其在不平衡搜索树上优势更显著。
4. **零样本适用性**：大型预训练模型（Qwen3-30B/35B）在正确提示下可以直接遵循 MPLM 指令，在 LongBench-v2 上获得与 RLM 相当或更高的准确率，同时延迟降低约 1.7–2.2 倍。这表明 MPLM 框架无需额外训练即可被现有关模型利用。

## 7. 优点
- **创新性**：首次将 MPI 风格的消息传递引入 LLM 并行推理，提出 spawn/send/recv/stop 原生指令机制，实现了线程间点对点通信和抢占，克服了 Fork-Join 的集中化瓶颈。
- **理论支撑**：给出了渐近上下文复杂度分析，证明 MPLM 的最大上下文比 Serial CoT 和 FJ 低 O(N/k) 因子（N 为线程数，k 为邻居数），提供了理论基础。
- **实验设计的严谨性**：在结构化任务中，通过控制变量（相同模型、相同训练数据）公平对比；在长上下文任务中，展示零样本 prompting 的能力，表明框架的实用潜力。
- **效率与性能兼得**：同时降低了上下文需求、延迟和顺序令牌数，在多个任务上达到或超越现有方法。

## 8. 不足与局限
- **应用场景局限**：当前实验主要在结构化推理任务（数独、SAT）上展示了优势，这些任务具有清晰的局部通信模式。对于开放域任务（如代码生成、数学证明），如何自动学习通信模式仍需进一步研究。
- **通信模式依赖**：MPLM 要求线程正确决定“与谁通信”及“何时通信”，在 Sudoku/SAT 中可通过固定策略预定义，但在复杂开放任务中可能需要精心设计 prompt 或微调。
- **模型规模限制**：Sudoku 和 SAT 实验仅微调了 0.6B 参数模型，未验证在更大模型（如 7B 或更大）上的通信协调能力，以及其在现代推理模型中的收益。
- **训练数据生成局限**：实验使用模拟的固定策略（裸单、DPLL）生成监督数据，可能限制了模型发现更高效通信模式的潜力；更理想的方式是通过强化学习或探索学习通信策略。
- **长上下文 QA 实验的公平性问题**：MPLM 的 prompt 模板需人工设计，而 RLM 也有自己的 prompt，可能存在不对等优化；且只测试了两个特定模型，不够广泛。
- **没有系统研究扩展性**：未系统比较 MPLM 在更多并行线程下的时延/吞吐量扩展曲线；也未评估在实际部署中的 KV 缓存管理、显存占用等工程因素。
