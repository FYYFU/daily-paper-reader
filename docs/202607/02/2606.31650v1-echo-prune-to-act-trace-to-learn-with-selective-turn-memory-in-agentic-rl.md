## 摘要
长视界语言智能体必须反复与工具交互、积累证据，并在有限的上下文窗口内做出决策。现有的上下文管理方法通过截断远距离历史、将过去轮次折叠为摘要或选择紧凑的记忆状态，使此类展开成为可能。然而，这些突破引入了两个耦合的限制。首先，随着轮次数量的增长，历史观察逐渐被移除或压缩为紧凑状态，使得策略更难复用细粒度证据。其次，一旦原始轮次不再具有源可寻址性，基于结果的强化学习就失去了将策略更新与支持成功最终答案的证据对齐的明确路径。为此，我们提出ECHO，一种选择性轮次记忆框架，通过基于源索引的重建联合解决历史崩溃与可追踪学习问题。具体而言，ECHO将每个已完成的环境轮次压缩为紧凑的记忆记录，通过从这些记录中选取来重建有界的策略上下文，并重用所选源索引将正面结果信用路由至支持成功答案的证据和选择动作。在BrowseComp-Plus上，ECHO达到43.4%的保留准确率，优于GRPO（28.9%）和滚动摘要基线SUPO（36.1%），同时使用的轮次和轨迹量少于SUPO（图1）。此外，训练得到的策略在稠密和MoE骨干网络上，跨多目标问答、代码生成和深度信息寻求基准测试中提升了零样本泛化能力。

## Abstract
Long-horizon language agents must repeatedly interact with tools, accumulate evidence, and make decisions under bounded context windows. Existing context-management methods make such rollouts feasible by truncating distant history, folding past turns into summaries, or selecting compact memory states. However, these breakthroughs introduce two coupled limitations. First, as the number of turns grows, historical observations are progressively removed or collapsed into compressed states, making it harder for the policy to reuse fine-grained evidence. Second, once the original turns are no longer source-addressable, outcome-based RL loses an explicit path for aligning policy updates with the evidence that supported a successful final answer. To this end, we propose ECHO, a selective turn-memory framework that jointly addresses history collapse and traceable learning through source-indexed reconstruction. Specifically, ECHO compresses each completed environment turn into a compact memory record, reconstructs bounded policy contexts by selecting from these records, and reuses the selected source indices to route positive outcome credit to the evidence and selection actions that support successful answers. On BrowseComp-Plus, ECHO reaches 43.4% held-out accuracy, outperforming GRPO (28.9%) and the rolling-summary baseline SUPO (36.1%), while using fewer turns and lower trajectory volume than SUPO (Figure 1). Additionally, the trained policy improves zero-shot generalization across multi-objective QA, code generation, and deep information-seeking benchmarks on both dense and MoE backbones.

- **核心问题**：长视界（long-horizon）语言智能体在有限上下文窗口（bounded context window）下反复与工具交互、积累证据并做出决策时，面临两个耦合的限制：
  1. **历史证据压缩**：随着交互轮次增多，细粒度的历史观察被移除或折叠为摘要，导致策略难以复用早期取得的证据。
  2. **基于结果的强化学习缺乏可追溯性**：原始轮次不再具有“源可寻址性”（source-addressable），因此无法将最终结果的正面信用（credit）明确地分配给那些支持成功答案的证据轮次，导致策略更新与有效证据之间缺乏显式对齐。

- **整体含义**：现有方法（如 GRPO、SUPO 滚动摘要）虽然通过截断、摘要、检索等技术使长序列展开可行，但导致了轮次激增、轨迹膨胀和训练效率下降。论文提出的 ECHO 框架通过选择性轮次记忆与源索引重建，同时解决了“压缩历史以行动”和“追踪证据以学习”之间的矛盾，使得上下文管理既能保持高效又能保留可追溯的信用分配路径，从而显著提升智能体长时域任务性能与零样本泛化能力。

## 2. 论文提出的方法论：核心思想、关键技术细节与算法流程

- **核心思想**：**使用源索引（source-indexed）的选择性记忆重建策略上下文，并利用同一组源索引将最终结果的正向信用路由回支持证据轮次、记忆发觉（findings）以及记忆选择动作**。这使得智能体在有限上下文下行动时仍能保留细粒度证据的链接，并且为基于结果的强化学习提供了显式的信用分配路径。

- **关键技术细节**：
  1. **源索引轮次记忆（Source-indexed Turn Memory）**：每完成一个工具交互轮次后，ECHO 对该轮次单独生成一个紧凑的“记忆记录”（memory record），包含该轮的源索引 \(i\)、动作的紧凑表示 \(\alpha_i\) 以及从该轮观察中解析出的“局部发觉”（local finding）\(m_i\)。这些记录形成一个非折叠的集合 \(M_j\)，每个记录独立可寻址。
  2. **自回归记忆选择（Autoregressive Memory Selection）**：当上下文预算即将超限时，ECHO 让策略基于当前状态和渲染后的记忆列表，输出一个选择命令，指定需要保留的源索引。通过修复操作 \(\rho\) 剔除无效、去重、限制选取数量（最多 \(S\) 个模型选择轮次 + 最近 \(K\) 个自动保留轮次）并满足预算约束，得到选中的索引集合 \(\hat{I}_j^{\text{sel}}\)。
  3. **有界上下文重建（Bounded Context Reconstruction）**：使用选中的记忆记录加上本地近期历史（\(\Phi_{\text{local}}\)）重建下一个策略上下文 \(c_{j,t}\)，确保其长度不超过预算 \(B\)。
  4. **溯源引导的信用分配（Provenance-guided Credit Assignment）**：对于每个轨迹 \(n\)，识别出以下令牌（token）作为信用令牌：
     - 最终答案段（final response segment）中的令牌；
     - 被选中进入最终重建上下文的源轮次中的动作令牌（\(\hat{I}_{\text{src}}^{(n)}\)）；
     - 这些源轮次的局部发觉令牌（finding tokens）；
     - 所有记忆选择动作令牌（selection tokens）。
     然后，只对具有正优势（\(A^{(n)+} = \max(A^{(n)}, 0)\)）的轨迹，将优势乘以硬信用掩码 \(\mu_q^{(n)}\)，得到 \(\tilde{A}_q^{(n)} = A^{(n)+} \cdot \mu_q^{(n)}\)，用于更新策略梯度。这种设计避免了将奖励平均分配给所有生成令牌（包括冗余搜索），而只强化被实际重用的证据和有助于构建上下文的动作。

- **算法流程（文字说明）**：
  - 初始 prompt \(x\) 进入工作上下文 \(C\)。
  - 每轮：策略采样动作 \(a_t\)（包括推理、工具调用、总结命令等），追加到上下文并记录可训练令牌。
  - 每完成一个工具轮次后，模型生成一个紧凑的“局部发觉”（<sum last turn> 块），并连同源索引存入记忆集。
  - 一旦上下文长度超过预算 \(B\)：
    1. 构造选择上下文 \(c_j^{\text{sel}}\)（当前上下文 + 记忆列表 + 选择指令）；
    2. 策略输出选择动作 \(a_j^{\text{sel}}\)，包含选中的源索引；
    3. 解析、修复得到最终选中索引 \(\hat{I}_j^{\text{sel}}\)；
    4. 记录选中的源索引、发觉令牌和选择令牌作为信用掩码的一部分；
    5. 将当前工作上下文 \(C\) 替换为 prompt + 选中记忆的渲染 + 继续提示 \(x_{\text{cont}}\)，然后继续交互。
  - 轨迹结束时，最终包含所有保存的生成段（包括多次压缩中的段），每个段附有对应的信用令牌掩码。
  - 训练时，对每个 prompt 采样 N 个轨迹，计算组相对优势 \(A^{(n)}\)，利用信用掩码计算 ECHO 目标函数 \(\mathcal{L}_{\text{ECHO}}^{(n)}(\theta) = \sum_q \tilde{A}_q^{(n)} \log \pi_\theta(y_q^{(n)}|...)\)，并求平均。

## 6. 论文的主要结论与发现

- ECHO 在 BrowseComp-Plus 上达到 43.4% 保留准确率，显著优于 GRPO（28.9%）和 SUPO（36.1%），同时使用的平均轮次（45.3）和轨迹体积（3.13）均低于 SUPO（62.5 轮、4.18 轨迹），表明其更高效地利用长时域搜索，而非鼓励冗余展开。

## 7. 优点：方法或实验设计上的亮点

- **创新的方法论整合**：将上下文重建与信用分配通过同一个源索引链结合，使得“行动时选择的证据”和“学习时强化证据”对齐，设计简洁且优雅。
- **解决了两大瓶颈**：同时缓解了历史压缩导致的信息丢失和信用分配噪声问题，而不是只优化其中一个。
- **丰富的消融与分析**：既做了记忆组件的消融（选择策略 vs. 检索、紧凑发觉 vs. 完整观察），又做了信用分配机制的消融，并展示了训练过程中的轮次和轨迹体积动态，充分阐明了每个组件的作用。
- **跨领域零样本测试**：在多目标 QA、代码生成、深度信息寻求三个完全不同场景中验证泛化，增强了结论的可信度。

- **信用分配的最终追踪近似**（Final-trace approximation）：ECHO 只将信用分配给最终重建上下文中选中的源轮次，而没有递归地追溯这些源轮次本身所依赖的更早轮次。对于具有深层嵌套依赖的极长序列，这可能导致某些关键历史轮次被低估信用。论文也承认这一点。
