## 摘要
多智能体大语言模型系统越来越多地集成检索、规划和推理，但本质上仍以文本为中心，要求智能体通过昂贵的预填充重复计算共享上下文。尽管已知单次推理可通过KV缓存管理加速，但这通常局限于本地服务范围。我们提出AAFLOW+，一种对智能体工作流算子的状态化扩展，将KV缓存提升为一等分布式系统对象。AAFLOW+将进程构建为通信感知图，同时优化数据、提示和可重用模型状态。它还提供KV物化、传输、分支、组合和驱逐等算子。其运行时支持零拷贝、传输感知执行，使智能体无需重新计算即可复用长上下文。基于经验硬件微基准参数化的分析成本模型，AAFLOW+将首令牌时延降低最多50.2倍，在16智能体规模下多智能体计算成本降低最多7.63倍，KV内存减少1.72-6.10倍，吞吐量提升超过7.74倍。结果表明，在中高带宽网络上KV传输优于重新计算，确保KV状态共享通过替代文本传递极大地提高了多智能体大语言模型系统的效率。

## Abstract
Multi-agent LLM systems increasingly integrate retrieval, planning, and reasoning, but remain fundamentally text-centric, requiring agents to repeatedly recompute shared context through expensive prefill. Although single-request inference is known to be accelerated by KV-cache management, it is usually restricted to local serving scopes. We introduce AAFLOW+, a stateful extension of agentic workflow operators that makes KV cache a first-class distributed systems object. AAFLOW+ builds processes into communication-aware graphs that concurrently optimize data, prompts, and reusable model state. It also provides operators for KV materialization, transfer, fork, composition, and eviction. Its runtime enables zero-copy, transfer-aware execution, allowing agents to reuse long context without recomputation. AAFLOW+ reduces TTFT by up to 50.2x, achieves up to 7.63x reduced multi-agent compute cost at 16-agent scale, reduces KV memory by 1.72-6.10x, and increases throughput by more than 7.74x, based on an analytical cost model parameterized by empirical hardware microbenchmarks. The results demonstrate that KV transmission outperforms recomputation on networks with moderate to high bandwidth, making sure KV-state sharing greatly increases efficiency in multi-agent LLM systems by replacing text passing.

- **核心问题**：当前多智能体大语言模型（LLM）系统在智能体间主要通过文本传递共享上下文（如检索结果、推理中间结果），导致下游智能体必须对相同的上下文重复进行昂贵的预填充（prefill）计算，即反复重建 KV 缓存（Key-Value Cache），造成严重的计算冗余、高延迟（TTFT）和内存浪费。

- **论文整体含义**：首次将 KV 缓存提升为一等分布式系统对象，提出状态化工作流抽象 AAFLOW+，通过显式的 KV 状态物化、分支、传输、组合和驱逐操作，实现智能体间零拷贝的状态传递，从而消除冗余预填充，大幅提升效率。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：将传统只关注数据流的 AAFLOW 工作流扩展为状态流（stateflow），使 KV 缓存成为可调度的、可传输的分布式对象。工作流被编译为包含数据边和状态边的执行图 \(G_s = (V, E_d, E_s)\)，其中状态边显式编码 KV 缓存的重用和移动。
- **关键技术细节**：
  1. **状态化算子定义**：每个状态算子 \(Op^s_i = (I_i, O_i, S^{in}_i, S^{out}_i, f_i, P_i, \sigma_i)\)，其中 \(S^{in}_i, S^{out}_i\) 为输入/输出状态对象，\(\sigma_i\) 为状态策略（传输、别名、分支、合并、驱逐或重计算）。
  2. **KV 状态对象**：\(S_{KV} = (M, \Theta, B, \Pi, \Lambda, \Gamma)\)，包含模型标识、配置、KV 块集合（每个块包括键/值张量、层索引、位置范围、设备位置）、位置元数据、血统元数据、放置和所有权信息。
  3. **状态算子家族**：
     - 物化（materialize）：将文本上下文转化为可重用 KV 状态。
     - 传输（transfer）：在节点间移动或别名 KV 块，使用 Apache Arrow 元数据 + UCX/MPI 零拷贝传输。
     - 分支（fork）：从一个共享前缀状态创建多个逻辑子状态，采用写时复制（copy-on-write）共享物理块。
     - 受限组合（restricted merge）：仅在兼容条件下进行非重叠顺序拼接或文本级归约，避免随意合并破坏位置语义。
     - 驱逐（eviction）：基于优先级评分（重用频率、大小、年龄）释放内存。
  4. **成本模型**：状态化执行优于文本执行的条件 \(T_{transfer}(S_{KV}) + T_{resume} + \Omega_{state} < T_{prefill}(L) + \Omega_{text}\)。对于 \(k\) 个分支，文本成本 \(kT_{prefill}\)，状态化成本为 \(T_{prefill} + T_{fork} + \sum T_{decode}\)。
  5. **运行时**：实现传输感知调度，根据带宽和 KV 大小动态选择传输或重计算；与 vLLM/SGLang 后端集成，通过 KV 感知 API 注入预计算状态跳过预填充。
- **算法流程**（文字说明）：
  - 编译阶段：解析工作流，识别共享上下文，插入显式状态算子，生成状态化 DAG。
  - 执行阶段：先对共享上下文执行一次预填充并物化 KV 状态；然后根据调度决策（传输或重计算）将状态分配到各智能体；智能体从接收到的状态继续解码，只产生分支特定的延续；最后通过受限合并输出结果。
