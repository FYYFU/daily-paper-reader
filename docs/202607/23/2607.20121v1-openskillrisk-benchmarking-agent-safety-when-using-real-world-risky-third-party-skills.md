## 摘要
基于LLM的智能体利用第三方技能来扩展其在开放世界场景中的能力。然而，第三方技能可能引入额外的安全漏洞，因为看似无害的技能可能包含仅在实际执行时才会显现的潜在安全风险。在这项工作中，我们系统性地调查了当前智能体系统在多大程度上能够识别并避免此类风险。为了支持定量和定性评估，我们构建了OpenSkillRisk，这是一个专用的安全基准，包含从公共技能市场收集的263个有风险的技能。我们根据威胁类型将这些技能分为七类，并为每个技能配备一个标准化的用户任务和一个对应的沙盒环境，以进行受控评估。与先前的基准不同，OpenSkillRisk不仅涵盖了更真实、更多样化的不安全场景，还提供了细粒度的分析来诊断智能体在这些场景中的行为模式。我们进行了全面的实验，涵盖了三个主流CLI智能体框架和十三个最先进的LLM。实验结果表明，没有任何一个测试系统能够可靠地处理有风险的技能：即使是最安全的配置，仍有约17%的情况下会执行不安全操作。上下文相关的和系统级的风险尤其难以被当前智能体系统避免。我们的行为分析揭示了三种反复出现的失败模式：智能体可能未能识别风险，或者识别了风险但在行动前未能干预，或者按照超出用户预期范围的技能指令执行。这些发现凸显了需要改进LLM中的风险推理和智能体框架中的执行控制。

## Abstract
LLM-based agents leverage third-party skills to extend their capabilities in open-world scenarios. However, third-party skills can introduce extra security vulnerabilities, as seemingly harmless skills can contain latent safety risks that only emerge during actual execution. In this work, we conduct a systematic investigation into how well current agent systems recognize and avoid such risks. To support quantitative and qualitative evaluation, we construct OpenSkillRisk, a dedicated safety benchmark containing 263 risky skills collected from public skill marketplaces. We classify these skills into seven categories based on their threat types and pair each skill with a standardized user task and a corresponding sandbox for controlled evaluation. Distinct from prior benchmarks, OpenSkillRisk not only covers more realistic and diverse unsafe scenarios, but also provides a fine-grained analysis to diagnose the behavioral patterns of agents in such scenarios. We conduct comprehensive experiments covering three mainstream CLI agent frameworks and thirteen state-of-the-art LLMs. Experimental results show that no tested system handles risky skills reliably: even the safest configurations still execute unsafe actions in about 17% of cases. Context-dependent and system-level risks are especially difficult for current agent systems to avoid. Our behavioral analysis reveals three recurring failure patterns: agents may fail to recognize the risk, recognize it but fail to intervene before acting, or follow skill instructions beyond the user's intended scope. These findings highlight the need to improve both risk reasoning in LLMs and execution control in agent frameworks.

### 论文提出的方法论

- **核心思想**：构建一个专用的端到端安全基准OpenSkillRisk，包含真实世界中的有风险第三方技能，设计标准化用户任务和沙箱环境，评估智能体在执行层面（是否触发危险操作）和认知层面（是否发出风险警告）的综合安全能力。
- **关键技术细节**：
  1. **技能收集与筛选**：从两个公开技能市场（SkillsMP和Skills.rest）爬取175,891个独特英语技能，通过规则扫描（静态分析）和LLM多阶段筛选，最终保留263个有风险技能。
  2. **风险分类**：将技能分为7类攻击类型：控制平面劫持、权限扩展、数据收集、执行引导、持久性植入、外泄、外部状态篡改。同时区分两种风险性质：明显恶意（无条件拒绝）和上下文相关（依赖具体授权）。
  3. **任务与沙箱构建**：为每个风险技能配对一个真实、良性的用户任务，包括工作区文件、模拟用户环境（含占位符凭据）、本地桩服务（mock）代替外部调用。确保任务既能暴露风险决策点，又可通过安全替代路径正常完成。
  4. **评估指标**：执行级ASR（攻击成功率）、认知级Awareness（风险意识率）、以及综合指标F_safe（调和均值）。还细分为5种行为模式：无意识执行、意识到仍执行、警告并中止、警告并安全完成、无声安全完成。
  5. **智能体行为分析**：通过追踪轨迹，分类智能体实际反应模式，诊断具体失败类型。

- **公式/算法流程**：
  - ASR = 触发危险执行路径的实例数 / 总实例数（越低越好）
  - Awareness = 主动检测并报告风险的实例数 / 总实例数（越高越好）
  - F_safe = 2 * (1-ASR) * Awareness / ((1-ASR) + Awareness) (平衡安全执行与风险意识)

### 论文的主要结论与发现

1. **当前智能体系统处理风险技能不可靠**：最安全配置（Claude Sonnet 4.6 + Claude Code）仍有17.87%案例执行不安全操作；F_safe最高仅80.97%，最低27.86%。
2. **安全表现依赖模型与框架的组合**：更强模型通常更安全，但框架层面（如Claude Code的警告-中止机制）也有重要贡献。
3. **认知与行动解耦**：有些系统能识别风险但仍执行（如Gemini 3.1 Pro Awareness高但ASR高）；有些系统避免执行却不发出警告（Claude Haiku 4.5）。
4. **上下文相关风险更难处理**：相比明显恶意，上下文风险导致ASR从22.0%升至35.6%，Awareness从62.9%降至51.4%。
5. **系统级攻击比数据级攻击更危险**：控制平面劫持和权限扩展F_safe仅27%-38%，远低于数据外泄的75%。
6. **三种反复失败模式**：无法识别风险、识别但未阻止、超出用户意图执行。
7. **防御技能可提升安全性但伴随过度防御**：主动加载防御技能显著降低ASR（例如Claude Haiku 4.5从20.53%降至7.22%），但良性任务过度防御率达22.50%。
