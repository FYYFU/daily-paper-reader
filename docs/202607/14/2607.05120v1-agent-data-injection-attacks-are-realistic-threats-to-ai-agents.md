## 摘要
AI智能体代表用户提示执行操作，消耗外部数据并根据智能体上下文采取行动。此前关于AI智能体安全的研究主要集中在间接提示注入上。其中研究最充分的是指令注入，即攻击者控制的不受信任数据被解释为指令。为此，研究人员提出了多种缓解措施来防止指令注入攻击。本文介绍了一种新的间接提示注入类别——智能体数据注入攻击。智能体数据注入将恶意数据伪装成受信任数据，例如安全关键元数据（如资源标识符或数据来源）或智能体上下文数据（如工具调用和响应格式）。结果，智能体在不知情的情况下基于攻击者控制的数据执行非预期操作。智能体数据注入与指令注入攻击具有相似的攻击影响，因为它会导致智能体行为异常并执行非预期操作。尽管影响相似，但智能体数据注入仍未得到充分探索，且易绕过现有间接提示注入防御。我们在真实世界的智能体中发现了几项关键漏洞，允许攻击者发起各种攻击：对Web智能体（Chrome中的Claude、Antigravity和Nanobrowser）的任意点击攻击，以及对编码智能体（Claude Code、Codex和Gemini CLI）的远程代码执行和供应链攻击。我们评估了现成模型和AI智能体中的智能体数据注入漏洞，发现智能体数据注入在独立LLM和AI智能体设置中均有效。智能体数据注入暴露了智能体安全中的一个关键缺口，表明当前的AI智能体并未采用基本安全原则：当前智能体未将受信任数据与不受信任数据隔离。

## Abstract
AI agents act on behalf of user prompts, consuming external data and taking actions based on the agent context. Prior research on AI agent security has primarily focused on indirect prompt injection (IPI). Its most well-studied category is instruction injection, where attacker-controlled untrusted data is interpreted as an instruction. In response, many mitigations have been proposed to prevent instruction injection attacks. In this paper, we introduce a new category of IPI, agent data injection attacks (ADI). ADI injects malicious data disguised as trusted data, such as security-critical metadata (e.g., resource identifiers or data origins) or agent context data (e.g., tool call and response formats). As a result, agents unknowingly execute unintended actions based on attacker-controlled data. ADI has similar attack impacts as instruction injection attacks, because it causes agents to misbehave and execute unintended actions. Despite the similar impact, ADI remains underexplored and easily bypasses existing IPI defenses. We found several critical vulnerabilities in real-world agents that allow an attacker to launch various attacks: arbitrary click attacks on web agents (Claude in Chrome, Antigravity, and Nanobrowser), and remote code execution and supply-chain attacks on coding agents (Claude Code, Codex, and Gemini CLI). We evaluate ADI vulnerabilities across off-the-shelf models and AI agents, and find that ADI is effective in both standalone LLMs and AI agent settings. ADI exposes a critical gap in agent security, signifying that current AI agents do not employ a fundamental security principle: current agents do not isolate trusted data from untrusted data.

# 论文总结：Agent Data Injection Attacks are Realistic Threats to AI Agents

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：AI 智能体（AI Agent）通过工具调用外部数据并据此执行用户指令，但现有安全研究主要聚焦于**间接提示注入（Indirect Prompt Injection, IPI）**中的**指令注入（Instruction Injection）**——攻击者控制的不受信任数据被 LLM 误解为指令。虽然已有多种防御（如模型强化、输入/输出护栏、双LLM等）能有效抑制指令注入，但安全社区忽略了**智能体内上下文数据内部的信任隔离问题**。
- **核心问题**：攻击者可以将恶意数据伪装成**受信任数据**（如资源标识符、数据来源、工具调用格式等），使 LLM 误解为合法的元数据或上下文，从而操纵智能体执行非预期操作。这种攻击被称为**智能体数据注入攻击（Agent Data Injection, ADI）**。
- **整体含义**：ADI 揭示了现有防御的根本缺陷——它们只关注指令与数据的分离，而未考虑智能体数据内部**受信任数据与不受信任数据**的隔离。这导致当前 AI 智能体缺乏基本的安全原则，面临严重威胁。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：ADI 攻击利用 LLM 对数据格式的**概率性解释**，通过注入**概率定界符（probabilistic delimiter）**，使 LLM 将攻击者控制的文本误解为受信任的结构元素（如 JSON 对象、DOM 元素、工具调用块等），从而篡改智能体对环境的认知。
- **关键技术：概率定界符注入**
  - 传统定界符注入（如 SQL 注入、XSS）依赖确定性解析器，必须注入精确匹配的定界符。而 LLM 以概率方式解释文本，即使注入不完全匹配的字符序列（如转义后的双引号 `\"`、相似符号 `”`、甚至无关符号 `$`），LLM 也可能将其视为有效定界符。
  - 示例：在 JSON 格式的邮件工具响应中，攻击者在邮件正文中插入 `\"` 和 JSON 结构，使 LLM 将其解释为第二个邮件对象，并伪造发送者字段。
- **形式化定义**：
  - 令智能体上下文 = 指令 `I` + 数据 `D = (D_T, D_U)`，其中 `D_T` 为受信任数据（工具生成），`D_U` 为不受信任数据（用户可控制的）。
  - 指令注入（II）：LLM(I, (D_T, D_U ∥ D_A)) ≈_II LLM(I ∥ D_A, (D_T, D_U))，即攻击者数据被误解为指令。
  - ADI：LLM(I, (D_T, D_U ∥ D_A)) ≈_ADI LLM(I, (D_T ∥ D_A, D_U))，即攻击者数据被误解为受信任数据。
- **攻击流程**：攻击者在外部资源（如网页评论、GitHub issue 评论、PR 描述）中嵌入概率定界符，伪装成受信任的元数据或工具响应。智能体读取后，LLM 错误解析，进而基于伪造的可信数据执行动作（如点击错误按钮、执行恶意命令、合并恶意 PR）。

## 3. 实验设计：数据集/场景、benchmark、对比方法

- **实验一：概率定界符注入的 LLM 独立评估**
  - **数据**：7 个类别（日历事件、云盘文件、GitHub 评论、邮件、GitHub issue、论文评审、Web DOM），共 157 个测试用例。前 6 类为 JSON 格式，Web DOM 使用特定格式。
  - **任务**：要求 LLM 从结构化响应中提取特定字段或进行聚合。
  - **攻击**：在不受信任字段中注入概率定界符（默认 JSON 用 `\"`，Web DOM 用真实方括号 `[ ]`）。评估一致攻击（构造完整伪造对象）和不一致攻击（仅插入不完整字段）。
  - **防御**：基线（无防御）、随机化（字段名附加随机 nonce）、清理（去除定界符类字符）。
- **实验二：智能体环境下的 ADI 攻击与防御评估**
  - **Benchmark**：基于 AgentDojo（模拟 Slack、云盘、邮件、银行、旅行规划环境），将工具响应格式从 YAML 改为 JSON。使用 96 个用户任务和 108 个 ADI 攻击；对比 935 个指令注入攻击。
  - **对比防御**：输入护栏（Llama Prompt Guard 2）、输出护栏（LlamaFirewall AlignmentCheck）、计划-执行（IsolateGPT）、智能体沙箱（Progent）、双LLM + 数据流追踪（CaMeL 的三变种：无策略、普通、严格）、随机化。
  - **模型**：GPT-5.2（ReAct 框架）。
- **实验三：真实世界智能体的 PoC 攻击**
  - 对 Claude in Chrome、Antigravity、Nanobrowser 进行元素 ID 注入；对 Claude Code、Codex、Gemini CLI 进行来源注入（远程代码执行）和工具调用注入（供应链攻击）。

## 6. 主要结论与发现

1. **LLM 对概率定界符注入高度脆弱**：在 JSON 上 ASR 达 31.3%–43.3%，Web DOM 上达 33.3%–100.0%。即使注入与真实定界符不匹配的字符，攻击仍有效。
2. **ADI 绕过现有 IPI 防御**：指令注入的 ASR 在几乎所有防御下接近 0%，而 ADI 的 ASR 为 22.2%–50.0%，仅 CaMeL 严格模式（完整数据流追踪）降至 0%，但实用性从 86.5% 降到 36.5%。
3. **随机化可缓解但有限制**：对 JSON 格式有效（ASR 降至 0–3%），但对列表添加类攻击无效（ASR 28.7%）；在 Web DOM 上当攻击者猜对 nonce 时失效。
4. **数据流追踪严格模式可完全防御但牺牲实用性**。
5. **真实世界智能体存在严重漏洞**：可导致任意点击、远程代码执行、供应链攻击。

## 7. 优点

- **新颖性与洞察**：首次系统提出 ADI 攻击类别，揭示“受信任与不受信任数据隔离”这一被忽视的攻击面，比指令注入更难防御。
- **技术贡献**：提出概率定界符注入技术，区别于传统确定性注入，并形式化区分指令注入与 ADI。
