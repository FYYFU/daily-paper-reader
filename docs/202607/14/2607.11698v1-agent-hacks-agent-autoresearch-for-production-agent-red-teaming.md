## 摘要
生产级大语言模型智能体（如Claude Code和Codex）在不可信内容、文件、命令和工作区状态下运行，使得安全失效直接可被利用。因此，红队测试必须跟上不断演进的模型和工具。现有方法主要优化攻击成功率并保留工件（如基准、载荷或攻击程序），这些工件记录了攻击成功的位置，但未揭示不安全智能体行为背后的促成条件。我们研究使用一个自主研究环境对生产级大语言模型智能体进行自动化红队测试，以发现关于另一个智能体的可复用漏洞知识。我们提出AHA，一个可证伪的发现循环：提出漏洞假设，构造证伪器，实例化有效攻击，在沙箱环境中执行，反思轨迹，并将已确认的发现提升到漏洞概念图（VCG）中。每个概念通过一个声明、促成条件、证伪器、迁移预测和支持证据，将面向攻击者的表面与不安全轨迹联系起来。在涵盖直接和间接攻击的三个场景中，针对Claude Code和Codex，发现的概念揭示了跨模型和智能体的可复用漏洞核心。一个冻结的VCG无需进一步搜索，在相同单次协议下比最强的冻结发现基线高出14.2个百分点，同时可跨场景和攻击通道迁移。生成的VCG为生产安全团队提供了可审计的工件，用于检查漏洞、验证补丁并积累可复用安全知识。我们的代码可在https://github.com/henrymao2004/Auto-research-red-teaming-in-sleep获取。

## Abstract
Production LLM agents such as Claude Code and Codex operate over untrusted content, files, commands, and workspace state, making safety failures directly actionable. Red-teaming must therefore keep pace with evolving models and tools. Existing approaches mainly optimize attack success and preserve artifacts such as benchmarks, payloads, or attack programs, which record where attacks succeed but not the enabling conditions behind unsafe agent behavior. We study automated red-teaming for production LLM agents using one agentic research environment to discover reusable vulnerability knowledge about another. We present AHA, a falsifiable discovery loop that proposes a vulnerability hypothesis, constructs a falsifier, instantiates a valid attack, executes it in a sandboxed harness, reflects on the trajectory, and promotes confirmed findings into a Vulnerability Concept Graph (VCG). Each concept links an attacker-facing surface to an unsafe trajectory through a claim, enabling condition, falsifier, transfer prediction, and supporting evidence. Across Claude Code and Codex on three scenarios covering direct and indirect attacks, the discovered concepts reveal a reusable vulnerability core across models and agents. A frozen VCG requires no further search and outperforms the strongest frozen discovery baseline by 14.2 percentage points under the same single-shot protocol, while transferring across scenarios and attack channels. The resulting VCG provides an auditable artifact for production safety teams to inspect vulnerabilities, validate patches, and accumulate reusable safety knowledge. Our code is available at https://github.com/henrymao2004/Auto-research-red-teaming-in-sleep.

### 1. 论文的核心问题与整体含义 (研究动机和背景)

- **核心问题**: 生产级LLM智能体（如Claude Code、Codex）在执行任务时会操作文件、调用API、执行命令。当它们处理不可信内容时，一次安全失败便可能造成真实损失（如文件被写、数据泄露）。
- **现有方法的局限**:
    - **“知其然，不知其所以然”**: 现有自动化红队方法主要目标是最大化攻击成功率，并保留攻击工件（如攻击载荷、攻击程序）。这些工件只记录了“攻击在哪里成功了”，但没有揭示“是什么条件导致了安全轨迹的产生”，即缺乏对**脆弱性的使能条件**的理解。
    - **知识不可复用**: 当模型更新、工具改变或场景变化后，这些一次性攻击迅速失效，红队工作需要从头开始，无法将一次发现的安全知识持续积累和复用。
- **研究意义 (整体含义)**: 本文提出了一种全新的范式，将红队测试从“寻找一次性漏洞”转变为“**累积可复用的安全知识**”。这为解决快速迭代的生产级AI系统安全评估的可持续性问题，提供了一个关键的基础框架。

### 2. 论文提出的方法论 (核心思想与关键技术)

- **核心思想**: 将生产环境智能体的红队测试视为一个**自动化研究 (Autoresearch)** 流程。该流程由一个智能体（研究代理）在受控环境中自动研究另一个智能体（受害代理），其最终产出不是单个攻击，而是**可迁移、可审计的脆弱性知识**。
- **关键技术：AHA (Agent Hacks Agent) 框架**:
    1.  **脆弱性概念发现循环**: 这是一个可证伪的自动循环，包含以下步骤：
        - **假设提出**: 研究代理在**设计攻击之前**，先对目标智能体可能存在的脆弱性提出一个可证伪的**假设**（例如：“受害代理可能会误将不安全的工具使用视为良性任务”）。
        - **证伪器承诺**: 假设中必须包含一个明确的**证伪器**，即“什么样的情况出现会反驳我的假设？”。这确保了发现的严谨性。
        - **攻击设计**: 攻击者代理基于该假设和场景，实例化一个具体的攻击载荷。
        - **沙盒执行**: 在隔离的沙盒中执行攻击，记录下完整的轨迹（消息、工具调用、观察结果）。
        - **反思与分类**: 反思者代理对比轨迹和证伪器，判断假设是**被证实、被证伪还是无法确定**。
        - **概念更新与提升**: 只有被多次（≥3次）证实的脆弱性假设，才会被提升到“冻结”的状态，存入 **脆弱性概念图（Vulnerability Concept Graph, VCG）**。
    2.  **脆弱性概念图**:
        - 这是AHA的核心产出，是一个结构化的知识库。每个“脆弱性概念”是一个可审计的单元，包含以下字段：脆弱性声明 (Claim)、使能条件 (Enabling Condition)、攻击模板 (Attack Template)、失败结果 (Failure Outcome)、迁移预测 (Transfer Prediction) 和证据 (Evidence)。
        - VCG不仅是攻击的集合，更是**对失效机制的抽象描述**，解释了“为什么”会失效，以及“在什么条件下”会失效。
    3.  **评估范式**: **冻结-单次-部署 (Freeze, Single-shot, Deploy)**。
        - 发现阶段结束后，VCG被**冻结**，不再进行任何搜索或优化。
        - 在评估阶段，对于每个新的（保留的）测试实例，V **一次性选取并实例化**一个最相关的概念，执行攻击。不允许重试或测试时再次搜索。
        - 这个严格的评估协议，直接衡量了知识本身（VCG）的**可复用性**和稳健性，而不是搜索优化的能力。

- **使用的数据集/场景**: 基于三个红队场景：

- **对比方法 (Baselines)**:

    - **AutoRISE***: 自动化程序搜索，产出攻击程序。

- **评估指标**: 主要指标是**攻击成功率 (Attack Success Rate, ASR)**，即在冻结的、单次评估协议下的成功率。

### 6. 论文的主要结论与发现

- **核心结论**: AHA将生产级智能体的红队测试从**一次性攻击发现**转变为**可复用的脆弱性知识发现**。其产出的VCG是可审计、可迁移、可积累的知识。
- **具体发现**:
    - **跨模型共享机制**: 发现的概念（如“冒认授权”）是**共享的机制**，在多个模型和智能体之间反复出现，而非特定目标的漏洞。

### 7. 优点 (亮点)

- **思想新颖且具有根本性**: 将红队工作的目标从“追求一次胜利”提升到“发现可复用的知识”，为解决AI安全评估的持续性和可扩展性问题提供了新思路。
- **严谨的科学研究方法**: 引入了“**可证伪假设**”和“**证伪器**”，使自动化红队过程从纯粹的工程搜索转变为一种科学发现的过程，有效避免了对结果的马后炮式解释。
- **极强的实践指导意义**: 产出的VCG具有高度的可审计性。安全团队可以直接阅读“使能条件”，修改智能体策略或工具边界来进行漏洞修复，然后在模型更新后**直接使用冻结的VCG进行回归测试**，无需重新启动一轮新的搜索。这大大降低了安全维护成本。
- **构建了可积累的安全知识体系**: VCG是累积性的，每次发现都补充到同一张知识图谱中，形成一份随着模型和产品演进而不断完善的“安全地图”。这为整个生态提供了治理和协同的基础。

### 8. 不足与局限

- **计算成本**: AHA的自动化研究循环，尤其是其中包含的多个智能体假设-执行-反思过程，计算开销可能**远大于**基于单一优化目标的基线方法。虽然产出结果更好，但这个开销是否是“值得的”需要进行更深入的成本-效益分析。
- **实验覆盖范围有限**: 实验场景集中在**编码和系统操作智能体**（Claude Code / Codex），以及特定的红队基准。对于更广泛类型的智能体（如聊天机器人、浏览器代理、、APIAgent、机器人等）的适用性尚未验证。
- **对基线方法的“冻结”可能不公**: 虽然这是为了公平对比知识本身，但AHA设计的评估协议本身就天然地倾向于其自身的VCG产出。这些基线方法（如T-MAP、IterInject）的本意是产出具体攻击，而非抽象知识，其被“冻结”后表现不佳部分程度上是不占优势的。
- **偏差风险**: 自动研究循环也面临自身寻优的风险。虽然文中采用了周期性审查来减轻这种风险，但无法完全杜绝。此外，VCG的构建依赖于研究模型（Claude-4.8-Opus/GPT-5.5）的认知和判断，该模型本身可能存在偏见，影响所发现概念的质量。
