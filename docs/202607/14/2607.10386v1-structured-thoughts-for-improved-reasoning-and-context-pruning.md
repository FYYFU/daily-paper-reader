## 摘要
大型语言模型（LLM）擅长生成长链思维，但冗长的推理轨迹往往冗长且内存效率低。在这项工作中，我们引入了结构化思维（Structured Thoughts），一个将推理组织成交替的<try>和<outcome>块的框架：<try>捕捉探索性的草稿工作，而<outcome>包含该步骤的提炼结论。我们通过将推理轨迹分割成<try>块，并提示LLM将每个步骤总结为对应的<outcome>，构建了一个结构化思维数据集。在这个重新格式化的数据上微调预训练基础模型，使模型采用结构化推理风格，与标准SFT相比，在推理基准上实现了高达8.08%的性能提升。显式结构还实现了上下文剪枝：在每个<try>/<outcome>对之后，可以剪枝<try>，使模型能够保留结论，而不必将完整的草稿工作保留在上下文中。一个概念验证的剪枝实现实现了平均85%的内存/上下文节省，而在数学任务上性能下降8.67%。

## Abstract
Large language models (LLMs) excel at generating long chains of thought, but long reasoning traces are often verbose and memory-inefficient. In this work, we introduce Structured Thoughts, a framework that organizes reasoning into alternating <try> and <outcome> blocks: <try> captures exploratory scratch work, while <outcome> contains the distilled conclusion of that step. We construct a dataset of structured thoughts by segmenting reasoning traces into <try> blocks and prompting an LLM to summarize each step into its corresponding <outcome>. Fine-tuning pretrained foundation models on this reformatted data produces models that adopt the structured reasoning style, leading to performance gains of up to 8.08\% on reasoning benchmarks compared to standard SFT. The explicit structure also enables context pruning: after each <try>/<outcome> pair, the <try> can be pruned, allowing the model to retain conclusions without keeping the full scratch work in the context. A proof-of-concept pruning implementation achieves an average of 85\% memory / context savings with an 8.67\% performance drop across mathematical tasks.

## 1. 核心问题与整体含义（研究动机和背景）

大型语言模型（LLM）在需要多步逻辑或数学推理的任务上常表现不佳，而推理模型通过生成中间步骤（如思维链 CoT）来扩展能力。然而，标准 CoT 轨迹往往冗长、内存效率低，且容易造成“迷失在中间”的问题（模型对长上下文中间部分信息利用不足）。此外，一旦生成完整轨迹，推理过程中大量草稿工作（scratch work）被保留在上下文中，导致 KV 缓存和注意力开销巨大。当前推理研究多聚焦于通过强化学习（RL）优化中间步骤奖励，但少有工作从推理轨迹的结构化组织本身出发，同时提升准确性和效率。

**本文的核心动机**：通过显式地将推理轨迹组织为交替的 <try>（探索性草稿）和 <outcome>（蒸馏结论）块，既改善推理质量，又支持后续的上下文剪枝（pruning），从而在保持性能的同时大幅降低内存和计算开销。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
将每个推理步骤分为两个阶段：
- **<try> 块**：包含探索性、细节性的中间计算或推理过程（如代数推导、验证、子问题求解）。
- **<outcome> 块**：提炼该步骤的核心结论，用简洁的一句话或一个公式概括。

通过这种结构化格式，模型可以更清晰地关注关键信息，并支持在后续步骤中丢弃 <try> 块，仅保留 <outcome> 作为上下文。

### 关键技术细节
**2.1 数据集构建（结构化 SFT 数据）**
- **输入**：现有推理数据集（如 Llama-Nemotron Math v1.1）中的原始轨迹。
- **步骤分割**：基于决策词（如 "Wait", "Hmm", "Let me try again"）启发式分割轨迹，将每个独立步骤包裹在 <try> 标签内。
- **摘要生成**：使用外部指令微调模型（Llama-70B-Instruct）为每个 <try> 块生成对应的 <outcome> 摘要（1~2 句话或紧凑方程），确保不引入新信息。
- **结果**：形成交替的 <try>/<outcome> 块结构，保留原问题和最终答案。

**2.2 阶段一：结构化监督微调（Structured SFT）**
- 在结构化数据集上使用标准交叉熵损失进行微调，使模型学会生成 <try>/<outcome> 格式的推理轨迹。

**2.3 阶段二：剪枝感知掩码训练（Pruning-aware Masking SFT）**
- 设计自定义注意力掩码：对于每个完整的 <try>/<outcome> 对，一旦 <outcome> 生成后，所有后续 token 禁止回看该 <try> 块，但可以访问 <outcome>。
- 实现：基于 Flex Attention（Dong et al., 2024）的规则掩码，存储每个步骤的 (try_start, try_end, outcome_end) 三元组，在注意力计算时屏蔽超过 outcome_end 的查询 token 对 try 区域内 key token 的访问。
- 目的：模拟推理时剪枝，训练模型在没有草稿工作的条件下仍能正确推理。

**2.4 阶段三：推理时剪枝**
- 在自回归解码中实现“生成-剪枝-继续”循环：
  1. 解码直到生成完整的 <try> + <outcome>。
  2. 立即将 <try> 块从上下文中移除（不重新计算位置编码，保留原 RoPE 索引）。
  3. 仅保留提示词、历史 <outcome> 和当前块继续解码。
- 实现：基于 HuggingFace Transformers 的自定义解码循环（因 vLLM 不支持细粒度 KV 缓存逐出）。

## 7. 优点

- **方法简单有效**：仅通过改变推理轨迹的格式（<try>/<outcome>）即可同时提升性能和实现剪枝，无需修改优化目标或奖励设计。
- **兼顾准确性与效率**：首次系统性地将结构化推理与上下文剪枝相结合，在保持较高准确率的同时大幅降低内存占用。
- **消融设计严谨**：Compute-Matched 实验有效控制了“更多 token = 更好”的替代解释，增强了结论的因果性。
- **可扩展性**：方法不依赖特定模型架构，可应用于不同推理模型；数据构建过程是自动化的（启发式分割+外部 LLM 摘要），可推广至其他领域。
- **开源潜力**：承诺开源代码，有利于社区复现和改进。
