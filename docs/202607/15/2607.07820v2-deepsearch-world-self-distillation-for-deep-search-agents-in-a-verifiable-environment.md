## 摘要
训练能够从自身经验中改进的工具使用智能体仍然具有挑战性，因为监督微调依赖于固定的教师蒸馏轨迹，而稀疏奖励强化学习为长程交互提供的监督信号较弱。我们提出DeepSearch-Evolve，这是一个针对网络智能体的自蒸馏框架，构建于DeepSearch-World之上——一个具有可复现搜索和页面阅读工具的确定性和可验证环境。DeepSearch-World包含从实体级随机游走构建的42万条多跳问答任务，并支持对自进化至关重要的关键智能体认知行为，包括进度验证、基于经验的反思和失败恢复。DeepSearch-Evolve迭代执行轨迹生成、过滤、数据混合和微调，以训练更强的智能体。在无需从更强模型蒸馏的情况下，DeepSearch-World-9B在与开源智能体的比较中取得了有竞争力的性能，在BrowseComp上达到31.2%，在GAIA上达到61.5%，在HotpotQA上达到93.4%，表明可验证环境能够实现长程网络智能体的可扩展自进化。我们将发布环境、42万训练池、验证集、模型和代码，以促进未来关于自改进深度搜索智能体的研究。

## Abstract
Training tool-use agents to improve from their own experience remains challenging, as supervised fine-tuning relies on fixed teacher-distilled trajectories, while sparse-reward reinforcement learning provides weak supervision for long-horizon interactions. We present DeepSearch-Evolve, a self-distillation framework for web agents built on DeepSearch-World, a deterministic and verifiable environment with reproducible search and page-reading tools. DeepSearch-World contains 420K multi-hop QA tasks constructed from entity-level random walks and supports key agentic cognitive behaviors useful for self-evolving, including progress verification, grounded reflection, and failure recovery. DeepSearch-Evolve iteratively performs trajectory generation, filtering, data mixing, and fine-tuning to train stronger agents. Without distillation from more capable models, DeepSearch-World-9B achieves competitive performance compared with open-source agents, reaching 31.2% on BrowseComp, 61.5% on GAIA, and 93.4% on HotpotQA, showing that verifiable environments enable scalable self-evolution for long-horizon web agents. We will release the environment, 420K training pool, validation set, model, and code to facilitate future research on self-improving deep search agents.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：如何使大规模语言模型（LLM）驱动的工具使用智能体（如搜索智能体）能够**自主地从自身交互经验中持续改进**，而无需依赖更强大模型的蒸馏或静态示范轨迹。
- **背景挑战**：
  - **监督微调（SFT）** 通常从教师模型（或自身）的正面轨迹中学习，但性能容易饱和，受限于骨干模型能力和轨迹多样性。
  - **强化学习（RL）** 利用奖励信号学习，但奖励通常是稀疏的轨迹级信号，难以在长程工具使用中提供有效的细粒度指导（例如搜索查询、工具选择、证据提取等中间步骤的成败）。
  - **在线策略自蒸馏（OPSD）** 试图通过匹配学生与教师的逐标记分布来缓解奖励稀疏问题，但在工具使用场景中，教师分布往往不可靠且难以获得过程级验证信号。
- **本文思路**：构建一个**确定性的、可验证的离线环境**（DeepSearch-World），使智能体在交互过程中能够获得**实体级的过程验证信号**（如每个工具调用是否找到了正确的证据实体），并基于此设计**自蒸馏训练框架**（DeepSearch-Evolve），让智能体从自身已验证的轨迹中迭代学习规划、记忆、纠错等高级认知行为。

### 2. 论文提出的方法论：核心思想、关键技术细节

**核心思想**：通过可验证的离线工具环境，为智能体提供**过程级的密集监督信号**，然后使用自蒸馏（Self-Distillation）将教师（当前模型）生成的已验证轨迹转化为标准ReAct格式的SFT数据，用于训练学生模型，从而实现模型的自我进化。

**关键技术细节**：

1. **DeepSearch-World（可验证环境）**：
   - 基于Wikipedia构建离线语料库（约1000万条目），支持确定性搜索（BM25索引）和页面阅读（SQLite偏移索引）工具。
   - 包含**42万条多跳QA任务**，通过实体级随机游走从Wikipedia超链接图中采样，要求智能体通过多次搜索和阅读来恢复目标实体。
   - 提供**实体级过程验证**：环境存储每个问题的真实实体集，每次工具调用后自动判断是否匹配到未解决的实体，从而给出客观的进度反馈（成功或失败）。
   - 失败时触发**分阶段规则反思**：首次失败给出通用修订信号，多次失败则提供更明确的指导（如规范化实体名称或模糊描述），从而生成“搜索-失败-反思-重试”的高质量轨迹。

2. **DeepSearch-Evolve（自蒸馏框架）**：
   - **教师（脚手架）轨迹生成**：教师模型在环境中执行多阶段推理（P LAN → A CT → E ND），显式维护工作记忆（completed_list, todo_list, experience, information）和状态更新，支持规划、记忆跟踪、失败恢复等行为。
   - **轨迹过滤**：保留答案正确且经过LLM质量过滤（剔除冗余、弱对齐、不一致的轨迹）的轨迹。
   - **脚手架到ReAct转换**：将教师的脚手架特定提示（如[REFLECTION]、[STATE_UPDATE]）和进度状态重写为标准的<think>格式，使学生模型直接学习规划、记忆和纠错能力。
   - **迭代微调**：每一轮用当前模型生成4,000条已验证轨迹（从10,000个问题中采样），重新混合（重要性采样，衰减因子γ=0.5）后训练一个epoch。训练使用Llama Factory，学习率5×10⁻⁶，余弦调度，深度ZeRO-2，BF16精度，最大序列长度32,768。
   - **额外GRPO精调**：在真实网络工具（Google SerpAPI + Jina页面检索）上额外进行1,600条实例的GRPO训练，以减少离线到实机的执行差距。

3. **训练目标**：
   - SFT损失函数：\(\mathcal{L}_{SFT}(\theta) = \mathbb{E}_{\tilde{\tau} \sim \tilde{D}^{(R)}} \sum_{t=1}^T KL(\delta_{y_t} \| \pi_\theta(\cdot|x, y_{<t}))\)（相当于负对数似然）。
   - 不同于OPSD的软目标分布，这里采用硬标签（已验证轨迹的真实token）进行教师强制优化，同时通过迭代生成部分符合当前策略分布（on-policy data generation with teacher-forcing optimization）。

### 6. 论文的主要结论与发现

1. **可验证环境是关键**：使用DeepSearch-World提供的确定性过程验证信号，使得智能体可以通过自蒸馏有效学习规划、记忆、纠错和证据整合能力。
2. **显著性能提升**：DeepSearch-World-9B相比骨干Qwen3.5-9B-Instruct在所有基准上大幅提升（+23.8% on BrowseComp, +37.6% on GAIA, +48.1% on HotPotQA等），且与使用更强模型蒸馏的开源智能体（如Marco-DR、MiroThinker）竞争力相当。
3. **自演化依赖数据多样性**：使用420K训练池比100K池获得了更高的验证性能饱和点、更低的格式错误率和更稳定的工具使用提升，说明大规模多样化任务支撑了更广泛的行为学习。
4. **长程交互与工具使用改善**：DeepSearch-World-9B平均交互轮数（18.0 vs. 4.7）、搜索调用（12.6 vs. 3.8）、访问调用（5.4 vs. 0.9）均显著高于骨干，高级能力评分从19%升至70%。
5. **轨迹过滤与转换策略的重要性**：拒绝采样（RS）+质量过滤（QF）组合效果最佳；状态内化（state internalization）和反思重写（reflection rewriting）均在脚手架→ReAct转换中起到关键作用。

### 7. 优点：方法或实验设计上的亮点

- **创新性**：首次构建了确定性、可验证的离线工具环境，并提供实体级过程验证信号，解决了长程智能体RL稀疏奖励和SFT固定轨迹的局限性。
- **实用性**：环境与真实网络工具兼容（schema对齐），训练后可无缝替换为在线工具；异步生成-训练流水线提高了效率。
- **可重复性**：承诺开源环境、42K QA任务、验证集、模型和代码，有助于社区复现和延伸研究。
- **综合评价**：在不使用更强模型蒸馏的情况下，达到了与顶级开源智能体相当的性能，展示了自演化范式的潜力。
- **分析全面**：对数据规模、过滤策略、转换策略、行为指标等进行了多角度消融和可视化，结论可靠。

### 8. 不足与局限

- **环境领域局限**：仅基于Wikipedia，域多样性有限，可能无法泛化到百科之外的信息源（如动态新闻、专业数据库、社交媒体等）。
- **训练数据偏差**：所有轨迹生成仅使用英文Wikipedia，导致中文基准（BrowseComp-ZH）提升有限，跨语言知识转移可能受限。
- **训练策略限制**：采用迭代SFT而非RL或OPSD更新，可能无法充分发挥过程信号的潜力，且SFT容易导致分布外泛化能力不足。论文也承认“如何将更高级能力注入RL训练仍未探索”。
- **评估局限性**：主实验中虽然使用真实网络工具进行评估，但验证集（DeepSearch-Val）仅377条，规模较小，且未在所有基准上运行消融实验（仅SearchQA和DeepSearch-Val）。
- **伦理风险**：Wikipedia可能包含社会偏见，模型可能继承这些偏见；虽然离线环境减少了实时网络访问风险，但安全对齐仍需进一步研究。
- **计算资源**：虽然异步设计高效，但完整实验仍需数天H20集群，对资源受限的团队有一定门槛。
