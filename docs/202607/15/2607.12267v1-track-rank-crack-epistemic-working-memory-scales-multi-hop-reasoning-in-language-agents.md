## 摘要
交织推理和工具使用的语言代理随着推理链的延长而急剧退化，即使每个单独步骤都很简单。我们将此归因于上下文稀释：代理的推理状态（已确认的内容、怀疑的内容以及仍需要的内容）仅隐含在不断增长的上下文窗口中，早期发现被后续检索所掩盖。我们引入了SLEUTH，通过结构化认知工作记忆使这一状态显式化并可操作：代理维护基于来源的已确认事实、按证据排序的主动假设以及直接驱动下一步行动的开放问题。在五个多跳基准和五个已有基线上，SLEUTH的优势随难度增长，从HotpotQA上的+5分到4跳链上的+11分，无需多轮即可超越Reflexion。分析剩余差距所在，我们识别出证据充分性问题：代理常能找到答案但未能确认，将预算浪费在无谓的验证上。轻量级承诺触发器解决了这一问题，但仅在代理已维护结构化状态时有效：应用于非结构化代理的相同触发器无改进，从而将组织化的认知状态确定为有效承诺的必要条件。最后，在较弱模型上强制遵循协议可在最难问题上恢复高达+19分，表明代理组织其推理的方式（而非原始模型能力）是扩展多跳推理的积极因素。

## Abstract
Language agents that interleave reasoning and tool use degrade sharply as reasoning chains lengthen, even when each individual step is easy. We trace this to context dilution: an agent's investigative state (what it has confirmed, what it suspects, and what it still needs) lives only implicitly in a growing context window, where early discoveries are buried under later retrievals. We introduce SLEUTH, which makes this state explicit and actionable through a structured epistemic working memory: the agent maintains Confirmed Facts grounded to sources, Active Hypotheses ranked by evidence, and Open Questions that directly drive its next action. Across five multi-hop benchmarks and five established baselines, SLEUTH's advantage grows with difficulty, from +5 points on HotpotQA to +11 on 4-hop chains, surpassing Reflexion without multiple episodes. Analyzing where the remaining gap lies, we identify the evidence sufficiency problem: agents often find the answer but fail to commit, exhausting their budget on needless verification. A lightweight commitment trigger fixes this, but only when the agent already maintains structured state: the identical trigger applied to an unstructured agent yields no improvement, isolating organized epistemic state as the necessary condition for effective commitment. Finally, enforcing protocol adherence on a weaker model recovers up to +19 points on the hardest problems, showing that how an agent organizes its reasoning, not raw model capability, is the active ingredient for scaling multi-hop reasoning.

- **整体含义**：论文的核心论点是：**对于语言代理，推理的组织方式（即如何管理认知状态）比推理能力本身更为重要**。这提供了一个新的设计维度，与单纯扩大模型规模或增加测试时计算量形成互补。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出 **SLEUTH** (Structured Layered Evidence-Updated Tracking of Hypotheses) 框架，通过**提示工程**，为一个语言代理装备一个**显式的、结构化的认知工作记忆**。这个工作记忆在每个步骤中都被读取和更新，使代理的推理状态变得清晰且可操作。
- **关键技术细节**：
    - **工作记忆结构**：在每个时间步 `t`，代理维护一个三元组状态 `Wt = (Ft, Ht, Qt)`。
        - **已确认事实 (Confirmed Facts Ft)**：从工具输出中提取的、可验证的陈述，并锚定到具体来源（如段落标题）。这些事实是单调累积的，一旦确认就不会被撤销，以防止上下文稀释。
        - **活跃假设 (Active Hypotheses Ht)**：根据现有证据排名的候选项答案。每个假设都关联到支持或反驳它的事实，并具有明确的置信度（高/中/低）。
        - **开放问题 (Open Questions Qt)**：悬而未决的疑问，其答案最有可能改变假设排名。每个问题都附带一个建议的搜索动作。
    - **更新协议**：每次工具调用后，代理执行一个结构化的更新过程：
        1.  **提取事实**：从工具输出中提取新的可验证信息，并作为已确认事实加入。
        2.  **更新假设**：根据新事实来调整现有假设的置信度，或生成新的假设。
        3.  **更新问题**：移除已被回答的问题，并生成新的、指向剩余不确定性的问题。
        4.  **选择动作**：执行优先级最高的开放问题所建议的搜索动作。如果无开放问题且某个假设置信度高，则调用 `finish(answer)` 给出答案。
    - **承诺机制 (Commitment Mechanism)**：
        - **问题**：结构化工作记忆可能导致“过度验证”，即代理即使找到答案，也因缺乏明确的停止标准而继续搜索，最终耗尽预算。这被称为**“证据充分性问题”**。
        - **解决方案**：一个轻量级的 **“nudge”（轻推）** 机制。当满足两个条件时触发：1) 代理已消耗至少 `α` 比例的预算（`t ≥ αT`）；2) 代理已积累至少 `τ` 个已确认事实。触发时，代理会收到一条合成指令，要求它**直接从其已确认事实和活跃假设中总结答案**，而不再进行搜索。`α=0.7` 和 `τ=2` 被证明是经验上的最优设置，实现了探索（收集证据）与利用（基于证据回答）之间的平衡。

### 3. 实验设计：使用的数据集、基准及对比方法

- **数据集**：使用了三个广泛认可的多跳问答数据集，包含2至4跳不等的问题：

    - **MuSiQue**：包含2/3/4跳问题的子集，共2,417个，用于控制难度分析。

- **基准方法（Baselines）**：对比了五种主流的代理推理范式：
    - **ReAct**：交织推理和行动。
    - **ReAct-CoT**：在每个动作前加入显式的思维链。
    - **Self-Ask**：将问题分解为子问题顺序回答。
    - **IRCoT**：交织检索与思维链。
    - **Reflexion**：在多次尝试后添加事后反思（本文使用2次尝试）。

    - **Facts-Only**：仅维护结构化事实。
    - **Facts+Hyp**：维护事实和假设。
    - **Facts+Q**：维护事实和开放问题。

### 6. 论文的主要结论与发现

- **SLEUTH有效性**：在所有五个基准测试中，SLEUTH均取得了最佳成绩，且优势随推理难度增加而扩大（从HotpotQA的+4.7分到4-hop MuSiQue的+10.9分），甚至无需多轮尝试就超越了Reflexion。
- **证据充分性问题**：识别并隔离了“证据充分性问题”，即智能体找到答案但无法在有限预算内做出承诺。
- **承诺机制的必要条件**：承诺机制（nudge）仅在代理拥有SLEUTH提供的**结构化认知状态**时才能有效工作。在非结构化代理上使用相同的机制没有效果。
- **协议遵循的重要性**：迫使较弱模型（GLM-5）严格遵守SLEUTH协议，其在最难问题上提升了高达**+19分**，这强烈证明了推理的组织方式比模型本身的原始能力更为关键。
- **核心发现**：**推理的组织方式（而非模型能力）是扩展多跳推理的积极因素**。

### 7. 优点

- **方法轻量且高效**：SLEUTH完全通过**提示工程**实现，无需任何模型微调、架构更改或额外的代码执行基础设施，具有极高的实用性和普适性。
- **实验设计精巧且严谨**：通过精心设计的消融实验和“必要条件”控制实验，清晰地证明了结构化工作记忆的价值，以及承诺机制与结构化状态之间的因果关系，而非简单的性能提升。
- **识别并解决了关键瓶颈**：系统性地诊断了长链推理中的两个主要失败模式：“信息丢失”和“信息瘫痪”，并分别提供了针对性的解决方案（结构事实追踪和承诺机制）。
- **可解释性强**：SLEUTH在每个步骤都显式输出其认知状态，使推理过程变得透明，便于人类监督和调试。

### 8. 不足与局限

- **承诺阈值可能不通用**：承诺机制的触发阈值（70%预算）是在特定基准上调优的，对于任务结构差异很大的场景（如探索需求远大于合成需求的任务）可能需要重新调整。
- **单调事实策略的潜在风险**：一旦确认事实被错误地认知为正确，其单调累积的特性会固化早期错误，无法进行修订，这在开放域或存在矛盾信息的场景中尤其危险。

- **实验设定局限性**：所有实验都在一个“封闭上下文”设定下进行，即所有支持证据都隐含在一个给定的文档集合中。论文方法在处理开放域、信息矛盾或时效性强的在线数据源时的表现尚待验证。
