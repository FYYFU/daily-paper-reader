## 摘要
大型语言模型（LLM）被日益信任于在诸如系统理论过程分析（STPA）等严谨过程中起草安全分析的产物，例如损失、危害、不安全控制动作（UCA）和安全约束。然而，这一迅速增长的文献中存在一个盲点：除了正在执行分析的LLM辅助工具本身——这是一个安全相关的系统，可能产生标准幻觉、发布不可验证的约束，并且从提示到产物没有留下审计轨迹——每个系统都得到了分析。我们认真对待该领域忽略的问题——
{谁分析分析器？}——并通过将该工具本身作为STPA分析对象来回答它。我们提出
\{宪法元STPA}，一个围绕闭环构建的LLM辅助STPA工具：该工具对AI辅助安全工具类别进行
{元STPA}，并{推导}而非断言其治理宪法，从结果中的损失$\to$危害$\to$UCA$\to$约束链中得到一部包含$21$条工具原则和$8$条元安全原则的已发布宪法，每条原则都绑定到代码执行点。我们将测量对象形式化为一个在原则集$P$（$|P|{=}29$）上的宪法边际覆盖算子，配备了一个将覆盖范围与模型和扫描器分离的可靠性引理，并报告四项发现。
{(i)~自推导：}一个前沿集成（{claude-opus-4.8}${+}${claude-sonnet-4}）从工具自身设计恢复了$18/21$条规范原则和全部$8/8$条治理原则，而一个较弱配对恢复了$12/21$条和$3/8$条，因此元层受限于模型而非宪法，且同样的$8/8$条原则从一个独立编写的第二个工具中再次出现。

## Abstract
Large language models (LLMs) are increasingly trusted to draft the artifacts of safety analysis such as, losses, hazards, Unsafe Control Actions (UCAs), and safety constraints, inside rigorous processes such as Systems-Theoretic Process Analysis (STPA). Yet a blind spot runs through this fast-growing literature: every system gets analysed except the LLM-assisted tool doing the analysing, which is itself a safety-relevant system that can hallucinate standards, emit unverifiable constraints, and leave no audit trail from prompt to artifact. We take seriously the question the field has skipped -- {who analyses the analyser?} and answer it by turning STPA on the tool itself. We present \{Constitutional Meta-STPA}, an LLM-assisted STPA tool built around a closed loop: the tool runs a {meta-STPA} of the class of AI-assisted safety tools and {derives} rather than asserts, its governance constitution from the resulting loss$\to$hazard$\to$UCA$\to$constraint chain, yielding a published constitution of $21$ Tool Principles and $8$ Meta-Safety Principles, each bound to a code enforcement point. We formalise the measured object as a constitution-marginal coverage operator over a principle set $P$ ($|P|{=}29$) with a soundness lemma that isolates coverage from model and scanner, and report four findings. {(i)~Self-derivation:} a frontier ensemble ({claude-opus-4.8}${+}${claude-sonnet-4}) recovers $18/21$ canonical and all $8/8$ governance principles from the tool's own design, while a weaker pair recovers $12/21$ and $3/8$, so the meta layer is model-limited, not constitution-limited, and the same $8/8$ re-emerge from a second, independently authored tool.

## 1. 核心问题与整体含义（研究动机和背景）

**核心问题**：大型语言模型（LLM）被越来越多地用于起草安全分析产物（如损失、危害、不安全控制动作UCA、安全约束），例如在系统理论过程分析（STPA）框架下。然而，现有研究存在一个系统性的盲点：**所有被分析的系统都得到了分析，但执行分析的LLM辅助工具本身却从未被分析**。该工具本身就是一个安全相关系统——它可能产生幻觉（虚构不存在的标准）、发布不可验证的约束、从提示到产物不留审计轨迹。论文提出的根本问题是：“谁分析分析器？”（Who analyses the analyser?）

**整体含义**：作者通过将STPA方法应用于LLM辅助安全分析工具本身，构建一个自验证闭环，使得分析工具能够自我推导出治理其行为的宪法原则，并确保这些原则被代码强制执行。这个闭环保证了分析工具的可靠性和可审计性。

## 2. 方法论

### 核心思想
- **Constitutional Meta-STPA**：一个LLM辅助STPA工具，其核心创新是运行一个**元STPA**——将AI辅助安全工具这个类别本身作为系统进行分析，从损失→危害→UCA→约束链中**推导**出工具的治理宪法，而非人工断言。
- 宪法包含**21条工具原则（Tool Principles, TP）**（行为层）和**8条元安全原则（Meta-Safety Principles, MSP）**（治理层），每条原则绑定到具体的代码执行点。
- 工具是一个五步流水线：输入系统描述S和宪法C，输出分析产物A = ⟨损失, 危害, 控制器, UCA, 约束⟩、验证报告R、导出决策g。宪法注入系统提示，但内容由模型生成，结构完整性由工具本身保证（确定性骨架枚举所有UCA槽位）。
- 覆盖框架：用宪法边际覆盖算子`cov(C, S, M, k)`测量生成的文本中匹配指定原则的集合。每个原则p有确定性见证谓词φp（通过正则表达式匹配）。引理1保证任何覆盖差异都基于生成文本中的词汇证据，而非模型自我报告或评判。

### 关键技术细节
- **元STPA推导**：将AI辅助安全工具的控制器识别为LLM、验证层、投票框架、导出门、工程师。遍历四种UCA类型（未提供、不安全提供、错误时序/顺序、过早停止/过长应用）得到8个元安全原则，如MSP-01（LLM审计日志）、MSP-02（运行清单）、MSP-05（宪法固定哈希）、MSP-06（语义匹配投票）、MSP-07（验证器门控导出）等。
- **语义匹配投票**：对于多模型投票，使用停用词过滤后的Token集计算Jaccard相似度（阈值0.4），替代原始基于ID相等性的投票（后者因同义控制器名称导致100%不一致）。同时提供句子嵌入（all-MiniLM-L6-v2）上界。
- **验证器门控导出**：导出（PDF、邮件）仅在8个验证器全部通过或工程师记录覆写时才允许。验证器包括完整性、四种UCA类型覆盖、置信度与证据、无捏造、可测量约束、责任分配、过度自信检测、限制披露。
- **可复现性清单**：每个运行记录`run manifest.json`（工具版本、git SHA、系统描述哈希、模型/温度/种子、宪法哈希）和`llm calls.jsonl`（每次调用的时间戳、步骤、模型、提示/响应SHA-256、令牌数等）。

- **20个对抗性探测**：用于行为实验，涵盖8个类别（完整性、协作、机密性、一致性、边缘情况、幻觉、过度自信、组合）。

### Benchmark与对比方法
- **宪法变体**（五步阶梯cv0–cv4）：

## 6. 主要结论与发现

1. **自推导成功**：前沿集成（claude-opus-4.8+claude-sonnet-4）从工具自身设计恢复18/21工具原则和全部8/8元安全原则；弱对恢复12/21和3/8。表明元层受限于模型能力而非宪法内容。同样的8/8条原则从第二个独立工具再次出现。
2. **行为效应**：工具原则（TP）将对抗性探测的平均安全得分从1.03提高到1.85（+79%，p<0.001，d=1.29），而通用Claude原则（cv1）无显著效果。元安全原则（MSP）对单轮行为无影响（预期内，因为它们在分析时运行，不改变单次回答）。
3. **覆盖指标是区分器而非剂量**：在固定系统上，元覆盖随宪法强度非单调变化（空宪法cv0反而最高）。但跨系统对比时，元覆盖能清晰区分AI工具（6/8）和硬件系统（0/8），成为系统区分器。
4. **可复现性**：固定种子和温度下，标准STPA的UCA变异系数≤12%（11/12个单元），gemini-2.5-flash完全可复现（CV=0%）。自系统分析中claude-sonnet-4种子忽略但元覆盖仍稳定（8/8）。
5. **非复现公开报告**：早期探索性运行报告了元覆盖在最强宪法下单调跳升，但确定性复现未再现；论文公开这一负面结果。

## 7. 优点

- **自验证闭环**：创新性地将STPA用于分析自身工具，推导而非断言治理原则，实现了“谁分析分析器”的自洽回答。
- **形式化覆盖框架**：宪法边际覆盖算子配合可靠性引理，确保任何覆盖变化基于生成的文本词汇证据，而非模型自我报告或评判。
- **多层面评估**：覆盖（写什么）和行为（做什么）分开评估，明确行为杠杆在工具原则层，治理在元层。
- **诚实与可复现**：提供完整的可复现性清单、每个运行的SHA-256哈希、运行清单。公开负面结果和不可复现发现，增强可信度。
- **稳健工程**：确定性的UCA枚举骨架保证四种类型全覆盖；验证器门控导出防止不完整分析被分发；语义投票解决同义标识符不一致问题。
- **跨系统泛化**：验证了元安全原则在第二个独立AI工具上也被恢复，而硬件系统保持0/8，说明原则是AI辅助安全工具类的属性。

## 8. 不足与局限

- **词汇扫描器假阳性/假阴性**：正则表达式的见证谓词容易被释义绕过。论文用LLM评判交叉检查（κ=0.39，中等）缓解，但未用人类专家校准。绝对覆盖计数应视为代理信号。
- **缺乏人类专家基线**：没有安全工程师独立评分相同分析或探测。行为评分和词汇银行均编码作者自身判断，外部效度待定。
- **泛化广度有限**：仅基于两个AI工具和三个硬件系统，单一分析方法（STPA），英语探测，一组前沿模型通过单一路由器。不能保证到其他工具类别、方法（FTA、FMEA、HAZOP）、非英语输入或未测试模型家族的通用性。
- **非确定性挑战**：即使固定种子，OpenRouter可能重路由后端，bit精确重放无法保证。清单仅能检测漂移而非阻止。
- **行为声明范围**：300响应的实验仅测量单轮文本行为，不涉及多轮交互、工具使用或下游部署结果。
