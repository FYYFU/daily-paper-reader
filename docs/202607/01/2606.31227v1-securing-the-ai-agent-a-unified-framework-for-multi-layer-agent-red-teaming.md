## 摘要
开源AI基础设施的快速增长——从模型服务引擎和智能体平台，到模型上下文协议（MCP）生态系统及语言模型本身——已超越了可用的安全防御工具。我们提出了AI-Infra-Guard，一个开源框架，它围绕一个核心观察组织AI红队测试：AI智能体的攻击面跨越多个层级（基础设施层、协议/工具层、智能体行为层和模型层），而单一检测范式无法适用于所有层级。因此，该框架为每个层级匹配一种范式：从基于超过75个AI组件和1400多个漏洞规则的确定性规则匹配，到LLM驱动的MCP服务器和智能体技能包的智能体审计，以及多轮黑盒智能体红队测试，再到涵盖16个数据集上26个以上攻击操作的越狱工具集。据我们所知，这是唯一涵盖所有这些方面的开源框架，包括对日益扩展AI智能体的智能体技能的供应链审计。我们将AI-Infra-Guard开源，以便“层级-范式匹配”能够作为智能体安全的实用基础，并成为社区共同发展的共享平台。

## Abstract
The fast growth of open-source AI infrastructure, from model serving engines and agent platforms to the Model Context Protocol (MCP) ecosystem and the language models themselves, has outpaced the security tooling available to defend it. We present AI-Infra-Guard, an open-source framework that organizes AI red teaming around a single observation: the attack surface of an AI agent is stratified across layers (infrastructure, protocol/tool, agent behavior, and model), and no single detection paradigm fits all of them. The framework therefore matches a paradigm to each layer, from deterministic rule matching over 75+ AI components and 1{,}400+ vulnerability rules, through LLM-driven agentic auditing of MCP servers and agent-skill packages and multi-turn black-box agent red teaming, to a jailbreak harness with 26+ attack operators over sixteen datasets. To our knowledge it is the only open-source framework to span all of these, including supply-chain auditing of the agent skills that increasingly extend AI agents. We release AI-Infra-Guard as open source so that \emph{layer-paradigm matching} can serve as a practical foundation for agent security and a shared base for the community to build on.

## 2. 方法论

### 核心思想：层-范式匹配（Layer-Paradigm Matching）

论文提出 **AI-Infra-Guard** 框架，将AI智能体攻击面分为四层，每层匹配最合适的检测范式：

| 层级 | 匹配范式 | 对应模块 |
|------|----------|----------|
| 基础设施（Infrastructure） | 确定性规则匹配 | M1: Infra-Scan |
| 协议/工具（Protocol/Tool） | LLM驱动的智能体审计 | M2: MCP-Scan |
| 智能体行为（Agent Behavior） | 多轮黑盒红队测试 | M3: Agent-Scan |
| 模型（Model） | 攻击算子枚举+模型判断 | M4: Prompt-Security |

此外，新增 **智能体技能（Agent Skills）** 供应链审计，作为协议/工具层的一部分。

### 关键技术细节

1. **基础设施扫描（M1）**：
   - 手写解释器执行指纹匹配语言（支持body/header/icon/hash字段，四种匹配算子，布尔逻辑）。
   - 版本规范化处理不规则版本字符串（如`b7824`、`latest`、`2.3.dev`等）。
   - 107条指纹规则 + 1443条漏洞规则，覆盖75个AI组件。
   - 发现分三级：已验证、基于版本、推断（空谓词规则）。

2. **MCP服务器审计（M2）**：
   - 静态模式（白盒）：分析源代码；动态模式（黑盒）：通过协议枚举工具+调用测试。
   - **Prompt-as-Rule**：将检测知识编码为自然语言规则，包含检测标准、排除条件。
   - 智能体化pipeline（agentic harness）：信息收集→代码审计→漏洞验证，带reason-act循环、上下文压缩、模型路由（不同子任务用不同模型）。
   - 扫描器自身防护：将工具响应视为不可信数据，避免间接提示注入攻击。

3. **智能体技能审计（M3-Skills）**：
   - 分析技能包（SKILL.md、安装脚本、依赖等），检测供应链投毒、隐藏注入、权限滥用等。
   - 预处理→静态风险线索检索→AI审计智能体→风险分类。
   - 不直接执行技能，通过受限工具环境探索。
   - 发布 **SkillTrustBench** 基准（5520个评估案例，9类威胁）。

4. **智能体红队测试（M3）**：
   - 通过对话接口进行多轮黑盒测试，由智能体代理自动生成攻击对话。
   - 四个风险家族：数据泄露、工具滥用、间接注入、授权绕过。
   - 使用升级阶梯（从直接探测到编码绕过）、停止规则、并行技能工人控制成本。
   - 客观锚定：SSRF用canary token，间接注入用标记（marker），将主观判断转为确定性检查。

5. **LLM越狱评估（M4）**：
   - 整合26+攻击算子（约70种编码/混淆、多轮策略如Crescendo、树搜索、Best-of-N等）。
   - 三个角色：模拟器模型（生成攻击）、目标模型、评估模型。
   - LLM-as-judge：通用越狱判断 + 伤害特定判断。
   - 16个数据集共约7248个有害提示。

## 6. 主要结论与发现

- **分层-范式匹配原则有效**：不同层级需要不同检测范式，混合使用可覆盖全栈攻击面。
- **AI-Infra-Guard是首个统一框架**：覆盖基础设施、协议/工具、行为、模型四层，并新增技能供应链审计。
- **Prompt-as-Rule范式可行**：将检测知识编码为自然语言规则，配合排除条件降低误报。
- **客观锚定提高信噪比**：canary token、标记注入等将主观判断转为确定性检查。
- **开放式架构可扩展**：规则库、攻击算子、检测模块均为可插拔，社区可贡献。
- **技能供应链审计是新关键**：随着智能体技能市场兴起，审计能力不可或缺。

## 7. 优点

1. **全覆盖性**：首个涵盖四层+技能审计的开源框架，解决工具碎片化问题。
2. **层-范式匹配设计思想清晰**：解释为何不同层需不同检测方法，具有理论指导意义。
3. **Prompt-as-Rule**：用自然语言规则表达检测逻辑，灵活且可维护，加入排除条件对抗LLM过度报告。

7. **可扩展性与开源**：插件系统、统一服务器-代理架构，便于社区贡献。
