## 摘要
推理模型在不同实例上花费不同的有用计算量，但尚不清楚学习到的停止规则何时优于简单的置信度或收敛阈值。我们使用LearnStop（一种无隐藏状态的检查点停止器）研究这一问题，该工具适用于推理语言模型。在固定预算检查点，LearnStop从当前推理前缀中提取简短答案，并根据在线特征（如答案置信度、熵、前缀投票份额、答案稳定性和回溯标记密度）预测前缀正确性。在涵盖GSM8K、MATH-500、MMLU-Pro、AIME-90、GPQA、Qwen3和DeepSeek-R1蒸馏的18个任务-模型设置中，答案是任务相关的。在自由形式数学任务上，学习到的多特征停止改进了固定预算边界，且通常优于标量退出：在GSM8K与Qwen3-32B上，经验边界达到事后峰值自适应增益+0.157，验证选择的工作点保持正增益，且相对于最强标量基线的配对增益为+0.028。在多项选择和极难设置中，标量置信度、熵或稳定性规则具有竞争力或更强。因此，我们不将学习停止视为标量退出的通用替代方案，而是视为一种价值取决于轨迹结构的工具。我们进一步提供了验证选择的工作点、配对bootstrap检验、有限网格失正确风险校准、KV分叉、前缀缓存和黑盒机制下的成本核算、H100服务配置文件、检查点调度扫描、迁移分析和鲁棒性检查。主要的实践发现是：当许多问题在完整预算之前变得正确，但未表现出单一的可靠标量停止信号时，学习停止是有用的；当置信度或答案收敛已解决停止问题时，其优势基本消失。

## Abstract
Reasoning models spend different amounts of useful computation across instances, but it remains unclear when a learned stopping rule improves over simple confidence or convergence thresholds. We study this question with LearnStop, a hidden-state-free checkpoint stopper for reasoning language models. At fixed budget checkpoints, LearnStop probes a short answer from the current reasoning prefix and predicts prefix correctness from online features such as answer confidence, entropy, prefix vote share, answer stability, and backtracking-marker density. Across 18 task-model settings spanning GSM8K, MATH-500, MMLU-Pro, AIME-90, GPQA, Qwen3, and DeepSeek-R1 distillations, the answer is task-dependent. On free-form math, learned multi-feature stopping improves the fixed-budget frontier and often beats scalar exits: on GSM8K with Qwen3-32B, the empirical frontier reaches a post-hoc peak adapt gain of +0.157, validation-selected operating points preserve positive gains, and the paired gain over the strongest scalar baseline is +0.028. On multiple-choice and very hard settings, scalar confidence, entropy, or stability rules are competitive or stronger. We therefore frame learned stopping not as a universal replacement for scalar exits, but as a tool whose value depends on trajectory structure. We further provide validation-selected operating points, paired bootstrap tests, finite-grid lost-correct risk calibration, cost accounting under KV-fork, prefix-cache, and black-box regimes, H100 serving profiles, checkpoint-schedule sweeps, transfer analyses, and robustness checks. The main practical finding is that learned stopping is useful when many questions become correct before full budget but do not exhibit a single reliable scalar stopping signal; its benefits largely disappear when confidence or answer convergence already solves the stopping problem.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程（用文字说明即可）

### 核心思想
LearnStop 是一种无隐藏状态的检查点停止器。它通过在每个固定预算检查点探测当前推理前缀得出的简短答案，提取在线特征，训练逻辑回归分类器预测前缀正确性，当预测概率超过阈值时停止生成。

### 关键技术细节
- **检查点探测**：在预算网格 B₀ < B₁ < ... < Bₘ₋₁ 的每个检查点 j，将当前推理前缀截断至 Bⱼ 个 token，追加“停止思考”标记，贪婪解码不超过 A 个 token 的答案 aⱼ。标签 cⱼ = 1{ aⱼ = yᵢ }（yᵢ为黄金答案）。
- **前缀可观测特征**（8个推荐特征）：归一化预算、归一化检查点索引、平均答案对数概率、答案 token 熵、当前答案是否与上一检查点匹配、当前答案的游程长度、前缀投票份额（仅用历史检查点）、回溯标记密度。其他特征（如是否已自然发出思考结束标记、观察到的思考长度）作为诊断上限，消融表明不影响结论。
- **训练与指标**：使用分组五折交叉验证（同一问题的所有检查点在同一折）训练逻辑回归。性能指标为**自适应增益** G(τ) = Acc_adapt(τ) - Acc_fixed(C_adapt(τ))，其中固定预算精度在平均思考 token 成本处线性插值。验证选择的工作点使用独立的验证集选择阈值。
- **成本核算**：总成本 C_total = C_think + (j+1) * A，A=48（除非另有说明）。区分三种部署场景：KV 缓存分叉（KV-fork）、前缀缓存（prefix-cache）、纯黑盒 API。
- **失正确风险校准**：定义 Lᵢ(τ) = 1{Fᵢ=1, Sᵢ(τ)=0}，控制 E[Lᵢ(τ)] 以约束与完整思考的精度损失上限。使用有限网格的同步 Hoeffding 上界 U(τ) 选择阈值，保证以概率 1-δ 满足风险上界 ≤ α。

## 6. 论文的主要结论与发现

1. **任务依赖性**：学习停止**最有用**于自由形式数学任务（GSM8K、MATH-500），在这些任务上它能改进固定预算边界，峰值自适应增益最高达 +0.157（GSM8K-32B），且验证选择的工作点保持正增益（+0.025~+0.136）。**不太有用**于多项选择（MMLU-Pro、GPQA）和极难任务（AIME-90），标量规则已足够或更好。
2. **机制分析**：学习停止的优势来自于当许多问题在完整预算前已正确但不存在单一可靠标量信号时，多特征互补捕获了这种复杂性；一旦置信度或答案收敛已解决停止问题，学到的收益消失。
3. **风险控制有效**：有限网格的失正确风险校准可在给定风险容忍度下选择工作点，所有测试风险低于目标（如 α=0.15 时，GSM8K-32B 测试风险 0.040，精度 0.915，总 token 节省 32.5%）。
4. **成本敏感性**：部署成本是关键。KV 缓存分叉/前缀缓存下节省显著；纯黑盒 API 可能消除或反转节省。检查点密度也应权衡：6个线性检查点保留 96% 的增益，而密度增加显著增加延迟。
5. **特征稳健性**：8特征版本（不含长度/端特征）与10特征版本性能无显著差异，推荐部署；逻辑回归与梯度提升/MLP 相比竞争力强且更易校准。
6. **迁移不对称性**：从困难数学到简单数学（MATH-500 → GSM8K）迁移效果好，反之较弱；跨族迁移效果差，需要目标校准。

## 8. 不足与局限：包括实验覆盖、偏差风险、应用限制等

- **任务覆盖有限**：AIME-90 仅 90 个问题，GPQA 也较小，统计显著性有限；结论主要基于数学和多项选择，未涵盖编程、科学推理等类型。
