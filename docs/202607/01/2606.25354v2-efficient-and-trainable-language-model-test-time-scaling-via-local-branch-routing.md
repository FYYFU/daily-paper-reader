## 摘要
测试时扩展改进了语言模型的推理能力，但现有方法往往面临艰难的权衡：长思维链采样仍然是单线程的，而句子级或解决方案级搜索可能计算成本高昂且难以端到端训练。我们引入了局部分支路由（LBR），这是一个令牌级测试时扩展框架，它扩展一个小型局部前瞻树，通过语言模型前向传播所有采样分支，并使用轻量级路由器选择要提交的深度为1的子树。通过在候选局部未来的隐藏状态上进行路由，LBR允许每个令牌决策使用超出根节点下一个令牌分布的证据，同时避免完整的解决方案级搜索。由此产生的剪枝-移位-增长解码过程保留了离散分支身份，并定义了一个可处理的树轨迹似然：新增长的节点在首次采样时被计数，路由器决策被分配显式概率。这使得能够使用可验证奖励进行端到端强化学习，在相同的似然比原则下联合优化基础模型和路由器，类似于离散令牌RLVR。在合成层次规划任务中，LBR表明候选隐藏状态提供了有用的路由证据。在数学推理基准上，LBR在Pass@1和Pass@32上均优于离散思维链、普通离散令牌RLVR以及与RL兼容的软令牌分支基线。这些结果表明，轻量级局部分支提供了一种高效、可训练且离散的语言模型测试时扩展形式。

## Abstract
Test-time scaling improves language-model reasoning, but existing approaches often face a difficult trade-off: long chain-of-thought sampling remains single-threaded, while sentence- or solution-level search can be computationally expensive and hard to train end-to-end. We introduce Local Branch Routing (LBR), a token-level test-time scaling framework that expands a small local lookahead tree, forwards all sampled branches through the language model, and uses a lightweight router to select the depth-1 subtree to commit. By routing over the hidden states of candidate local futures, LBR allows each token decision to use evidence beyond the root next-token distribution while avoiding full solution-level search. The resulting prune-shift-grow decoding process preserves discrete branch identities and defines a tractable tree-trajectory likelihood: newly grown nodes are counted when first sampled, and router decisions are assigned explicit probabilities. This enables end-to-end reinforcement learning with verifiable rewards, jointly optimizing the base model and router under the same likelihood-ratio principle as discrete-token RLVR. On synthetic hierarchical-planning tasks, LBR shows that post-candidate hidden states provide useful routing evidence. On mathematical reasoning benchmarks, LBR improves both Pass@1 and Pass@32 over discrete chain-of-thought, vanilla discrete-token RLVR, and RL-compatible soft-token branching baselines. These results suggest that lightweight local branching offers an efficient, trainable, and discrete form of language-model test-time scaling.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：测试时扩展（test-time scaling）已证明能提升语言模型推理能力，但现有方法存在根本性权衡：
  - 长思维链（Chain-of-Thought）采样本质上是单线程的，无法并行探索替代路径；
  - 句子级或解决方案级搜索（如 Tree-of-Thoughts、MCTS）虽引入宽度，但计算成本高，且难以作为单一可训练解码策略进行端到端优化。
- **目标**：探索一种**轻量级、可训练、离散**的 token 级测试时扩展方法，在不进行全局搜索的前提下，让每个 token 决策能利用局部未来隐藏状态作为证据。

## 2. 方法论

### 2.1 核心思想

- **Local Branch Routing (LBR)**：在自回归解码的每一步，维护一个滚动的小型局部前瞻树（深度 \(L\)，宽度 \(K\)），树的每个节点都被前向传播计算出隐藏状态。然后一个轻量级路由器（router）基于所有节点隐藏状态，选择并提交其中一个深度为1的子树（即下一个 token）。未选中的子树被剪枝，选中的子树则向前移位并重新生长一层，恢复深度 \(L\)。
- 本质：将 token 级决策从仅依赖根节点 next-token 分布，提升为利用局部候选项的**后候隐藏状态**（post-candidate hidden states），从而提供更丰富的决策信号。

### 2.2 关键技术细节

1. **局部树构建**：
   - 每一步从已提交前缀 \(x_{<t}\) 出发，采样 \(K\) 个 child tokens（来自过滤后的模型分布 \(\tilde{\pi}_\theta\)），若 \(L=1\) 则仅一层；若 \(L=2\) 则每个候选 token 再进一步展开 \(K\) 个 continuation。
   - 所有 sampled 节点都通过语言模型前向传播，计算隐藏状态 \(h_t(v) = f_\theta(x_{<t}, \text{path}(v))\)。

2. **路由**：
   - 路由器 \(\rho_\phi\) 接收局部树所有隐藏状态，输出一个分布选择深度-1子树。
   - 路由器架构：集合注意力（Set-Attention）——先将每个候选子树编码为向量，再通过跨候选子树的集合注意力进行对比，最后用共享 MLP 打分，softmax 得到路由分布。这保证了排列不变性，且支持对比性决策。

3. **剪枝-移位-重新生长（Prune–Shift–Grow）**：
   - 选中子树后，其根 token 被提交为 \(x_t\)，其他子树被丢弃。
   - 选中的子树的剩余节点成为下一步的局部树根部，然后从新的前沿节点生长一层新节点（再次采样和前向传播），恢复深度 \(L\)。

4. **轨迹似然与训练**：
   - 树轨迹似然可分解为：新生长节点的 log 概率之和 + 路由器选择 log 概率之和。复用节点（从先前步骤保留）的 log 概率只在首次生长时计入。
   - 训练采用 RLVR（Reinforcement Learning from Verifiable Rewards），使用 GRPO 风格的目标函数，损失函数为优势加权的负对数似然，同时优化基础模型 \(\theta\) 和路由器 \(\phi\)。当 \(K=1\) 时退化为标准离散 token RLVR。

### 2.3 公式（文字说明）

- 轨迹似然：\(\log p_{\theta,\phi}(\mathcal{F}|q) = \sum_{t} \sum_{v \in \mathcal{G}_t} \log \tilde{\pi}_\theta(\text{tok}(v)|\text{ctx}(v)) + \sum_t \log \rho_\phi(k_t^* | x_{<t}, \mathcal{A}_t; \theta)\)。
- 训练损失：\(\mathcal{L}_{\text{LBR}}(\theta,\phi;\mathcal{F}) = -A(\mathcal{F}) \log p_{\theta,\phi}(\mathcal{F}|q)\)，其中 \(A\) 为基于验证奖励计算的 advantage。

## 6. 主要结论与发现

1. **LBR 提升数学推理**：在 1.5B 和 7B 基座上，LBR 在 Pass@1 和 Pass@32 上均一致优于离散 CoT、普通离散 RLVR 和软 token RLVR。尤其在小模型上提升显著。
2. **局部前瞻树提供有用信号**：合成任务表明，后候选隐藏状态比前分支状态更可预测最终答案，且路由器能够利用这一信号。
3. **对比性路由优于独立评分**：集合注意力路由器相比独立路由器提升明显，说明跨分支比较是关键。
4. **联合训练适应**：路由器熵和基础模型在路由位置的熵同时下降，说明两者协同优化。
5. **离散分支优于软混合**：概念身份探针显示 LBR 保持分支身份信息，而软 token 方法会模糊概念身份，导致性能下降。

## 7. 优点

- **高效轻量**：仅需局部小树前向传播和轻量路由器，计算开销远低于全局搜索。
- **可端到端训练**：树轨迹似然分解简单，可直接纳入 GRPO 等 RLVR 框架，无需额外过程奖励或 MCTS。
- **提供对比性证据**：路由器利用多个候选分支的隐藏状态进行相对决策，而非独立重排序。
- **保持离散性**：不引入连续嵌入混合，保留 token 级别的离散分支，避免概念模糊。
- **提升多样性与准确性**：同时改善 Pass@1（单样本）和 Pass@32（多次采样），说明路由不会坍缩至确定性。

## 8. 不足与局限

- **局部性**：LBR 的规划仅基于局部前瞻窗口（深度 \(L\)），不能实现长距离规划或全局搜索。对于需要远距离依赖或复杂探索的任务可能效果有限。
- **深度-速度权衡**：增大 \(L\) 可提供更丰富信号，但显著增加前向传播成本（指数级增长），在实践中需针对具体任务调整。
- **路由器依赖基础模型隐藏状态**：路由器输入包含来自基础模型的隐藏状态，训练时需联合优化，计算图较复杂。
- **实验范围有限**：仅在数学推理和合成规划上验证，未在更多领域（如代码、科学问答、多步骤推理）测试泛化性。
- **未与更先进的搜索方法直接对比**：如 Tree-of-Thoughts、MCTS 等方法，这些方法通常在更粗的单元上搜索，虽不可比且成本更高，但可作为上限参考。论文未讨论这一点。
- **路由器温度固定**：实验中 \(\tau=1\) 固定，未探索温度调度对路由探索-利用平衡的影响。
