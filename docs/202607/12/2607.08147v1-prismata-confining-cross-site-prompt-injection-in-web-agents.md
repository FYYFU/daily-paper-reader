## 摘要
自主Web代理有望自动化日常浏览任务，但继承了Web最古老的攻击面之一。跨站脚本攻击已经证明，混合可信与不可信内容是危险的，即使在良性的页面上也是如此。代理通过将自然语言解释为指令重新引发了这一风险，使得第三方和用户生成的内容能够通过提示注入劫持代理。核心挑战在于，推导任务特定的安全策略需要对页面结构进行推理，而页面结构又与攻击者的内容纠缠在一起。我们提出Prismata，这是一种为Web代理实施上下文最小权限的防御机制，既限制代理能看到的内容，也限制其能执行的操作。Prismata的动态信任推导为页面内容生成权限标签，并具有结构限制保证，这借鉴了经典完整性模型，能够限定任何标签错误，使得标签只能降低权限，且错误标记被限制。Prismata的机械限制通过编辑内容和限制代理能力来强制执行这些标签。重要的是，这些机制不需要开发者注释，因此Prismata支持长尾网站。在近期发表的Web代理攻击（包括自适应变体）中，Prismata在保持良性任务效用的同时，显著降低了攻击成功率。

## Abstract
Autonomous web agents promise to automate everyday browsing tasks, but inherit one of the web's oldest attack surfaces. Cross-Site Scripting proved that mixing trusted and untrusted content is dangerous, even on benign pages. Agents resurface this risk by interpreting natural language as instructions, allowing third-party and user-generated content to hijack the agent via prompt injection. The core challenge is that deriving a task-specific security policy requires reasoning over page structure that is entangled with the attacker's content.   We present Prismata, a defense enforcing contextual least privilege for web agents, constraining both what the agent sees and what it can do. Prismata's dynamic trust derivation produces permission labels for page content, with structural confinement guarantees, inspired by classical integrity models, that bound any labeling errors so that labels can only decrease in privilege and mislabelings are bounded. Prismata's mechanical confinement enforces these labels by redacting content and restricting agent capabilities. Importantly, these mechanisms require no developer annotations, so Prismata supports the long tail of websites. Across recent published web agent attacks, including adaptive variants, Prismata substantially reduces attack success while preserving benign task utility.

# 论文《Prismata: Confining Cross-Site Prompt Injection in Web Agents》详细中文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **问题定义**：自主Web代理（如BrowserGym、WebArena等）在执行用户任务时，会遭遇跨站提示注入（Cross-Site Prompting, XSP）攻击。攻击者通过在良性网站上的第三方或用户生成内容（如产品评论、广告）中嵌入恶意指令，劫持代理执行非授权操作（如泄露用户数据、重置密码等）。
- **核心挑战**：与传统的跨站脚本（XSS）不同，XSP的载荷是自然语言或图像，而非可执行代码，因此无法通过输入清理或沙箱隔离来防御。更根本的是，**Web缠绕问题（Web Entanglement Problem）**：安全策略需要从页面结构中推导，而页面结构本身与不可信内容纠缠在一起，使得防御机制在分离结构信号时暴露于攻击者操纵。
- **现有防御局限**：模型级防御（如指令层次训练）易被自适应攻击绕过；系统级防御（如CaMeL）依赖预定义工具语义，无法推广到数据依赖的Web任务；细粒度页面级安全策略需要开发者手动构建站点地图，不具扩展性。
- **论文贡献**：提出**Prismata**，一种无需开发者注释的上下文最小权限防御，同时限制代理“能看到什么”和“能做什么”，基于经典Biba完整性模型（1977）提供结构限制保证。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **关键路径（Critical Path）**：从DOM根节点到每个可交互元素（按钮、链接等）的完整祖先链。观察发现，这些路径几乎全部由开发者编写的结构元素（容器div、导航栏等）组成，很少包含不可信内容。
- **动态信任推导（Dynamic Trust Derivation）**：利用关键路径的特性，将每个可交互元素独立评估，且评估时仅暴露路径上的内容，避免攻击者影响全局判断。
- **Biba解析（Biba Parsing）**：受到Biba完整性模型（no-read-down, no-write-up）启发，递归遍历关键路径，在暴露子节点之前先评估祖先节点的结构线索（如标题、类名、ARIA属性），一旦检测到不可信边界，立即限制后续全部权限。
- **机械限制（Mechanical Confinement）**：确定性代码强制执行标签结果：删除不可信内容、降级元素功能（如只读）、拒绝超过权限的动作。

### 2.2 系统架构（三阶段）
1. **动作门（Action Gate）**：
   - 枚举页面上所有可交互元素，对每个元素独立获取其关键路径。
   - 仅使用用户任务描述 + 该路径的结构上下文（不含其他页面内容），由LLM判断该元素是否在任务范围内。
   - 若路径中不包含不可信内容，则标签是“无注入”的（Case 1）。
2. **Biba解析（Biba Parsing）**：
   - 对每个路径递归分类内容来源（开发者、用户、托管方、外部）。
   - 每个节点标签只依赖自身和祖先，子节点和兄弟节点被屏蔽（no-read-down）；标签只能比父节点更严格（no-write-up）。
   - 当结构线索（如“Reviews”标题）出现在不可信内容之前时，标签自动级联到所有后代，无需进一步查询（Case 2）。
   - 不可信子树根据任务需求被裁剪或限制为只读。
3. **机械限制（Mechanical Confinement）**：
   - 策略模型根据用户任务决定允许哪些来源（例如取消订阅不需要用户评论）。
   - 确定性代码执行：从观察中删除不可信节点、降级ID、拒绝超出能力的动作。所有动作均经过BrowserGym的有限动作空间，无任意JavaScript执行。

### 2.3 三种情况与安全保证
| 情况 | 注入位置 | 标签是否无注入 |
|------|----------|----------------|
| Case 1 | 注入不在可交互元素的关键路径上 | ✅ 动作门独立决定 |
| Case 2 | 注入在关键路径上，但有结构线索先于注入出现 | ✅ Biba解析在暴露注入前已限制 |
| Case 3 | 注入在关键路径上且无结构线索 | ❌ 残余风险（实测仅占0.1%） |

- **安全证明概要**：动作门只看到路径，Biba解析递归屏蔽；攻击者必须同时绕过两者才可能提升权限，且机械限制是确定性代码不可绕过。

## 3. 实验设计

### 3.1 数据集与场景
- **网站语料**：来自Common Crawl（Tranco top 10K域，每个域100页）和Mind2Web任务轨迹，共5,664个DOM（283,200个存档DOM）。用于验证关键路径属性和结构线索。
- **攻击环境**：WebArena（GitLab、购物、Reddit论坛、CMS等生产级站点）。
- **攻击模板**：
  - **Shortcut攻击**：弹出“一键快捷方式”诱使点击。
  - **Fake Completion攻击**：显示“任务已完成”消息诱导代理终止。
  - **Ignore Instruction攻击**：覆盖原始指令。
  - 额外测试：WASP（Web Agent Security Probe）和自适应攻击（WA20）。

### 3.2 对比方法
- 主要对比为 **无Prismata的基线代理**（使用gpt-5.4-mini）。

## 6. 主要结论与发现

- **攻击防御效果**：Prismata将平均攻击成功率从85.5%降至0.7%（ΔASR = -84.8个百分点），并将受攻击时的任务成功率从4.5%提升至23.0%（ΔTSR = +18.5个百分点）。
- **良性效用保持**：无攻击时任务成功率从29.9%略降至26.6%（降幅3.3%）。
- **WASP和自适应攻击**：Prismata完全阻止了WASP的plain-goal成功，并消除所有自适应WA20攻击的成功。
- **标签精度**：最佳标签模型（GPT-5.4-nano）达98.69%精度，F1最高为95.14%（Gemini 3 Flash），表明标签错误率低且主要导致保守限制（召回率稍低）而非安全薄弱点。
- **结构性质验证**：在90,408个不可信路径实例中，仅1.2%包含可交互后代；其中仅0.1%缺少结构线索（Case 3），在最佳实践站点上降至0.017%。

## 7. 优点

1. **无需开发者协作**：Prismata自动从DOM结构中推导标签，支持长尾网站，降低部署门槛。
2. **基于经典安全模型**：借鉴Biba完整性模型，提供可证明的“标签只能降级”保证，机械限制确定性不可绕过。
3. **细粒度上下文最小权限**：同时限制观察内容和动作能力，且通过关键路径隔离实现无注入标签（Case 1 & 2）。
4. **实验覆盖全面**：包含多种攻击模板、自适应变体、真实网站语料验证、人工标注标签验证，结果统计严谨。
5. **缓存友好**：利用DOM结构重复性，标签可重用，显著降低延迟和成本。
6. **系统设计清晰**：动作门、Biba解析、机械限制三阶段解耦，便于未来扩展。

## 8. 不足与局限

1. **仅限文本输入**：未处理多模态攻击（如图像对抗性扰动、非文本指令），当前评估基于Accessibility Tree，多模态环境需额外扩展。
2. **依赖模型准确性**：标签错误可能导致过严格（降低效用）或过宽松（减弱安全性）。虽然精度高，但最坏情况下的单点失误仍可被利用。
3. **Case 3残余风险**：注入在关键路径上且无结构线索时（约0.1%），攻击仍有成功可能。虽然要求攻击者精确放置注入，但理论上存在。
4. **未与其他系统防御对比**：如CaMeL、ceLLMate等，这些防御需要预规划或手动注释，无法直接比较，但限制了性能基准的全面性。
5. **高权限任务局限性**：若任务需要所有内容来源（如读取用户评论并执行操作），Prismata无法降级，只能依赖模型级鲁棒性。
6. **假设网站遵循最佳实践**：如输入过滤、CSP、沙箱广告等。若站点本身存在XSS漏洞，则超出XSP攻击范围。
7. **计算开销**：虽然缓存缓解，但Prismata仍增加约459秒（策略模型）+1098秒（动作门）+790秒（Biba解析），成本上升约50%（$1.105→$1.623），在延迟敏感场景可能受限。
8. **评估环境限制**：仅在WebArena上进行，未测试真实电商、社交平台等复杂性更高的网站，可能存在环境偏差。
