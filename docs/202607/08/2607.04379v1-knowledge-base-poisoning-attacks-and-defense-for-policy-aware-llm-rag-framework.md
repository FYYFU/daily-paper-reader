## 摘要
本文对面向战场物联网（IoBT）任务控制的策略感知大语言模型检索增强生成（PA-LLM-RAG）框架进行了对抗性安全研究。我们提出了一种新颖的攻击——查询无关的语义检索投毒，该攻击将语义精心设计的规则注入IoBT知识库，在不需了解运行时提示的情况下，在所有操作员查询类型中实现高检索排名。该攻击通过单条注入规则（1.6%投毒率）即可实现85%的LLM上下文污染，并在7.7%投毒率下达到饱和，表明即使对知识库的最小程度破坏也足以破坏任务决策。为应对这一威胁，我们提出了CLD-KB（知识库网络分层防御），一种双检测器异常检测框架，结合单类支持向量机边界检测与新颖的基于成员的类别扩散分析，利用IoBT三类策略分类法在决策LLM接触到投毒规则之前识别它们。CLD-KB在投毒检测和知识保留方面显著优于五种基线方法（包括DBSCAN、LOF、K-Means、孤立森林和单类SVM），每个任务仅增加7ms计算开销，从而成为面向LLM驱动的IoBT任务系统的一种有效且可部署在边缘的防御方法。

## Abstract
This paper presents an adversarial security study of the Policy-Aware LLM Retrieval-Augmented Generation (PA-LLM-RAG) framework for Internet of Battlefield Things (IoBT) mission control. We propose Query-Agnostic Semantic Retrieval Poisoning, a novel attack that injects semantically crafted rules into the IoBT knowledge base achieving high retrieval ranking across all operator query types without requiring knowledge of runtime prompts. The attack achieves 85% LLM context corruption from a single injected rule (1.6% poisoning rate) and saturates at 7.7% poisoning, demonstrating that even minimal knowledge base compromise is sufficient to corrupt mission decisions. To counter this threat, we propose CLD-KB (Cyber-Layered Defense for Knowledge Base), a dual-detector anomaly detection framework combining One-Class SVM boundary detection with a novel Member-Based Category Spread analysis that exploits the three-category IoBT policy taxonomy to identify poisoned rules before they reach the decision LLM. CLD-KB significantly outperforms five baseline methods including DBSCAN, LOF, K-Means, Isolation Forest, and One-Class SVM in both poisoning detection and knowledge preservation, with only 7ms computational overhead per mission, establishing it as an effective and edge-deployable defense for LLM-driven IoBT mission systems.

## 1. 核心问题与整体含义（研究动机和背景）
- **背景**：战场物联网（IoBT）任务控制需要严格遵循操作策略（工作流、交战规则、能力约束）。作者团队先前提出了PA-LLM-RAG框架，通过检索增强生成（RAG）将LLM决策锚定在结构化策略知识库中，并利用Judge LLM进行执行前验证。该框架假设知识库完全可信。
- **核心问题**：在对抗环境下，对手若获得知识库写权限，可以注入精心设计的“投毒规则”，这些规则在语义检索中排名靠前，从而污染LLM的上下文，导致自主智能体做出违反策略的灾难性决策（如误判目标、错误部署）。现有RAG安全研究主要针对开放域文档，未关注结构化策略规则库及IoBT场景。
- **整体含义**：本文首次系统研究了面向IoBT策略感知LLM-RAG的知识库投毒攻击，并提出领域特定的防御机制，证明最小程度的库破坏即可严重威胁系统安全。

## 2. 方法论：核心思想、关键技术细节
### 2.1 攻击方法：查询无关语义检索投毒（Query-Agnostic Semantic Retrieval Poisoning）
- **核心思想**：对手无需知道运行时操作员查询的具体内容，通过构造一条或多条跨类别（WF、ROE、CAP）词汇的“通用”规则，使其嵌入向量接近整个查询分布的质心，从而在所有查询类型下均获得高余弦相似度排名。
- **关键技术细节**：
  - 分析IoBT操作词汇高频词（如gate, drone, UAV, UGV, perimeter等），将资产类型和边界词汇嵌入规则文本。
  - 构造的规则同时包含WF风格的指令、ROE风格的紧迫性和CAP风格的资产引用，使其嵌入等距于三个类别中心。
  - 攻击形式化：设合法规则集K，注入M条投毒规则P，目标使任意查询q的top-k上下文中至少包含一条pj。
  - 投毒率ρ = M/(N+M)，实验从1.6%（1条规则）到25%（20条规则）。
- **公式/算法**（文字说明）：
  - 使用nomic-embed-text得到768维嵌入，余弦相似度检索。
  - 投毒规则针对查询分布质心ē_Q（由20个操作员提示的嵌入平均而得）优化，使sim(ē_Q, φ(pj)) ≥ τ_ret。
  - 攻击成功标准：LLM上下文中出现任何非合法库中的规则（通过Judge LLM交叉验证）。

### 2.2 防御方法：CLD-KB（Cyber-Layered Defense for Knowledge Base）
- **核心思想**：双检测器框架，在检索层拦截投毒规则。利用IoBT策略的三分类结构检测异常。
- **检测器1：One-Class SVM (OC-SVM)**
  - 在合法规则嵌入上训练，学习包围正常分布的超球面边界，边界外的规则被标记为异常。
  - 参数：ν=0.2, RBF核γ=0.1，阈值τ_svm设为训练得分的最低3百分位，保证零假阳性。
- **检测器2：基于成员的类别扩散（Member-Based Category Spread）**
  - 利用合法规则严格属于单一类别（WF/ROE/CAP）的特性：合法规则与其自身类别成员的余弦距离小，与其他类别成员距离大，因而跨类别最小距离的跨度大（>0.20）。
  - 投毒规则因混用三类词汇，与各类别成员距离相近，跨度小（<0.15）。
  - 定义：spread(r) = max_c d_min(r,c) - min_c d_min(r,c)，阈值τ_sp=0.17。
- **联合检测**：任一检测器标记即判定为投毒。搜索时，检索top-10候选规则，过滤直到收集3条干净规则。
- **自适应黑名单**：被标记的规则从知识库永久移除，后续检索中不会出现。

## 6. 主要结论与发现

- **防御有效性**：CLD-KB在所有投毒率下保持100%上下文完整性（CIR=100%），且零假阳性（F1=1.0）。对比基线中，DBSCAN和LOF彻底失败（F1=0）；K-Means和孤立森林仅部分检测（F1约0.68-0.70）；单类SVM召回率100%但有一个假阳性（F1=0.98）。CLD-KB唯一完美。
