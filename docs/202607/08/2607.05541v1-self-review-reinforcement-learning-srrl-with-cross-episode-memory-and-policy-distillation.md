## 摘要
强化学习通常用于利用环境反馈训练大型语言模型。在实际应用场景中，环境通常提供稀疏或延迟的反馈。这使得模型难以确定其推理过程中哪些动作导致了成功或失败。因此，有效从这些信号中学习是很困难的，因为模型必须确定每个失败应如何指导后续迭代中有意义的行为修正。我们提出了一种训练框架——自我审查强化学习，该框架在每个强化学习回合中嵌入明确的自我审查步骤。当首次响应失败时，模型会生成自我审查以识别问题所在，从而为改进的第二次尝试创造条件。与推理时反思方法（如Reflexion）不同，该框架使用策略梯度优化自我审查，并通过选择性提炼将改进内化到基础策略中，确保其在未来的回合中持续存在。跨回合记忆会保留成功的自我审查，以便在后续训练中遇到类似任务时复用。我们在GSM8K基准上，使用GRPO优化器，对两个语言模型Qwen 3-4B和OLMo-3-7B，将SRRL与标准RLVR基线进行了评估。SRRL在最终奖励性能上始终优于RLVR，并通过成功将反馈转化为行为改进，获得了更高的学习效率。

## Abstract
Reinforcement Learning is commonly used to train large language models using environmental feedback. In applied settings, the environment usually provides sparse or delayed feedback. This makes it difficult for the model to pinpoint which actions in its reasoning led to success or failure. So, learning effectively from these signals is hard because the model must determine how each failure should inform meaningful behavioral corrections in subsequent iterations. We introduce a training framework, Self-Review Reinforcement Learning, that embeds an explicit self-review step into each RL episode. When a first-pass response fails, the model generates a self-review to identify what went wrong, which conditions an improved second attempt. Unlike inference-time reflection approaches, such as Reflexion, the framework optimizes self-review with policy gradients and internalizes improvements into the base policy via selective distillation, ensuring they persist across future episodes. A cross-episode memory keeps successful self-reviews for reuse when encountering similar tasks in future episodes during training. We evaluate SRRL against a standard RLVR baseline using the GRPO optimizer across two language models, Qwen 3-4B and OLMo-3- 7B, on GSM8K benchmark. SRRL consistently outperforms the RLVR in final reward performance and achieves greater learning efficiency by successfully transforming feedback into behavioral improvement.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：大语言模型（LLM）在强化学习（RL）训练中通常只能获得稀疏或延迟的环境反馈（如简单的成功/失败标量奖励），导致模型难以定位推理过程中的具体错误步骤，无法有效从失败中学习，学习效率低下。
- **现有局限**：
  - 标准 RLVR（Reinforcement Learning with Verifiable Rewards）仅依赖标量奖励，缺乏结构化指导，模型需通过大量探索才能隐式推断错误原因。
  - 推理时反思方法（如 Reflexion）虽能在部署时进行自我修正，但计算成本高，且改进不持久。
  - 监督微调（SFT）无法利用交互反馈。
- **动机**：设计一种在训练过程中显式利用结构化自我审查（self-review）以转化反馈为持久策略改进的框架，同时避免推理时额外开销。

## 2. 方法论：核心思想、关键技术细节与流程
### 核心思想
- 在每次 RL 回合（episode）中嵌入一个显式的自我审查步骤：如果第一次回答失败，模型生成自我审查（分析错误、提出修正策略），然后基于审查进行第二次回答。通过策略梯度优化自我审查过程，并通过选择性蒸馏（selective distillation）和跨回合记忆（cross-episode memory）将成功的修正策略内化到基础策略中，使得改进在后续回合和部署时无需再次审查。

### 技术细节与流程
1. **第一次回答**：给定问题 x，模型 πθ 生成回答 y1，环境返回标量奖励 r1 ∈ {0,1} 和文本反馈 f1。
2. **条件触发**：若 r1 < 阈值 τ（τ=1），则进入自我审查阶段。
3. **自我审查生成**：模型基于 x, y1, r1, f1 以及从跨回合记忆检索到的相关记忆 m，生成自我审查 Δ（分析错误并给出改进指南）。
4. **第二次回答**：模型基于 x 和 Δ 生成 y2，环境返回 r2, f2。
5. **跨回合记忆**：若 y2 成功（r2 > τ），则将 Δ 存入记忆 m，供未来相似问题检索。
6. **策略蒸馏**：仅对成功的 y2 应用蒸馏损失 `L_distill(θ) = -E[I{r2>τ} log πθ(y2|x)]`，使基础策略学会直接产生类似 y2 的质量，而不依赖自我审查。
7. **RL 目标**：采用 GRPO（Group Relative Policy Optimization）优化策略。自我审查 Δ 的奖励设为 r2。完整目标：`L_SRRL = L_policy + L_distill + β·L_KL`。其中 L_policy 覆盖所有生成步骤（y1、Δ、y2），蒸馏损失仅在 r2 成功时生效。
8. **部署**：仅使用 `y ~ πθ(·|x)`，无需自我审查、记忆或蒸馏，零延时。
